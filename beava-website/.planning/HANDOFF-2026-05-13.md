## Repo facts worth memorising

- Beava server: `cargo build --release --bin beava` → `target/release/beava` (3.6 MB stripped, arm64 macOS).
- Python wheel: `cd python && maturin build --release` → `target/wheels/beava-*.whl` (2.3 MB).
- Auto-deploy fires on `Publish edge image to Docker Hub` workflow_run completion. Only takes effect for `beava-website/**` changes that the image build noticed; non-image-affecting changes still rsync because the workflow always runs.
- `/srv/beava-website/` on Hetzner is NOT a git checkout — it's rsync-managed by the workflow.
