# beava-website

The beava.dev site. Hand-rolled static HTML + a few React-via-Babel pages, served by Cloudflare Pages.

## Local development

```bash
cd beava-website
npm install                  # installs Pagefind for the search build
python3 -m http.server 8000 --directory project
# open http://localhost:8000/
```

The project is mostly plain HTML/CSS/JSX — no build step for content. The two scripts under `scripts/` rebuild the search index and audit internal links:

```bash
npm run build:search         # crawl project/ + emit project/_pagefind/ search index
npm run check:links          # post-build link audit (must exit 0 before pushing)
```

## Layout

```
project/
  index.html                 # marketing home (React+Babel)
  guide/                     # /guide/ landing + chapter-1 (React+Babel)
  docs/                      # hand-written docs pages (React+Babel)
    index.html               # /docs/ landing
    get-started/             # quickstart, push-events, query-features, define-a-pipeline
    concepts/                # streams, tables, windows, freshness, get-and-batch-get
    vision/                  # why-beava, non-goals
  js/                        # SiteHeader, SiteFooter, DocsSidebar, DocsTOC, DocsShared
  styles/                    # colors_and_type.css, site.css, docs-kit.css
  assets/                    # mascots, favicon, banner
  _pagefind/                 # generated by build:search
scripts/
  build-search.mjs           # Pagefind 1.5 index builder
  check-links.mjs            # internal-link audit (exits non-zero on BROKEN)
  scrub-slop.mjs             # one-off scrub helper for AI-slop patterns
```

The Phase 13.0 spec docs (markdown sources at `docs/*.md` in repo root) and the auto-render pipeline (`render-docs.mjs`) were nuked in commit `f99a09e1`. The current site is hand-written; new pages are added by writing the HTML file directly under `project/docs/`.

## Deploy (Cloudflare Pages)

Auto-deploy is configured via `wrangler.toml` + Cloudflare's dashboard. One-time setup (run by the repo owner, not Claude):

1. Log in to https://dash.cloudflare.com → Pages → "Create a project" → "Connect to Git"
2. Authorize the GitHub org owning `beava` (likely `beava-dev`)
3. Select the `beava` repository
4. Build settings (Cloudflare auto-detects from `wrangler.toml`):
   - Production branch: `main`
   - Build command: `cd beava-website && npm install && npm run build:search`
   - Build output directory: `beava-website/project`
   - Root directory: (repo root)
5. Click "Save and Deploy" — first build takes ~2-3 min.
6. Once green: visit the auto-generated `<pages-project>.pages.dev` URL and smoke-test:
   - `/` — home loads
   - `/docs/` — docs landing with sidebar
   - `/docs/get-started/quickstart/` — quickstart
   - `/guide/chapter-1/` — guidebook chapter 1
   - Search box — type "table" → expect a hit at `/docs/concepts/tables/`
7. Add custom domain: Pages project → Custom domains → set up `beava.dev` + `www.beava.dev`.
8. DNS (Cloudflare DNS for beava.dev — same account if already there; else manual):
   - CNAME `beava.dev` → `<pages-project>.pages.dev` (proxied)
   - CNAME `www.beava.dev` → `<pages-project>.pages.dev` (proxied)
9. Wait ~5 min for SSL provisioning, then visit beava.dev.

After setup: every push to `main` auto-deploys. PR branches get preview URLs at `<branch>.<pages-project>.pages.dev`.
