Events don't add memory. The sketch tiers and window TTL do the work: distinct-count promotes through ExactArray (≤16, ~128 B) → HashSet (≤1024, ~32 KB) → HLL p=12 (~4 KB, constant) per-user as cardinality grows, and any user inactive past the widest window (7d here) ages out of the hot set entirely. One billion events on 1M hot users is still well under a gigabyte. Only a new distinct user or a new metric column grows the footprint.
