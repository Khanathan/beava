# GitHub Repo Surface Runbook (INFRA-09)

**Decision locks:** D-16 (description), D-17 (topics), D-18 (social preview).

These are manual maintainer steps. The GitHub UI does not have a fully
API-first path for topics / description / social preview that fits our launch
cadence. Execute each step once at launch time against the production repo.

---

## 1. Repo Description (D-16)

Navigate to your repo homepage on GitHub and click the gear icon next to
"About" in the right-hand sidebar (or go to Settings → General → About).

Paste exactly:

```
Real-time feature server for ML. Single binary, event-time, HTTP + CDC fork.
```

This is 67 characters — within GitHub's soft ~110-char limit and stays on one
line at common viewport widths.

**Website field:** set to `https://beavadb.github.io/beava` or `https://beava.dev`
if that domain is live. If neither is ready, leave blank — an empty field is
better than a fabricated URL.

---

## 2. Topics (D-17)

Same gear icon → Topics field. Add exactly these 8 topics (one per entry,
no spaces — GitHub lowercases and slugifies them automatically):

| Topic | Discovery intent |
|-------|-----------------|
| `feature-server` | direct category search |
| `real-time` | streaming / low-latency audience |
| `rust` | language-ecosystem discovery |
| `streaming` | event-streaming searchers |
| `ml` | ML-platform evaluators |
| `apache-2-0` | license-driven searches |
| `event-time` | niche but high-signal for the right audience |
| `feature-store` | covers "Feast alternative" searches |

8 topics — well within GitHub's 20-topic cap. These cover all major discovery
queries: "rust feature server", "streaming ml", "feature store alternative".

## 3. Social Preview (D-18)

The asset is committed at `site/assets/social-preview.png` (1280×640 PNG,
generated by ImageMagick from the `beaver.png` mascot).

**Upload steps:**

1. Go to **Settings → General → Social preview** (scroll down past "Features").
2. Click **Edit** next to the social preview section.
3. Click **Upload an image…** and select `site/assets/social-preview.png`
   from your local clone of the repo.
4. Click **Save**.

The image is used as the Open Graph `og:image` tag for GitHub, LinkedIn,
Twitter/X, Bluesky, and Hacker News preview cards when someone shares the
repo URL.

**Verify after upload:**

- Paste the repo URL into https://opengraph.xyz — confirm the 1280×640 card
  renders with the beava wordmark and tagline.
- Or use Twitter's card validator: https://cards-dev.twitter.com/validator

## 5. Optional: Enable GitHub Pages for Docs

If you want to serve the `docs/` tree as a GitHub Pages site:

1. Go to **Settings → Pages**.
2. Under **Source**, select **Deploy from a branch**.
3. Branch: `main`, folder: `/docs` (or `/` root if you prefer the `site/`
   directory via a custom build step).
4. Click **Save**. GitHub will publish at `https://<org>.github.io/<repo>`.

This is optional for launch — Docker + curl from README is the primary onboarding
path. Enable Pages if/when docs traffic justifies it.

---

## 6. Rollback

**Description / Topics:** Edit the About gear icon and revert to previous values.
No downtime — changes take effect immediately.

**Social preview:** Re-upload a simpler version (dark background + wordmark only)
or click "Remove image" to fall back to GitHub's auto-generated preview.
