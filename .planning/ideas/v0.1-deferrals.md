## SDK deferrals (from 2026-05-03 design session)

| Item | Why deferred | Workaround in v0 |
|---|---|---|
| **Java SDK** | Not in top-3 popularity for beava's startup/modern-stack target audience; large maintenance lift (Maven/Gradle/JVM compat); slower adoption velocity | Wire spec doc enables community port |
| **Async / `AsyncApp`** | Sync covers v0; async is parallel API, easy additive change | `asyncio.to_thread(app.push, ...)` |
| **`app.deregister(name)`** | No server-side opcode yet; force-re-register handles drop case | `app.register(*reduced, force=True)` |
| **`app.migrate(...)`** | Non-trivial schema evolution (re-key, fold, reshape) — needs design work | Additive-only or `force=True` drop+rebuild |
| **Lifecycle hooks** (`before_push`, `after_register`, retry callbacks) | Niche middleware feature; not blocking launch | Wrap `app.push()` in user code |
| **Structured `ErrorCode` enum** (Python) | String-match on `exc.code` works | `if exc.code == "unknown_table": ...` |
| **Schema introspection** (`app.list_descriptors()`, `app.get_schema(name)`, `app.diff_pending(*new)`) | Useful for tooling but not v0 launch-critical | `app.ping()` returns `registry_version`; `dry_run=True` returns the diff |
| **Embed binary version-introspection** beyond version-pin (`app.embed_info()`) | Niche debugging feature | Check `beava --version` separately |
| **`OP_PUSH_BATCH`** (multi-event push frame) | Single connection means low overhead; loop-push is fine | `for event in batch: app.push(...)` |
| **`OP_PUSH_SYNC`** (acks=all durable push) | Phase 6.1 designed but not wired through; v0 push is acks=1 (Kafka-style) | Use disk-mode embed for tests; production accepts acks=1 |
| **Multi-conn pool** inside one `App` | Single-conn + auto-reconnect handles common cases; horizontal concurrency via multiple processes | Spawn multiple `App` instances |
| **HTTP/2** | HTTP/1.1 + keep-alive sufficient for v0 | n/a |
| **msgpack-on-HTTP** (currently TCP-only) | TCP is the perf path; HTTP is the curl/WAF path | Use TCP for production; HTTP for setup |
| **Brew formula** (`brew install beava`) | Optional polish; pip + docker cover most install paths | `pip install beava` or `docker run beava/beava` |

## How to add an item to this list

When deferring something in a session, append to the appropriate section above with:
- Item name
- Why deferred (1-2 sentence rationale)
- Workaround in v0 (so users have a path today)
- Estimated effort (LOC) for v0.1 implementation

For larger directions (e.g. historical engine, session windows), create a dedicated doc in `.planning/ideas/` and link it from § Big architectural directions.
