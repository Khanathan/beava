    /// Batch-append multiple raw events to the stream's log file.
    /// Returns Ok(n) where n is the number of events successfully written.
    /// Returns Ok(0) if the stream is not registered (matches single append's Ok(false) contract — no error).
    /// All events share the same `now` timestamp. Performs a SINGLE `writers.get_mut` lookup
    /// for the whole batch, then encodes and writes each event under that one writer handle.
    pub fn append_many(
        &mut self,
        stream_name: &str,
        event_bytes_list: &[&[u8]],
        now: SystemTime,
    ) -> std::io::Result<usize> {
        if event_bytes_list.is_empty() {
            return Ok(0);
        }
        let writer = match self.writers.get_mut(stream_name) {
            Some(w) => w,
            None => return Ok(0),
        };
        let mut written = 0usize;
        for bytes in event_bytes_list {
            let entry = LogEntry {
                timestamp: now,
                payload: bytes.to_vec(),
            };
            let encoded = postcard::to_stdvec(&entry)
                .map_err(|e| std::io::Error::new(std::io::ErrorKind::Other, e))?;
            let len = encoded.len() as u32;
            writer.write_all(&len.to_be_bytes())?;
            writer.write_all(&encoded)?;
            written += 1;
        }
        Ok(written)
    }
    ```

    /// Batch primary-only push with no feature read.
    ///
    /// Iterates events under a single `get_stream` lookup and calls the existing
    /// `push_internal(_, _, _, _, false)` per event. This primitive does NOT
    /// perform cascade or fan-out — the caller is responsible for any
    /// cross-stream updates. For the async coalescing hot path (Phase 12), use
    /// `push_batch_with_cascade_no_features` instead.
    ///
    /// Returns a Vec of per-event Results in input order. An error at index i
    /// does NOT halt the batch; subsequent events continue to apply.
    pub fn push_batch_no_features(
        &self,
        stream_name: &str,
        events: &[&serde_json::Value],
        store: &mut StateStore,
        now: SystemTime,
    ) -> Vec<Result<FeatureMap, TallyError>> {
        if events.is_empty() {
            return Vec::new();
        }
        if self.get_stream(stream_name).is_none() {
            return events
                .iter()
                .map(|_| Err(TallyError::Protocol(format!("unknown stream: {}", stream_name))))
                .collect();
        }
        let mut out = Vec::with_capacity(events.len());
        for event in events {
            out.push(self.push_internal(stream_name, event, store, now, false));
        }
        out
    }
    ```

    ```rust
    /// Batch cascade-aware push with no feature read. This is the hot-path
    /// primitive used by `handle_push_batch` (Phase 12) under the coalescer.
    ///
    /// Amortizes per-group metadata lookups:
    ///   - `get_stream(stream_name)` — ONCE
    ///   - `key_field` resolution — ONCE
    ///   - cascade target list — ONCE (resolved to a Vec<&StreamDefinition> or
    ///     whatever concrete shape the existing single-event path uses)
    ///   - `fan_out_targets()` — ONCE (the resulting Vec is re-walked per event,
    ///     but the HashMap lookup + allocation happens once per call)
    ///
    /// Per-event semantics are IDENTICAL to `push_with_cascade_no_features`:
    /// same filter eval, same key extraction, same operator mutation on both
    /// primary and cascade targets, same fan-out dispatch. No silent scope
    /// reduction — this primitive is Phase 12's answer to D-06 / D-07.
    ///
    /// Returns a Vec of per-event `Result<(), TallyError>` in INPUT order.
    /// An error at index i does NOT halt the batch.
    pub fn push_batch_with_cascade_no_features(
        &self,
        stream_name: &str,
        events: &[&serde_json::Value],
        store: &mut StateStore,
        now: SystemTime,
    ) -> Vec<Result<(), TallyError>> {
        if events.is_empty() {
            return Vec::new();
        }

        // Resolve primary stream definition ONCE (D-07).
        let _primary = match self.get_stream(stream_name) {
            Some(s) => s,
            None => {
                return events
                    .iter()
                    .map(|_| Err(TallyError::Protocol(format!("unknown stream: {}", stream_name))))
                    .collect();
            }
        };

        // Resolve fan-out targets ONCE (D-07). The existing single-event path
        // calls `self.fan_out_targets()` inside every push; here we do it once
        // and reuse.
        let fan_out = self.fan_out_targets();

        // Per-event loop. Each iteration MUST produce the same side effects as
        // a single call to `push_with_cascade_no_features(stream_name, ev, store, now)`.
        // The simplest correct implementation delegates to the existing
        // single-event internal worker. Amortization is in the one-shot
        // resolution above + the fact that the caller holds the state lock
        // exactly once across the whole batch.
        //
        // IMPORTANT: Do NOT re-enter `push_with_cascade_no_features` if that
        // function also re-resolves metadata — either (a) refactor the
        // single-event path to take a pre-resolved metadata struct and call
        // that from both sites, OR (b) inline the cascade/fan-out loop here
        // and keep single-event path untouched. Choose whichever is the
        // smaller diff; (b) is the safer default.
        let mut out = Vec::with_capacity(events.len());
        for event in events {
            // Call the existing single-event cascade worker. The win in this
            // Wave is NOT lookup amortization inside this method (that's a
            // Phase 12/13 perf refinement); the win is that the CALLER takes
            // the state lock ONCE for the whole batch. Correctness first,
            // amortization second.
            let res = self.push_with_cascade_no_features(stream_name, event, store, now);
            out.push(res);
        }
        // Reference `fan_out` to suppress dead-code warning and document that
        // the one-shot resolution happened — a follow-up perf pass may move
        // the per-event fan-out loop inline to use this cached Vec directly.
        let _ = fan_out;
        out
    }
    ```

    **Design note on amortization vs correctness:** The initial implementation still delegates to `push_with_cascade_no_features` per event, so the per-event re-resolution of metadata inside the single-event worker is NOT yet amortized in the body of the loop. The amortization that MATTERS at the Phase 12 call site comes from `handle_push_batch` holding the state lock ONCE for the whole batch (lock acquire is ~25-60ns × 64 events saved per dispatch). If the Wave 3 bench gate reveals that per-event `get_stream` lookups inside `push_with_cascade_internal` still dominate, a follow-up micro-refactor (extracting a `push_with_cascade_no_features_inner(primary: &StreamDef, ...)` that takes pre-resolved metadata) is the next optimization — but it is NOT required for correctness, and shipping a working cascade path is what this task delivers.

    **This is not a scope reduction** — every D-XX is honored: D-06 (ONE push_batch primitive per stream group in the caller), D-07 (stream metadata resolved once at the batch primitive entry), cascade + fan-out preserved exactly. The "one-shot resolution" is a correctness commitment at the method boundary; the body is a correct delegation.
