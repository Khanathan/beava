<objective>
Switch TCP `/get` to MessagePack body + response by default while leaving HTTP `/get` JSON-only. The wire-byte plumbing landed in Plan 12-07 (`body_format: u8` is on TcpGet / TcpMGet / TcpGetMulti variants); this plan makes the dispatch helpers actually branch on it.

**What this fixes:** the post-12-07 read path serialises and deserialises every TCP `/get` body+response through `serde_json` even when the caller would prefer the cheaper msgpack codec. Per the diagnosis in `12-09-SCOPE.md`, JSON parse + serialize is **~54% of `dispatch_get_batch` apply work**. The fast-path-eligible content_type byte already rides on every TCP /get frame; the apply-side dispatch just ignores it (`body_format: _` in `apply_shard.rs:235-300`).

**What this delivers:**

1. `dispatch_get_batch_sync(app, body, body_format)` and `dispatch_get_single_sync(app, feature, key, body_format)` branch on `CT_JSON` vs `CT_MSGPACK` for body parse and response encode.
2. `GlueResponse::QueryResult { body, format: u8 }` extension; encoder emits the byte. Same-format-as-request response semantics (msgpack-in → msgpack-out).
3. `apply_shard::dispatch_one` arms pass body_format through (no longer `_`-ignored).
4. HTTP `/get` stays JSON-only — `HttpGet { body }` arm passes a `CT_JSON` constant; `encode_glue_response_http` always emits `Content-Type: application/json` regardless of `format`.
5. Python SDK `App.get(...)` over `tcp://` URLs defaults to msgpack body+response; over `http://` stays JSON. New `python/benches/read_bench_tcp.py` driver exercises the path.
6. Tests covering: TCP+msgpack round-trip, TCP+JSON unchanged, HTTP+JSON unchanged, dispatch unit tests for both content_types, error path for unsupported byte.
7. Criterion microbench `phase12_09_msgpack_get.rs` comparing JSON vs msgpack at single-cell, 10×5, 100×5 shapes — proves ≥40% lift on Apple-M4.
8. Throughput rebaseline: bench-v18 + read sweep across `(small, fraud-team) × (tcp, http) × (json, msgpack)` /get matrix; rows in `.planning/throughput-baselines.md`.
9. Phase 12-09 SUMMARY + STATE.md "Current Focus" + ROADMAP.md tick.

Purpose: closes the JSON-on-the-read-path overhead identified by the 12-08 perf trace. Delivers the foundation Plan 12-10 (push-and-get) needs — by 12-10's wave 1, msgpack /get must already be the TCP default.

Output: working msgpack /get over TCP end-to-end, microbench + throughput baselines, Plan 12-09 SUMMARY.
</objective>
