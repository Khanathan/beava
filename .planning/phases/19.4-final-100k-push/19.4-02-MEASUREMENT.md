**OVERALL: HALT**

All three primary measurement floors miss. The structural-correctness invariant (spill-counter test) PASSES — the cap was widened correctly, the spill is eliminated for the fraud-team-shape — but the macroscopic measurements cannot demonstrate the predicted -530 ns lift above the system-load variance band.

### Why the measurement floors miss

1. **Criterion bench is structurally insensitive (same as Plan 01).** The synthetic fixture rows have 7 fields — below the 8-cap threshold — so neither cap=8 nor cap=16 produces a spill on this bench. The SmallVec optimization-target mode (length > 8) is never engaged. Plan 01 hit the same shape on CountDistinct (only 2/14 features in HashSet mode); Plan 02 hits it more strongly (zero of 14 features even reach the spill threshold).

2. **System load variance dominates the live signal.** The 9 outer-trace runs span 8,521–13,954 ns (5.4k ns range, ±25% of the 11.6k ns mean). The 22 EPS runs span 51,643–93,323 (42k EPS range, 60% of the median). Plan 01 was measured during a quieter system state (3 runs, 78,447–81,188 EPS, 2.7k EPS spread). Today's system has higher background load (`uptime` 4.5–7.7) which expands the noise band by ~10×, making any -397 ns predicted lift inseparable from the ±2,500 ns measurement floor.

3. **The cap-8→16 change has no effect on the steady-state apply path for cluster sizes ≤ 8.** Most fraud-team aggs are on clusters with ≤8 fields per source. Only the TxnByUser cluster (10 fields) hits spill at cap=8. The plan's predicted -530 ns assumes ALL apply iterations hit this cluster; in reality, fraud-team apply iterations distribute across 5 clusters, so only ~1/5 of events hit the 10-field cluster, giving an amortized predicted lift of ~-100 ns/event — already well below the ±400 ns trace measurement noise floor on this loaded system.

### What is correct

- The cap was widened from 8 to 16 at exactly one line (`agg_op.rs:232`).
- All 611 lib tests pass (was 609; +2 new spill-counter tests in `agg_op::tests`).
- The two RED-state tests (cap≥12, cap≥16 invariants) flip to GREEN against the new cap.
- `cargo clippy ... -D warnings` and `cargo fmt --all --check` are clean.
- Anti-pattern grep over `agg_op.rs` (`spawn|rayon|par_iter`) returns 0 matches; no architectural changes were introduced.
- Threat model unchanged: stack frame grew by 8 × 16 = 128 bytes per `ExtractedFields`; well below stack-overflow risk.

### What is uncertain

Whether the cap-8→16 widening produces a real -397+ ns/event lift on a quiet measurement system. The data we have suggests:
- Mean-basis (across 9 runs): -448 ns = clears the floor (but inside variance band)
- Median-basis (across 9 runs): -40 ns = misses by 357 ns
- Quiet single-run (run 6 or 7): 8,521–8,645 ns, suggesting -3,000 ns drop versus 11,667 baseline (clears floor by 7×)
- Loaded single-run (run 9): 13,954 ns, suggesting +2,287 ns regression

Per CONTEXT D-03 HALT-GATE: write `19.4-02-DEVIATION.md`, do NOT proceed to Plan 19.4-03 — the orchestrator must surface this to the user with three options:

1. **Re-measure on a quieter system** (load avg < 1, no concurrent http server / Claude processes; cold reboot if needed) — the 9-run trace median is well within the per-run variance band, so a tighter measurement may resolve PASS/HALT cleanly.

2. **Accept measured behavior** — note that the spill-counter test passes (the structural fix is correct; cap≥16 invariant holds); the absolute predicted lift (-530 ns) is below the measurement noise floor of the present hardware/load combination. Re-derive Plan 03/04 floors using the observed prediction-realization rate (similar to how Plan 01 was accepted with a re-derived floor model).

3. **Revert cap to 8** — discard the GREEN commit (`c47c792`); leave `ExtractedFields<'a>` at cap=8. The fraud-team-shape spill remains a future-phase optimization opportunity. Loses the structural-correctness invariant (cap holds 12+ fields without spill) and the spill-counter test would fail RED again.

The strong recommendation is **option 1** (re-measure on a quiet system) before any architectural decision. The current data is too noisy to declare either PASS or genuine REGRESSION; the structural fix is correct (RED→GREEN), and the macro signal is dominated by system-load variance.
