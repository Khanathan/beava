## Decisions Made

- **Stack-allocated SmallVec for hoist, not thread_local + unsafe.** The plan's pseudocode mentioned a thread_local with raw pointers + SAFETY blocks. The simpler stack-only approach works because the `'a` lifetime of `ExtractedFields<'a>` is bound to `&row`'s lifetime which lives for the whole `apply_event_to_aggregations` call — no need for buffer reuse across events.
- **Per-feature field_idx_into_event_extracted shape.** Empty for fieldless features (Count, Ratio); length = agg.field_names.len() for features with declared fields, indexed via feat.descriptor.field_idx. This shape passes the Task 4.2.a test AND works with the apply-path consumer's `[field_idx]` indexing.
- **Re-point lat_idx/lon_idx into per-source union.** Both resolvers overwrite feat.descriptor.ext.lat_idx/.lon_idx with union positions when source_union is non-empty. Without this, the geo update_at fast path would index into the wrong slot under the hoist.
- **Test-only thread_local Cell<u64> for EXTRACTED_BUILD_COUNT.** Per-thread storage avoids cross-test pollution under cargo's parallel test runner. Production data plane is single-threaded so per-thread storage is also semantically correct there.
- **Three INDEPENDENT codepath differences in the legacy oracle.** The Task 4.4.a test would otherwise be tautological. The plan called for these explicitly; my Task 4.4.b implementation satisfies all three (audited via `grep` on the legacy fn body).
- **HALT on trace floor failure even though EPS hit 100k goal.** Per plan rules, both floors must clear; one failing triggers HALT. The right action is to surface the deviation, get explicit user override, then advance — not to silently mark PASS.

### One bench-fixture detail

The criterion bench (`apply_path_bench.rs`) uses a synthetic 14-feature, 3-cluster registry on a single Txn source. The hoist eliminates 2 redundant per-desc rebuilds (D=3 → 1 build).
