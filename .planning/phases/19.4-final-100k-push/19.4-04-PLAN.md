<objective>
Hoist `ExtractedFields` build out of the per-descriptor loop and up to the per-event scope. Per `.planning/phases/19.3-windowed-pre-extraction/19.3-COST-MODEL.md §4`, the current code rebuilds `ExtractedFields` 5× per event (once per descriptor), costing ~500 ns × 5 = 2,500 ns/event of scaffolding. Per `.planning/phases/19.3-windowed-pre-extraction/19.3-FLAMEGRAPH.md §2 row #2`, `apply_event_to_aggregations` self-time is 11.13% of apply CPU = 1,467 ns/event, with per-desc rebuild being the dominant component.

The plan implements CONTEXT D-02's "field-union scope = union across ALL descriptors at register-time" decision in 4 task-pairs (each red→green per CLAUDE.md TDD discipline), plus a measurement gate:

- Task 4.1: Populate `EventDescriptor.apply_field_names` at registration time (union of fields any agg on this event source consumes; per RESEARCH §6 amendment, currently `vec![]` at every registry construction site, including registry.rs::tests fixtures).
- Task 4.2: Re-resolve `field_idx` against the per-event union (introduce `field_idx_into_event_extracted: Vec<u8>` on each agg state struct; populated by extending `resolve_field_indices_for_agg_mut` / `_inner` from registry.rs:564 / 604).
- Task 4.3: Hoist `ExtractedFields` build to the existing apply-loop entry (single per-thread local Vec alongside the existing per-event scaffolding) — `clear() + resize_with(union_size, || None)` per event, reused across events.
- Task 4.4: Bit-identical state regression test via `f64::to_bits()` strict equality on a deterministic 14-feature fraud-team-shape event sequence; the test-only `legacy_apply_event_to_aggregations` oracle allocates a fresh per-descriptor `ExtractedFields` (no thread-local, no event-level hoist) so the comparison exercises an INDEPENDENT codepath, not the same code with different inputs.
- Task 4.5: Measurement gate — criterion + live trace + EPS; HALT if 75% floor missed (-900 ns / 99k EPS).

Per CONTEXT D-02 + memory `project_no_same_key_batching` + memory `project_no_sharded_apply`: NO same-key batching, NO multi-thread changes; this is a structural hoist, not a rewrite.

Purpose: Fourth lever in the Phase 19.4 sequential 100k-EPS push (D-01 in `19.4-CONTEXT.md`); the largest predicted-lift plan in 19.4 and the final code-changing lever before the verification phase (Plan 19.4-05).
Output: Event-level ExtractedFields hoist + field-union resolver + bit-identical regression test + measurement gate against 75% floor.

**Target shape (post-19.4-04):**
```rust
fn apply_event_to_aggregations(...) {
    // Per-thread local Vec; reused across events. Single-thread apply forever per
    // memory `project_no_sharded_apply` makes thread_local! reuse safe.
    EXTRACTED_BUF.with(|cell| {
        let mut extracted = cell.borrow_mut();
        let union_field_names = &source_event_descriptor.apply_field_names;  // populated at register
        let union_size = union_field_names.len();
        extracted.clear();
        extracted.resize_with(union_size, || None);
        for (idx, field_name) in union_field_names.iter().enumerate() {
            extracted[idx] = row.get(field_name.as_str());        // ONE SCAN OVER UNION
        }
        #[cfg(test)]
        EXTRACTED_BUILD_COUNT.fetch_add(1, std::sync::atomic::Ordering::Relaxed);

        for desc in event_descriptors_for_source {
            for (i, feat) in desc.features.iter().enumerate() {
                // Re-resolve pre_val via the agg's field_idx_into_event_extracted mapping;
                // the existing `update_with_extracted` signature accepts &ExtractedFields
                // and is unchanged — just receives the hoisted union slice now.
                let pre_val = if feat.descriptor.field_idx != FIELD_IDX_NONE {
                    let union_idx = feat.descriptor.field_idx_into_event_extracted
                        [feat.descriptor.field_idx as usize];
                    extracted.get(union_idx as usize).copied().flatten()
                } else { None };
                entity_row[i].update_with_extracted(
                    pre_val, event_time_ms, feat.descriptor.where_expr.as_ref(),
                    row, feat.descriptor.field.as_deref(), feat.descriptor.field_idx,
                    &extracted,
                    feat.descriptor.ext.lat_idx,    // already maps into the union (see below)
                    feat.descriptor.ext.lon_idx,
                );
            }
        }
    });
}
```

Note: `lat_idx`/`lon_idx` (resolved by Plan 19.4-03) ALREADY reference per-agg `field_names` indices. Under the hoist they must instead reference the per-source `apply_field_names` union indices. The Plan 19.4-03 resolver patch must be re-pointed to use `apply_field_names` (the per-source union), not the per-agg list. Since the per-agg `field_names` is a SUBSET of the per-source `apply_field_names` union, the mapping is straightforward: for each agg-local idx `i`, the union idx is `field_idx_into_event_extracted[i]`.

**Why per-source field-union scope (per CONTEXT D-02):**
- fraud-team has 1 dominant source (Txn) with ~12 distinct fields across all 88 features
- Per-source-schema (one union per Source) is functionally equivalent and adds zero source-dispatch overhead beyond what already exists
- Per-cluster (5 separate ExtractedFields) doesn't capture full predicted savings — still 5× rebuild per event
- Wasted slots ~30 unused entries vs 12 used; indexed access is 1-2 ns; waste is negligible

**Why per-thread local Vec (per RESEARCH Q1):**
- Single-threaded apply forever (memory `project_no_sharded_apply`) — `thread_local!` reuse is safe
- `clear() + resize_with(N, || None)` zero-allocates after the first event grows the SmallVec to its inline cap (cap=16 from Plan 19.4-02)
- The lifetime `'a` of `ExtractedFields<'a>` is constrained per-event; the thread_local stores a `'static`-bound buffer and is re-typed via a transient mut borrow per call. (Implementer choice: store as `Vec<Option<&'static Value>>` with a SAFETY comment explaining the per-event re-borrow, OR store as plain `Vec<Option<crate::row::Value>>` if cloning is acceptable. Pick the former; row Values live for the apply call duration; the thread_local is cleared at function entry.)

**Bit-identity criterion (Task 4.4) — STRICTER per checker feedback:**
- Deterministic 14-feature fraud-team-shape event sequence, 1000 events, seeded.
- For every numeric agg state field (Count, Sum, etc.): compare `f64::to_bits()` exactly between two parallel state machines.
- The "legacy" path is `legacy_apply_event_to_aggregations` — a `#[cfg(test)]` function that allocates a FRESH `ExtractedFields` (`SmallVec::new()`, NOT borrowed from the per-thread `EXTRACTED_BUF`) per descriptor, populates it from `desc.field_names` (NOT `apply_field_names`), and dispatches via the existing `update_with_extracted`. This means:
  - The legacy path does NOT touch `EXTRACTED_BUF` (different storage)
  - The legacy path uses the per-agg `field_names` list (different field union scope)
  - The legacy path allocates fresh `SmallVec` per descriptor (different allocation pattern)
  These three independent codepath differences make the `f64::to_bits()` strict equality a meaningful regression check, NOT a tautology.
