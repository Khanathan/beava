## StoreView Enum Design

```rust
pub enum StoreView<'a> {
    Legacy(&'a crate::state::store::StateStore),
    Sharded(&'a mut crate::shard::Shard),
}

impl<'a> StoreView<'a> {
    /// get_or_create_entity: returns &mut EntityState for mutation
    /// For Legacy: delegates to DashMap RefMut (via a local shim)
    /// For Sharded: uses AHashMap::entry().or_default()
    pub fn get_or_create_entity_mut<F, R>(&mut self, key: &str, f: F) -> R
    where F: FnOnce(&mut crate::state::store::EntityState) -> R;

    /// get_entity: read-only lookup
    /// For Legacy: delegates to DashMap Ref guard
    /// For Sharded: returns reference into AHashMap
    pub fn get_entity(&self, key: &str) -> Option<&crate::state::store::EntityState>;
}
```

Note: Because `get_or_create_entity` on StateStore returns a DashMap RefMut guard
(which borrows from the DashMap), we cannot return `&mut EntityState` with the same
lifetime as a generic reference. The cleanest approach is a closure-based API
`with_entity_mut(key, |entity| { ... })` that lets each arm manage its own borrow
scope. Alternatively, refactor `push_internal` to accept an explicit `&mut EntityState`
after the lookup, keeping the guard out of the signature. Both approaches avoid
introducing a trait object.
