# Phase 50.5 Plan 01: Shard Thread Completion — Cascade Wiring Summary

Wires the shard thread as the exclusive entity-state owner at N>1, replacing the fire-and-forget stub with a real cascade push against the per-shard AHashMap. The legacy DashMap `StateStore` is no longer written on the N>1 path.

## What Was Built

**Task 1 (Wave 0 grep-and-count):** Counted store method call sites in `push_internal` and `push_with_cascade_internal`. Found 4 call sites, 2 distinct methods (`get_or_create_entity`, `get_entity`). Selected enum shape (StoreView) over trait shim per the <5 threshold. Result documented in `50.5-01-CASCADE-SHAPE.md`.

**Task 2 (Contract + RED):** Widened `ShardResult::Ok` to carry `FeatureMap`. Updated `spawn_shard_threads` to accept `Arc<ConcurrentAppState>`. Added `push_with_cascade_on_shard` stub to PipelineEngine. Added `StoreView<'a>` enum with `unimplemented!()` bodies. Wrote 4 failing integration tests (all assert DashMap empty at N>1, which fails before Task 3).

**Task 3 (GREEN):**
- `StoreView::with_entity_mut`: DashMap arm uses `store.get_or_create_entity(key)` guard, Sharded arm uses `shard.state.entry(key).or_default()`
- `StoreView::get_entity_ref`: DashMap arm uses `store.get_entity(key).map(...)`, Sharded arm uses `shard.state.get(key).map(...)`
- `push_internal_on_shard`: parallel to `push_internal` but writes to `shard.state: AHashMap<EntityKey, EntityState>` with no DashMap lock
- `push_with_cascade_on_shard`: full cascade walk (leaf-stream fast path, EnrichFromTable read-only lookup, StreamStreamJoin buffer, keyed/keyless downstream push) all using `push_internal_on_shard`
- `shard_event_loop`: parses JSON bytes, calls `push_with_cascade_on_shard`, advances `shard.watermark`, emits metrics, sends `ShardResult` via oneshot
- `handle_push_core_ex` at N>1: computes shard_hint, sends to SPSC inbox, returns early with empty FeatureMap (no DashMap write). Inbox-full → `BeavaError::Protocol` returned to client

**3. [Rule 2 - Missing critical functionality] push_internal_on_shard instead of StoreView refactor**
- **Found during:** Task 3 analysis
- **Issue:** Refactoring `push_internal` to take `StoreView` would require changing the DashMap RefMut guard lifetime across all call sites (complex borrow checker issue)
- **Fix:** Added parallel `push_internal_on_shard` that takes `&mut Shard` directly (AHashMap `entry().or_default()` returns `&mut EntityState` — same type as DashMap guard dereference, but no guard needed)
- **Impact:** StoreView enum still declared and its method bodies implemented (usable for future refactors), but `push_with_cascade_on_shard` uses `push_internal_on_shard` directly
