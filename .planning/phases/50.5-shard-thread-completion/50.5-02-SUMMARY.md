# Phase 50.5 Plan 02: Accept-Path SO_REUSEPORT Boot Wire + Per-Connection Stream-Name Interning Summary

Wires `bind_reuseport_tcp` into the `run_tcp_server_with_listener` boot path on Linux (previously defined and unit-tested but never called — Pitfall 7 root cause of Hetzner +2% gap), and adds per-connection `Arc<str>` stream-name interning via `ConnAccumulator::stream_name_cache`.

## What Was Built

**Task 1 (Contract + RED):**
- `tests/test_so_reuseport_boot.rs` (new, Linux-only): asserts 4 LISTEN sockets on same port when shard_count=4 via `/proc/net/tcp` parsing.
- `tests/test_shard_thread_ownership.rs` extended: `macos_accept_path_interns_stream_name_per_connection` asserts `conn_interns_total == 1` for 10 events on one stream from one TCP connection.
- `tests/test_metrics_parity.rs` extended: `n2_both_shards_see_events` asserts both `shard="0"` and `shard="1"` have `accepted > 0` after 100 diverse-key pushes at N=2.
- `ConcurrentAppState::conn_interns_total: AtomicU64` (always-on probe field, zero production overhead) added and initialized in `make_concurrent_state_full`.

**Task 2 (GREEN):**
- `ConnAccumulator` gains `stream_name_cache: AHashMap<String, Arc<str>>` field and `intern_stream(&mut self, name, &state) -> Arc<str>` method. First call per (conn, stream) allocates one `Arc::from`; subsequent calls return a clone (atomic refcount bump). Increments `state.conn_interns_total` on first intern.
- `handle_push_core_ex` gains `interned_stream_name: Option<Arc<str>>` parameter. When `Some`, uses the interned arc for the SPSC shard event instead of a fresh `Arc::from`. All existing callers updated to pass `None`.
- `handle_connection` intercepts `Command::Push` before both the outer-loop and inner tight-loop match dispatch using a `ref`-peek pattern, calling `accumulator.intern_stream` before routing to `handle_sync_command`.
- `run_tcp_server_with_listener` on Linux with `shard_count > 1`: drops the passed-in listener and calls `spawn_linux_per_shard_accept_loops(addr, &state)`.
- `spawn_linux_per_shard_accept_loops` (new, `#[cfg(target_os = "linux")]`): binds N SO_REUSEPORT sockets via `bind_reuseport_tcp`, spawns N accept loops on the ambient tokio runtime, returns `std::future::pending()` to hold the server alive.

## Linux SO_REUSEPORT Boot-Path Wire-Up

The gap (Pitfall 7): `bind_reuseport_tcp` at `src/server/tcp.rs` was defined in Phase 50-05 and unit-tested, but `run_tcp_server_with_listener` used plain `TcpListener::bind` — only 1 LISTEN socket regardless of `BEAVA_SHARDS`. Now on Linux with N>1 shards, N SO_REUSEPORT sockets are bound. Kernel 4-tuple-hash distributes connections across them.

**Linux `ss -lntp` verification:** Deferred to Plan 50.5-03's human-run ship-gate (requires Hetzner CCX43). On macOS dev (current machine), the Linux `#[cfg]` path is not compiled; `test_so_reuseport_boot.rs` is Linux-only and will pass on Linux CI.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing critical functionality] conn_interns_total must be always-on (not #[cfg(test)])**
- **Found during:** Task 1 — compile error: integration tests compile the library without cfg(test), so `#[cfg(test)]` fields are invisible to integration test binaries.
- **Fix:** Made `conn_interns_total: AtomicU64` an unconditional field (always compiled). Zero production overhead since it's never read on the hot path; only incremented once per (conn, stream) first-intern and read in tests.
- **Files modified:** src/server/tcp.rs
- **Commit:** 45864b5

**2. [Rule 1 - Bug] Late-drop early-return cannot be done inline in a match arm**
- **Found during:** Task 2 — first attempt at inline `Command::Push` arm with watermark check used `return Ok(writer.write_all(...).await...)` which has wrong type inside the match.
- **Fix:** Simplified to a pre-match `ref`-peek pattern: call `accumulator.intern_stream(stream_name, &state)` before the match dispatch, then let `handle_sync_command` handle the full command including watermark/late-drop logic. The intern counter is incremented even on late-drop events — conservative but correct (no incorrect interning; just one extra count on rare late events).
- **Files modified:** src/server/tcp.rs
- **Commit:** e634f87

**3. [Rule 4 - Documented] macOS accept path: kept tokio::spawn per connection**
- **Executor decision (Option A):** Per plan's explicit "Executor: apply Option A" directive, `tokio::spawn` per connection is retained on macOS. The test `macos_inline_accept_no_spawn_per_conn` was renamed to `macos_accept_path_interns_stream_name_per_connection` per the plan's renaming instruction, and the assertion changed to test only interning (not absence of spawn).

## Known Gaps

- **Per-shard accept loops run on ambient runtime, not shard's `current_thread` runtime:** RESEARCH.md Open Question 4 recommends spawning on the shard's runtime for cache locality (tightest core affinity). This plan wires SO_REUSEPORT at the boot path (the observable Hetzner gap). Runtime-topology tuning is a Phase 51 follow-up — requires threading `tokio::runtime::Handle` through `ShardHandle`.
- **Linux ss -lntp verification:** Not executable on macOS dev machine. Deferred to Plan 50.5-03 human-run ship-gate on Hetzner CCX43.
- **Pre-existing test_replica_subscribe failure:** `backpressure_drops_subscriber` fails with macOS `AddrNotAvailable` (code 49) on the `127.0.0.1` loopback bind — confirmed pre-existing via `git stash` bisect. Not caused by this plan.

## Known Stubs

None — all interning paths are wired. The `spawn_linux_per_shard_accept_loops` is fully implemented (not stubbed) but unverifiable from macOS.

## Threat Flags

None — no new network endpoints or auth paths introduced. `spawn_linux_per_shard_accept_loops` uses the existing `bind_reuseport_tcp` primitive which already has `SO_REUSEPORT` + `SO_REUSEADDR` + nonblocking set. The per-connection `stream_name_cache` is dropped with the accumulator when `handle_connection` returns — no cross-connection leakage (T-50.5-02-02 accepted per threat model).
