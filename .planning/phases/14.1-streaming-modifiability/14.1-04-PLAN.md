<objective>
Redesign every Tier 3 operator's state so it can be reconstructed from a
`&[ModLogEntry]` slice. Ship a generic default implementation
(`Self::default()` + loop over log calling existing `update()`) plus per-op
overrides only where benchmarks show ≥ 2× speedup. Expected net result for
14.1: zero custom overrides land; the generic default is sufficient everywhere.

Ship a proptest `replay_equivalence.rs` that generates random event sequences
and asserts:
- **Tier 1/C ops:** `apply_sorted(events) == replay_from_log(shuffle(events))`
  byte-exact.
- **Tier A ops:** within documented ε per op.
- **Tier B ops:** not tested here (analyzer rejects them at register time;
  runtime uses the generic default for defence-in-depth but we don't
  guarantee semantic equivalence).

Purpose: Plan 05's slow-path calls `AggState::replay_from_log` inside the
apply `borrow_mut`; this plan provides the mechanics.

Output: `AggState::replay_from_log` dispatch method + per-variant helper +
exhaustive proptest.

    Add to `AggState` enum (`agg_op.rs`):
    ```rust
    impl AggState {
        /// Reconstruct state from a sorted slice of modification log entries.
        /// Caller MUST ensure the log is sorted by (event_time_ms, event_id)
        /// ascending (Plan 02 invariant).
        pub fn replay_from_log(
            &self,  // used for type-dispatch; fresh state is built inside
            log: &[ModLogEntry],
            desc: &AggOpDescriptor,
        ) -> Self {
            match self {
                AggState::Count(_) => {
                    let mut s = CountState::default();
                    for e in log {
                        s.update_with_row(&e.row, e.event_time_ms, ...);
                    }
                    AggState::Count(s)
                }
                AggState::Sum(_) => { ... }
                AggState::Ewma(_) => {
                    let mut s = EwmaOp::new_from_desc(desc);
                    for e in log { s.update(&e.row, e.event_time_ms, ...); }
                    AggState::Ewma(s)
                }
                // ... every variant ...
                AggState::Windowed(w) => {
                    // WindowedOp state spans a sliding bucket — insert-replay
                    // for windowed ops requires folding ALL log entries into a
                    // fresh WindowedOp with matching window/bucket params.
                    // Defer to Plan 05 to decide whether windowed ops need
                    // special handling (likely: windowed ops are all Tier C
                    // inside the window, so slow-path replay is redundant for
                    // them — Plan 05 may skip calling replay_from_log on
                    // Windowed variants entirely).
                    let mut fresh = WindowedOp::new_from_desc(desc);
                    for e in log { fresh.apply(&e.row, e.event_time_ms, ...); }
                    AggState::Windowed(Box::new(fresh))
                }
            }
        }
    }
    ```

    Key implementation notes:
    - The generic pattern is "build a fresh state from descriptor, loop update
      over log." Don't try to in-place modify `self` — replay always produces
      a new state.
    - Op-specific params (half_life_ms, buffer capacity, etc.) come from
      `desc`. This may require adding `AggOpDescriptor::half_life_ms()` and
      similar accessors if not already present.
    - For ops whose `update` signature takes a `where_matched: bool`, the log
      entry's row is already pre-filtered (Plan 05 filters at log-insert
      time; confirm in Plan 05); pass `where_matched=true` during replay.
      Alternatively: store `where_matched` in the `ModLogEntry` itself — add
      a `pub where_matched: bool` field in Plan 02 if Plan 05 needs it.
      Resolve during execution; whichever chosen must be consistent with
      how Plan 05 inserts into the log.

    Exhaustive match ensures compile-time coverage.
  </action>
