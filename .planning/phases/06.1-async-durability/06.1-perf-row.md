## Regression-gate call

This is a **new** bench (Phase 6.1 introduces SyncMode::Periodic; no
prior baseline exists in this hw-class). It establishes the baseline
that future phases will regress-check against. Per CLAUDE.md the
10%/25% thresholds apply from this commit forward.

The Phase 6 PerEvent bench (`wal/append_fsync_default_coalesce`)
remains in the codebase (`benches/phase6_wal.rs`) and continues to
report ~7.4 ms — that's the strict-mode baseline used by
`/push-sync`. No regression on the PerEvent path expected (Plan 6.1-01
left PerEvent semantics unchanged).
