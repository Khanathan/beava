<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> REGISTER handler | Untrusted register payload with new backfill field |
| re-registration -> state store | Schema diff must not corrupt existing operator state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | FeatureDefRequest.backfill | mitigate | `#[serde(default)]` defaults to false; `Option<bool>` rejects non-bool via serde deserialization |
| T-08-02 | Denial of Service | Repeated re-registration flooding | accept | Single-threaded lock prevents concurrent re-registration; register() is fast (no I/O); same as existing REGISTER rate |
| T-08-03 | Information Disclosure | SchemaDiff in REGISTER response | accept | Diff only reveals feature names the client already sent; no internal state leaked |
| T-08-04 | Tampering | Type change bypass via re-register | mitigate | `same_operator_type()` using `std::mem::discriminant` rejects type changes; user must remove+add with new name |
| T-08-05 | Denial of Service | Orphan operator memory accumulation | mitigate | Lazy GC during `clone_for_snapshot_with_gc()` filters orphan operators; periodic snapshots bound memory growth |
</threat_model>
