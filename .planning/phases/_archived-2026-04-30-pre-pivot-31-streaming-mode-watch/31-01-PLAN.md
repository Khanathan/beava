<objective>

a two-socket subscribe-first buffered-replay dance that sits on top of
Phase 28-04's historical session and Phase 27's `Scope` codec. No mode state
machine beyond `Subscribing -> BufferedReplay -> Live -> Stopped`. No
`LOG_FETCH`. No `Catchup`. No global `seq` in the event payload.

generator + `tally sync` CLI + `SubscriberDroppedError` on top of what this
plan exports.

  <behavior>
    - `StreamingClient::connect(remote: &str, scope: Scope, token: &str) -> Result<StreamingClient, ClientError>` executes the Option K dance in this exact sequence:
      1. **Subscribing phase.** Open socket #1 (`TcpStream::connect(remote)`, `set_nodelay(true)`). Call `session::subscribe_handshake(&mut sock1, token, &scope)`. `try_clone()` the socket for out-of-band shutdown; store clone on the `StreamingClient` as `socket_shutdown`.
      2. **Start buffer-fill thread.** Spawn a `std::thread::spawn` closure that owns sock1's read half (or the whole sock1 if using `Arc<TcpStream>`), a shared `Arc<Mutex<Vec<BufferedEvent>>>`, the `stop_flag`, the `stop_reason`, and the `last_applied_ts_nanos`. The thread's loop:
         ```
         loop {
             if stop_flag.load(Acquire) { set_stop_reason(UserRequested); return; }
             match read_frame(&mut sock1) {
                 Ok(frame_bytes) => {
                     let ev: BufferedEvent = decode_push_frame(&frame_bytes)?; // { timestamp, stream, key, event }
                     // Phase selector: during Subscribing/BufferedReplay, push to buffer.
                     // During Live, apply directly. Gated by an AtomicU8 mode enum shared with the main thread.
                     match mode.load(Acquire) {
                         MODE_BUFFERING => buffer.lock().push(ev),
                         MODE_LIVE => {
                             store.write().apply_event(&ev.stream, &ev.key, &ev.event);
                             last_applied_ts_nanos.store(to_nanos(ev.timestamp), Release);
                             if let Some(d) = dispatcher.as_ref() {
                                 d.on_applied(ev.timestamp, &ev.stream, &ev.key, &ev.event);
                             }
                         }
                     }
                 }
                 Err(Eof) if !stop_flag.load(Acquire) => {
                     let at_ts = from_nanos(last_applied_ts_nanos.load(Acquire));
                     set_stop_reason(ServerDropped { at_timestamp: at_ts });
                     if let Some(d) = dispatcher.as_ref() { d.on_stopped(&ServerDropped{..}); }
                     return;
                 }
                 Err(Eof) => { set_stop_reason(UserRequested); return; }
                 Err(Io(e)) => {
                     let r = if stop_flag.load(Acquire) { UserRequested } else { Io(e.to_string()) };
                     set_stop_reason(r); return;
                 }
             }
         }
         ```
         Use `sock1.set_read_timeout(Some(Duration::from_millis(500)))` so the loop can periodically observe `stop_flag` even when no events arrive. `WouldBlock` / `TimedOut` → continue.
      3. **Snapshot phase.** Open socket #2 (`TcpStream::connect(remote)`, nodelay). Call `session::fetch_snapshot(&mut sock2, token, &scope)`. Receive `(snapshot_taken_at, BaseSnapshotState)`. Drop sock2 (let go out of scope — its Drop closes the FD).
      4. **BufferedReplay phase.** Acquire `store.write()`. Call `StateStore::bulk_load(snapshot)` under the lock. Drop the lock.
         Then swap the shared buffer out: `let drained: Vec<BufferedEvent> = { let mut g = buffer.lock(); std::mem::take(&mut *g) };` (Any further events the bg thread tries to push during the split will now land in the fresh empty buffer — but we're about to flip the mode atomic to LIVE, so the window is tiny. Document: events that land in `drained` AND are later re-pushed by the bg thread as live events would double-apply. Option K accepts this because `apply_event` operates on an immutable `timestamp` and a deterministic payload — applying the same event twice yields the same state. If a particular event type is non-idempotent, that is a separate v0 issue tracked in the SUMMARY.)
         Sort `drained` by `timestamp`. For each event:
         - If `timestamp <= snapshot_taken_at` → drop.
         - Else → `store.write().apply_event(...)`, update `last_applied_ts_nanos`, and **release the write lock between events** (the explicit `drop(guard)` after each apply, not holding it across the loop) so readers can interleave. Dispatch via `dispatcher.on_applied` if set.
         If `drained.is_empty()` AND the first live event (once we flip to LIVE) has `timestamp <= snapshot_taken_at`, ignore it; the gap-close is handled by timestamp comparison inside the live-apply branch too — add that check for belt-and-suspenders, document in the `<verify>` section.
      5. **Live transition.** `mode.store(MODE_LIVE, Release)`. From here, the background thread applies directly. `connect()` returns `Ok(StreamingClient { apply_handle: Some(handle), .. })`.
