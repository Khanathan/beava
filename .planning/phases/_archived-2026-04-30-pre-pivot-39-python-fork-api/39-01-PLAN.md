# Plan 39-01 — Python `tl.fork()` DX layer

## Goal

Ship a single Python entry point that a data scientist uses to define scope + streams + pipelines and get a queryable local replica — no shell commands, no JSON files.

## Task T1 — Python module + unit tests

**Files:**
- `python/tally/_fork.py` (new) — ~150 lines. Houses `fork()`, `ForkedReplica`, the error hierarchy.
- `python/tally/__init__.py` (modified) — re-export `fork`, `ForkedReplica`, `ForkError`, `ForkTimeoutError`, `ForkSubprocessError`, `ForkValidationError`.
- `python/tests/test_fork_unit.py` (new) — unit tests for validation, serialization, and error paths (no subprocess spawn — mock out `subprocess.Popen`).

**Subtasks:**

1. **Arg validation** (`_validate_fork_args`):
   - `streams` non-empty, every element has the `@tl.stream` marker attribute (same duck-typing `_app.py` does).
   - `keys xor key_prefix` — `ForkValidationError` if both set or neither (with a warn if neither, since that means "all keys for the streams"; actually allow that, just no filter).
   - `token` resolves: arg → `TALLY_REPLICA_TOKEN` env → raise.
   - `pipelines` optional; if present, every element must duck-type to `_to_register_json()`.
   - Streams: reject duplicate class objects.

2. **Pipeline serialization** (`_build_seed_file`):
   - Build REGISTER doc(s). Pattern: for each stream + pipeline, call `_to_register_json()` OR reconstruct the register JSON using whatever the existing `App.register` flow does.
   - **STOP-AND-REPORT** if the existing SDK doesn't expose a clean serialization step that produces the same bytes Phase 36's `seed_pipelines_from_file` consumes. In that case flag the gap — we may need to tweak `_app.py` to factor out the serializer.
   - Write to a temp file, return the path.

3. **Subprocess spawn** (`ForkedReplica._start`):
   - `subprocess.Popen([binary, "fork", "--remote", remote, "--streams", ",".join(stream_names), ...])`.
   - Capture stdout/stderr to temp log files; keep paths on the instance.
   - Compute `local_port` if None by binding to port 0 first, closing, reusing the number.

4. **Readiness polling** (`ForkedReplica._wait_ready`):
   - Loop `GET http://127.0.0.1:{port}/debug/ready` every 200ms up to `ready_timeout`.
   - On timeout, read last 50 lines of stderr log and raise `ForkTimeoutError` with that context.

5. **Query surface**:
   - `.get(pipeline_or_stream, key)`: resolve feature/stream name (`__name__` attribute works for classes; `@tl.table`-decorated function has `_register_name`). Call `GET /debug/key/{key}` and return the `computed_features[feature_name]` subset OR None if absent.
   - `.inspect()`: iterate declared scope keys, call `.get()` for each, return `{stream_name: count_of_keys_seen}`.
   - `.local_url` property.
   - `.stop()`: SIGTERM → 5s wait → SIGKILL. Clean temp files. Idempotent.
   - `__enter__` / `__exit__`.

6. **Unit tests** (mock `subprocess.Popen` + HTTP):
   - Validation happy + each error path.
   - Seed-file writes expected JSON content.
   - Port auto-allocation.
   - `.stop()` is idempotent.
   - Timeout path surfaces stderr tail.

**Done when:** `pytest python/tests/test_fork_unit.py -v` green. `python -c "import tally; tally.fork"` imports without error.

## Task T2 — Python-native E2E integration test

**File:** `python/tests/integration/test_fork_python_api.py` (new).

**Flow:**
1. Start prod server subprocess on a free port (reuse helpers from `test_fork_demo.py` or adapt).
2. Push fixture events via existing Python SDK's `App.push()` or equivalent.
3. Author pipelines purely in Python:
   ```python
   import tally as tl

   @tl.stream
   class Transactions:
       user: str
       amount: float

   @tl.table(depends_on=[Transactions])
   def txn_summary(t):
       return t.group_by("user").agg(
           count=tl.count(),
           total=tl.sum(t.amount),
       )
   ```
4. `with tl.fork(remote=f"127.0.0.1:{prod_port}", streams=[Transactions], keys=["u1","u2"], token="test-token", pipelines=[txn_summary]) as fork:`
5. `assert fork.get(txn_summary, key="u1") == {"count": 3, "total": 60.0}`.
6. Push 2 more events to prod (u1, u3-out-of-scope).
7. Wait 2s.
8. `assert fork.get(txn_summary, key="u1") == {"count": 4, "total": 160.0}` — live tail.
9. `fork.get(txn_summary, key="u3")` — expect None or out-of-scope semantics matching the replica.
10. Exit context → `.stop()` fires.

**Done when:** test passes deterministically (3× consecutive local runs).

## Stop-and-report triggers

- Existing `_to_register_json()` doesn't produce the exact format Phase 36's `seed_pipelines_from_file` reads — report the mismatch, propose either: (a) tweak the serializer, (b) tweak the reader, (c) adjust plan to bundle differently. Don't guess.
- `tl.table`-decorated functions don't expose a stable `_register_name` or similar handle to identify the feature — report what's there.
- Test flakiness (e.g., `/debug/ready` returns 200 but a moment later the query returns stale data). Document the retry pattern.
- Existing `tl.fork` name conflicts with something (e.g., git fork helper). Pick `tl.replica_fork` or `tl.Fork` if needed and flag.
