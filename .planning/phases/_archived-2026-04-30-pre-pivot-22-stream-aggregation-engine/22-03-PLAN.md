# PLAN 22-03: Hybrid sketch operators + performance gate

## Goal

Ship the three sketch-backed hybrid operators — `percentile` (exact sorted Vec → UDDSketch), `count_distinct` (exact HashSet → per-bucket HLL), `top_k` (exact HashMap → CMS+heap) — with correct exact→sketch transition, ring-buffer retraction where applicable (UDDSketch has `decrement()`; HLL and CMS are bucket-granular), and full observability via `/debug/key/:key`. Close the phase by running the full benchmark matrix; if any cell regresses > 5% vs the v2.0 baseline, the phase does not close. This plan is non-autonomous — it ends with a checkpoint that pauses for the user to review the benchmark results.

## Success Criteria

1. **UDDSketch vendored + patched.** `src/engine/uddsketch.rs` contains the Timescale-derived UDDSketch with a new `decrement(value: f64)` method (saturates at 0). `current_alpha()` exposed for telemetry. Underflow test passes.
2. **CMS in-house.** `src/engine/cms.rs` implements w=2048, d=4 Count-Min Sketch with signed-int counters (i32), supporting `insert(value, count)` and `decrement(value, count)`. 4 pairwise-independent hash functions seeded deterministically.
3. **TopKHeap.** Paired with CMS: binary heap of `(estimated_count, value)` bounded at k. On insert: update CMS, then if new value's CMS estimate > heap's min, push and pop. On decrement: adjust CMS; rebuild the heap from the CMS estimates of the current candidate set (candidates = union of heap items across the last ~k insert rounds; see cms.rs doc-comment for the exact heuristic).
4. **HLL reuse.** `src/engine/hll.rs` already exists and contains HyperLogLog++ at precision=14. Reuse for sketch-mode count_distinct. For bucket-granular retraction: per-bucket mini-HLL at precision=10 (1KB each × 60 buckets = ~60KB per feature per entity — within budget); merge across buckets on read; discard whole mini-HLL on bucket expiry.
5. **Exact→sketch transition (one-way).**
   - `percentile`: transition at 257th event. At transition: construct UDDSketch with α=0.01, insert every element from the exact sorted Vec in order, clear the exact Vec, flip mode. Retention list (per-bucket Vec<f64> of raw values) begins tracking from transition onward (for retract on bucket expiry).
   - `count_distinct`: transition at 1025th unique value. Insert every existing HashSet member into per-bucket mini-HLLs (routing by the bucket each value originally landed in — requires the exact mode to track per-bucket HashSets, not a single flat HashSet; trade-off for correct retraction).
   - `top_k`: transition at 1025th unique value. Seed CMS with all counts from the exact HashMap; seed heap with top-k keys by count; clear the HashMap.
6. **Retraction on bucket expiry.**
   - `percentile` sketch mode: per-bucket retention list holds every value in bucket; on bucket expiry, iterate and `UDDSketch.decrement(v)` for each. Retention list is the memory cost (~2× sketch memory during window lifetime; released on expiry).
   - `count_distinct`: bucket-granular — mini-HLL per bucket; merge on read; drop bucket's HLL on expiry.
   - `top_k` sketch mode: bucket-granular — per-bucket CMS + global heap; on bucket expiry, CMS.decrement each value in the bucket's retention list; rebuild heap from candidates.
7. **Observability.** `/debug/key/:key` JSON includes per feature:
   ```json
   { "name": "p95_amount", "op": "percentile", "mode": "sketch",
     "exact_count": 256, "transition_at": 256, "sketch_alpha_current": 0.015,
     "memory_bytes": 2408 }
   ```
   Extend the `Operator` trait with `fn hybrid_telemetry(&self) -> Option<HybridTelemetry>` (default `None`); override for the three hybrid ops.
8. **Benchmark matrix passes.** `benches/matrix.rs` (or a shell script driving `bench.py`) runs 9 cells: {small 10k keys / medium 100k / large 1M} × {1 core / 4 cores / 8 cores}. Each cell is compared against the recorded baseline from 19-05 (1.1M eps on the reference cell). Any cell regressing > 5% **fails the build**. Baseline numbers captured in `.planning/phases/22-stream-aggregation-engine/BASELINE.json` and committed before running.
9. **Final checkpoint.** After all tests + the benchmark matrix pass, pause for user review. Show the full matrix table in the checkpoint so the user can sign off on the regression posture.

## Implementation Steps

1. **Vendor UDDSketch.**
   - Copy the Timescale Rust port's core algorithm into `src/engine/uddsketch.rs`. Keep the file self-contained — no crate dependency.
   - Add `fn decrement(&mut self, value: f64)`: mirror the insert logic; saturate bucket counter at 0 on underflow; track `total_count` and saturate that too.
   - Add `fn current_alpha(&self) -> f64`: after N collapses, α′ = 2α / (1+α²) applied N times starting from α₀=0.01. Store the current α on the struct rather than recomputing.
   - Add `fn estimated_bytes(&self) -> usize`.
   - Unit-test: insert 10k gaussian samples, delete half, query quantile — should be within α′ of the ground-truth on the surviving half.
2. **Implement CMS + heap.**
   - `CountMinSketch { counters: [[i32; W]; D] }` where W=2048, D=4. Hash functions: four MurmurHash3 seeds fixed via `const`. Chosen for simplicity, pairwise-independence is the only requirement.
   - `insert(hash: u64, delta: i32)` updates all D rows at column `(hash rotated per-row) % W`.
   - `estimate(hash: u64) -> i32` = min across D rows.
   - `decrement(hash, delta)` same as insert with negative delta (saturate at 0 per row).
   - `TopKHeap { heap: BinaryHeap<Reverse<(i32, Value)>>, k: usize, candidates: AHashSet<Value> }`. Push on insert if new estimate > heap.peek() or len < k. Rebuild on decrement: query CMS for every `candidate`, reinsert into a fresh heap bounded at k.
3. **PercentileOp.**
   - State: `{ mode: PercentileMode, retention: RingBuffer<Vec<f64>>, threshold: usize, alpha: f64 }`.
   - `PercentileMode::Exact { values: Vec<f64> }` — keep sorted (binary-search insert) so read is O(1) at `values[(q*(n-1)) as usize]`.
   - `PercentileMode::Sketch { sketch: UDDSketch }`.
   - `push(event, enr, now)`: extract field; advance ring buffer; push to current bucket's retention list; if Exact: sorted-insert + check threshold → if exceeded, transition (construct UDDSketch, insert every existing value, swap mode). If Sketch: `sketch.insert(value)`.
   - On bucket expiry (detected in `advance_to`): if Sketch mode, drain the expiring bucket's retention list and `sketch.decrement(v)` for each. If Exact mode: remove each expiring value from the sorted Vec (binary-search find + Vec::remove — acceptable because pre-transition size ≤ 256).
   - `read`: exact → `values[floor(q*(n-1))]`; sketch → `sketch.quantile(q)`.
   - `hybrid_telemetry`: always returns Some with current mode + counts + alpha (if sketch) + estimated_bytes.
4. **CountDistinctOp.**
   - State: `{ mode, threshold, bucket_mini_hlls: RingBuffer<MiniHll> }` (mini-HLL precision=10 always present; used in both modes as the retraction primitive after transition).
   - Wait — re-read 22-CONTEXT.md decision: exact→sketch at 1024 unique. Exact mode uses per-bucket `AHashSet<Value>` (so per-bucket granularity is preserved). Sketch mode uses per-bucket mini-HLL.
   - Simplest state: `PerBucketExact(RingBuffer<AHashSet<Value>>)` → `PerBucketSketch(RingBuffer<MiniHll>)`. The union cardinality across buckets is the reading.
   - On transition: for each non-expired bucket, convert the bucket's HashSet to a mini-HLL by inserting each member.
   - `read`: exact → cardinality of union of buckets' HashSets; sketch → merge all buckets' mini-HLLs, return count.
   - Bucket expiry: drop the bucket (both modes).
5. **TopKOp.**
   - State: `{ mode, threshold, k, bucket_retention: RingBuffer<Vec<Value>>, sketch: Option<(CountMinSketch, TopKHeap)> }`.
   - Exact mode: `AHashMap<Value, u64>` counts across all buckets; per-bucket retention list exists from day one so transition can bucket values correctly.
   - Transition at 1025th unique: construct CMS, insert each `(value, count)` from the exact map; construct heap; flip mode.
   - On push: update exact map or CMS+heap; push to current bucket's retention.
   - On bucket expiry: for each value in expiring bucket's retention: exact map → decrement (remove if 0); sketch → CMS.decrement, then rebuild heap from candidates.
   - `read` → list of (value, count) pairs length k.
6. **Plumb telemetry.**
   - Extend `Operator` trait with `fn hybrid_telemetry(&self) -> Option<HybridTelemetry> { None }`.
   - `HybridTelemetry` struct in `src/engine/operators.rs`: `{ op: String, mode: &'static str, exact_count: usize, transition_at: usize, sketch_alpha_current: Option<f64>, memory_bytes: usize }`.
   - `/debug/key/:key` handler in `src/server/http.rs` iterates the entity's operators, calls `hybrid_telemetry`, and includes the result in the JSON response.
7. **Snapshot serde.** Three new `Serializable*` structs for the three hybrid ops. UDDSketch serde via `#[derive(Serialize, Deserialize)]` on its internal representation (counters map). CMS: 2D counter array serializes directly. Round-trip tests in `test_snapshot_v0_ops.rs` (from 22-02) extended.
8. **Tests.**
   - `test_percentile_hybrid.rs`: exact correctness on [0..256]; transition at 257; sketch correctness on 10k uniform; decrement saturation at 0; bucket-expiry retraction preserves the remaining sketch.
   - `test_count_distinct_hybrid.rs`: exact on 1024 unique; transition at 1025; sketch on 100k unique within ±2%; bucket expiry drops the bucket's HLL.
   - `test_top_k_hybrid.rs`: exact heavy-hitters on 1024 unique; transition; sketch top-k matches true top-k on Zipf distribution (top-10 recall ≥ 0.9).
   - `test_hybrid_transitions.rs`: assert that `{percentile, count_distinct, top_k}` all produce values within tolerance *immediately after* transition (no "lost" values). Record `memory_bytes` before/after transition; expect a jump at transition (exact state freed) then steady state.
9. **Baseline capture.** Before running the matrix, commit `BASELINE.json`:

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Benchmark regression > 5% (most likely cause: per-bucket retention list doubles memory pressure on percentile) | (a) Profile with `perf` / `cargo flamegraph`; (b) reduce retention to `SmallVec<[f64; 16]>` inline for small buckets; (c) if still red, downgrade sketch_mode threshold to 512 to reduce time spent in expensive mode. Escalate to user via the final checkpoint if still regressing. |
| UDDSketch's `decrement` mis-drifts α on heavy retract workloads (literature-confirmed: α drifts under collapse, not under decrement, but implementations sometimes conflate) | Dedicated test: 100k insert, 100k decrement (same set) → cardinality back to 0, quantile returns NaN (documented). Interleaved insert/decrement test: 10k insert, 1k decrement, 10k more insert → quantile within α of ground truth. |
| CMS produces over-estimates for rare items, heap fills with non-hitters | Pre-filter: only push to heap if CMS estimate > per-feature `min_count_to_consider` threshold (default: `total_count / (2k)`). Documented as a tuning knob in the AggOp (future — for now, hard-coded). |
| Per-bucket mini-HLL at precision=10 under-counts on small buckets | Use HLL's small-range bias correction (already in `hll.rs`). Sparse representation for buckets with < 32 distinct values. |
| CMS hash-function collisions bias top-k on adversarial inputs | Seeds chosen via `const` rather than `rand` so attacks need offline hash-collision finding; not in v0 threat model beyond accepting T-22-03. |
| Snapshot size balloons because of per-bucket retention lists | Skip retention lists from snapshots (reconstructable: after snapshot reload, the sketch reflects all values since startup, retention starts empty — so retract-on-bucket-expiry is correct for values inserted *after* reload; values from before reload get dropped from the sketch on expiry, which is wrong but bounded). Document this in snapshot.rs. Phase 25 (TTL/warnings) may surface this. Accept for v0. |
| Final checkpoint blocks for too long | If user doesn't respond in 24h, executor should post a best-estimate summary and ping. Not in scope for this plan — covered by the GSD execution loop. |
