## Decisions Made

- **D-01 enforced via `windows(4).any(|w| w == b"\r\n\r\n")`:** scan the whole buffer for the boundary marker on each parse-attempt. This is O(buf.len()) per attempt, but the buffer is bounded by the Content-Length cap = 4 MiB. The httparse library would also accept `Status::Partial` to signal "no header-end yet", but a direct scan is simpler and decouples the cap-check from the httparse invocation.
- **Cap-check ordering:** the new boundary check runs BEFORE `httparse::parse` is invoked. This preserves the original spirit of the early-return — we never hand a 4-MiB buffer to httparse for header parsing if the header bytes alone exceed 8 KiB.
- **No env-tunables:** D-04 deferred. 4 MiB body cap covers fraud-team.json (15 KiB) by 250×. If a user reports needing more, revisit then.

## Deviations from Plan
