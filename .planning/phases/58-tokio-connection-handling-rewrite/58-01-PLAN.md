<objective>
Wave 1 replaces the Linux TCP PUSH accept pattern ("single accept loop OR
spawn_linux_per_shard_accept_loops that tokio::spawns per conn") with a per-shard
accept loop owned by the shard thread's own `current_thread` tokio runtime, driven
by `FuturesUnordered` — NO `tokio::spawn` per connection. Each Linux shard thread binds
its own `TcpListener` via `bind_reuseport_tcp` (existing Phase 50 helper, REUSED not
re-implemented) on the public port; the kernel's 4-tuple hash distributes connections.

Purpose: Satisfies D-A1/D-A2/D-A3/D-A4. Eliminates the per-connection tokio task churn
measured at 25–40% CPU in Phase 54 pprof / ~60% of leaf samples. Flips the Linux half
of Wave 0's two RED tests to GREEN.

Output: Production Linux TCP PUSH path now runs inline per-shard with no tokio::spawn.
macOS preserves its Phase 50.5-02 behavior in this wave (Wave 2 rewrites macOS). HTTP
path untouched.
</objective>

            .build()

        rt.block_on(async move {
            // Phase 58-01 D-A1/D-A2: per-shard SO_REUSEPORT accept loop on Linux.
            #[cfg(target_os = "linux")]
            let mut accept_state: Option<PerShardAcceptState> = match &accept_cfg {
                Some(cfg) => Some(PerShardAcceptState::build(shard_index, cfg)?),

            // Existing ShardOp inbox drain loop gets wrapped in tokio::select! with
            // the accept future + FuturesUnordered drain. If accept_state is None,
            // the select! arms for accept are no-op (future::pending).
            loop {
                #[cfg(target_os = "linux")]
                if let Some(ref mut acc) = accept_state {

                        biased;

                        // 1. Drain in-flight connection futures (no tokio::spawn; D-A3).
                        maybe_done = acc.inflight.next(), if !acc.inflight.is_empty() => {
                            let _ = maybe_done; // connection completed/errored; counters updated in handle_connection via existing observability

                                    let st = state.clone();

                                        let _ = crate::server::tcp::handle_connection(stream, st).await;
                                    }));
                                }
                                Err(_) => {
                                    // EMFILE/ENFILE etc — continue; other N-1 shards unaffected.
                                }
                            }
                        }

                        // 3. Pump the existing ShardOp inbox (try_recv to avoid blocking).
                        //    Fall through to process one ShardOp per tick.
                        _ = tokio::time::sleep(std::time::Duration::from_micros(50)) => {}
                    }
                }

                // Existing ShardOp drain — identical to pre-Phase-58 body.
                while let Ok(op) = rx.try_recv() {
                    // ... existing ShardOp dispatch (unchanged) ...
                }

                // ... existing idle/timeout handling (unchanged) ...
            }
        });

        shard_index: usize,
        listener: tokio::net::TcpListener,
        inflight: futures::stream::FuturesUnordered<std::pin::Pin<Box<dyn std::future::Future<Output = ()> + Send>>>,
