<objective>
Wave 2 rewrites the macOS TCP PUSH accept path to match Linux semantics: each shard
gets a dedicated `std::thread` running a blocking `TcpListener::accept` loop (D-B1),
and per-connection PUSH dispatch runs blocking (no tokio per-conn task). Decoded
events are handed off via the existing SPSC inbox primitive that Phase 50 already
uses — no new channel plumbing.

Purpose: Satisfies D-B1 (dedicated-thread-per-shard macOS default) and D-B2
(`BEAVA_SHARDS_SINGLE_LISTENER=1` single-listener fallback — preserves Phase 50.5
behavior as an escape hatch). Flips the macOS half of Wave 0's RED tests to GREEN.
Keeps HTTP untouched (D-B3).

Why blocking on macOS: SO_REUSEPORT is BSD-compat-but-unreliable on Darwin (connections
pile on one socket); a dedicated `std::thread::accept` per shard gives real per-CPU
accept parallelism without depending on kernel hashing. The existing per-shard thread
already runs blocking code between tokio ticks — adding a sibling accept thread that
feeds its inbox is additive.

Output: Production macOS TCP PUSH path runs one accept thread per shard (default) or
one round-robin thread (fallback). Tokio runtime on macOS is used only for HTTP (axum).
</objective>

    ```rust
    #[cfg(not(target_os = "linux"))]
    struct MacosConnSlot {
        inflight: std::sync::Arc<std::sync::atomic::AtomicUsize>,
    }

    #[cfg(not(target_os = "linux"))]
    impl MacosConnSlot {
        fn try_acquire(inflight: &std::sync::Arc<std::sync::atomic::AtomicUsize>, cap: usize) -> Option<Self> {
            // CAS loop: read current, if >= cap return None, else CAS-incr.
            loop {
                let cur = inflight.load(std::sync::atomic::Ordering::Acquire);
                if cur >= cap { return None; }
                if inflight.compare_exchange(cur, cur + 1, std::sync::atomic::Ordering::AcqRel, std::sync::atomic::Ordering::Acquire).is_ok() {
                    return Some(Self { inflight: std::sync::Arc::clone(inflight) });
                }
            }
        }
    }

    #[cfg(not(target_os = "linux"))]
    impl Drop for MacosConnSlot {
        fn drop(&mut self) {
            self.inflight.fetch_sub(1, std::sync::atomic::Ordering::AcqRel);
        }
    }
    ```

    ```rust
    #[cfg(not(target_os = "linux"))]
    pub fn spawn_macos_per_shard_accept_threads(
        accept_addr: std::net::SocketAddr,
        shard_handles: Vec<ShardHandle>,
        state: std::sync::Arc<ConcurrentAppState>,
        max_conns_per_shard: usize,
    ) -> std::io::Result<Vec<std::thread::JoinHandle<()>>> {
        let shard_count = shard_handles.len();
        let mut threads = Vec::with_capacity(shard_count);
        for shard_index in 0..shard_count {
            let listener = std::net::TcpListener::bind(accept_addr)?;
            listener.set_nonblocking(false)?;  // blocking accept (D-B1)
            let inflight = std::sync::Arc::new(std::sync::atomic::AtomicUsize::new(0));
            let state = std::sync::Arc::clone(&state);
            let t = std::thread::Builder::new()
                .name(format!("beava-accept-{shard_index}"))
                .spawn(move || {
                    state.accept_threads_spawned_total.fetch_add(1, Ordering::Relaxed);
                    loop {
                        match listener.accept() {
                            Ok((stream, _peer)) => {
                                // Enforce BEAVA_MAX_CONNS_PER_SHARD via MacosConnSlot RAII.
                                match MacosConnSlot::try_acquire(&inflight, max_conns_per_shard) {
                                    Some(slot) => {
                                        let state = std::sync::Arc::clone(&state);
                                        // Worker thread per connection: bounded via slot.
                                        let _ = std::thread::Builder::new()
                                            .name(format!("beava-conn-{shard_index}"))
                                            .spawn(move || {
                                                let _ = handle_connection_blocking(stream, state, shard_index, slot);
                                            });
                                    }
                                    None => {
                                        // At cap: write SHARD_OVERLOAD and close.
                                        let mut s = stream;
                                        let _ = std::io::Write::write_all(&mut s, &[0x10]);
                                        drop(s);
                                    }
                                }
                            }
                            Err(_) => continue, // EMFILE etc — log-and-continue.
                        }
                    }
                })?;
            threads.push(t);
        }
        Ok(threads)
    }
    ```

    **Important:** The accept thread spawns a worker `std::thread` per accepted
    connection, cap-protected by `MacosConnSlot`. This is NOT per-connection
    tokio task churn — std::thread spawn is rare (connections are long-lived, typically
    1 per client for the lifetime of the session). Per-event cost is zero — the worker
    thread runs a blocking frame-read loop for the life of the connection. This matches
    D-B1's "dedicated thread owns the accept loop" + natural per-connection worker model.
