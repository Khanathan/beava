<objective>
Land `OP_SNAPSHOT_FETCH` (0x12): the "easiest for v0 and demo" snapshot endpoint. In-memory
filter of the current `BaseSnapshotState.entities` by scope, wrapped in a header +
payload frame pair. Also deliver the shared `Scope` struct, its wire codec, and the scope
validator — `27-02` reuses all three for `OP_SUBSCRIBE`.

Purpose: Deliver the "historical" half of the Phase 31 client's subscribe-first
buffered-replay dance. No new persistence format, no streaming snapshot reader, no global
seq counter. The snapshot is one postcard blob today and we keep it that way.

Output: Two Rust source files (new `replica.rs` module + modifications to `protocol.rs`
and `tcp.rs`), one Rust integration test, one Python asyncio integration test.
</objective>
