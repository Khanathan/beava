<objective>
Land `OP_SUBSCRIBE` (0x11), replacing the Phase 25-01 reserved stub. Wire the
SubscriberRegistry (DashMap-based, same pattern as Phase 14), hook `notify_subscribers`
into the ingest path, enforce the 10k bounded-channel backpressure drop, and publish
metrics + signals. No global seq counter; per-connection accept-order only.

Purpose: Delivers the "live" half of the Phase 31 client's subscribe-first
buffered-replay dance. Combined with `27-01`'s SNAPSHOT_FETCH, this is the minimum
v0 replica surface.

Output: Updates to `protocol.rs`, `replica.rs`, `tcp.rs`, `http.rs`, `signals.rs`,
`pipeline.rs`; one Rust integration test; one Python asyncio multi-subscriber test.
</objective>
