<objective>
Implement scatter-gather for `GET /streams` and `GET /streams/{name}` — TPC-PERF-05.

Purpose: Stream listing touches every shard because stream definitions are replicated to all shards at registration time but watermarks are shard-local. The scatter-gather fans out to all N shards via `futures::join_all`, merges results (dedup names, watermark = min), and tracks the fanout with the Phase 50 metrics counter.

Output:
- `src/routing/scatter.rs` — `scatter_gather<Req, Resp>` generic over shard message type and merge function
- `src/server/http.rs` — `GET /streams` and `GET /streams/{name}` handlers updated to use scatter_gather
- `Cargo.toml` — `futures` explicit dep added (if not already transitive)
- Tests (TDD): duplicate-dedup test (N=4 shards same list → no duplicates), watermark-min test
</objective>
