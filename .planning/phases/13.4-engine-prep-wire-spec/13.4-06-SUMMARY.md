# Phase 13.4 Plan 06: D-01 force=true diff matrix + dry_run=true register flag — Summary

**Implements D-01 (USER-LOCKED) STRICT diff matrix and `dry_run=true` flag (CONTEXT scope item #6) by adding `RegisterDiff`/`DiffEntry` categorized-lists types, a pure `classify_register_diff` walker, and the `register_check_force_required` gate — wired into the POST /register dispatch chain in `apply_shard.rs` AFTER the existing JSON-prelude shims and BEFORE the legacy compute_diff path. force=true pre-removes conflicting descriptors via the new `Registry::force_remove_descriptors` helper so the additive apply path bumps `registry_version` + writes the RegistryBump WAL record. dry_run=true short-circuits to a 200 + `{diff, would_apply: false}` response without mutating state. Both flags share the same pure diff classifier; sort_key-based idempotency means two dry_run calls with the same payload produce byte-identical responses.**

## Accomplishments

- **`RegisterDiff` + `DiffEntry` types** in `crates/beava-core/src/registry_diff.rs` — categorized-lists (additive / destructive) with serde-tagged `kind` discriminator. Sort-key infrastructure built in for idempotency.
- **`classify_register_diff` walker** in `crates/beava-core/src/register_validate.rs` — pure function over `(prev_registry, new_payload)`. Walks the events / tables / derivations maps, emits per-class entries (Rename via paired removal+addition heuristic; TypeChange via field-by-field schema comparison; OpRemoval via ops-list-length shrink; AggRemoval via agg-map deletion; WindowChange via per-feature `params.window` comparison; KeyColsChange via group_by `keys` or table `primary_key` comparison; NewDescriptor / NewAgg / NewField for additive deltas). Both vectors sorted at the end.
- **`register_check_force_required` gate** — consumes the diff + the boolean force flag, returns `Err(ForceRequiredError)` when destructive entries exist without force=true.
- **`ForceRequiredError`** — `{code: "force_required", reason: "Destructive registry change requires force=true. See diff for details.", diff: RegisterDiff}`. Embedded directly in the 409 body.
- **`Registry::force_remove_descriptors`** in `crates/beava-core/src/registry.rs` — removes named descriptors plus their compiled chains/aggregations/feature_index/aggregations_by_source entries so the subsequent additive apply path treats them as new.
- **Dispatch wiring** in `crates/beava-server/src/apply_shard.rs::dispatch_one`'s WireRequest::Register arm:
  1. After all 4 existing JSON-prelude shims (12.6-04 / 12.6-06 / 12.7-01 / 12.8-01) and after strict serde, parse the body JSON Value to extract `force` + `dry_run` flags.
  2. Snapshot the registry; classify the diff.
  3. dry_run=true → 200 + `{diff, would_apply: false}` — fires BEFORE the force gate so dry_run+force=true is treated as dry_run.
  4. Destructive non-empty AND !force → 409 + force_required error envelope.
  5. force=true on destructive → pre-remove conflicting descriptors so `execute_register_with_wal`'s additive apply path bumps `registry_version` + RegistryBump WAL record persists.
- **`docs/error-codes.md`** — appended `force_required` entry with full payload shape, recovery guidance, alphabetical placement, and HTTP-status-table cross-link.
- **9 force tests + 4 dry_run tests** — covering all 6 destructive classes (Test 9 agg_removal `#[ignore]` until Plan 09 plumbs `output_kind=table`), additive paths, first-time register, force=true success path, dry_run on destructive, dry_run on additive, dry_run+force interaction, dry_run idempotency.

## Decisions Made

- **Force-then-replace** (vs threading force deep into apply): force=true removes conflicting descriptors first, then the existing additive `apply_registration` path bumps version + writes WAL. Minimal surgery; preserves the existing apply-AFTER-fsync invariant.
- **Categorized-lists payload coexists with legacy `registration_conflict`** — both codes valid; SDKs dispatch on `code` discriminator.
- **AggRemoval as descriptor-level-removal sentinel** — when a descriptor disappears entirely (no paired rename), classifier emits `AggRemoval { table: <name>, agg: "" }` rather than introducing a new "DescriptorRemoval" variant. Keeps the wire surface compact.
- **Sort-key idempotency contract** — `DiffEntry::sort_key` returns `(u8 class_tag, String primary_field, String secondary_field)`. Any future variant addition must extend this match.
