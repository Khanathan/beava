## Deferred items + known divergences (surfaced for v0.0.x or downstream phases)

### From Plan 03 — wire-spec doc cohesion gap

Plan 03 dispatch uses body shape `{table, entity_id}` + per-tuple errors (`results[i].error`); the published Phase 13.0 wire-spec doc + fixture (`examples/wire/batch_get-heterogeneous.{request,response}.json`) uses `{table, key}` + whole-frame OP_ERROR_RESPONSE. Plan 03 followed plan must_haves verbatim (executor contract); reconciliation is a Phase 13.0 doc revisit OR Phase 13.5/13.6 SDK contract negotiation. **Disposition: Phase 13.0 doc revisit** (defer to Phase 13.5/13.6 SDK landing — SDKs are the canonical client; if SDKs adopt `{table, entity_id}` per the engine's existing implementation, the doc updates to match. If SDKs adopt `{table, key}` per the doc, the engine accepts both via a thin alias).

### From Plan 03 — OP_BATCH_GET DoS cap

`dispatch_batch_get_sync` walks `requests[]` without a cap; sibling `dispatch_get_batch` enforces `BATCH_CAP = 10_000` at the cell level. **Disposition: defer to v0.0.x** — adding the cap is a single-line check; out of scope for the doc-only Plan 10 closure but should land before v0 GA if not absorbed by Phase 13.5/13.6 SDK testing.

### Pre-existing test issues (not Phase 13.4's responsibility)

1. **`phase5_smoke::sc6_aggregation_on_table_rejected`** (HTTP 409 vs 400). The test asserts that registering a derivation `output_kind=table` that aggregates on another `output_kind=table` returns 400 with code `aggregation_on_table_not_supported`. ADR-001's partial overturn (Plans 05 + 09) revives derivation `output_kind=table`; the test's premise is now stale. The 409 is Plan 06's force_required diff classifier firing on the second register's window-change (`5m → 1h`) treated as destructive. **Disposition: known-divergence-to-resolve in v0 SDK shake-out** (per orchestrator prompt option). The test predates ADR-001 and the cheaper fix is to retire/rewrite during Phase 13.5 Python-SDK acceptance test consolidation.

2. **`phase2_5_smoke::criterion_6_pipelined_registers_return_in_order`** (`left=65535, right=1`). Pre-existing flake on `phase2_5_smoke.rs`; verified pre-existing by Plan 08 + Plan 09 stash-and-rerun checks. Last touch on the test file is `7ad84c5` (Phase 12.6-06), pre-13.4. **Disposition: pre-existing infrastructure issue; not Phase 13.4's responsibility.** Tracked for Phase 13.7+ test infrastructure cleanup.

3. **`phase5_smoke::sc2_push_then_get_returns_count`** (parallel-run flakes on WAL tempdir collision). Documented in orchestrator known-issues. **Disposition: parallel-test infrastructure issue; not Phase 13.4's responsibility.**
