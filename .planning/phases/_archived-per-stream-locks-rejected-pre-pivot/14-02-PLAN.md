**Snapshot serialization (D-09):**

The snapshot system needs to reconstitute a flat `AHashMap<EntityKey, SerializableEntityState>` from the per-stream DashMaps for serialization (the existing snapshot format expects this).

Add a function in snapshot.rs:
```rust
pub fn collect_snapshot_state(
    stream_stores: &DashMap<String, StreamStore>,
    static_store: &DashMap<String, AHashMap<String, StaticFeature>>,
) -> (AHashMap<String, SerializableEntityState>, AHashSet<String>, AHashSet<String>)
```

This function:
1. Iterates each stream's DashMap via `.iter()` (acquires read locks on DashMap shards — safe for background path per RESEARCH.md D-09)
2. Reassembles per-entity state: for each `(entity_key, StreamEntityState)` in each stream, merge into a `HashMap<EntityKey, SerializableEntityState>` that groups all streams for that entity
3. Collects dirty_keys and deleted_keys from each stream's tracking sets (lock each stream's `dirty_keys` Mutex briefly)
4. Overlays static features from `static_store.iter()`

For delta snapshots: collect only dirty entities. For each stream, lock `dirty_keys`, clone the set, clear it. Then for each dirty key, collect that entity's state from the relevant stream DashMaps.

Update `clone_for_snapshot_with_gc` to accept `&DashMap<String, StreamStore>` + `&DashMap<String, AHashMap<String, StaticFeature>>` instead of `&mut StateStore`. Or add a parallel function `collect_for_snapshot_concurrent(...)` that does the same.

Update main.rs snapshot timer to call the new function instead of `state.lock() -> clone_for_snapshot_with_gc`.

Update main.rs post-snapshot metrics to use `state.metrics.lock()` and `state.snapshot_*` individual locks.

Update http.rs `trigger_snapshot` to use the same concurrent collection path.

**Eviction (per-stream via DashMap retain):**

Refactor `evict_expired_keys` to work with ConcurrentAppState:
```rust
pub fn evict_expired_concurrent(
    state: &ConcurrentAppState,
    now: SystemTime,
) {
    let engine = state.engine.read();
    for entry in state.stream_stores.iter() {
        let stream_name = entry.key();
        let stream_store = entry.value();
        let ttl = engine.get_stream_entity_ttl(stream_name);
        if let Some(ttl) = ttl {
            stream_store.entities.retain(|key, entity_state| {
                let keep = match entity_state.last_event_at {
                    Some(t) => now.duration_since(t).unwrap_or_default() <= ttl,
                    None => true,
                };
                if !keep {
                    stream_store.deleted_keys.lock().insert(key.clone());
                }
                keep
            });
        }
    }
    // Also evict orphaned static features where entity has no streams left
    // (optional — can defer if complex)
}
```

Update main.rs eviction timer to call `evict_expired_concurrent(&state, now)` instead of locking global state.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| snapshot iteration | Background thread reading DashMap while hot path writes |
| eviction retain() | Background removing entries while hot path accesses them |
| event log writer | Multiple streams writing to their log files concurrently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | T (Tampering) | Snapshot captures inconsistent cross-stream state | accept | Existing "lose ~30s on crash" contract; per-stream DashMap iter is shard-level consistent, cross-stream is eventually consistent |
| T-14-06 | D (Denial of Service) | DashMap iter() during snapshot blocks hot-path writes briefly | accept | DashMap iter read-locks shards sequentially; each shard lock held briefly (~microseconds); writes to other shards proceed |
| T-14-07 | T (Tampering) | Eviction retain() removes key while PUSH is inserting it | mitigate | DashMap retain() holds per-shard write lock; PUSH to same shard waits. After retain releases, PUSH re-inserts via entry(). No data loss — just re-initialized state. |
</threat_model>
