## Critical anti-patterns (avoid these on resume)

| Pattern | What it is | Severity | Prevention |
|---|---|---|---|
| Process-global env vars in tests | `TestServerBuilder.tcp_max_frame_bytes()` set BEAVA_TCP_MAX_FRAME_BYTES via `std::env::set_var` and never restored. Leaked across parallel TestServer instances → criterion_6_pipelined_registers race. Same family applies to BEAVA_WAL_*, BEAVA_IO_THREADS, BEAVA_TEST_MODE, BEAVA_MEMORY_GOV_ENFORCE. | advisory | Always plumb per-server config through `ServerV18Config` / `WorkerConfig` struct fields — never `std::env::set_var` from test code. Architectural-test tripwire planned for Phase 999.1. |
| Stale release binary in test fixtures | `phase12_07_main_uses_v18_test::ensure_release_binary_built` only rebuilds `target/release/beava` if missing. Stale binary masks wire-shape regressions (saw it this session — predated Plan 13.4.1-04 `keys`→`key` change). | advisory | When tests use `target/release/<bin>`, the helper should rebuild when source is newer than binary, or always rebuild on each test run. Filed as v0.0.x follow-up. |
| Workspace test bash wrapper exit code | `cargo test ... \| tail \| ...; tail -X` returns the exit of the LAST tail in the chain, NOT cargo test's exit. Misleading "exit 0" doesn't mean tests passed. | advisory | Use `set -o pipefail` or capture cargo's exit explicitly via `${PIPESTATUS[0]}`. |
