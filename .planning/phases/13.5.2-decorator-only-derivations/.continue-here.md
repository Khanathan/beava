<decisions_made>

- **FirstN/LastN/HourOfDayHistogram** switched from string-encoded / map-with-padded-string-keys to `Value::List`. Matches modern `ReservoirSampleState` pattern; cleaner Python list / JS array surface. Blast radius minimal — only `test_buffer.py`/`test_point_ordinal.py` consume these and they want lists.
- **ValueChangeCountState** stores `last_value` as `Option<Value>` (was `f64` + `initialized: bool`). Generic `Value` comparison works for any type. Old impl silently dropped string/bool transitions.
- **`query_time_ms = max(watermark, SystemTime::now() ms-since-epoch)`** in `runtime_core_glue.rs`. Preserves D-06 replay-determinism (watermark wins during replay) AND lets live `time_since`/`age` advance with wall-clock after `time.sleep()`. The existing zero-fallback used `Instant::now().elapsed()` which is always ~0 — that was a latent bug too.
- **reservoir_sample:** server now reads `desc.ext.samples.or(desc.ext.k)`. SDK sends `samples`; server was hardcoded to `k`. Fallback handles both names so any older client/test using `k` still works.
