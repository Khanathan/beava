# Plan 19.1-01 — Bench wall_clock measurement fix

## Objective

Fix the `crates/beava-bench/src/bin/beava-bench-v18.rs` wall_clock-bug per memory `project_phase19_bench_wallclock_fix` and CONTEXT D-06/D-07/D-08. The bug under-reports EPS by ~5× for any N where bench-time < 1s because `let elapsed = start.elapsed();` is captured AFTER `let _ = get_task.await;` and `let _ = rss_task.await;` — both of which can sleep up to 1000ms / 500ms before exiting their inner loop. Phase 19 verdict was PASS-WITH-DEFICIT only because of this bug; the real number clears the M4 threshold by 2.5×.

Fix:
1. Move `let elapsed = start.elapsed();` from after the worker/get_task/rss_task awaits to BEFORE that block.
2. Convert `get_task` and `rss_task` from raw `loop { if stop { break; }; sleep(...); ... }` patterns to `tokio::select! { _ = stop_rx.recv() => break, _ = sleep(...) => { /* sample */ } }` so they exit promptly when workers stop sending events.
3. TDD red commit lands first (per CLAUDE.md); test asserts source-text contract that the fix isn't quietly reverted.

## TDD Pair

### Task 1.a (RED): Source-text regression test for capture order + tokio::select!

**File:** `crates/beava-bench/tests/bench_wallclock_capture_order.rs`

**Behavior:** Read `crates/beava-bench/src/bin/beava-bench-v18.rs` as a string and assert:
- `let elapsed = start.elapsed();` appears BEFORE `let _ = get_task.await;` and `let _ = rss_task.await;` (both in source-byte-offset terms).
- `get_task` definition uses `tokio::select!` macro (not just `loop { sleep }`).
- `rss_task` definition uses `tokio::select!` macro.

These three checks pin the structural fix in place. Test fails on the current (buggy) source; passes after the fix.

**Run:** `cargo test --workspace bench_wallclock_capture_order` — MUST FAIL before the fix.

**Commit:** `test(19.1-01): RED — wall_clock captured before background-task awaits`

### Task 1.b (GREEN): Apply the elapsed-move + tokio::select! fix

**File:** `crates/beava-bench/src/bin/beava-bench-v18.rs`

**Behavior:**
1. Add `tokio::sync::watch::channel::<()>(())` (or `oneshot`/`broadcast` per fit) for signaling background tasks to stop.
2. Convert `get_task` and `rss_task` to `tokio::select!` — exit promptly on stop signal.
3. After workers finish, capture `let elapsed = start.elapsed();` BEFORE awaiting `get_task` / `rss_task`.
4. Send the stop signal, then await both background tasks (clean shutdown).

**Run:**
- `cargo test --workspace bench_wallclock_capture_order` — MUST PASS.
- `cargo build -p beava-bench --release` — MUST succeed.
- Optional: sample run at `--total-events 100000 --blast-shape zipfian --pipeline crates/beava-bench/configs/small.json --transport tcp --wire-format msgpack --parallel 16 --pipeline-depth 1024 --isolation-mode --no-ledger` reports `wall_clock_ms < 1000`.

**Commit:** `feat(19.1-01): GREEN — capture elapsed before background tasks; use tokio::select!`
