      - `expr.referenced_fields()` must be a subset of self's schema; else schema_mismatch_error for each unknown field
      - Serialized op: {"op": "filter", "expr": expr.to_expr_string()}

    .map(**derivations) — short alias for with_columns when all outputs are new fields; accepts kwargs of name=_ExprAST.
      - Actually: collapse .map into .with_columns for simplicity — `.map()` is a no-op alias. Document in docstring.
      - (Rationale: DataFrame-parity list had both; keep both names but identical behavior.)

    .select(*field_names):
      - All field_names must exist in schema; else schema_mismatch_error with suggest()
      - Output schema = only the named fields, in the order given
      - Serialized: {"op": "select", "fields": [...]}

    .drop(*field_names):
      - All field_names must exist; else schema_mismatch_error
      - Output schema = schema minus dropped fields (order preserved)
      - Table key fields cannot be dropped → error "cannot drop key field 'X' from Table '<name>'"
      - Serialized: {"op": "drop", "fields": [...]}

    .rename(**mapping):
      - mapping keys must exist in schema; mapping values must NOT already be in schema (else error "rename target 'new' collides with existing field 'new' — drop it first")
      - Output schema: renamed fields, same types
      - Table: renaming a key field also renames the key in the descriptor's _key list
      - Serialized: {"op": "rename", "mapping": {...}}

    .with_columns(**derivations):
      - Each kwarg value must be an _ExprAST; references validated against self's schema
      - Adding a column with a name that already exists REPLACES the type (inferred from the expression — v0 inference is minimal: treat arithmetic as float, comparison as bool, literals by type; otherwise fall back to "any")
      - Serialized: {"op": "with_columns", "exprs": {name: expr_string, ...}}

    .cast(**type_map):
      - Each key must exist; each value must be a string in {"int", "float", "str", "bool"} (v0 type set)
      - Output schema: field types updated
      - Serialized: {"op": "cast", "casts": {field: type, ...}}

    .fillna(**defaults):
      - Each key must exist; each value must be a JSON-serializable scalar
      - Output schema: field's optional flag becomes False (nulls are filled)
      - Serialized: {"op": "fillna", "defaults": {...}}

    All ops return an instance of the same outer type (Stream or Table) wrapping the upstream + the appended op. The new instance's `.describe()` reflects the transformed schema and carries an `_ops` list and an `_upstream` reference.
