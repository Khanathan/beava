3. **Implement `render-docs.mjs`:**
   - Load `render-docs-config.json` from script directory
   - For each page, read the source markdown file at `<repo-root>/<src>`
   - Strip the leading H1 (it becomes the page `<h1>` in the shell, not body content)
   - Use `markdown-it` (with `html: false, linkify: true, typographer: false`) to render the rest to HTML
   - Post-process the rendered HTML:
     - **Code fences** → replace `<pre><code class="language-X">CONTENT</code></pre>` with `<Code lang="X">{\`CONTENT\`}</Code>` (use a regex; CONTENT is HTML-escape-decoded back to source so React renders it raw)
     - **Inline links to `.md` files** → rewrite `./operators/core/count.md` → `/docs/operators/core/count/`, `../concepts/embed-mode.md` → `/docs/concepts/embed-mode/`, etc. (resolve relative to source file dir, then map onto target route via the `pages[].src→out` map; emit warnings for unresolved targets but do NOT fail the build — capture the warnings to stderr + a `render-docs-warnings.txt` artifact in the repo root for the link-audit task in Plan 02)
     - **Anchor links** (`#section-id`) — preserve as-is
     - **Cross-repo links** (`.planning/decisions/ADR-001-...md` or `examples/wire/...`) — leave as-is for now (Plan 02 link-audit handles)
     - **Headings**: harvest H2 IDs (markdown-it's `slugify` default = lowercase + dash) and emit them as the TOC array passed into `DocsTOC`
   - **Auto-derive TOC**: every H2 in the body becomes a TOC entry `{ id: slug(h2_text), label: h2_text }`
   - **Auto-derive metaDescription**: first paragraph after the H1 (truncated to ~160 chars at word boundary, no closing punctuation added) — UNLESS the page has an explicit `metaDescription` in config, in which case use that
   - **Render the shell**: emit `<!doctype html>...<head>...<style>...</style></head><body class="beava"><div id="root"></div><script src="https://unpkg.com/react@18.3.1/...">...<script type="text/babel">const TOC = [...]; const App = () => <><SiteHeader.../><DocsSidebar active="..."/>...<main className="bv-content"><div className="bv-crumbs">...crumbs...</div><h1>...</h1>BODY_HTML<Pager.../></main><DocsTOC items={TOC}/>...</>; ReactDOM.createRoot(...).render(<App/>);</script></body></html>`. The `stylesRelativeBase` in config is the prefix used in `href="../../../styles/..."` — derive per page from output depth.
   - **HTML output style**: stable indentation, sorted attributes, deterministic so re-running produces byte-identical files (crucial for git diffs and CI)
