Decision rationale: the existing comment in `build-search.mjs` notes "When/if docs migrate to static HTML, switch to `index.addDirectory({ path: 'project' })`." Plan 01's rendered pages still ship as React+Babel client-rendered (the prose lives in JSX template literals inside `<script type="text/babel">`, not as raw HTML body). So `addDirectory` will NOT extract content from them either. We must keep using `addCustomRecord` and feed it from a new source.

Steps:

1. **Add a helper `extractContentFromBabelTemplate(htmlPath)`** in `build-search.mjs`:
   - Reads the rendered HTML file
   - Locates the `<script type="text/babel">...</script>` block
   - Extracts the substring between `<main className="bv-content">` and `</main>`
   - Strips JSX braces, `<Code lang=...>{...}</Code>` blocks (keep the inner text), html-escapes, and React component tags (`<SiteHeader/>`, etc.)
   - Returns plain text for indexing

2. **Auto-generate PAGES entries for Plan-01 pages**: at the top of `build-search.mjs`, import the `render-docs-config.json` page list. For each entry whose `out` path starts with `beava-website/project/docs/`, derive:
   - `url`: the `out` path mapped to a route (e.g. `beava-website/project/docs/operators/core/count/index.html` → `/docs/operators/core/count/`)
   - `title`: read the `<title>` from the HTML or use the config's `title` field
   - `section`: derived from URL prefix (e.g. `/docs/operators/` → "Operators", `/docs/sdk-api/` → "SDKs", `/docs/architecture/` → "Architecture", etc.)
   - `content`: the extracted prose from `extractContentFromBabelTemplate(out)`

3. **Merge with existing PAGES**: the legacy hand-curated PAGES list (Quickstart, Define a pipeline, etc.) STAYS — those routes (`/docs/get-started/quickstart/`, etc.) are NOT touched by Plan 01 and still have empty source HTML. De-duplicate by URL: if an auto-generated entry has the same URL as a hand-curated one, the auto-generated wins (Plan 01 produced fresher content).

   **Ambiguous URLs to resolve**:
   - Plan 01 produces `/docs/quickstart/` (from `docs/quickstart.md`)
   - The existing PAGES list has `/docs/get-started/quickstart/` (the legacy React-rendered quickstart)
   - These are DIFFERENT URLs → both index entries kept. The new `/docs/quickstart/` is the canonical Phase-13.0 quickstart; the old `/docs/get-started/quickstart/` is the prior site quickstart. Plan 03 may consolidate; for now keep both.

**Action: build a simple Node link-checker, run it against the rendered docs tree, fix any broken internal links.**

Steps:

1. **Create `beava-website/scripts/check-links.mjs`** (Node ESM, ~150-250 LOC):
   - Recursively walk `beava-website/project/docs/` for all `index.html` files
   - For each file, extract every `<a href="...">` URL inside the `<script type="text/babel">` body (since the React+Babel pages embed links in JSX)
   - Classify each href:
     - **Relative `/docs/.../`**: must resolve to an `index.html` on disk under `beava-website/project/<that path>index.html`
     - **Anchor-only `#...`**: skip (anchor validity is best-effort; not in scope)
     - **External `http(s)://...`**: skip (not in scope; Plan 01 left these as-is)
     - **Repo-relative paths (`.planning/...`, `examples/...`, `crates/...`)**: report as WARNING (not BROKEN — these would resolve via GitHub web view)
   - Output: structured report to stdout. Format: `<status> <source-file> -> <href>` (status ∈ {OK, BROKEN, WARN, EXTERNAL_SKIP}). Final summary line: `BROKEN_COUNT=N WARN_COUNT=M`. Exit code: 0 if BROKEN_COUNT=0, 1 otherwise.

2. **Add npm script**: in `beava-website/package.json` add `"check:links": "node scripts/check-links.mjs"`.

3. **Run the link-checker**: `cd beava-website && npm run check:links`. Capture output to `link-audit-report.txt` (do NOT commit this report; it's a working artifact). Inspect BROKEN entries.

4. **Fix broken internal links**:
   - **Cause A: link in source markdown points at a `.md` file Plan 01's converter couldn't auto-resolve.** Fix: extend `render-docs-config.json` with a `linkOverrides` section that maps the unresolvable source href to a target route, then re-run `npm run build:docs` to regenerate the affected pages. Re-run `check:links` until it exits 0.
   - **Cause B: link points at a page that doesn't exist (typo in source markdown).** Fix: this is a Phase 13.0 source-content bug; either (a) fix the source markdown in `docs/<file>.md` and re-render, or (b) drop the link if it was always bogus. Pick (a) for any link that was clearly intentional; pick (b) only if the linked target genuinely doesn't exist anywhere.
   - **Cause C: cross-repo link (`.planning/decisions/ADR-001-...md`)** — not a BROKEN, but a WARNING; leave as-is. CONTEXT explicitly accepts cross-repo links to `.planning/` files.

5. **Re-run** `npm run check:links`. Exit code MUST be 0 (BROKEN_COUNT=0). WARN_COUNT may be non-zero (cross-repo links).

6. **Commit**: `docs(13.7-02): cross-link audit — npm run check:links exits clean (NN warnings retained for cross-repo links)` (single commit if no source markdown fixes needed; two commits if Plan-01 source markdown was patched + re-rendered).
