<decisions>
## Implementation Decisions

### D-01 — Static site generator: MkDocs Material
- Picks: MkDocs Material (Python; Material theme; built-in search; mature plugin ecosystem; matches DuckDB / Polars docs feel)
- Rejected: Docusaurus (React-heavy; more JS than needed); Sphinx (Python-native but uglier default theme); raw HTML (rebuilds painful)
- Repo location: `beava-website/` (existing dir; MkDocs config goes there)
- Source `docs/` (top-level) is the SOURCE; MkDocs reads it via `docs_dir: ../docs/` config
- Build output: `beava-website/site/` (gitignored); deployed to beava.dev

### D-02 — Theme + visual identity
- Material theme with custom palette + accent matching beava brand (TBD — defer to plan-phase research)
- Dark mode toggle (Material built-in)
- Mono font: JetBrains Mono (or Fira Code)
- Sans font: Inter
- Code-block syntax highlighting: Material's default + extras for Rust/Python/TS/Go

### D-03 — Information Architecture per `project_beava_website_ia`
- Top-nav: Home / Guide / Docs / Community
- Home (`/`): hero per Q17 — combined latency + throughput + memory headlines (`<10ms P99 / 100K+ EPS / ~6KB per entity`); Quickstart CTA; 3 vertical guides preview
- Guide (`/guide/`): 3 vertical tutorials (adtech / fraud / ecommerce) wrapping the bundled demos
- Docs (`/docs/`): the rendered `docs/` tree (wire-spec / sdk-api / pipeline-dsl / operators / etc.)
- Community: links to GitHub, Discord, blog
- Cloud banner: top-of-page "Self-host today; managed cloud coming v0.1+" non-intrusive banner

### D-04 — Quickstart polish: animated GIF of `bv.demo("adtech")` running
- Record screen of `pip install beava && python -c "import beava as bv; app = bv.demo('adtech'); print(app.get('UserClicks', 'alice'))"` — under 60 seconds
- Embed at top of `docs/quickstart.md` as `quickstart.gif`
- `asciinema` for the recording; convert to GIF via `agg`

### D-05 — 3 vertical guides
- Each guide: ~500-800 LOC of prose + code wrapping the bundled demo
- adtech: "Build a real-time click attribution system in 50 lines"
- fraud: "Detect velocity-based fraud across 30 features in <10ms"
- ecommerce: "Track per-user purchase behavior with reservoir sampling"
- Each has: data model walkthrough + step-by-step register/push/query + expected output + cross-link to operator docs + cross-link to ADR if relevant

### D-06 — Search + cross-linking
- Material's built-in client-side search (lunr.js)
- Cross-link discipline: every operator doc links back to family index + master catalog + cost-class.md
- Auto-generated "See also" sections from doc headers
- Copy-to-clipboard buttons on every code block (Material extension)

### D-07 — Deploy: Cloudflare Pages
- Repo: `beava-website/` connected to Cloudflare Pages
- Build command: `mkdocs build`; output dir: `site/`
- Custom domain: beava.dev (Cloudflare DNS)
- Auto-deploy on push to `v2/greenfield` (or `main` post-merge)
- Preview deploys on every PR
