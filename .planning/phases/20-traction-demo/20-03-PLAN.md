    **`deploy/tally.service`:**
    ```ini
    [Unit]
    Description=Tally real-time feature server
    After=network-online.target
    Wants=network-online.target

    [Service]
    Type=simple
    User=tally
    Group=tally
    ExecStart=/usr/local/bin/tally --public-mode --tcp-bind 127.0.0.1 --tcp-port 6400 --http-port 6401 --snapshot-dir /var/lib/tally
    Environment=RUST_LOG=info
    EnvironmentFile=/etc/tally/admin.token.env
    Restart=always
    RestartSec=5
    StateDirectory=tally
    StandardOutput=journal
    StandardError=journal
    LimitNOFILE=65536
    # TCP port 6400 bound to 127.0.0.1 explicitly (--tcp-bind) — never reachable from the internet

    [Install]
    WantedBy=multi-user.target
    ```
    Note: `/etc/tally/admin.token.env` contains `TALLY_ADMIN_TOKEN=<hex>` — created by provision.sh with mode 600, owner tally:tally.

    **`deploy/Caddyfile`:**
    ```
    demo.tally.dev {
        encode gzip
        header {
            Access-Control-Allow-Origin "*"
            Strict-Transport-Security "max-age=31536000"
            X-Content-Type-Options "nosniff"
        }
        rate_limit {
            zone public_zone {
                key {remote_host}
                events 120
                window 1m
            }
        }
        reverse_proxy 127.0.0.1:6401 {
            # Block admin paths at the edge as defense-in-depth
            # (the router already enforces this, but belt+suspenders is cheap)
        }
        @admin path /pipelines* /snapshot* /debug/*
        handle @admin {
            respond "Not found" 404
        }
    }
    ```
    The `@admin` matcher returns 404 for admin paths from the public side. Admin access is done via `ssh -L 6401:127.0.0.1:6401 demo-vm` + localhost which bypasses Caddy entirely. (Router middleware accepts loopback unconditionally.)

    If the `rate_limit` module isn't bundled in the default caddy binary, fall back to a simple `@rate` matcher using caddy's built-in `handle` — or accept no rate limit for the 5-day demo window. Document the tradeoff.

    **`deploy/provision.sh`:**
    ```bash
    #!/usr/bin/env bash
    set -euo pipefail
    DOMAIN="${1:?usage: provision.sh <domain>}"
    ADMIN_TOKEN="${TALLY_ADMIN_TOKEN:-$(openssl rand -hex 32)}"

    # 1. System user
    id -u tally >/dev/null 2>&1 || useradd --system --home /var/lib/tally --shell /usr/sbin/nologin tally

    # 2. Caddy
    apt-get update
    apt-get install -y debian-keyring debian-archive-keyring apt-transport-https curl
    curl -1sLf https://dl.cloudsmith.io/public/caddy/stable/gpg.key | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
    curl -1sLf https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt | tee /etc/apt/sources.list.d/caddy-stable.list
    apt-get update
    apt-get install -y caddy

    # 3. Tally binary (assume pre-built tarball at ./tally.tar.gz alongside this script)
    install -m 755 ./tally /usr/local/bin/tally

    # 4. Admin token
    install -d -m 750 -o tally -g tally /etc/tally
    umask 077
    echo "TALLY_ADMIN_TOKEN=$ADMIN_TOKEN" > /etc/tally/admin.token.env
    chown tally:tally /etc/tally/admin.token.env
    chmod 600 /etc/tally/admin.token.env
    # Also write the plain token for operator access
    echo "$ADMIN_TOKEN" > /etc/tally/admin.token
    chmod 600 /etc/tally/admin.token
    chown tally:tally /etc/tally/admin.token

    # 5. systemd unit
    install -m 644 ./tally.service /etc/systemd/system/tally.service
    systemctl daemon-reload
    systemctl enable --now tally.service

    # 6. Caddy
    sed "s/demo.tally.dev/${DOMAIN}/g" ./Caddyfile > /etc/caddy/Caddyfile
    systemctl reload caddy

    # 7. Firewall — allow SSH (22) + HTTP (80) + HTTPS (443); deny Tally's TCP (6400) + HTTP mgmt (6401) on public iface
    apt-get install -y ufw
    ufw --force reset
    ufw default deny incoming
    ufw default allow outgoing
    ufw allow 22/tcp    # SSH
    ufw allow 80/tcp    # HTTP (Caddy ACME challenge + redirect)
    ufw allow 443/tcp   # HTTPS (Caddy)
    ufw deny 6400/tcp   # Tally TCP protocol — loopback-only, double-enforced by --tcp-bind
    ufw deny 6401/tcp   # Tally HTTP mgmt — reached only via Caddy reverse_proxy on 127.0.0.1
    ufw --force enable

    # 8. Disable kernel auto-updates during demo window
    systemctl disable --now unattended-upgrades || true

    # 9. Journald cap
    mkdir -p /etc/systemd/journald.conf.d
    echo -e "[Journal]\nSystemMaxUse=500M" > /etc/systemd/journald.conf.d/tally.conf
    systemctl restart systemd-journald

    # 10. Wait for health
    for i in {1..24}; do
      if curl -fsS "https://${DOMAIN}/health" >/dev/null 2>&1; then
        echo "OK: https://${DOMAIN}/health"
        echo "Admin token: $(cat /etc/tally/admin.token)"
        exit 0
      fi
      sleep 5
    done
    echo "FAIL: health check timeout" >&2
    journalctl -u tally -n 50 >&2
    exit 1
    ```

    **`deploy/smoke.sh`:**
    ```bash
    #!/usr/bin/env bash
    set -euo pipefail
    BASE="${1:?usage: smoke.sh <base-url>}"
    FAIL=0
    check() { if eval "$2"; then echo "[OK] $1"; else echo "[FAIL] $1"; FAIL=1; fi; }

    # 1. /health returns 200
    check "health" "curl -fsS '$BASE/health' | grep -q '\"status\":\"ok\"'"

    # 2. /public/stats has all 6 fields
    check "stats shape" "
      resp=\$(curl -fsS '$BASE/public/stats')
      for f in events_total current_eps p99_push_us p50_push_us uptime_seconds keys_total; do
        echo \"\$resp\" | grep -q \"\\\"\$f\\\"\" || exit 1
      done
    "

    # 3. Admin endpoint returns 403 (or 404 via Caddy edge block)
    check "admin denied" "
      code=\$(curl -s -o /dev/null -w '%{http_code}' -X DELETE '$BASE/pipelines/Transactions')
      [[ \"\$code\" == \"403\" || \"\$code\" == \"404\" ]]
    "

    # 4. Replay eps floor — TCP 6400 is loopback-only, so replay runs ON the VM via SSH
    if [[ "${2:-}" == "--with-replay" && -n "${TALLY_SSH_HOST:-}" ]]; then
      check "replay eps floor" "
        out=\$(ssh \"\$TALLY_SSH_HOST\" 'cd /opt/tally && python3 benchmark/replay/replay_30d.py --events 1000000 --workers 4 --host 127.0.0.1 --port 6400' 2>&1)
        eps=\$(echo \"\$out\" | grep -oE 'events_per_sec=[0-9]+' | cut -d= -f2)
        [[ -n \"\$eps\" && \"\$eps\" -gt 500000 ]]
      "
    fi

    # 5. Crash-recovery: record keys_total, trigger restart via ssh, wait, verify keys_total restored
    #    (this case assumes SSH access — run from a machine with the deploy key)
    if [[ -n "${TALLY_SSH_HOST:-}" ]]; then
      check "crash recovery" "
        before=\$(curl -fsS '$BASE/public/stats' | grep -oE '\"keys_total\":[0-9]+' | cut -d: -f2)
        ssh \"\$TALLY_SSH_HOST\" 'sudo systemctl restart tally'
        sleep 15
        after=\$(curl -fsS '$BASE/public/stats' | grep -oE '\"keys_total\":[0-9]+' | cut -d: -f2)
        # Accept equality or within-10% drift (snapshot cadence may have lost a few keys)
        [[ \"\$after\" -ge \$((before * 90 / 100)) ]]
      "
    fi

    # 6. TCP 6400 must NOT be reachable from the public interface
    PUBLIC_HOST=\$(echo '$BASE' | sed 's|https\\?://||;s|/.*||')
    check "tcp 6400 closed publicly" "! nc -z -w 2 \$PUBLIC_HOST 6400"

    exit \$FAIL
    ```

## Risks & Mitigations

- **TLS cert issuance fails** (Let's Encrypt rate-limit or DNS misconfigured). → Caddy's auto-TLS has robust retry; DNS must propagate before provision.sh runs. README notes the pre-check: `dig demo.tally.dev` should return the VM IPv4 before running provision.
- **Binary built on dev box doesn't run on Debian 12 VM** (glibc mismatch). → Build with `--target x86_64-unknown-linux-gnu` on matching glibc, or use `musl` static target. Provision README documents both options.
- **Snapshot recovery loses > 10% of state.** → Phase 9 full snapshot cadence is every 30s; a restart at peak load might lose up to 30s of events, which on a mostly-idle demo instance is near-zero. If it exceeds 10% during smoke, tighten to every 10s for the demo window via CLI flag.
- **Caddy rate-limit module not bundled.** → Provision script falls back to no rate-limit; document this in README. HN-sized traffic unlikely for a launch blog.
- **kernel auto-upgrade reboots the VM mid-run.** → `systemctl disable unattended-upgrades` in provision.sh. Apply manually after the 5-day window closes.
- **Replay eps measurement on small VM is far lower than dev-box measurement.** → This is the actual number that goes in the blog. If CX22 only hits ~400k eps, the headline is honest: "30M events in ~75 seconds on a $5/mo VM" is still a strong story. Accept reality; do not re-run on a bigger box and mislead.
