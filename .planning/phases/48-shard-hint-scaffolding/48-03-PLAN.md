  **Step 2 — Create `.github/workflows/bench-nightly.yml`.**

  ```yaml
  # Nightly criterion bench runner — shard_hint Wave 0 (Phase 48, D-07).
  # Runs on schedule to avoid per-PR false alarms from shared-runner variance.
  # Criterion output is uploaded as an artifact; does NOT auto-commit to repo.
  # To update benchmark/shard_scaffold/README.md: download the artifact and
  # commit the relevant numbers manually after a stable reference run.

  name: Bench Nightly

  on:
    schedule:
      # 02:00 UTC every day — off-peak for GitHub Actions shared runners.
      - cron: "0 2 * * *"
    # Allow manual trigger for ad-hoc reference runs.
    workflow_dispatch:

  env:
    CARGO_TERM_COLOR: always

  jobs:
    bench-shard-scaffold:
      name: shard_scaffold criterion bench
      runs-on: ubuntu-latest
      timeout-minutes: 30

      steps:
        - uses: actions/checkout@v4

        - uses: dtolnay/rust-toolchain@stable

        - uses: Swatinem/rust-cache@v2
          with:
            # Separate cache key from CI to avoid evicting test build artifacts.
            key: bench-nightly

        - name: Run shard_scaffold bench
          run: |
            cargo bench --bench shard_scaffold 2>&1 | tee bench_output.txt
          # Note: criterion exits 0 even when a benchmark finishes normally;
          # tee ensures the output is captured even if the process exits non-zero.

        - name: Upload criterion output
          uses: actions/upload-artifact@v4
          with:
            name: shard-scaffold-bench-${{ github.run_id }}
            path: |
              bench_output.txt
              target/criterion/shard_hint/
            retention-days: 90
          # 90-day retention: keeps a rolling 3-month window of nightly results
          # without consuming excessive storage.
  ```
