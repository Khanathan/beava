## Implementation Decisions

### TestServerV18 migration shape
- **D-01:** Drop-in rewrite. Preserve the `TestServer` name + builder API surface (`post_json`, `get_json`, `tcp_client`, `base_url`, `registry`, `app_state`, `force_snapshot_now`, `tcp_addr`). Rewrite internals to spawn `ServerV18` (calling `ServerV18::bind() / serve_with_dirs()`) instead of legacy `Server`. ~20 test files compile unchanged or with one-method renames. Planner figures out the shim layer for white-box accessors (especially `app_state()` and `dispatch_wire_request` substitutes — likely call into `apply_shard.rs::dispatch_*_sync` directly from white-box tests).

### phase11_smoke handling
- **D-02:** Rewrite the failing assertion (line 235, `v["a"].as_f64().expect("a share")`) to assert on the SET of expected `type_mix` entries rather than specific keys. Treat as a HashMap iteration nondeterminism artifact in the test, not an operator regression. Lands inline during the TestServer drop-in rewrite work — no separate `/gsd-debug` cycle. Risk acknowledged: if this ever surfaces as a real op regression, root-cause investigation is a separate follow-up.

### Wire schema bump posture
- **D-03:** Hard rip everywhere — zero `event_time_ms` compat at any layer. v0 isn't released; no users to break.
  - **Push wire:** `event_time_ms` deleted from JSON + framed-TCP push payload schemas. Server-side parser doesn't accept the field; `serde(deny_unknown_fields)` or equivalent enforces this.
  - **WAL format:** Schema version bumps cleanly (no event_time field). Recovery does NOT handle pre-pivot WAL records — pre-pivot WALs/snapshots are dev artifacts that get discarded. No migration path.
  - **Python SDK:** `event_time_field` decorator + `tolerate_delay` parameter deleted. Compile errors at register-time if any user-side fixture still references them.
  - **No deprecation window, no parse-and-strip, no warn-then-error.**

### Plan 12-10 push-and-get fate
- **D-04:** Plan 12-10 stays deferred — dropped from v0 critical path entirely. Phase 12.6 does NOT migrate or wire push-and-get on mio; v0 ships without it. Plan file (`.planning/phases/12-server-side-async-push-coalescing/12-10-PLAN.md`) stays in place for v0.0.x or v0.1+ revisit; Phase 12.5 dir (`.planning/phases/12.5-push-and-get/`) gets formally retired during Phase 12.6's archive sweep with a SUPERSEDED-AND-DEFERRED banner. Users do 2-RT (push → get) in v0.

## Existing Code Insights

### Reusable Assets
- **`ServerV18` struct (server.rs:370+):** Already production binary entry post-Plan 12-07. Drop-in TestServer rewrite wraps `ServerV18::bind() → serve_with_dirs() → shutdown()`.
- **`apply_shard.rs::dispatch_*_sync` (line 394+):** Canonical hot-path entry. White-box tests that previously called `dispatch_wire_request` migrate here.
- **`runtime_core_glue::dispatch_get_batch_sync` (line 365):** Sync read path — survives axum kill; only async wrappers deleted.
- **Phase 12-07/08/09 + phase18 test patterns** (16 test files) — already on ServerV18; serve as references for the drop-in rewrite shape.

### Established Patterns
- **mio data plane = single hot-path entry:** All push/get/upsert/delete dispatches via `apply_shard.rs::dispatch_*_sync`. Phase 12.6 makes this a locked architectural invariant (mio-only enforcement test).
- **Sync dispatch on the apply thread:** No `.await` inside the apply borrow. Glue layer (`runtime_core_glue.rs`) keeps the sync helpers; async wrappers go.
- **Single-threaded data plane:** Locked per `project_no_sharded_apply`. Phase 12.6 doesn't introduce any concurrency primitives; it removes them where legacy axum had them.
- **Drop-in TestServer pattern:** Builder API (`.tcp_enabled() / .with_persistence() / .build()`) shapes the test surface. Rewrite preserves this; tests barely change.

### Integration Points
- **Wire schema bump:** Both `crates/beava-core/src/snapshot_body.rs` (event-time field) and the WAL record encoder need a coordinated bump. Recovery in `recovery.rs` reads the version byte and refuses pre-pivot records (D-03 hard rip).
- **Python SDK update:** `python/beava/_events.py` + `_app.py` + `_validate.py` + `_wire.py` need `event_time_field` / `tolerate_delay` decorator removal. Some SDK tests will fail at register-time until updated — surfaces hidden dependencies.
- **Register-time validator (`crates/beava-core/src/register_validate.rs`):** Add early-rejection arms for residual `OpNode::Join` / `OpNode::Union` references with structured error codes (compatible with the existing 409 destructive-change error shape).
