<objective>
Land the Phase 12.6 microbench per CLAUDE.md §Performance Discipline. After Plans 05 (Path X — windowed-op time-source swap) + 06 (event-time strip) + 07 (legacy axum kill) land, the apply hot path has changed shape: SystemTime::now() syscall replaces row.event_time read, and the deletion of axum-based push.rs removes a parallel data-plane that wasn't on the hot path anyway but did pollute trace overhead. This plan measures the post-12.6 hot path with criterion and appends the baseline to perf-baselines.md.

The bench has three cells:
1. **simple counter** — `Tx.group_by("user_id").agg(cnt=bv.count())`. Dominant cost: HashMap lookup + bookkeeping. Sensitive to: dispatch overhead.
2. **sketch-heavy** — `Tx.group_by("user_id").agg(unique_users=bv.count_distinct("session_id", window="1h"))`. Dominant cost: HLL update. Sensitive to: identity-hasher path (Phase 19.4-A landed the fix).
3. **windowed rolling 60s sum** — `Tx.group_by("user_id").agg(amt_60s=bv.sum("amount", window="60s"))`. Dominant cost: WindowedOp bucket index + fold. Sensitive to: Path X parameter rename + the SystemTime::now() syscall in dispatch_push_sync.

Comparison: most recent prior baseline is post-19.4 from `.planning/perf-baselines.md` and `.planning/throughput-baselines.md` (Apple-M4). The bench reports criterion's `time/iter` plus a verdict (PASS / WARN / BLOCK) per cell against the prior baseline.

Wave 6 placement: depends on Plans 05, 06, 07 — all the code-deletion work that changes the apply path. Plan 04 (joins/unions) is a register-time change with no hot-path impact, so not a dep. Plan 08 (Python SDK) doesn't touch Rust, no dep. Plan 10 (architectural test) doesn't change runtime, no dep. Plan 11 measures the result of all four code-changing plans together.
