## Simple-fraud TCP Verdict + rationale

**Verdict: PASS.**

- **Median delta: -0.94%** (700,571 vs 707,237) — well within the ±10% PASS band.
- **Mean delta: -1.5%** (696,420 vs 707,237) — also PASS.
- **Run variance:** stddev 23,845 = 3.4% — matches post-12-08's 3% pattern; consistent with running on a busy box (load avg 5–11).
- **Range:** 653,380 (low) to 724,383 (high) across 6 runs. Both extremes are within ±10% of the post-12-08 baseline — no individual run triggered WARN/BLOCK.

**Architectural rationale (why we expected ~neutral):** Phase 12.6 was a deletion sweep, not an optimisation. The push hot path (mio data plane → `apply_shard.rs::dispatch_push_sync` → `apply_event_to_aggregations`) was already the only data-plane runtime post-Plan-12-07. The Phase 12.6 changes (legacy axum kill, event_time_ms wire-strip, joins/unions removal, Path X now_ms() swap) all touch the **cold path or the wire-parse layer**, not the inner apply loop. Two competing forces:

1. **Lift:** Removed legacy code is no longer compiled in the binary → fewer instructions in icache → potential ~0–5% lift on small/tcp.
2. **Headwind:** Path X switched windowed-op time source from `event_time_ms` (parsed from payload) to server-side `now_ms()` (`SystemTime::now()` syscall). At ~10–30 ns per syscall × ~500 ns mean apply = ~2–6% headwind.

Net: we expected ~neutral. Observation matches: median -0.94%, mean -1.5%. No surprise; no architectural concern. Confirmed by independent cells: medium/tcp +0.7%, fraud-team/tcp +0.5%, small/http +1.6% — production-relevant cells all within ±2%.

## Methodology details

**Bench invocation pattern (per Phase 12-08 / 19.1 convention):**
```bash
CARGO_MANIFEST_DIR=$(pwd)/crates/beava-bench \
./target/release/beava-bench-v18 \
  --pipeline {small|medium|large|fraud-team} \
  --transport {tcp|http} \
  --wire-format {msgpack|json} \
  --total-events 1000000 \
  --parallel 16 \
  --pipeline-depth 1024 \
  --continuous-pipeline true \
  --blast-shape zipfian --zipf-alpha 1.0 \
  --cardinality 10000 \
  --isolation-mode \
  --get-sample-interval-ms {50|500|1000} \
  --get-batch-keys 100 \
  --no-ledger
```

**Variance characterization:** small/tcp run 6 times back-to-back (median + mean + stddev reported per Phase 12-08 convention). All other cells run once with `/get` sampling for batch-get P99 measurement.

**Build:**
```bash
cargo build -p beava-bench --release
```
Binary: `target/release/beava-bench-v18` (in-process ServerV18 — bench spawns server, registers pipeline, pre-warms, runs workload).

**Cargo fmt --all --check:** clean (no formatting drift).
