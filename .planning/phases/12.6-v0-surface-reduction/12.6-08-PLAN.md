<objective>
Strip event-time and join/union surface from the Python SDK. After this plan lands, the public Python SDK API has no event_time field on @bv.event, no tolerate_delay parameter, no bv.join helpers, no bv.union helper. Users authoring fraud-detection pipelines see only the post-pivot architecture: events are processing-time-bucketed, no joins, no event-time field on the wire.

Why Wave 4: depends on Plans 04 (Rust-side joins/unions deletion — server rejects join DAGs with structured error code), 06 (Rust-side EventDescriptor field deletion — server's deny_unknown_fields posture on push), and 07 (legacy axum kill — eliminates any remaining HTTP path that might silently accept stale fields). With those landed, this plan brings the Python SDK in sync.

Coordination with Plan 04 (joins): Plan 04 deleted the Python tests `test_v0_join_stubs.py`, `test_v0_stream_table_join.py`, `test_watermark_e2e.py` already (Task 3.b in Plan 04). This plan deletes the helper FUNCTIONS those tests called.

Output:
- python/beava/_events.py — `event_time_field`, `_event_time_field`, `tolerate_delay`, `_tolerate_delay_ms` deleted from the @bv.event decorator and EventDescriptor; the schema parser no longer special-cases an event_time field.
- python/beava/_validate.py — `_check_event_time_field` (rule 5) deleted; `_get_event_time_field` deleted.
- python/beava/_app.py — line 135 doc-comment "event_time_field validity" deleted; any helper that references _event_time_field deleted.
- python/beava/_wire.py — any wire-format helper that includes event_time_ms in the push body deleted.
- python/beava/__init__.py — bv.join / bv.union exports removed (if they exist).
- python/beava/_agg.py / _events.py / _tables.py — bv.join / bv.union function definitions deleted (if they exist).
- python/tests/test_events.py / test_validate.py / test_app.py — fixtures using event_time_field / tolerate_delay updated; new tests added asserting decorator-time TypeError when event_time / tolerate_delay are passed.
