# Phase 12.6 Plan 08: Python SDK event-time / joins / unions removal — Summary

**The Beava v0 Python SDK (`python/beava/`) no longer accepts `event_time` field declarations, the `tolerate_delay` parameter, or the `event_time_field` parameter on `@bv.event`. The strict-deny fires at decorator time with a structured TypeError that explicitly cites the 2026-04-30 no-event-time pivot. `bv.join` / `bv.union` are confirmed absent from the `bv` namespace; the legacy `tally` package's join/union surface is out of plan scope and stays untouched.**

This implements the public-API layer of `project_redis_shaped_no_event_time_ever` (locked 2026-04-30) at the Python SDK boundary. Plan 04 deleted the server-side join/union OpNode variants. Plan 06 deleted the server-side event-time wire schema. Plan 08 brings the Python decorator surface in sync — every stale fixture now fails fast at decoration time, with a clear migration path in the error message.

## Surface deletions

### 1. `@bv.event` decorator — strict-deny on event_time field

`python/beava/_events.py::_decorate_event_class` was:

```python
event_time_field: str | None = None
if "event_time" in schema:
    py_t = schema["event_time"].py_type
    if py_t is not int and py_t is not datetime.datetime:
        raise TypeError(
            f"event_time field must be int (milliseconds epoch) or datetime.datetime, "
            f"got {py_t!r}; if omitted the server stamps wall-clock on receipt"
        )
    event_time_field = "event_time"
```

Now:

```python
if "event_time" in schema:
    raise TypeError(
        "event_time field on @bv.event is no longer supported per the "
        "no-event-time pivot (2026-04-30). Beava is processing-time only; "
        "remove the event_time field from your event class. The server "
        "stamps wall-clock arrival time on every push automatically."
    )
```

### 2. `@bv.event` decorator — strict-deny on legacy keyword arguments

`python/beava/_events.py::event(...)` signature:

```python
def event(
    arg: Any = None,
    *,
    keep_events_for: str | None = None,
    dedupe_key: str | None = None,
    dedupe_window: str | None = None,
    **legacy_kwargs: Any,
) -> Any:
    if "tolerate_delay" in legacy_kwargs:
        raise TypeError(
            "tolerate_delay parameter on @bv.event is no longer supported per "
            "the no-event-time pivot (2026-04-30). Remove the parameter; the "
            "server stamps processing time on every push so there is no late-"
            "arrival window to tolerate."
        )
    if "event_time_field" in legacy_kwargs:
        raise TypeError(
            "event_time_field parameter on @bv.event is no longer supported "
            "per the no-event-time pivot (2026-04-30). Remove the parameter; "
            "the server stamps processing time on every push automatically."
        )
    if legacy_kwargs:
        first = next(iter(legacy_kwargs))
        raise TypeError(
            f"event() got an unexpected keyword argument {first!r}"
        )
    # … rest of decorator …
```

## Architectural commitment posture

This plan implements the SDK side of `project_redis_shaped_no_event_time_ever` (locked 2026-04-30). Key constraints honored:

1. **No parse-and-strip / no warn-then-error** — D-03 verbatim. The strict-deny raises TypeError with structured codes at decorator time, NOT silently drops the legacy surface.
2. **No "deprecation period" wording** — error messages say "no longer supported per the no-event-time pivot" and explain the migration path. They do not say "will be removed in a future version."
3. **No `force=True` override or compat-mode toggle** — there is no `BEAVA_LEGACY_EVENT_TIME=1` Python env-var escape hatch. Removed means removed.
4. **No silent import-time downgrade** — `bv.event(tolerate_delay=...)` and `bv.event(event_time_field=...)` raise TypeError immediately; they don't get logged as warnings or auto-stripped.
5. **bv.join / bv.union HARD removed** — the success criterion was "DELETE entirely (no parse-and-strip; hard remove)". Verified by `hasattr(bv, 'join')` and `hasattr(bv, 'union')` returning `False`. The fact that they were already absent from `python/beava/__init__.py` (never re-added post-Plan-04 audit) means hard removal is satisfied.

## Decisions Made

See frontmatter `key-decisions:` for the full list. Key calls:

- **`**kwargs`-capture for legacy keyword strict-deny** — preserves DX with explicit, structured TypeError messages.
- **Schema-side detection in `_decorate_event_class`** — single check catches both `event_time: int` and `event_time: datetime` annotations.
- **Field rename rather than deletion in test fixtures** — `event_time: int` → `ts: int` preserves op-chain shape semantics.
- **VALIDATION_ERROR_KINDS narrowed** — `event_time_field_invalid` is dead code post-pivot; surfacing the deletion in the kind-set tightens the contract.
