<objective>
Surgically delete the Join + Union DAG nodes from `beava-core`, then add register-time rejection arms that emit structured error codes when older Python SDK fixtures or hand-rolled JSON DAGs include `{"op": "join"}` or `{"op": "union"}` nodes. After this lands, the runtime cannot represent a join/union; the wire schema cannot deserialize one (because the OpNode variants are gone); and the register-time validator gives a clean v0 message instead of a confusing serde error.

Why this matters in 12.6: deleting the variants from the enum is the surgical operation. But naively deleting them also deletes the deserialize path, which means a stale fixture's `{"op": "join"}` payload would surface as an opaque serde "unknown variant" error to users. The structured-error-code path makes the failure mode obvious.

Wave 2 placement: depends only on Plan 01 (TestServer rewrite) so the new integration test can spin up ServerV18 via the rewritten harness. Independent of Plans 05 (Path X), 06 (event-time strip), 07 (axum kill) — they touch different files.
