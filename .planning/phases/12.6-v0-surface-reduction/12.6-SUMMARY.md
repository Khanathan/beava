## Executive summary

Phase 12.6 is the architectural moment where Beava v0's surface collapses to exactly what `project_redis_shaped_no_event_time_ever` (locked 2026-04-30) defines as the v0 architecture. **mio is now the SOLE data-plane runtime** per `project_phase18_no_dual_runtime`; the legacy axum data plane (~7475 LOC across `push.rs` / `http.rs` / `push_and_get.rs` / `tcp.rs` / legacy `Server` struct + co-located router) is **deleted**; the tokio admin sidecar (`http_admin.rs::BoundAdminServer`) is preserved on its own port (`cfg.admin_addr`) for `/health`, `/ready`, `/metrics`, `/registry` only.

The **event-time hard rip** (D-03) deletes `event_time_ms` / `event_time_field` / `tolerate_delay_ms` from every layer — push wire schema (HTTP JSON + framed-TCP), register wire schema, `EventDescriptor` struct, `DevAggState` global, WAL record schema (v1→v2), snapshot body + header schemas (v1→v2 each), and the Python SDK decorator surface. Stale fixtures hit structured 400 errors with codes `unknown_field_event_time_v0` / `unknown_field_tolerate_delay_v0` per CONTEXT D-03 verbatim — no parse-and-strip, no warn-then-error, no compat shim, no v(N-1) WAL replay.

**Joins and unions** (`OpNode::Join`, `OpNode::Union`, `JoinType { Inner, Left }`) are **deleted** from `beava-core/src/op_node.rs`; the register-time validator rejects residual DAG references at the JSON layer with structured codes `feature_removed_no_joins_v0` / `feature_removed_no_unions_v0`. The Python SDK confirms `bv.join` / `bv.union` absent from the `bv` namespace; the legacy `tally` package is out of scope per D-05.

**Path X** (windowed-op time-source swap) renames the `event_time_ms: i64` parameter to `now_ms: i64` across the AggState trait, every concrete state struct, every WindowedOp dispatch arm, the apply_shard, and the AggOp wrapper enum's 71 dispatch arms. "Rolling 60s sum" now means 60s of arrival-time. The 55-operator catalogue is preserved; no operators added or removed.

**Quantitative results:** ~7475 LOC removed by Plan 07 (legacy axum kill); ~3500 LOC of Plan 04 (joins/unions) + Plan 06 (event-time) deletions; the workspace test count is **1067 passed / 0 failed / 3 ignored** at HEAD `1e318b1`. Plan 11's criterion microbench captures 3 cells (simple_counter @ 810 ns/event, sketch_heavy @ 884 ns/event, windowed_60s_sum @ 883 ns/event) as the post-axum-kill baseline. Plan 12's throughput rebaseline captures 8 cells (small/medium/large/fraud-team × http+tcp); the regression-gate cell (small/tcp) is at **-0.94%** vs the post-12-08 baseline (700,571 EPS vs 707,237 EPS) — well within the ±10% PASS band. The fraud-team primary tuning bench is at **+0.5%** (102,788 EPS vs 102,291 EPS).

The Python SDK's `@bv.event` decorator now raises `TypeError` at decoration time for `event_time` field declarations and `tolerate_delay` / `event_time_field` keyword arguments, with structured error messages that explicitly cite the no-event-time pivot. `bv.join` / `bv.union` are confirmed absent from the public namespace.

**Section ownership** for shared docs (`STATE.md`, `ROADMAP.md`) was partitioned between Plan 09 (Wave 4: Phase 12.5 archival, worktree-status block, dir banners) and Plan 13 (Wave 8: Phase 12.6 entry header, v0 critical-path block, phase-progress block, Current Focus line); the boundary held with no collisions.

## Architectural outcomes

### Locked architectural commitments (post-12.6, all enforceable)

1. **mio is the SOLE data-plane runtime.** All push/get/upsert/delete/retract/table_get traffic dispatches through `apply_shard.rs::dispatch_*_sync`. Plan 10's architectural test `phase12_6_mio_only_dataplane.rs` walks the workspace at `cargo test` time and fails on any third caller of `apply_event_to_aggregations` or `axum::*` symbol outside `http_admin.rs`. Companion: `phase12_6_legacy_axum_killed.rs` asserts the deleted files stay deleted.

2. **No event-time anywhere.** `event_time_ms` / `event_time_field` / `tolerate_delay_ms` are gone from push wire (HTTP + framed-TCP) + register wire + WAL records (v=2) + snapshot body (v=2) + snapshot header (v=2) + EventDescriptor + DevAggState + Python @bv.event decorator. Strict-deny at register/push/decorator boundaries with structured error codes. Pre-pivot WALs and snapshots fail with `SchemaVersionMismatch`.

3. **No joins, no unions.** `OpNode::Join` + `OpNode::Union` + `JoinType { Inner, Left }` deleted from `beava-core/src/op_node.rs`. Register-time validator rejects stale `{op:join}` / `{op:union}` payloads at the JSON layer with `feature_removed_no_*_v0`. `bv.join` / `bv.union` absent from Python `bv` namespace.

4. **Windowed ops use server-side `now_ms()`.** Path X swap renamed the parameter across the AggState trait + 22 concrete state structs + every WindowedOp dispatch arm + the AggOp wrapper enum's 71 dispatch arms + agg_apply + apply_shard. "Rolling 60s sum" = 60s of arrival-time. 55-operator catalogue preserved.

5. **Tokio admin sidecar restricted to admin endpoints.** `http_admin.rs::BoundAdminServer` lives on `cfg.admin_addr`; the data-plane port speaks pure mio + custom-framed TCP + minimal HTTP routing.

## Lessons learned

These patterns emerged worth saving as memories for future feature-deletion phases:

### 1. Pre-deserialize JSON-prelude shim for permanently-removed features (Plans 04, 06)

When an architectural pivot retires a feature, ship a JSON-layer pre-check shim BEFORE the strict deserialize path so stale fixtures get a structured `feature_removed_*` / `unknown_field_*_v0` error code instead of an opaque "unknown variant" serde error. Same shim pattern used twice — Plan 04 for joins/unions, Plan 06 for event-time keys — landing in `crates/beava-core/src/register_validate.rs` (single source of truth) and wired into all 3 `/register` dispatch entry points (mio sync apply thread, legacy async glue, legacy axum router for the brief window before Plan 07's deletion sweep).

**Reusable for:** any future v0.x or v0.1+ feature retirement where backward-compat error semantics matter and silent-deserialize-failure is unacceptable DX.

### 2. TDD ordering for variant-delete-with-rejection (Plan 04)

When deleting an enum variant whose previous payloads should still produce structured error codes, land the structured-error-emitting shim FIRST (with variants still alive — shim catches at JSON layer) in commit N, then delete the variants in commit N+1 (rejection path stays correct because shim runs before serde reaches the now-deleted variant). No commit re-architects the validator mid-task. Each commit's contribution is unambiguous; rollback is a clean revert of one commit.

### 3. Hard-rip + fixture-sweep posture (Plan 06)

D-03 hard-rip means no silent-ignore at any layer. The cost is migrating ~50 test fixtures, NOT a backwards-compat shim. Surface the cost openly: the test-fixture sweep is its own RED→GREEN gate, run as the final commit of the rip plan. This decouples the structural cleanup (variants/fields gone) from the migration (fixtures updated), keeping each commit's purpose unambiguous.

### 4. TestServer accessor shim (Plan 01)

`ServerV18::bind_with_state` provides eager `AppState` / `Registry` / `WalSink` / `WalWriter` construction so a TestServer harness can clone `Arc` handles BEFORE `serve_with_dirs` consumes the server. The accessor methods are gated `#[cfg(any(feature = "testing", test))]` so the production binary never sees them. Reusable for any future white-box-test harness that needs internal handles without re-architecting the production constructor.

### 5. SystemTime::now() vDSO cost (Plan 05 Path X)

~10-30 ns/event on macOS/Linux; well within the 500 ns agg-stage budget. Suggests Path X is a near-zero-cost architectural change for the operator catalogue. Plan 12's throughput rebaseline confirmed: net delta ~neutral on the production-relevant cells (small/tcp -1%, fraud-team/tcp +0.5%).

### 6. Architectural-test-as-grep (Plan 10)

Runtime-grep tests in `phase12_6_mio_only_dataplane.rs` catch import-by-string-name bypasses that `#[deny]` lints miss (e.g., a `let _: Option<axum::Router> = None;` line in production code). Cheaper than refactoring the linter; equally effective for one-of-a-kind invariants. Add a sanity test (`sanity_allowlisted_callers_actually_contain_patterns`) so the test doesn't silently become a no-op if a refactor moves the canonical sites.

### 7. Section-ownership contract for shared docs (Plans 09 + 13)

When two plans in different waves edit the same doc (`STATE.md`, `ROADMAP.md`), partition the doc by section in advance and call out the partition in BOTH plans' frontmatter. Verify the boundary at commit time via `git diff` inspection BEFORE the commit lands. Prevents Wave-N's edits from colliding with Wave-M's. The contract worked perfectly in Phase 12.6 (Plan 09 owned Phase 12.5 entry + worktree-status block; Plan 13 owns Phase 12.6 entry + v0 critical-path block + phase-progress block + Current Focus line).

### 8. WalWriter-required-for-bench (Plan 11)

Bench bootstrap helpers spawning a `WalBufferRing` MUST also spawn a `WalWriter` thread. Without a writer, sealed buffers never return to FREE state and `WalBufferRing::append` blocks indefinitely on `free_condvar` once the 3 × 16 MiB ring exhausts. Documented in the helper's module-level docstring + a long inline comment so the reason isn't lost on future readers.

### 9. Plan-author estimates can underestimate gap-closure scope (Plans 14, 15)

Plan 01's "the ~20 test files barely change" assumption was true at the source level (no test code needed to change for `.spawn()` / `.builder()` API) but required substantial mio-side wire-shape additions (HTTP /register response shape parity, /ready and /registry routes on data-plane port, 404/405 errors, Content-Type 415 enforcement, idem-cache replay byte-identity, snapshot recovery on mio path, WalWriter shutdown-flag join, TCP error-frame surface for op_not_implemented / unsupported_content_type / frame_too_large, dev_endpoints flag on AppState). Plans 14+15 closed all 44 broken tests inline rather than splitting to a separate plan or escalating as architectural checkpoints — the executor latitude granted by Rules 1+2 covered the scope expansion. Document the discoveries clearly (see `12.6-14-SUMMARY.md` Deviations + `12.6-15-SUMMARY.md` Deviations) so the audit trail captures what was unplanned vs planned.
