# Phase 12.6 Plan 14: Mio data-plane HTTP gap closure — Summary

Plan 01's TestServer→ServerV18 swap surfaced a fleet of broken tests because the legacy axum HTTP surface (`/upsert`, `/delete`, `/retract`, `/table`, `/registry` dev-gating, Content-Type 415) was not mirrored on mio. Plan 14 closes those gaps end-to-end. **27 tests across 4 binaries flip from RED to GREEN**, all wire shapes match legacy axum byte-for-byte, no architectural deviations from the locked-in mio data-plane posture.

## What landed

### Task 1 — POST /upsert/:table on mio
- **Helper:** `temporal_http::upsert_via_mio(app, table_name, body_bytes) -> (u16, Vec<u8>)` — extracted from the legacy axum `upsert_handler` body; same JSON parse, descriptor lookup, entity-key encoding, WAL `TableUpsert` append, MVCC store apply (temporal only), event_id_index update, `next_event_id` bump.
- **Wire:** `Route::Upsert { table }` → `WireRequest::HttpUpsert { table, body }` → `GlueResponse::TemporalResponse { http_status, body }`.
- **Apply path:** `ApplyShard::dispatch_one` runs the helper on a temp current-thread tokio runtime (cold-path; same approach as register dispatch).

### Task 2 — POST /delete/:table on mio
- Same pattern. `delete_via_mio` helper, `Route::Delete`, `WireRequest::HttpDelete`. Returns 200 on temporal-table tombstones, 400 on non-temporal tables, 404 on key-not-found, 503 on WAL unavailable.

### Task 3 — POST /retract on mio
- Same pattern. `retract_via_mio` helper, `Route::Retract`, `WireRequest::HttpRetract`. Preserves D-12 / D-17 error shapes:
  - 404 `event_id_not_found` for unknown event_id
  - 501 `stream_retraction_unimplemented` for stream events (per D-12)
  - 400 `table_not_temporal` for non-temporal table writes (per D-17)
  - 409 `event_id_already_retracted` for idempotent re-retracts
  - 200 with `{ack_lsn, target_event_id, table, restored_to_lsn}` on success

### Task 3b — GET /table/:table?key=...&as_of=... on mio (scope expansion)
- Surfaced during scope discovery: phase11_5 + phase18_07 tests round-trip upsert → table_get and the existing mio router had no `/table` route. Added inline rather than splitting to a separate plan (same architectural shape as the upsert/delete/retract trio).
- **Helper:** `table_get_via_mio(app, table_name, raw_query) -> (u16, Vec<u8>)` — pure sync (no WAL/async I/O). Honors `as_of` query param against temporal stores; returns 400 `as_of_requires_temporal` for non-temporal tables; 404 on key-not-found; 501 on non-temporal table get (Phase 12 deferred).
- **Router:** `Router::route` now strips the query-string before path-segment matching so `/table/foo?key=bar` correctly routes as `Route::TableGet { table: "foo" }`.

### Task 4 — GET /registry dev_endpoints gating on mio
- Added `Arc<AtomicBool> dev_endpoints` field to `AppState` + `dev_endpoints_enabled()` accessor.
- `build_runtime_state` (mio path) and legacy `Server::bind` both honor `BEAVA_DEV_ENDPOINTS=1` env var (production binary contract).
- TestServer's `.dev_endpoints(true)` builder method now flips the flag on the shared `AppState` post-spawn — was a no-op pre-Plan 14.
- Mio dispatch returns `GlueResponse::HttpRouteNotFound { path: "/registry" }` (404) when flag is false; full registry dump when true.

### Task 5 — POST /register Content-Type 415 enforcement
- Added `WireRequest::HttpUnsupportedMediaType { received, path }` and matching `GlueResponse` variant.
- HTTP listener parses the Content-Type header; gates POST /register against `application/json` (case-insensitive media-type match, ignores `; charset=utf-8`).
- Scope: only POST /register validates Content-Type — matches legacy axum's `register::is_json_content_type` exactly. (See Deviations below for the over-broad initial impl that regressed event_loop_smoke; fixed in commit 9c119a1.)

## Deviations from Plan

### Auto-fixed issues (Rule 1 — bug)

**1. [Rule 1 - Bug] Over-broad Content-Type 415 enforcement.**

- **Found during:** Workspace test run after the GREEN commit. `event_loop_smoke::http_post_push_with_content_length_produces_http_push`, `http_post_push_chunked_body_concatenated_correctly`, and `http_three_pipelined_requests_all_parsed` regressed — they deliberately omit Content-Type to verify the parser handles header-absent cases.
- **Issue:** Initial impl applied 415 to ALL POST endpoints (Push/PushSync/PushBatch/Get/Upsert/Delete/Retract/Register). Legacy axum had Content-Type validation only on `register::post_register`; other POST handlers attempted JSON parse and returned 400 on failure.
- **Fix:** Narrowed the 415 gate to `route == Route::Register && method == "POST"` in `parse_http_request`.
- **Files modified:** `crates/beava-runtime-core/src/http_listener.rs`
- **Commit:** `9c119a1`

### Scope expansion (Rule 2 — missing critical functionality)

**2. [Rule 2 - Missing functionality] GET /table/:name route.**

- **Found during:** Initial test compilation. `phase11_5_temporal_smoke::temporal_table_upsert_retract_returns_prior_value` and `phase18_07::test_upsert_stores_row_readable_via_table_get` round-trip upsert→table_get; the mio router had `/upsert /delete /retract` planned but no `/table`.
- **Issue:** Without `/table` GET, even after impl-ing upsert/delete the integration tests can't verify state because lookup returns 404 from mio's NotFound shim.
- **Fix:** Added inline as Task 3b. Same architectural shape as upsert/delete/retract — pure helper extraction (`table_get_via_mio` is sync since no WAL involved), Route → WireRequest → GlueResponse → encoder.
- **Files modified:** `crates/beava-runtime-core/src/router.rs`, `crates/beava-runtime-core/src/wire_request.rs`, `crates/beava-runtime-core/src/http_listener.rs`, `crates/beava-server/src/temporal_http.rs`, `crates/beava-server/src/runtime_core_glue.rs`, `crates/beava-server/src/apply_shard.rs`
- **Commit:** Bundled with the GREEN feat commit (`ba5048e`)

The plan-author flagged scope-expansion latitude in the prompt (*"If during scope discovery you find the gap is bigger than expected ... surface it inline by extending Plan 14's scope"*); calling this out for Plan 14's audit trail.
