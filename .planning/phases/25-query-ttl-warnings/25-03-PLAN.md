```rust
// src/engine/warnings.rs

#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Serialize, Deserialize)]
pub enum Severity { Info, Warning, Error, Critical }
// NOTE: derive Ord such that Critical > Error > Warning > Info
// (reverse-order in sort for the feed).

#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
pub enum Category { Config, DataQuality, Operational, Safety, Performance }

#[derive(Debug, Clone, Serialize)]
pub struct Warning {
    pub id: String,              // stable identifier for dedupe
    pub severity: Severity,
    pub category: Category,
    pub title: String,
    pub detail: String,
    pub action: Option<serde_json::Value>,
    pub details: Option<serde_json::Value>,   // rich evidence
    pub evidence_url: Option<String>,
    pub first_seen: SystemTime,
    pub last_seen: SystemTime,
}

pub struct WarningRegistry {
    inner: DashMap<String, Warning>,
}
impl WarningRegistry {
    pub fn new() -> Self;
    pub fn upsert(&self, w: Warning);            // same id → update last_seen + details
    pub fn resolve(&self, id: &str);              // remove — signal cleared
    pub fn snapshot(&self) -> Vec<Warning>;       // severity-sorted
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin HTTP (`/debug/warnings`) | Existing loopback-or-token gate |
| WarningRegistry (in-process) | Upsert from multiple async tasks |
| Bench harness → server | Local-only; no external trust surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-03-01 | Information Disclosure | /debug/warnings exposes pipeline names, stream names, registration errors | mitigate | Admin-gated via existing loopback-or-token middleware (same as /debug/config-recommendations) |
| T-25-03-02 | Denial of Service | Unbounded WarningRegistry growth via distinct ids | mitigate | Cap at 1024 entries; evict oldest-first when full; counter `tally_warnings_evicted_total` for observability |
| T-25-03-03 | Tampering | Race between signal-check ticker upsert + admin resolve | accept | DashMap entry-level locking; worst case a resolve followed by immediate re-upsert (signal still present) — idempotent |
| T-25-03-04 | Denial of Service | 1 Hz ticker blocks the runtime | mitigate | Ticker is a dedicated tokio task with work-capped checks (each signal bounded by O(tables) or O(streams)); benchmark gate catches regressions > 5% |
| T-25-03-05 | Information Disclosure | Registration-failure warning leaks submitted JSON | mitigate | Warning.detail is truncated to 256 chars + contains pipeline_name only; full error in server logs for operator, not /debug |
| T-25-03-06 | Tampering | Benchmark gate false-pass due to host warmup variance | mitigate | 7-run median per cell; explicit host-load warmup before first cell (carry pattern from 24-05) |
</threat_model>
