<objective>
Fix the `agg_windowed` bucket-reset silent-data-loss bug by widening `bucket_ms` from `window / 64` to `(window + tolerate_delay) / 64`. Thread `tolerate_delay_ms` from the source `EventDescriptor` through the aggregation compile pipeline so each `WindowedOp` is sized against `window + tolerate_delay`. Land a regression test encoding the original bug's repro so we can never regress it.

Purpose: Today, an in-tolerance late event (within `tolerate_delay` of the watermark) can silently destroy a neighbouring fresh event's contribution via the `bucket_epoch_start_ms[idx] != bucket_epoch` stale-reset branch. Widening the bucket so that within-tolerance late events always share an epoch with the fresh neighbour removes this class of bug by construction.
Output: Updated `WindowedOp` constructor signature, updated `agg_compile`, regression test.
</objective>
