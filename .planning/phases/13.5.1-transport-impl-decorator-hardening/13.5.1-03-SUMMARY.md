## D-03 Coverage Confirmation (PART 1 + PART 2)

The plan-checker iter-1 fix that locked **both** Test 4 and Test 4b into this plan is preserved verbatim:

- **PART 1 — App.get features filter (Test 4):**
  ```python
  narrowed = transport_app.get("UserTxn", "alice", features=["c"])
  assert set(narrowed.keys()) == {"c"}
  assert narrowed["c"] == 4
  ```
  Catches the wire-level `features` kwarg on `Transport.send_get` AND the `App.get(features=[...])` plumbing. Without this, an HTTP-only or TCP-only `send_get` impl that hard-codes `features` to `None` would slip through.

- **PART 2 — App.batch_get per-entry features filter (Test 4b):**
  ```python
  results = transport_app.batch_get([
      {"table": "UserTxn", "key": "alice"},
      {"table": "UserTxn", "key": "bob", "features": ["c"]},
  ])
  # If dict-shape raises TypeError/ValueError, fall back to tuple-of-3:
  # results = transport_app.batch_get([("UserTxn", "alice"), ("UserTxn", "bob", ["c"])])
  assert set(results[0].keys()) == {"c", "s"}  # full row
  assert set(results[1].keys()) == {"c"}        # narrowed
  ```
  Heterogeneous batch matches `examples/wire/batch_get-heterogeneous.request.json`. The `try/except (TypeError, ValueError)` future-proofs against Plan 05 shipping either dict-shape (matches wire spec exactly) OR tuple-of-3 widening — EITHER design satisfies this test.

## Import Pattern Pinning (rewrite-blocker fix)

The plan-checker iter-1 blocker was `from python.tests.v0.conftest import cold_start_equivalent` — that path is NOT importable because `python/` has no `__init__.py` (pytest collects `conftest.py` via plugin mechanism, not as a regular module). This file uses the canonical pattern from `test_core.py:20`:

```python
from ._helpers import _engine_available, cold_start_equivalent
```

`python/tests/v0/_helpers.py` already re-exports both names from `conftest` (lines 22-27), so this is a one-hop relative import that every other test file in `tests/v0/` uses. NO inline-helper copy was needed; NO fallback `try/except ImportError`; NO absolute `python.tests.v0.*` form.

## Anti-Pattern Guards (D-05 USER-LOCKED)

- **MagicMock count:** 0 (verified by `grep -c "MagicMock\|unittest.mock"`).
- **All transport methods exercised against a real engine** — `bv.App(test_mode=True)` for embed; `bv.App(url=…)` against `spawn_embedded_server(test_mode=True)` for http/tcp.
- **Per-test `@bv.event` / `@bv.table` definitions** — keeps test bodies self-contained, prevents cross-test decorator side-effects.

- **try/finally subprocess teardown** — http/tcp branches wrap the `bv.App(url=…)` context in `try: yield ... finally: teardown_process(proc)` so a failing test still releases the spawned binary.
