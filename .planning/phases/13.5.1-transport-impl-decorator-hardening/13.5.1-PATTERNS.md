### Target: `HttpTransport.send_push / send_get / send_batch_get / send_reset`

**Analog:** `python/beava/_transport.py:132-158` (`HttpTransport.send_register`).

The new methods mirror this exactly: same `httpx.Client` instance, same `r = self._client.post(<verb-route>, content=<json-bytes>, headers={"Content-Type": "application/json"})` shape, same 200-vs-error branching that raises `RegistrationError` (or a sibling `BeavaError` if D-01 lands a separate class).

```python
# python/beava/_transport.py:132-158 — HttpTransport.send_register
def send_register(self, payload_json: bytes) -> dict:  # type: ignore[type-arg]
    """POST /register with JSON payload.

    Args:
        payload_json: UTF-8 JSON bytes matching the Phase 2 wire contract.

    Returns:
        Parsed success body dict (``status='ok'``, ``registry_version=N``, …).

    Raises:
        RegistrationError: Server returned 4xx or 5xx with a JSON error body.
    """
    r = self._client.post(
        "/register",
        content=payload_json,
        headers={"Content-Type": "application/json"},
    )
    body = r.json()
    if r.status_code == 200:
        return body  # type: ignore[no-any-return]
    error = body.get("error", {})
    raise RegistrationError(
        code=error.get("code", "unknown"),
        path=error.get("path", ""),
        message=error.get("reason") or error.get("message", ""),
        errors=[],
    )
```

**Wire contract bindings** (verb-style, Phase 13.4-locked, see `docs/http-api.md:161-430`):

| Method | HTTP route | Request body shape | Success-200 body shape |
|---|---|---|---|
| `send_push(event_name, fields)` | `POST /push` | `{"event_name": "...", "fields": {...}}` | `{"ack_lsn": int, "registry_version": int}` |
| `send_get(table, key, features?)` | `POST /get`  | `{"table": "...", "key": "..."|[...], "features"?: [...]}` | row-shape flat dict (cold = `{}`) |
| `send_batch_get(requests)` | `POST /batch_get` | `{"requests":[{"table","key","features"?}...]}` | `{"results":[<row-shape>, ...]}` |
| `send_reset()` | `POST /reset` | `{}` | `{"status":"ok"}` (403 `reset_disabled` if not test_mode) |

**Note on `send_batch_get` return shape:** the base-class signature returns `list[dict]`, but the wire response is `{"results":[...]}`. The transport MUST unwrap `body["results"]` so the App layer (`_app.py:306-319`) can return the list directly without re-handling.

**Note on `send_reset` 403:** when `enable_reset_op=false` (i.e. server not in test_mode per Phase 13.4 D-03), HTTP returns `403 reset_disabled`. The transport must surface this as a typed exception (`RegistrationError(code="reset_disabled", ...)` is fine in v0; a dedicated `ResetDisabledError` is allowed but not required).

### Target: `TcpTransport.send_get / send_batch_get / send_reset`

**Analog:** `python/beava/_transport.py:264-286` (`TcpTransport.send_push`). The OP_PUSH envelope already shows the shape: encode the body dict as JSON bytes via `json.dumps(...).encode("utf-8")`, send through `encode_frame(OP_*, CT_JSON, envelope)`, read the response with `read_frame(...)`, JSON-decode the payload, return the dict.

```python
# python/beava/_transport.py:264-286 — TcpTransport.send_push (mirror this for get/batch_get/reset)
def send_push(
    self,
    *,
    event_name: str,
    fields: dict[str, Any],
) -> dict[str, Any]:
    """Send an OP_PUSH frame with the given event name and body fields.

    Encodes the envelope ``{"event": event_name, "body": fields}`` as JSON
    and sends it as an OP_PUSH frame. Default JSON encoding for v0
    compatibility with the server's CT_JSON path.

    Returns:
        Parsed JSON ACK dict from the server (e.g. ``{"ack_lsn": 42}``).
    """
    envelope = json.dumps(
        {"event": event_name, "body": fields}, ensure_ascii=False
    ).encode("utf-8")
    sock = self._ensure_connected()
    sock.sendall(encode_frame(OP_PUSH, CT_JSON, envelope))
    frame = read_frame(sock, self.max_frame_bytes)
    result: dict[str, Any] = json.loads(frame.payload.decode("utf-8"))
    return result
```

**Per-op TCP body shapes** (D-02 = JSON default; matches `docs/wire-spec.md:270-397`):

```python
# python/beava/_events.py:241-269 — _make_event_derivation
def _make_event_derivation(fn: Callable[..., Any]) -> EventDerivation:
    sig = inspect.signature(fn)
    params = list(sig.parameters.values())
    if not params:
        raise TypeError(
            f"@bv.event function {fn.__name__!r} must take at least one parameter"
        )
    try:
        resolved = get_type_hints(fn)
    except Exception:
        resolved = {}
    upstream_proxies: list[Any] = []
    for p in params:
        ann = resolved.get(p.name, p.annotation)
        if isinstance(ann, str):
            ann = fn.__globals__.get(ann, ann)
        upstream_proxies.append(ann)
    result = fn(*upstream_proxies)
    if not isinstance(result, EventDerivation):
        raise TypeError(
            f"@bv.event function {fn.__name__!r} must return a chain "
            f"(filter/select/group_by/agg/...); got {type(result).__name__}"
        )
    result._name = fn.__name__
    return result
```
