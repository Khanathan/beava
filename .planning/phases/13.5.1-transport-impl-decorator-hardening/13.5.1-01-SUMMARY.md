## D-01 Verbatim TypeError Substring Asserted

Per `13.5.1-CONTEXT.md` §D-01 (USER-LOCKED — em-dash U+2014 preserved in Plan 04 impl):

```
@bv.table function 'UserClicks' parameter 'clicks' must be annotated
with the upstream event class — e.g. def UserClicks(clicks: Click): ...
```

The RED tests assert this contract via:

1. **Stable-prefix regex** (passed as `pytest.raises(..., match=...)`):
   ```
   @bv\.table function '\w+' parameter '\w+' must be annotated
   ```
   Resilient to whitespace tweaks; uses `\w+` placeholders for the function and parameter names so the same regex covers all three RED cases.

2. **Three substring assertions** on `str(excinfo.value)`:
   - The function name (`'UserClicks'` / `'UserPageClicks'` / `'TotalClicks'`)
   - The parameter name (`'clicks'`)
   - The verbatim suffix `must be annotated with the upstream event class`

The em-dash (U+2014) and full corrected-signature snippet (`e.g. def Fn(clicks: Click): ...`) deliberately live OUTSIDE the regex to keep the test resilient to whitespace / hyphenation tweaks; they will be exercised verbatim in Plan 13.5.1-04's GREEN test pass via the substring assertion's `must be annotated with the upstream event class` anchor.

**Failure mode (canonical anti-pattern A2 from PATTERNS.md):**

```
AttributeError: type object '_empty' has no attribute 'group_by'
AttributeError: type object '_empty' has no attribute 'agg'
```

Currently, `python/beava/_table.py:48-70::_resolve_upstream_proxies` silently
returns `inspect.Parameter.empty` for missing annotations (lines 65-69):

```python
for p in params:
    ann = resolved.get(p.name, p.annotation)   # <- yields _empty sentinel
    if isinstance(ann, str):
        ann = fn.__globals__.get(ann, ann)
    proxies.append(ann)
return proxies
```

The sentinel then propagates into `_make_table` (line 77: `result = fn(*proxies)`) where the user's body invokes `.group_by(...)` / `.agg(...)` on the `_empty` class object — surfacing as `AttributeError` rather than the strict `TypeError` D-01 mandates. Plan 13.5.1-04 GREENs this by inserting:

```python
if ann is inspect.Parameter.empty:
    raise TypeError(
        f"@bv.table function {fn.__name__!r} parameter {p.name!r} "
        f"must be annotated with the upstream event class — "
        f"e.g. def {fn.__name__}({p.name}: Click): ..."
    )
```

The positive-control test (Test 4) is GREEN at HEAD AND will stay GREEN
post-Plan-04 — D-01 only adds a raise on the empty path; the happy path
through `_resolve_upstream_proxies` is untouched.
