# Plan 13.5.1-07d ‚Äî Global-agg test data hygiene (Category 5)

## Objective

Repair the four `python/tests/v0/test_global.py` tests flagged by Plan 13.5.1-07 triage as Category 5 ("test design issues with empty / unrelated event sources"). Pure test-data hygiene; no SDK or server changes. After this plan, `pytest python/tests/v0/test_global.py -v` is fully GREEN (8/8).

## Context

Source triage: `13.5.1-07-PLAN.md` lines 140-150 + 292-302. Locked SDK contract: `docs/sdk-api/python.md` global-agg section. Semantic spec: ADR-003 (`.planning/decisions/ADR-003-global-aggregation-and-bv-lit.md`).

The Plan 05 SUMMARY narrative ‚Äî "EmptyTotal aggregation registered against UnrelatedEvent" ‚Äî is the *literal* root cause of `test_global_cold_start`. The other three Category-5 tests share the same archetype (test-data design errors that mis-encode the test's intent), but with mechanically distinct symptoms:

| Test | Confirmed RED symptom (this plan, fresh `pytest` run) | Mechanical fix |
|------|--------------------------------------------------------|----------------|
| `test_global_top_k_pages` | `assert {'count': 507, 'value': 'page-0'} == 'page-0'` ‚Äî assertion compares the dict-shaped `top_k` element to a bare string | Extract `.value` (or dict-aware key) when reading the top-1 page |
| `test_global_form_equivalence` | `RegistrationError: aggregation_feature_name_collision_across_aggregations` on feature name `t` (declared twice across `TotalForm1` + `TotalForm2`) | Use distinct feature names per form (`t1`, `t2`) and assert their values match |
| `test_global_app_get_arity_mismatch` | Same collision on feature name `total` shared between `PageTotal` (global) + `UserPageTotal` (per-entity) | Use distinct feature names (`page_total`, `user_page_total`) |
| `test_global_cold_start` | `assert {'total': 500} == {'total': 0}` ‚Äî `EmptyTotal` registered against `UnrelatedEvent`, then 500 `UnrelatedEvent` events get pushed ‚Üí counter advances to 500 | Re-bind `EmptyTotal` to `TargetEvent` so `UnrelatedEvent` traffic doesn't feed it |

All four are test-side errors. The SDK and engine semantics around global aggregation, feature-name uniqueness, and `top_k` output shape are correct; the tests had encoding bugs that only surfaced when the global-agg engine started actually working in Plan 05.

## Decisions

- **D-01 ‚Äî Distinct feature names across aggregations.** The server enforces that feature names are globally unique across aggregation nodes (`aggregation_feature_name_collision_across_aggregations`). Test 6 + Test 7 must respect this; rename collided features to per-form / per-table distinct names. (No SDK or server change.)

- **D-02 ‚Äî `top_k` returns dict elements, not bare strings.** `bv.top_k` outputs `[{value, count}, ...]`; the test was written assuming bare-string elements. Update assertion to read `actual[0]["value"]`. The existing branch (`isinstance(actual[0], (list, tuple))`) is dead ‚Äî `top_k` never returns tuples per the live SDK. Replace with the correct dict-shape extraction.

- **D-03 ‚Äî `test_global_cold_start` event-source binding.** Re-bind `EmptyTotal(targets: TargetEvent)` so that pushing 500 `UnrelatedEvent` events yields zero entries on `EmptyTotal`. Matches the test's docstring intent verbatim ("EmptyTotal's source (TargetEvent) gets ZERO events").
