<declaration_site_contract>
## `@bv.event` / `@bv.table` upstream-class resolution contract (formalized)

The resolver MUST find an upstream class by name in this order, and STOP at the first hit:

1. **`fn.__globals__`** — module-level declarations. The canonical, mypy-friendly location.
2. **Enclosing closure cells** — `fn.__closure__` paired with `fn.__code__.co_freevars`. Captures lexically-enclosed names from outer fn scopes (so `def factory(): class Inner: ...; @bv.table def Fn(x: Inner): ...` works without frame-walking).
3. **Single-frame caller `f_locals`** — exactly one frame back (`inspect.currentframe().f_back`), bounded. Captures the pytest-fixture pattern: `def test_x(): @bv.event class Txn: ...; @bv.table def UserTxns(t: Txn): ...`.

**NOT walked:**
- Multiple frames back (the old 8-frame walk) — fragile; if the user wraps `@bv.table` in `functools.lru_cache` or runs it through a decorator stack, the frame depth shifts.
- `f_globals` of caller frames — `fn.__globals__` already covers the module that defined `fn`; caller-frame globals are usually the same module anyway.

**Supported declaration sites (documented contract — exhaustive):**

| Site | Mechanism | Example |
|---|---|---|
| Module-level | `fn.__globals__` lookup | `@bv.event class Click: ...` at module top |
| Function-local (pytest fixture) | Caller-frame `f_locals` (one frame back) | `def test_x(): @bv.event class Click: ...; @bv.table def Fn(c: Click): ...` |
| Inner-class (in enclosing fn) | Closure cells | `def factory(): class Click: ...; def make(): @bv.table def Fn(c: Click): ...` |
| `lru_cache`-wrapped factory | Closure cells (the wrapped fn keeps its own closure) | `@functools.lru_cache def make_table(): @bv.table def Fn(c: Click): ...` |
| Class method | Caller-frame `f_locals` (the class body's frame) | `class TestSuite: def setup(self): @bv.table def Fn(c: Click): ...` |

**NOT supported (raise `NameError` at decoration time with a clear hint):**
- Lambdas (lambdas can't host the `@bv.table` body anyway — the decorator requires a real `def` for the upstream-proxy call).
- Names imported via `from x import *` AFTER `@bv.event` resolves them — Python's late-binding import order means our resolver runs at decoration time; if the name doesn't exist yet, it can't be found.

This contract replaces the Plan 05 8-frame magic walk. The resolver is more predictable (no surprising depth), more robust to decorator-stack wrappers (lru_cache works because closure cells are inspected directly, not by frame walk), and the failure mode is a clean `NameError` instead of silent wrong-class resolution at frame N.
</declaration_site_contract>
