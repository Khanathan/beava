<decisions>

### D-01 (5.1-A) — `@bv.table` empty-annotation = strict TypeError (USER-LOCKED)

When a `@bv.table`-decorated function has an unannotated parameter, raise `TypeError` with a helpful message:

```
TypeError: @bv.table function 'UserClicks' parameter 'clicks' must be
annotated with the upstream event class — e.g. def UserClicks(clicks: Click): ...
```

**Rationale:** Predictable, mypy-friendly, mirrors the existing `@bv.event` convention. The 68 acceptance tests that currently use `def UserClicks(clicks):` will be amended to add annotations as part of this phase (mechanical edit). Auto-infer / generic `EventStream` proxy was rejected (adds magic; weakens mypy story).

**Implementation site:** `python/beava/_table.py::_resolve_upstream_proxies`. Replace the silent `inspect.Parameter.empty` fallback with the explicit `TypeError` raise.

### D-02 (5.1-B) — TCP read-path wire format default = JSON (USER-LOCKED)

`TcpTransport.send_get` / `send_batch_get` default to JSON (`CT_JSON`) — matches `send_push`. Users opt into msgpack via the existing `wire_format="msgpack"` kwarg pattern carried over from Plan 12-09's `tcp_get_single`.

**Rationale:** One fewer runtime dep on the happy path; `msgpack` package only needed by users who actively flip the kwarg. Simpler v0; fast-path users still have the escape hatch. Performance regression-gate already passes with JSON-only (Phase 13.5 small/tcp -7.1% PASS).

### D-03 (5.1-C) — `send_get` exposes optional `features` filter (USER-LOCKED)

The base-class `Transport.send_get(*, table, key)` is amended to:

```python
def send_get(
    self,
    *,
    table: str,
    key: str | list[Any],
    features: list[str] | None = None,
) -> dict[str, Any]:
    ...
```

Default `features=None` returns the full row (Redis-shaped behavior). Passing `features=["count"]` narrows the response. Plumbed through HTTP (`POST /get` body `{table, key, features?}`) and TCP (frame body identical).

`bv.App.get(table, key)` and `bv.App.get(table, key, features=[...])` both work; `App.batch_get` per-entry shape similarly accepts an optional `features` per entry.

### D-04 (5.1-D) — Legacy `*_get_single` helpers → private, removed in v0.0.x (USER-LOCKED)

Rename:
- `TcpTransport.tcp_get_single` → `TcpTransport._tcp_get_single`
- `HttpTransport.http_get_single` → `HttpTransport._http_get_single`
- `EmbedTransport.tcp_get_single` → `EmbedTransport._tcp_get_single`

Update internal tests that call them. Public surface = `send_get` only. Schedule full removal for v0.0.x cleanup (track in `.planning/ideas/v0.1-deferrals.md` or equivalent).

### D-05 (5.1-E) — v0 acceptance test harness = embed-mode + small cross-transport smoke (USER-LOCKED)

The 68 v0 acceptance tests under `python/tests/v0/test_<family>.py` run against `bv.App(test_mode=True)` (embed subprocess; `Persistence::Memory`; spawn the binary via `_embed.py`). pytest fixture pattern:

```python
@pytest.fixture
def app():
    with bv.App(test_mode=True) as a:
        yield a
```

A separate transport-equivalence smoke (~6 tests) runs the same canonical fraud-team flow across HTTP / TCP / Embed and asserts identical results. Lives at `python/tests/v0/test_transport_equivalence.py` (new file).

**Anti-pattern reminder (CRITICAL):** integration tests MUST hit a real engine. **Never `MagicMock`.** The 0/68 deficit was masked at Phase 13.5 Plan 11 because internal tests used `MagicMock` as the Transport — `send_push` stubs returning `NotImplementedError` never surfaced until the real engine ran. Plan checker for 13.5.1 must reject any new test using `MagicMock` against the Transport surface.
