## 5.1-B — TCP read-path wire format default

**Question:** Default wire format for `TcpTransport.send_get` / `send_batch_get`?

**Options presented:**
- (a) JSON default (Recommended) — match `send_push`; one fewer dep on happy path; msgpack stays opt-in via `wire_format="msgpack"` kwarg.
- (b) msgpack default — match Plan 12-09 `tcp_get_single` Phase 12 D-A/D-B production fast-path; +30% smaller frames.

**User selection:** (a) JSON default.

**Notes / follow-ups:** Plan 12-09 `_tcp_get_single` (now private per 5.1-D) keeps msgpack as its default since that was the existing locked Phase-12 contract; new public `send_get` is JSON-default. No semantic conflict — different methods, different defaults, both honored.

## 5.1-C — `send_get` `features` filter parameter

**Question:** Should `send_get` expose an optional `features` filter at the SDK level?

**Options presented:**
- (a) Add `features: list[str] | None = None` (Recommended) — full row by default; opt-in narrowing.
- (b) Omit — always full row in v0; `features?` is wire-spec-only escape hatch.

**User selection:** (a) Add `features` parameter.

**Notes / follow-ups:** Base-class signature `Transport.send_get(*, table, key, features=None)` plumbs through HTTP (`POST /get` body field) + TCP (frame body field). `App.get(table, key, features=[...])` is the public surface. `App.batch_get` similarly accepts an optional `features` per entry.

## 5.1-E — v0 acceptance test harness topology

**Question:** How should the 68 v0 acceptance tests be wired to a real engine?

**Options presented:**
- (a) Embed-mode + small cross-transport smoke (Recommended) — 68 tests via `bv.App(test_mode=True)` + ~6 transport-equivalence tests.
- (b) HTTP-bound external server — pytest fixture binds server on port; tests connect via URL.
- (c) Parametrized 3-way (all 68 × 3) — every test runs against embed AND HTTP AND TCP.

**User selection:** (a) Embed-mode + small cross-transport smoke.

**Notes / follow-ups:** Embed-mode is closest to user-experience (`bv.App()` no-URL → spawn local binary). Transport-equivalence smoke catches HTTP/TCP/Embed semantic divergence on a single canonical fraud-team flow without paying the 3× test-runtime cost of full parametrization. Hard plan-checker gate for this phase: NO `MagicMock` in any integration test. The 0/68 deficit was specifically caused by `MagicMock` masking `NotImplementedError` stubs.
