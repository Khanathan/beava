All four nonetheless fall inside Plan 07d's stated scope ("Pure test edit", line 294 of Plan 07): every repair is a test-side encoding fix, no SDK / server / engine modification. 1. **`app.get(table, key)` on a global table returns `RegistrationError` with code `key_parse_failure`** — this is correct behavior but unintuitive. Consider mapping to `KeyError` for SDK polish.

Both observations should be added to `.planning/ideas/v0.1-deferrals.md` v0.0.x section by Plan 07e or a separate housekeeping pass; out of scope for 07d.
