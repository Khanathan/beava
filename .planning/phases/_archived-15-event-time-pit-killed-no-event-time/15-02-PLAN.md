<objective>
Compute `derived_retention_ms` at register time by walking the stream→table DAG. Replace the wall-time retention sweep with an event-time sweep driven by watermark. Emit a `BV-I-RETENTION-GROWTH` diagnostic in the register response when a newly-registered stream grows retention for an existing temporal table.

Purpose: Lock in correct retention bounds the moment a user declares streams + tolerate_delays, without hand-tuning `retention_ms`. Keep memory bounded by event-time (the correct axis for PIT) rather than wall-clock.

Output: Registry carries derived retention per temporal table; temporal store sweeps on event-time; register response surfaces retention growth.
</objective>

// registry.rs
pub struct TableDescriptor {
    // ... existing fields ...
    /// Register-time-computed retention from the stream DAG (D-04, D-11).
    /// Set by Registry::compute_derived_retention_ms on every register.
    /// `None` means "no streams feed this table yet; no sweep"; otherwise
    /// the sweeper uses max(derived_retention_ms, retention_ms) as the bound.
    #[serde(default)]
    pub derived_retention_ms: Option<u64>,
}

impl Registry {
    /// Walk the DAG: for each temporal table T, find all streams S whose

    /// Returns a map table_name -> new derived_retention_ms.
    pub fn compute_derived_retention_ms(&self) -> BTreeMap<String, u64>;
}

impl TemporalStore {
