  <files>crates/beava-core/src/temporal.rs</files>
  <behavior>
    - `retract(&mut self, key, target_lsn, retract_lsn) -> Result<Option<u64>, RetractError>`:
      1. Find the chain entry whose inner-key is `(_, target_lsn)` (scan the chain — LSN is unique within a chain).
      2. If not found → `TargetNotFound`.
      3. Check if any existing `Retracted { undo_of: target_lsn, .. }` exists → `AlreadyRetracted`.
      4. Inherit `event_time_ms` from the target version.
      5. Insert `Retracted { undo_of: target_lsn, event_time_ms: target_event_time }` at composite key `(target_event_time, retract_lsn)`.
      6. Compute `prior` as the visible version strictly below `(target_event_time, target_lsn)` using the same retraction-skip walk, return its LSN.
    - `lookup_at_event_time` retraction-skip pass: gather the set of LSNs that are `undo_of` of any `Retracted` marker with `event_time_ms ≤ as_of`. Walk newest-first; skip Retracted markers AND skip any entry whose inner-key `.1` (lsn) is in that set.
    - Late-upsert self-heal works naturally because BTreeMap range query over composite key is total-order-correct; no special code needed beyond the chain-key refactor.
  </behavior>
