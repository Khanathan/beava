# Plan 37-01 — `tally fork` CLI + E2E demo

## Goal

Ship the scientist-facing entry point and the one test that proves the Option M workflow end-to-end.

## Task T1 — `tally fork` subcommand

**Files:** `src/main.rs` (add `fork` subcommand dispatch on the main `tally` binary — one binary for scientists) OR `src/bin/tally_cli.rs` (fallback location).

**Choice rationale:** `tally fork` on the main binary is the better scientist ergonomics story — one binary to install, one command to run. If the main binary's arg parsing is shared with `tally serve`, add `fork` as a sibling subcommand; otherwise add a small dispatcher.

**Subtasks:**
- Parse `tally fork` flags:
  - `--remote HOST:PORT` (required)
  - `--streams s1,s2` (required)
  - `--keys k1,k2` OR `--key-prefix P` (optional; mutex)
  - `--since <ISO-8601 or ms>` (optional; default = "1970-01-01T00:00:00Z" = full history)
  - `--token T` (required; env var `TALLY_REPLICA_TOKEN` overrides)
  - `--local-port PORT` (optional; default 7400)
  - `--pipeline-file PATH` (optional)
- Translate to `--replica-*` flags and exec the server binary (in-process: call the same startup function `tally serve` uses; no shell subprocess).
- Log banner: "🍴 tally fork — remote=<HOST> scope=<STREAMS> since=<TS> → localhost:<PORT>". (Banner can be plain text if user wants no emojis — follow existing CLI style.)
- Unit tests: flag parsing happy + error paths (missing required, invalid --since, mutex keys violation).

**Also (minor):**
- Add `/debug/ready` HTTP endpoint (if not present) that returns 200 once the replica-catchup-done oneshot has fired. ~10 lines of axum. Plan 36-01 may already add it — confirm.

**Done when:** `cargo test` new fork-flag tests green; `cargo build --bin tally` green; `tally fork --help` prints the flag list.

## Task T2 — E2E pytest `test_fork_demo.py`

**File:** `tests/integration/test_fork_demo.py` (new).

**Helpers** (put in `tests/integration/conftest.py` or the test file):
- `start_prod(port, events)` — spawn Tally server, push fixture events via existing Python SDK, return subprocess handle.
- `start_fork(prod_port, local_port, since, streams, keys, token, pipeline_file)` — spawn `tally fork` subprocess.
- `wait_for_ready(local_port, timeout=30s)` — poll `GET http://localhost:{port}/debug/ready` until 200 or timeout.

**Test flow (one `test_scientist_fork_workflow` function):**
1. Allocate two free ports (`prod_port`, `local_port`).
2. `prod = start_prod(prod_port, events=[…])` — events for stream `Transactions`:
   - u1 @ T+1s, amount=10
   - u1 @ T+2s, amount=20
   - u1 @ T+3s, amount=30
   - u2 @ T+1s, amount=5
   - u2 @ T+2s, amount=15
3. Author scientist pipeline in Python using existing `@tl.stream` + `@tl.table` decorators:
   ```python
   @tl.stream
   class Transactions:
       user: str
       amount: float

   @tl.table(depends_on=[Transactions])
   def txn_summary(t):
       return t.group_by("user").agg(
           count=tl.count(),
           total=tl.sum(t.amount),
       )
   ```
4. Serialize the pipeline's REGISTER payload via `tl.serialize_register(txn_summary)` (whatever the existing SDK helper is) to `/tmp/fork_demo_pipeline.json`.
5. `fork = start_fork(prod_port, local_port, since="1970-01-01T00:00:00Z", streams="Transactions", keys="u1,u2", token="test-admin", pipeline_file="/tmp/fork_demo_pipeline.json")`.
6. `wait_for_ready(local_port)` — expect ready within 30s.
7. `replica = tl.Client(remote=f"localhost:{local_port}", token="local-noauth")`.
8. Assert `replica.get("u1", stream="txn_summary") == {"count": 3, "total": 60.0}`.
9. Assert `replica.get("u2", stream="txn_summary") == {"count": 2, "total": 20.0}`.
10. Push 2 more events to prod: `push(Transactions, u1, amount=100)` and `push(Transactions, u3, amount=50)` (u3 is OUT of scope).
11. Wait 2s.
12. Assert `replica.get("u1", stream="txn_summary") == {"count": 4, "total": 160.0}` — live tail worked.
13. Assert out-of-scope: `replica.get("u3", stream="txn_summary")` raises `OutOfScopeError` OR returns None (confirm v0 behavior — likely None since we're not enforcing client-side scope checks on the replica).
14. Attempt `replica.push(Transactions, u1, amount=999)` against replica — expect HTTP 403 / "replica mode: local PUSH disabled".
15. Teardown both processes.

**Done when:** `pytest tests/integration/test_fork_demo.py -v` passes deterministically (run 3× locally to catch flakiness).
