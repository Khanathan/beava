<decisions>
## Implementation Decisions

### D-01 — Python SDK structure: core flat + advanced submodules
- Public namespace: `bv.App`, `bv.event`, `bv.table`, `bv.col`, `bv.lit` (NEW per ADR-003), `bv.demo`, all 53 op helpers (`bv.count`, `bv.sum`, ...), `bv.session_window` (alias of `window=`)
- Submodule: `beava.test` (`fixture`, `replay`, `assert_features_eq`, `MockApp`)
- Submodule: `beava.cli` (entry point for `beava bench`, `beava demo`, etc.)

### D-02 — DELETE the 5 stale modules; preserve git history (no rename)
- Delete `python/beava/_events.py`, `_agg.py`, `_schema.py`, `_validate.py`, `_col.py` (~2000 LOC removed)
- Pure delete (not move) — prior content lives in git; new content built clean from spec docs
- Keep names available for new implementation (NEW `_events.py` + `_agg.py` + `_col.py` written from scratch following 13.0 sdk-api/python.md)

### D-03 — KEEP + bug-fix the 4 modules
- `_wire.py` — fix `OP_PUSH = 0x0010` opcode bug (Python had `0x0002`); also add `OP_BATCH_GET = 0x0024`, `OP_RESET = 0x0040` per Phase 13.4 wire spec
- `_transport.py` — keep HTTP + framed-TCP transport; add `verb-style POST + JSON body` per D-04 (HTTP); update opcode constants for TCP
- `_errors.py` — keep exception hierarchy; add new codes from `error-codes.md` (force_required, dry_run_passed, wrong_arity_global_table, unsupported_op_in_lifetime_mode)
- `_embed.py` — keep binary-discovery + spawn; add `Persistence::Memory` flag passthrough per Phase 13.4 D-07

### D-04 — Public `bv.lit` export (ADR-003, V0-LIT-01)
```python
# python/beava/__init__.py
from beava._col import _Literal as _Literal
def lit(value):
    return _Literal(value)
```
~5 LOC. The internal `_Literal` AST node already exists at `_col.py:195` — pure namespace promotion.

### D-05 — Global aggregation surface (ADR-003, V0-GLOBAL-AGG-02)
- `events.group_by()` with no args: flip `python/beava/_events.py:170-172` rejection to acceptance; return `GroupBy(_keys=[])` (~10 LOC)
- `events.agg(**aggs)` direct shorthand (no group_by required): equivalent to `events.group_by().agg(**aggs)` (~30 LOC on EventSource + EventDerivation)
- `@bv.table` without `key=` kwarg: declares global table; sets `_global = True` on the descriptor (~15 LOC)
- `App.get(table_name)` 1-arg overload: when global table, omit entity_id; raises `KeyError("wrong_arity_global_vs_entity_table")` if mismatched (~30 LOC)
- Total: ~85 LOC. Acceptance: `python/tests/v0/test_global.py` (8 tests) + `test_lit.py` (5 tests) from Plan 13.0-16 must PASS.

### D-06 — Pipeline DSL: 53 op helpers + Polars-style chains
- Each op = module-level function (`bv.count(window=, where=)`, `bv.sum(field, window=, where=)`, etc.)
- `bv.col("x")` operator overloading: `+ - * /`, `< > <= >= == !=`, `& | ~`, `.isnull()`, `.cast(type)`, `.alias(name)`
- Chain: `events.filter(bv.col("x") > 100).group_by("user").agg(name=bv.count(...))`
- `window=` kwarg per-op (NOT `windowed=`); per Q1 Path B, `bv.sum(field: str)` only (no `_ExprAST` field)
- `bv.session_window(...)` is a v0.1+ stub — raises `NotImplementedError("session windows are v0.1+; use windowed primitives")` per `project_v0_events_only_scope`

### D-07 — Demo loader: `bv.demo("adtech"|"fraud"|"ecommerce")`
- Bundled datasets in `python/beava/demos/{adtech,fraud,ecommerce}/{schema.json, events.jsonl}` (~300 LOC across 3 demos)
- `bv.demo("adtech")` returns a fresh `App` with the demo registered + first 100 events pushed; ready for `app.get(...)` exploration
- Idempotent (calling twice returns 2 separate App instances)

### D-08 — `beava.test` fixtures
- `beava.test.fixture(persistence="memory")` — pytest fixture returning `App`; auto-close + reset between tests
- `beava.test.replay(events_iter, app)` — replay events with `assert_features_eq` checkpoints
- `beava.test.assert_features_eq(actual, expected, tolerance=None)` — sketch-aware tolerance (HLL ±5%, quantile ±2pct)
- ~150 LOC

### D-09 — PEP 563 fix (postponed evaluation of annotations)
- Replace raw `param.annotation` reads with `get_type_hints(cls)` to handle string annotations correctly
- Fixes class-form decorators (`@bv.event class Click: user_id: str`) when `from __future__ import annotations` is present
- ~20 LOC across `_events.py` + `_agg.py`

### D-10 — `beava bench` CLI: 3 modes + interactive walkthrough
- 3 modes: `throughput` (sustained EPS), `mixed` (read-write ratio), `memory` (RSS / per-entity overhead)
- Interactive: `inquire` crate for prompts; `--yes` for non-interactive
- Pre-run memory estimator: per-derivation feature count × Phase 12.9 `size_of` math = projected RSS at N entities
- Output: human (default) / `--json` / `--markdown` / `--append=ledger.jsonl`
- 3 bundled dataset generators: adtech, fraud, ecommerce (each ~300 LOC)
- ~2000 LOC total
