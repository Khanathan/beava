<decisions>
## Implementation Decisions

### D-01 — mypy --strict + documented Any escapes (USER-LOCKED)

- `mypy --strict` clean overall — every helper typed; no implicit Any
- **Documented Any escapes** for AST-style helpers where exhaustive Generic[T] coverage is impractical:
  - `bv.col(...)` chains return Any (operator overloading produces dynamic AST)
  - `bv.lit(value)` returns Any (literal of arbitrary type)
  - Each escape: `# type: ignore[no-any-return]` (or equivalent) with one-line comment explaining
- Acceptance: `cd python && mypy beava --strict 2>&1 | grep -c "error:"` returns 0 (after explicit ignores)
- **Why:** catches real bugs; doesn't fight the dynamic AST shape; matches Polars convention

### D-02 — In-package demo datasets (USER-LOCKED)

- Bundled with `pip install beava` — adds ~3 MB to install size
- Path: `python/beava/demos/{adtech,fraud,ecommerce}/{schema.json, events.jsonl}`
- `bv.demo("adtech")` works offline immediately after install
- Each dataset: ~1 MB (~10K events) — enough to exercise sketch/decay/velocity ops meaningfully
- **Why:** simplest first-run UX; no extra install steps for the locked 60-second quickstart

### D-03 — `beava bench` CLI: subcommand + long-form flags (USER-LOCKED)

- Pattern: `beava bench <mode> --workload=<name> --size=<small|medium|large> --duration=<spec>`
- 3 modes (subcommands): `throughput`, `mixed`, `memory`
- Examples:
  - `beava bench throughput --workload=fraud --size=medium --duration=60s`
  - `beava bench mixed --workload=adtech --read-write-ratio=70/30 --duration=2m`
  - `beava bench memory --workload=ecommerce --entities=1000000`
- Interactive walkthrough: `beava bench` (no args) → `inquire` prompts for mode + workload + size; `--yes` for non-interactive
- Output formats: human (default) / `--json` / `--markdown` / `--append=ledger.jsonl`
- **Why:** mirrors cargo/git/kubectl conventions; mode-specific options discoverable via `beava bench <mode> --help`

### D-04 — Module structure: keep current flat (USER-LOCKED)

- `python/beava/_*.py` private modules (underscore-prefixed)
- Public flat namespace via `python/beava/__init__.py`:
  - Core: `bv.App`, `bv.event`, `bv.table`, `bv.col`, `bv.lit` (NEW per ADR-003), `bv.demo`
  - 53 op helpers (`bv.count`, `bv.sum`, ..., `bv.distance_from_home`)
  - Errors: `bv.BeavaError` + subclasses
- Submodules:
  - `beava.test` — pytest fixtures (`fixture`, `replay`, `assert_features_eq`, `MockApp`)
  - `beava.cli` — entry point for `beava bench` subcommand + future CLI commands
- Import pattern: `import beava as bv` (canonical; matches Plan 13.0-16's 68 tests)
- **Why:** simplest; no breaking change for python/tests/v0 users; matches Polars/pandas convention

### D-05 — Cross-amendment from 13.4 D-03 — `bv.App(test_mode=True)` kwarg + `beava.test.fixture()` defaults

- New SDK kwarg: `bv.App(url=None, test_mode=False, ...)`
- Embed mode (`url=None`) + `test_mode=True`: SDK passes `BEAVA_TEST_MODE=1` env var to spawned binary
- Network mode (`url=...`) + `test_mode=True`: emits warning ("test_mode kwarg ignored for network mode; server controls"), proceeds without effect
- `beava.test.fixture()` defaults to `test_mode=True` so test code doesn't need to repeat
- Wire-level: SDK sends reset request normally; server responds with `reset_disabled_in_production` if not in test mode
