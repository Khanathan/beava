## Notes

**Fsync mode wraps `force_snapshot_now()`** as a proxy for the group-commit
watermark crossing. Phase 6.1's microbench measured the actual fsync cost;
this mode times the per-push latency *including* the fsync wait by issuing
`force_snapshot_now()` after each push. When push_sync wire op exposes a
per-call watermark in v0.1+, the fsync-mode harness should switch to that.

**Decision: workloads back onto existing config files.** Rather than
hand-writing register payloads (which would risk wire-shape drift from the
real `{"nodes": [...]}` schema with `upstreams` / `ops` / `agg` / `params`),
the workloads module loads from `crates/beava-bench/configs/*.json` — the
same canonical configs the throughput regression-gate uses. `fraud-team.json`
is the primary tuning benchmark per `project_fraud_team_primary_bench`.

**Tracing output redirected to stderr** so `--json` output stays parseable.
This was the last green-up flake — without it, register-validate WARN
messages from beava-core polluted stdout.
