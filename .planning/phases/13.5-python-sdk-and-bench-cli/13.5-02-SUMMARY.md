## Transport factory note

The existing `_transport.py` already provides `HttpTransport`, `TcpTransport`,
`EmbedTransport` with `send_register/send_ping/send_push` (TCP) and
`send_register/send_ping/http_get_single` (HTTP). The new `make_transport`
factory dispatches on URL scheme and adds `test_mode` plumbing for embed mode.

**Plan 11 (Wave 7) handoff:** `send_get / send_batch_get / send_reset` are
called by `bv.App` but not yet implemented on `HttpTransport` or `TcpTransport`
subclasses — Plan 02's tests use `unittest.mock` to stub them. Plan 11
integration tests against the real engine will surface the need to wire these
to the actual HTTP routes (`/get`, `/batch_get`, `/reset`) and TCP opcodes
(OP_GET 0x0020, OP_BATCH_GET 0x0024, OP_RESET 0x0040).
