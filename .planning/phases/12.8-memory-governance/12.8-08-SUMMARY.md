## Lessons Learned

**Lesson:** **The inter-cell delta IS the inline-cheap gate; the cross-phase delta is investigation signal.** Plan 08's two-cell microbench cleanly isolated the cold-TTL feature's marginal cost (-2.6% — inline-cheap as promised). The disabled cell's +11.3% vs Phase 12.7's simple_counter is the cumulative Phase 12.8 hot-path overhead — primarily Plan 06's per-event `entity_count_resident` sum + atomic store. This is real overhead that DID land on the disabled path, but it's the cost of the 5-metric Prometheus family Plan 06 chose (per-event sample for gauge freshness). Phase 13 has the budget to amortize this.

The Plan 08 design choice (two-cell A/B with same pipeline shape) was correct — cross-phase comparisons against Phase 12.7's `simple_counter` cell would have conflated Plan 06's metric-sampling cost with Plan 03's cold-TTL check cost. The two-cell A/B isolates the cold-TTL feature in isolation; the cross-phase reading (informational) shows the cumulative cost of all Plan 12.8 changes including those NOT under test in this plan.

**Lesson:** **Strict-deny shims work as designed — and force the test surface to align.** The fraud-team.json config rejection during the throughput run was a real catch by Plan 04's lifetime-bound shim. The fix was trivial (kwarg rename) but the moment-of-truth was valuable: this is exactly the v0 commitment in action, refusing pre-12.8-style configs that don't declare explicit bounds. Phase 13's SDK polish should include a config-validator pass over all checked-in JSON configs.
