## Cold-entity TTL config surface

| Option | Description | Selected |
|--------|-------------|----------|
| **Per-source decorator only — `@bv.event(cold_after='7d')`** | Granular per-source policy; declarative; survives restart via durable registry. Reuses Phase 5 duration parser. | ✓ |
| Global env var only — `BEAVA_COLD_TTL=7d` | Simple deploy-level config; one TTL applies to all sources; loses per-source granularity. | |
| Both — env var default + decorator override | Maximum flexibility; two surfaces to document; potential config drift. | |
| Runtime API — `app.set_cold_ttl(...)` | Imperative configuration; less aligned with declarative `@bv.event` philosophy. | |

**User's choice:** Per-source decorator only — captured as **D-01**.
**Notes:** Default = `cold_after=None` (omitted) → no expiry. Aligned with existing `dedupe_window` / `keep_events_for` parameter precedents.

## `lifetime=True` SDK kwarg shape

| Option | Description | Selected |
|--------|-------------|----------|
| **Implicit — omit `windowed=`** | Existing behavior stays canonical; no new SDK surface. Phase 12.8's contribution is the per-op memory-bound contract, not new syntax. | ✓ |
| Explicit `lifetime=True` kwarg | Self-documenting at call site; 54 op signatures touched; possible confusion with implicit usage. | |

**User's choice:** Implicit — captured as **D-02**.
**Notes:** Operator docs (Phase 13) will document the omission semantics. No new SDK surface in 12.8.

## Memory-bound enforcement for currently-unbounded ops

| Option | Description | Selected |
|--------|-------------|----------|
| **Hard reject at register-time** | New JSON-prelude shim `pre_check_unbounded_op_in_lifetime_mode`. Forces explicit memory budget. | ✓ |
| Auto-default to sensible cap with warning metric | More forgiving; risks silent over-budget memory if metric unread. | |
| Document only — no runtime enforcement | Cheapest implementation; weakest contract; fails phase goal. | |

**User's choice:** Hard reject — captured as **D-03**.
**Notes:** Fourth JSON-prelude shim alongside 12.6 Plans 04+06 and 12.7 Plan 01. Error code: `unbounded_op_in_lifetime_mode`. Error message names the op + suggests cap kwarg. Error framing: "requires explicit memory bound in v0" (forward-looking, NOT retrospective). Catalog audit during plan phase identifies ops needing cap-kwarg additions (likely candidates: `histogram`, possibly `seasonal_deviation`).

## Bucket reclaim trigger mechanism

| Option | Description | Selected |
|--------|-------------|----------|
| **Per-event only — trailing bucket eviction during `update_at()`** | Existing mechanism; zero new hot-path cost; preserves single-thread invariant. | ✓ |
| Periodic compact pass during snapshot writer | Snapshot duration grows linearly with entity count; per-event already covers it. | |
| Eager scan via apply-thread heartbeat | Apply-thread amortized cost; overkill given Option 1 + cold-TTL. | |
| Background reclaim thread | Violates `project_no_sharded_apply`; rejected. | |

**User's choice:** Per-event only — captured as **D-04**.
**Notes:** Tier 2 is "always on" because the existing per-event reclaim is always on. Phase 12.8 verifies + tests this contract; doesn't add new mechanism. Idle entities hold state until either Tier 1 cold-TTL evicts (D-01, opt-in) or next event triggers cleanup.

## Claude's Discretion (decided autonomously per `feedback_logistics_autonomy`)

## Deferred Ideas

- Per-feature TTL (different cold thresholds for different aggregations on the same entity) — too complex for v0.
- Resurrection with persisted history — would need disk; conflicts with v0 in-memory-only commitment.
- Memory-pressure backpressure with automatic action — metric emission only; v0.0.x or v0.1+ if demand surfaces.
- Background reclaim thread — violates `project_no_sharded_apply`; rejected.
- Periodic compact pass during snapshot writer — per-event reclaim covers all cases.
- `bv.session()` revival via bucket-reclaim primitive — Phase 25 stays archived-indefinitely.
- Per-event-type cold logic within a multi-event-type source — per-source uniform only.
- SSD overflow / tiered eviction — locked permanently OUT.
- `cold_after` as global env-var override — explicitly rejected per D-01.
- `lifetime=True` SDK kwarg — explicitly rejected per D-02.
