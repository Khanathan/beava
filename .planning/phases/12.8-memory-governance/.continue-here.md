**Critical gotchas (re-discovered if forgotten):**

1. **`--total-events N` triggers `Pool=N` pre-build** — bench encodes ALL N frames upfront into `Vec<Bytes>` per worker. RAM cost: ~2 GB per worker per 250k frames. With `--total-events 5_000_000 --parallel 4`, that's ~30 GB pool memory → OOM in 16 GiB pod. **Solution:** use smaller `--total-events` (250k-1M) OR omit it (duration-only mode, smaller pool default).
2. **`--duration-secs N` is the push-deadline, NOT process-exit deadline.** After deadline: drain in-flight pushes (~30s for parallel=16 × pd=256) + HDR percentile analysis + 5s server graceful shutdown. Total wall = `duration_secs` + ~60s. Don't `pkill` mid-drain or you lose the summary output.
3. **`--pipeline-depth 1` (default) gets ~30k EPS; `--pipeline-depth 256` gets ~175k EPS** on the same hardware. PD is the dominant client tuning knob (5× lift). Always specify `--pipeline-depth 256`.
4. **`--blast-shape=fixed` (default) uses tiny key pool** (~50-100 entities from fraud-team.json's `keys` arrays). For realistic entity counts, use `--blast-shape zipfian --cardinality K` where K = desired entity count.
5. **`pkill -9 beava-bench-v18` is mandatory before re-runs** — earlier rejected commands can leave processes running that compete for CPU and skew measurements.
