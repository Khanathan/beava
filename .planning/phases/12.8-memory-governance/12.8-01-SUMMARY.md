## Decisions Made

See key-decisions in the frontmatter for the load-bearing list. Quick recap of the most consequential:

1. **Forward-looking error framing** (`unbounded_op_in_lifetime_mode`, NOT `feature_removed_*`) per D-03 verbatim — v0 is the FIRST public release.
2. **Wave 1 env-gate default-OFF** (`BEAVA_MEMORY_GOV_ENFORCE` defaults unset) keeps the workspace green while Plan 04 fills the per-op bound table; Plan 06 flips the default to ON in Wave 3.
3. **Per-call env-var read** (NOT a OnceLock cache) on the COLD register path — preserves per-test env-flip semantics inside a single integration-test binary; explicitly contrasted with the OnceLock pattern used on the HOT apply path's `trace_apply_enabled()` cache.
4. **`tokio::sync::Mutex` (NOT `std::sync::Mutex`) in the integration test file** — async-aware lock held across `.await` points satisfies clippy's `await_holding_lock` lint; functionally equivalent for in-test serialization.
5. **Stub `lifetime_bound_for_op_str` returns Unbounded for every input** — Plan 04 replaces the helper body with the 54-op classification table in a single in-place edit.
