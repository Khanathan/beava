## Lessons Learned

- **Match-exhaustiveness on AggKind via test-not-Rust-pattern.** Because `lifetime_bound_for_op_str` takes `&str` rather than `AggKind`, Rust's exhaustive-match enforcement doesn't apply. Plan 04 uses a test (`every_aggkind_has_classified_bound`) that enumerates every op-string by name to catch missing classifier rows. Trade-off: writers must remember to add new ops to BOTH the agg_compile.rs `parse_agg_kind` match AND the test enumeration; the architectural test in Plan 07 will close this gap.
- **Histogram canonical kwarg verification mid-plan.** Plan 04 plan body originally referenced `num_buckets` as the histogram cap kwarg — execution found the wire convention is `buckets` (Vec<f64>). Verified in 30s via `grep "histogram" agg_compile.rs`; updated classification table to match. The plan body had explicit "VERIFY DURING EXECUTION" instruction, which prevented spec drift.
- **Plan 01 fixture surgery anticipated.** Plan 01's tests were intentionally written with `count` because the stub returned Unbounded for everything — the choice didn't matter. Plan 04 had to update them. Future plans that introduce a new shim with a stubbed classifier should note in their SUMMARY which test fixtures are "transitional" so the next plan's executor knows where the surgery lands.
- **Per-test custom env vars in Python require tmp_path data dirs.** Python pytest's `monkeypatch.setenv` only affects the test process; subprocesses (`Popen`) inherit env from a snapshot. To control `BEAVA_MEMORY_GOV_ENFORCE` per-test, the fixture spawns each subprocess with a fresh `env=` and per-test `BEAVA_WAL_DIR`/`BEAVA_SNAPSHOT_DIR` via `tmp_path` so multi-test runs don't collide.
