<objective>
Implement cold-entity TTL eviction on the mio apply hot path. Read `descriptor.cold_after_ms` (added by Plan 02) once per event; if `Some(N)` and `now_ms - last_seen_ms > N` for the entity being touched, clear the entity's `Vec<AggOp>` BEFORE applying the new event (FRESH state on resurrect, Redis TTL pattern). Inline-cheap when `cold_after_ms = None` (single Option branch, ~1 ns).

Per CONTEXT D-04: "no background scan thread; eviction logic must be inline-cheap or amortized". This plan does the inline-cheap variant. The check runs at apply time only, so an idle entity with cold state continues to consume memory until next event arrival — D-04 explicitly accepts this trade.

**Storage decision (planner discretion per CONTEXT D-04):** the `last_seen_ms` is currently NOT tracked at the `AggStateTable` level — it's tracked per-AggOp inside the `SeenState` struct (used by `last_seen` op, line 1857 of agg_state.rs). Plan 03 introduces a parallel `HashMap<EntityKey, u64>` sidecar in `AggStateTable` indexed by the same shape (single_u64 / single_str / multi) — this avoids touching the per-AggOp state and keeps the change isolated. Implementation: 3 parallel sidecar maps `last_seen_u64: HashMap<u64, u64>` / `last_seen_str: HashMap<CompactString, u64>` / `last_seen_multi: HashMap<EntityKey, u64>` mirroring the existing 3-shape dispatch.

Rationale: a per-entity `last_seen_ms` sidecar costs ~16 bytes/entity (key replication + u64) but is dispatch-shape compatible. The alternative — augmenting the `Vec<AggOp>` value to `(Vec<AggOp>, u64)` — would cascade through `init_row`, `query_feature`, `iter_sorted` (snapshot), and 30+ test sites. Sidecar has tighter blast radius.

Output: Plan 02's `cold_after_ms` field becomes operative. Microbench cell (Plan 08) confirms <5% per-event regression in the cold-TTL-DISABLED case (Plan 02 baseline) and <10% in the cold-TTL-ENABLED case. Eviction visible to tests: register source with `cold_after='1s'`, push event for entity_A, sleep 1.1s, push another event for entity_A — second event sees fresh state (e.g. count=1, not count=2).

<!-- apply_event_to_aggregations contract — extends the per-source loop -->

The apply path's `apply_event_to_aggregations` (in `crates/beava-core/src/agg_apply.rs`) iterates over aggregations whose `source_node_name == event_name` and updates each via `state_table.get_or_init_by_shape(...)`. Plan 03 inserts the cold-TTL check IMMEDIATELY before each `get_or_init_by_shape` call site — OR (cleaner) at the apply_shard.rs orchestration layer (one-shot per push, before locking the tables).

**Decision: orchestrate at apply_shard.rs layer.** Plan 03 reads `descriptor.cold_after_ms`, computes the entity-key shape from the row (using existing `descriptor.apply_field_names` machinery — see `crates/beava-core/src/agg_apply.rs::apply_event_to_aggregations`), and clears the per-entity Vec for ALL aggregations sourced from this event in one pass. Reuses `tables` lock; no new lock.

Caveat: a single source event may feed multiple aggregations with different group_keys (e.g. group_by user_id AND group_by user_id+merchant). The cold-TTL check must clear ALL aggregations that source from this event for ALL entity-shapes that include the row's data. The simplest correct implementation: iterate the registry's `aggregations_for_source(event_name)`, derive each agg's entity-key from the row, and per-agg apply the eviction check.

        /// Plan 12.8-03 D-01/D-04: cold-TTL eviction check.
        ///
        /// Reads the entity's last_seen_ms (from the appropriate sidecar map);
        /// if older than `now_ms - cold_after_ms`, REMOVES the entity from the
        /// state map (Vec<AggOp> dropped) AND removes the last_seen_ms entry,
        /// returning `true`. Returns `false` if entity is warm or absent.
        ///
        /// Caller MUST update last_seen_ms via `record_last_seen_by_shape` after
        /// the apply call (regardless of whether eviction fired).
        ///
        /// Per CONTEXT D-04 (Redis TTL pattern, locked permanent): on resurrect
        /// the entity is treated as fresh — no partial-state preservation. The
        /// `get_or_init_by_shape` call after eviction allocates a new
        /// `Vec<AggOp>` via `init_row`.
        pub fn evict_entity_by_shape_if_cold(
            &mut self,
            shape: &EntityKeyShape,
            now_ms: u64,
            cold_after_ms: u64,
        ) -> bool {
            // Read last_seen_ms for this shape; bail if absent (first event).
            let last_seen_ms = match shape {
                EntityKeyShape::SingleU64(k) => self.last_seen_u64.get(k).copied(),
                EntityKeyShape::SingleStr(_, s) => self.last_seen_str.get(s.as_str()).copied(),
                EntityKeyShape::Multi(ek) => self.last_seen_multi.get(ek).copied(),
            };
            let last_seen = match last_seen_ms {
                Some(t) => t,
                None => return false, // first event for this entity — no eviction
            };
            // Check cold threshold.
            if now_ms.saturating_sub(last_seen) <= cold_after_ms {
                return false; // warm
            }
            // COLD — remove the Vec<AggOp> and last_seen entry. Caller's
            // get_or_init_by_shape will allocate a fresh Vec for the new event.
            match shape {
                EntityKeyShape::SingleU64(k) => {
                    self.single_u64.remove(k);
                    self.last_seen_u64.remove(k);
                }
                EntityKeyShape::SingleStr(_, s) => {
                    self.single_str.remove(s.as_str());
                    self.last_seen_str.remove(s.as_str());
                }
                EntityKeyShape::Multi(ek) => {
                    self.multi.remove(ek);
                    self.last_seen_multi.remove(ek);
                }
            }
            true
        }

        /// Plan 12.8-03 D-01: record the wall-clock arrival time for this
        /// entity. Called from the apply path AFTER apply_event_to_aggregations
        /// — only when source has `cold_after_ms = Some(_)`.
        pub fn record_last_seen_by_shape(&mut self, shape: &EntityKeyShape, now_ms: u64) {
            match shape {
                EntityKeyShape::SingleU64(k) => {
                    self.last_seen_u64.insert(*k, now_ms);
                }
                EntityKeyShape::SingleStr(_, s) => {
                    use hashbrown::hash_map::RawEntryMut;
                    match self.last_seen_str.raw_entry_mut().from_key(s.as_str()) {
                        RawEntryMut::Occupied(mut o) => { *o.get_mut() = now_ms; }
                        RawEntryMut::Vacant(v) => { v.insert(s.clone(), now_ms); }
                    }
                }
                EntityKeyShape::Multi(ek) => {
                    use hashbrown::hash_map::RawEntryMut;
                    match self.last_seen_multi.raw_entry_mut().from_key(ek) {
                        RawEntryMut::Occupied(mut o) => { *o.get_mut() = now_ms; }
                        RawEntryMut::Vacant(v) => { v.insert(ek.clone(), now_ms); }
                    }
                }
            }
        }
