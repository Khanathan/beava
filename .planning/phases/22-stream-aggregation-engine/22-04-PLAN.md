## Implementation Steps

1. **Capture BASELINE.json first** ‚Äî check out the last commit before 22-01 (before any v0 engine changes), run `bench.py` full 9-cell matrix, record eps + p99 per cell, commit to `BASELINE.json` on the main branch. This locks the comparison baseline. Then return to main.

2. **TCP REGISTER dispatch.** In `src/server/tcp.rs`, detect v0 REGISTER payload (distinguishing from v2.0 ‚Äî can be field-based, e.g. presence of `aggregation` key in the JSON). Dispatch to `register_v0` from `src/engine/register.rs`. If v2.0 detection matters: either keep both paths (with a `--legacy-register` flag), or delete v2.0 REGISTER parsing entirely (since Phase 21 removed the SDK surface).

3. **PUSH path for v0 aggregations.** Ensure `pipeline.rs` routes events to the new v0 operators built via `build_operator`. Verify the cascade through 22-02's linear ops and 22-03's sketch ops works end-to-end.

4. **GET path for v0 features.** Verify `/get` returns inferred schema values from the v0 operators via the existing EntityState lookup.

5. **Un-skip Python round-trip test** (`python/tests/test_v0_register_roundtrip.py::test_full_tcp_roundtrip_register_push_get`). It should PUSH events, GET features back, and assert correct aggregation results for count/sum/max + at least one sketch operator.

6. **Install criterion.** Add `criterion = "0.5"` to `Cargo.toml` dev-dependencies. Port the 3 bench files (`uddsketch_ops.rs`, `cms_ops.rs`, `hll_ops.rs`) from `std::time::Instant` to `criterion` harness. Run and record; confirm 22-03's numbers reproduce.

7. **TopKHeap optimization.** Replace the linear-scan `heap.observe` in `src/engine/cms.rs` with a proper binary heap + HashMap<Value, heap_position>. Insert becomes O(log k); lookup of existing value becomes O(1). Target: top_k.push drops from 1487ns ‚Üí < 300ns.

8. **Full benchmark matrix.** Run `bench.py` 9-cell matrix against the live v0 TCP path. Record results. Compare each cell to `BASELINE.json`; no cell > 5% regressed.

9. **Commit and write 22-04-SUMMARY.md.**

## Risks & Mitigations

- **TCP backward-compat break** ‚Äî if v2.0 TCP clients exist on-disk (tests), they break. Mitigation: since Phase 21 deleted the old SDK, all tests should use v0 API already. Verify with `grep -r "@tl.source\|@tl.dataset" tests/` returning zero results.
- **Benchmark regression > 5%** ‚Äî if v0 path is slower than v2.0, this plan fails. Mitigation: profile the hot path with `perf`/`cargo flamegraph`; common causes are allocator churn in hybrid-mode operators, over-eager cloning in dispatch, or the new operator trait indirection. Optimize in place before declaring regression.
- **BASELINE capture requires git checkout** ‚Äî ~15 minutes to check out pre-v0 commit, run matrix, return to main. Mitigation: document the SHA; run once; the numbers don't change after.
- **TopKHeap optimization could introduce correctness bug** ‚Äî heap + position-index must stay in sync. Mitigation: existing 22-03 integration tests cover top_k semantics; optimization must keep them green.
