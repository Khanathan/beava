Target Cow shape:
```rust
fn value_to_key_string(v: &Value) -> Option<Cow<'_, str>> {
    match v {
        Value::Str(s) => Some(Cow::Borrowed(s.as_str())),     // ← no alloc
        Value::I64(n) => Some(Cow::Owned(n.to_string())),     // ← still alloc for I64
        Value::F64(f) => Some(Cow::Owned(format!("{f:?}"))),
        // ...
    }
}
```

Snapshot back-compat: `EventTypeMixState.allowed` serialized as `Option<Vec<String>>` today. Changing to `Option<AHashSet<String>>` changes the serde shape. Either:
  (a) Keep `Vec<String>` for serde + materialize to AHashSet on first update (lazy init via OnceLock-on-state).
  (b) Add `#[serde(deserialize_with = "...")]` to convert from Vec → Set on load.
v0 has no on-disk data; pick (a) — store as Vec for serde, build a `RefCell<Option<AHashSet>>` cache on first update. Simpler and closer to today's struct shape.

Actually a cleaner shape: don't cache, just convert at construction time AND keep an internal `allowed_set: Option<AHashSet<String>>` populated from `allowed: Option<Vec<String>>` at `EventTypeMixState::new`. Serde keeps Vec; in-memory access uses Set. This avoids the serde-shape question entirely and is what CONTEXT.md D-04b actually specifies (line 312-314: "swap Vec<String> allowlist for AHashSet<String> at EventTypeMixState::new").
