# Phase 19.2 Plan 05: EventTypeMix AHashSet + Cow refactor + update_at migration Summary

EventTypeMix allowlist O(n) → O(1) via AHashSet; `str_from_row` and `value_to_key_string` return `Cow<'_, str>` for zero-alloc borrow on `Value::Str`; EventTypeMix migrated to `update_at` consuming Plan 19.2-01's pre-extracted `ExtractedFields` array.

## What Was Built

**agg_buffer.rs:**
- Added `allowed_set: Option<ahash::AHashSet<String>>` field to `EventTypeMixState` alongside existing `allowed: Option<Vec<String>>` (kept for serde stability with `#[serde(skip, default)]` on the set)

## AHashSet shape + serde-vs-runtime split

```
EventTypeMixState {
    counts: BTreeMap<String, u64>,    // serde-stable, hot-path keys
    total: u64,
    max_categories: usize,
    allowed: Option<Vec<String>>,     // serde-stable: Vec persists in snapshot
    #[serde(skip, default)]
    allowed_set: Option<AHashSet<String>>,  // runtime-only: rebuilt at new() or on first update
}
```

Serde skips `allowed_set`; on snapshot load the field defaults to `None`. The next `update()` or `update_at()` call triggers `ensure_allowed_set()` which pays O(allowed.len()) once. For `v0` (no long-lived on-disk data), this path is rarely exercised.

## Cow refactor pattern

Both helpers now follow the same contract:

```
Value::Str(s)     → Cow::Borrowed(s.as_str())   // zero alloc — borrows from CompactString
Value::I64(n)     → Cow::Owned(n.to_string())   // alloc required (derived string)
Value::F64(f)     → Cow::Owned(format!("{f:?}")) // alloc
Value::Bool(b)    → Cow::Owned(b.to_string())   // alloc
Value::Datetime   → Cow::Owned(ms.to_string())  // alloc
Null/Bytes/...    → None
```

Consumers call `cow.as_ref()` → `&str` before passing to `BloomFilter::insert` / `EntropyHistogram::insert`. For the common `Value::Str` input this is a pure pointer dereference — no heap activity.

| `BloomMemberStateWrap::update_pre` | same | same |
| `EntropyStateWrap::update` | same | same |
| `EntropyStateWrap::update_pre` | same | same |

### Auto-fixed Issues

**1. [Rule 1 - Bug] value_to_key_string migrated in Task 1.b (not Task 2.b)**
- **Found during:** Task 1.b
- **Issue:** Test file imports `value_to_key_string` and tests its `Cow<'_, str>` return type. Keeping it `fn value_to_key_string(...) -> Option<String>` causes a compile-time type mismatch (not a runtime failure), blocking tests 1/4/5 from running.
- **Fix:** Made `value_to_key_string` `pub` and updated its return type to `Cow<'_, str>` in Task 1.b; updated Bloom/Entropy `.as_ref()` consumers at the same time. Task 2.b then adds the Bloom pointer probe only.
- **Files modified:** `agg_state.rs`
- **Commit:** `d81b849`

**2. [Rule 1 - Bug] Bloom pointer probe uses feature = "test-utils" not #[cfg(test)]**
- **Found during:** Task 2.b
- **Issue:** `#[cfg(test)]` functions in a library are not visible to integration test crates (separate compilation units). The probe compiled silently to "not found" at the integration test link stage.
- **Fix:** Switched both the `thread_local!` and `_last_bloom_insert_ptr()` to `#[cfg(feature = "test-utils")]`; registered the integration test in `Cargo.toml` with `required-features = ["test-utils"]`.
- **Files modified:** `bloom.rs`, `Cargo.toml`
- **Commit:** `3961503`

**3. [Rule 1 - Bug] Bloom probe pointer capture must read from Row not from pre-move Value**
- **Found during:** Task 2.b test execution
- **Issue:** Capturing `val.as_str().as_ptr()` before `Row::new().with_field("token", val)` moves `val` — CompactString may be at a different stack address after the move. Pointer comparison failed.
- **Fix:** Capture the pointer via `row.get("token")` after insertion, which borrows from the stable in-row address.
- **Files modified:** `tests/event_type_mix_optimization.rs`
- **Commit:** `3961503`

**4. [Rule 2 - Missing functionality] update_with_extracted extended with field_idx + extracted**
- **Found during:** Task 1.b wiring
- **Issue:** `update_at` requires `(extracted, field_idx)` but `update_with_extracted` only had `pre_val`. Routing EventTypeMix through `update_at` required the call site to pass the full `ExtractedFields` array and `field_idx`.
- **Fix:** Extended `update_with_extracted` signature with two new parameters; updated single call site in `agg_apply.rs`; added `#[allow(clippy::too_many_arguments)]`.
- **Files modified:** `agg_op.rs`, `agg_apply.rs`
- **Commit:** `d81b849`
