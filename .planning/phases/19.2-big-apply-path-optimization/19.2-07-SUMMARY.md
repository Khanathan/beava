# Phase 19.2 Plan 07: Cost-Class Catalogue + /debug/op-cost Endpoint Summary

One-liner: hand-maintained 53-op cost-class catalogue (38/6/9 tier split) + dev-gated
`GET /debug/op-cost` endpoint that exposes the latest TRACE_AGG per-kind snapshot as JSON.

---

## What was built

### D-06: `docs/operators/cost-class.md`

170-line hand-maintained markdown catalogue covering all 53 surviving operators
(post-Plan-19.2-06 removal of `bv.unique_cells` + `bv.geo_entropy`). Three tier sections:

- **Tier 1 (38 ops):** Phase 5 core (8) + Phase 8 point/recency/streak (15) + Phase 9
  decay/velocity/z-score (14) + Phase 11 direct-array buffer ops (HourOfDayHistogram,
  DowHourHistogram, SeasonalDeviation = 3). Algo floor 4-22 ns; post-19.2 expected 25-42 ns.
- **Tier 2 (6 ops):** OutlierCount, CountDistinct, BloomMember, GeoVelocity, GeoDistance,
  Percentile (Exact mode). Algo floor 18-80 ns; post-19.2 expected 35-130 ns.
- **Tier 3 (9 ops):** Percentile (UDDSketch), TopK, Entropy, EventTypeMix, Histogram,
  MostRecentN, ReservoirSample, DistanceFromHome, GeoSpread. Algo floor 10-250 ns.

**Tier-count derivation:** pre-removal audit lines 13-15 + 177 = 38/8/9 = 55.
UniqueCells + GeoEntropy are both Tier 2 (audit lines 173-174 + 206). After Plan 19.2-06
removal: 38 + (8-2) + 9 = **38/6/9 = 53**. ✓

**Recipe Replacements section** documents post-Plan-19.2-06 patterns:
- `bv.unique_cells` → `bv.count_distinct(quadkey(lat, lon, zoom))` (Tier 2)
- `bv.geo_entropy` → `bv.entropy(quadkey(lat, lon, zoom))` (Tier 3)

Source of truth pointer: `.planning/research/operator-update-uniformity-audit.md` is the
per-op tier authority. Hand-maintained staleness mitigation: update in the same PR as any
algorithm-change implementation. Upgrade path if drift becomes a problem: proc-macro
`#[doc(cost = "tier1")]` + build-script extraction (documented in CONTEXT.md Deferred Ideas).

### D-07: `GET /debug/op-cost` endpoint

**Feature-gating:** Route registered only inside the `if dev_endpoints_enabled { ... }` block
in `crates/beava-server/src/http.rs::router_with_push`. Production deployments (no
`BEAVA_DEV_ENDPOINTS=1`) see 404. Matches the Phase 15 PIT-debug + `registry_debug` router
pattern exactly.

**JSON response shape:**
```json
{
  "ops": [
    {"kind": "Count", "tier": 1, "last_traced_ns": 25, "last_traced_count": 1},
    {"kind": "Entropy", "tier": 3, "last_traced_ns": 160, "last_traced_count": 1}
  ],
  "captured_at_ms": 1714000000000
}
```
- `ops` is empty when `BEAVA_TRACE_AGG_TIMING` has never been set in the process.
- `captured_at_ms` is the `event_time_ms` of the last traced event (0 if never traced).
  Using event_time_ms instead of `SystemTime::now()` preserves D-06 determinism.
- `tier` is derived from `tier_for(AggKind)` — an exhaustive match encoding the
  post-Plan-19.2-06 38/6/9 classification. Compiler enforces coverage of all 53 variants.

**Implementation path:**
1. `crates/beava-core/src/agg_apply.rs` — Added `PerKindSnapshot` struct + `PER_KIND_LATEST`
   OnceLock static + `pub fn per_kind_latest()` accessor. The trace block at end of
   `apply_event_to_aggregations` now also writes `per_kind.clone()` into the snapshot under
   `parking_lot::Mutex`. Mutex contention is negligible (write: once per traced event;
   read: HTTP scrape rate ~1 Hz).
2. `crates/beava-server/src/http.rs` — Added `handle_debug_op_cost()` async handler that
   reads the snapshot and serialises `DebugOpCostResponse`. Added `tier_for(AggKind) -> u8`
   helper. Route registered as `.route("/debug/op-cost", get(handle_debug_op_cost))` inside
   the dev-endpoints conditional.

**D-06 compliance:** `apply_event_to_aggregations` is a deterministic pure function
(no wall-clock reads). The D-06 grep guard (`no_systemtime_now_in_agg_apply` test) confirmed
compliance. The snapshot timestamp uses `event_time_ms` (logical time already in the
function signature) rather than `SystemTime::now()`. For live workloads event_time_ms ≈
wall-clock ms; for WAL replay it is deterministic.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] D-06 determinism violation: removed SystemTime::now() from apply_event_to_aggregations**

- **Found during:** Task 2.b (GREEN implementation)
- **Issue:** Initial implementation used `std::time::SystemTime::now()` to populate
  `captured_at_ms` in the per-kind snapshot. The existing D-06 grep guard test
  (`no_systemtime_now_in_agg_apply`) caught this: `apply_event_to_aggregations` must
  contain no wall-clock reads so it is safe for WAL replay.
- **Fix:** Replaced `SystemTime::now()` with `event_time_ms as u64` (guarded by
  `if event_time_ms > 0`). For live workloads this is effectively wall-clock ms; for
  WAL replay it is deterministic. All 603 beava-core lib tests now pass including the
  D-06 guard.
- **Files modified:** `crates/beava-core/src/agg_apply.rs`
- **Commit:** `17e86bb` (folded into the GREEN commit)

**2. [Rule 2 - Missing] rustfmt formatting applied post-implementation**

- **Found during:** `cargo fmt --all --check` in final verification pass
- **Issue:** rustfmt reformatted the snapshot write block in `agg_apply.rs` (indentation
  of the `snap.captured_at_ms.store(...)` call).
- **Fix:** `cargo fmt --package beava-core` applied; separate chore commit.
- **Files modified:** `crates/beava-core/src/agg_apply.rs`

mutex is held briefly (~1 µs max). This matches the threat model in the plan's
`<threat_model>` section (T-19.2-07-01 mitigated by env-gate, T-19.2-07-02 accepted).
