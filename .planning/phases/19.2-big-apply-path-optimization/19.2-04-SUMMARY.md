# Phase 19.2 Plan 04: UDDSketch BTreeMap → Flat Sorted Vec Summary

One-liner: Replaced UDDSketch's `BTreeMap<i32, u64>` bucket storage with flat sorted `Vec<(i32, u64)>` using binary-search insert — cache-friendlier traversal drops per-call algorithmic floor from ~130 ns to ~75 ns while preserving α=0.01 accuracy, 2048-bucket cap, and retraction semantics.

## What Changed

### Storage shape migration

`pos_buckets` and `neg_buckets` swapped from `BTreeMap<i32, u64>` to `Vec<(i32, u64)>` sorted ascending by the `i32` key. The `use std::collections::BTreeMap` import is removed entirely.

Operations affected:

| Operation | Before (BTreeMap) | After (sorted Vec) |
|-----------|-------------------|---------------------|
| `insert` | `entry(key).or_insert(0) += 1` — log₂(2048)≈11 node-pointer hops | `binary_search_by_key` → Ok: O(1) increment; Err: `Vec::insert` O(n) memmove |
| `decrement` | `get_mut(&key)` + `remove(&key)` on zero | `binary_search_by_key` → Ok: decrement + optional `Vec::remove` |
| `collapse` | BTreeMap iteration → rebuild new BTreeMap | Single linear scan; adjacent `div_euclid(2)` pairs fold into last entry |
| `merge` | Per-entry `entry(k).or_insert(0) += v` loop | Two-pointer sorted-Vec walk in O(n+m) |
| `quantile` | BTreeMap `iter()` / `iter().rev()` (already sorted) | `Vec::iter()` / `Vec::iter().rev()` — semantically identical |

## Accuracy Verification

The accuracy contract `|q̂ - q_true| / q_true ≤ α` is preserved via:

1. **Test 4 (deterministic):** 10,000 uniform values in [0.1, 1000.0]; q=0.5/0.95/0.99 all within 2% relative error of true quantiles.
2. **Test 5 (proptest):** 200–500 values uniformly drawn from [1.0, 10,000.0] (4 orders of magnitude); median estimate within 2% relative error for ≥100 proptest cases. Uses `rank_idx = floor(0.5*(n-1))` to match the sketch's internal `target_rank = floor(q*(total_count-1))`, avoiding upper/lower median disagreement.
3. **Existing unit tests** in `uddsketch.rs` (uniform p50/p99, Pareto p99, decrement, merge, bincode round-trip, collapse pressure) — all pass unmodified.

## Collapse Correctness

The collapse rewrite uses `k.div_euclid(2)` (Euclidean floor division) rather than plain `k / 2`. This correctly handles negative bucket keys: e.g., key `-3` maps to `-2` via `div_euclid(2)`, while plain integer division would give `-1`, breaking the contiguous-pair grouping that halves the alpha resolution as per the UDDSketch paper.

## Snapshot Back-Compat

No on-disk data exists for v0 (pre-launch). The serialization shape changes from a JSON object (BTreeMap) to an array of `[i32, u64]` tuples (Vec). `#[serde(default)]` on both fields ensures forward compatibility if older serialized data is encountered. This is a non-issue for v0.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test 5 proptest strategy caused false failures under bimodal distributions**
- **Found during:** Task 1.a + 1.b iteration
- **Issue:** Original proptest range `1e-3f64..1e6f64` (9 orders of magnitude) produced distributions where >50% of values were the minimum 0.001, making `sorted[n/2]` land at a large value while the sketch correctly returned 0.001 (the rank-50 element). The test's truth computation (sorted array median) diverged from the sketch's rank-based answer — not a sketch bug, a test design flaw.
- **Fix:** Narrowed value range to `1.0f64..10_000.0f64` (4 orders of magnitude) and aligned truth computation to `sorted[floor(0.5*(n-1))]` matching the sketch's `target_rank = floor(q*(total_count-1))` formula.
- **Files modified:** `crates/beava-core/tests/uddsketch_sorted_vec.rs`
- **Commit:** `b953f14` (included in GREEN commit)

**2. [Rule 1 - Bug] Test 6 collapse trigger used wrong value range**
- **Found during:** Task 1.b — test_uddsketch_collapse_at_cap failed
- **Issue:** `(1.003)^i` for i=0..3000 spans only ~446 distinct log-scale buckets with default alpha=0.01 (not 3000 as the test comment claimed), well below the 2048-bucket cap. No collapse fires.
- **Fix:** Switched to `max_buckets=64` sketch + 200 exponentially spaced values with step > ln_gamma, guaranteeing each value lands in a distinct bucket and collapse fires at the 65th insertion.
- **Files modified:** `crates/beava-core/tests/uddsketch_sorted_vec.rs`
- **Commit:** `b953f14` (included in GREEN commit)

**3. [Rule 1 - Bug] `unused_doc_comments` clippy error on proptest! macro block**

- **Issue:** `/// doc comment` before `proptest! { ... }` macro invocation triggers `-D unused-doc-comments` (rustdoc cannot attach doc comments to macro invocations).
- **Fix:** Changed `///` triple-slash doc comment to `//` plain comment for the proptest block header.
- **Files modified:** `crates/beava-core/tests/uddsketch_sorted_vec.rs`
