## Seven open questions for `/gsd-discuss-phase 19.2`

1. **Field pre-extraction storage shape**: indexed array (`Vec<&Value>` with field-idx-into-row) vs hashmap (`HashMap<&str, &Value>`). _Lean: indexed array._
2. **HLL hasher choice**: FxHasher (fastest, non-keyed) vs AHasher-cached (HashDoS-resistant). _Lean: FxHasher (fraud-internal workload, no HashDoS concern)._
3. **EntityKey single-u64 collision approach**: Approach A (numeric only, zero collision, narrow) vs C (hybrid: SingleU64+SingleStr+Multi). _Lean: Approach C._
4. **Cluster storage shape**: shared `Vec<AggOp>` across clustered aggs (split by agg_id via secondary indirection at update time) vs separate Vec<AggOp> per agg with shared row lookup. _Lean: split-by-agg_id (simpler)._
5. **UniqueCells/GeoEntropy cap behavior at threshold**: drop new cells (truncate) vs fall back to HLL approximation vs LRU eviction. _No lean — needs discussion._
6. **Cost-class doc surface**: source attribute (`#[doc(cost = "tier1")]`) vs separate markdown table in operator catalogue docs site. _No lean — needs discussion._
7. **Observability endpoint gating**: `/debug/op-cost` always-on vs feature-gated behind `BEAVA_DEV_ENDPOINTS=1`. _Lean: feature-gated (matches Phase 15 pattern for dev endpoints)._
