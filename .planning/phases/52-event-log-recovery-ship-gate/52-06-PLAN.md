<objective>
Implement LSN-based dedup for fork/replica (TPC-CORR-06 scope expansion, CONTEXT D-09
through D-11). Every log entry gains a `lsn: u64` field packed as
`(upstream_shard_id: u8 << 56) | (stream_ord: u16 << 40) | (seq: u40)`. The replica tracks
`max_lsn_seen(stream, upstream_shard_id)` persistently in snapshot v8 `replica_lsn_map`.
On OP_SUBSCRIBE reconnect, events with LSN <= max_lsn_seen are silently dropped, closing
the upstream-rolling-restart double-emit window identified in PITFALLS.md §5.2.

Purpose: Without LSN dedup, an upstream rolling restart causes the replica to replay already-
applied events, doubling feature counts. This is a user-accepted scope expansion to
TPC-CORR-06 (2026-04-18). CONTEXT D-10: 40-bit seq supports ~1T events per (stream, shard)
pair — sufficient for any realistic stream lifetime.

Output: `lsn: u64` on `LogEntry`, pack/unpack helpers, per-(stream,shard) seq counters in
`EventLog::append`, dedup filter in replica ingest, `replica_lsn_map` in SnapshotV8.
</objective>

<!-- LSN packing (CONTEXT D-10) -->
<!-- bits 63-56: upstream_shard_id (u8) -->
<!-- bits 55-40: stream_ord (u16) -->
<!-- bits 39-0:  seq (u40, max ~1 trillion) -->
```rust
pub fn lsn_pack(upstream_shard_id: u8, stream_ord: u16, seq: u64) -> u64 {
    ((upstream_shard_id as u64) << 56)
        | ((stream_ord as u64) << 40)
        | (seq & 0x00FF_FFFF_FFFF)
}

pub fn lsn_unpack(lsn: u64) -> (u8, u16, u64) {
    let upstream_shard_id = (lsn >> 56) as u8;
    let stream_ord = ((lsn >> 40) & 0xFFFF) as u16;
    let seq = lsn & 0x00FF_FFFF_FFFF;
    (upstream_shard_id, stream_ord, seq)
}
```
