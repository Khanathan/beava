# Phase 52 Plan 01: Snapshot v8 Migration Summary

**One-liner:** Snapshot format bumped to v8 with `shard_count: u16` and `replica_lsn_map`; hard-fail boot guard (TPC-CORR-02) refuses boot when snapshot shard_count != BEAVA_SHARDS; full v6/v7/v8 read compat with zero-downtime N=1 upgrade path.

## What Was Built

### Task 1: Snapshot v8 Struct + v7/v8 Read Dispatch

`src/state/snapshot.rs`:
- `SNAPSHOT_FORMAT_VERSION` bumped from 7 to 8
- Added `LEGACY_V7_FORMAT: u8 = 7` constant
- Added `BaseSnapshotStateV8` struct: same fields as `BaseSnapshotState` plus `shard_count: u16` and `#[serde(default)] replica_lsn_map: HashMap<(String, u8), u64>`
- `BaseSnapshotStateV7` type alias for `BaseSnapshotState` (same on-disk layout)
- `load_snapshot_file`: v6 → promote to v8 (shard_count=1, empty lsn_map); v7 → promote to v8; v8 → decode directly; unknown → return None (T-52-01-01)
- `SnapshotFile::Base` variant now carries `BaseSnapshotStateV8`
- `save_base_snapshot_v8` — explicit v8 writer
- `save_base_snapshot` — wraps `BaseSnapshotState` to v8 with shard_count=1 (existing call sites unchanged)
- `save_base_snapshot_v7_for_test` — test helper to write frozen v7 bytes
- `load_snapshot` legacy API updated to handle v7 bytes on top of existing v5/v6/v8 handling

`tests/fixtures/snapshot_v7_sample.bin`: frozen 117-byte v7 binary with one entity (key="u3", 3 CountOp events). Generated by `generate_v7_fixture` `#[ignore]` test, committed as a regression artifact.

`tests/test_snapshot_v8_migration.rs`: 9 behavioral tests:
1. v7 fixture loads with shard_count=1 and empty replica_lsn_map
2. v8 round-trip: all fields (entities, pipelines, shard_count, replica_lsn_map) survive postcard encode/decode
3. v7 bytes → v8 promotion: entities intact, shard_count=1
4. shard_count=3 vs BEAVA_SHARDS=8 → exact TPC-CORR-02 error string
5. shard_count=1 vs BEAVA_SHARDS=1 → success
6. v7 promoted (shard_count=1) vs BEAVA_SHARDS=1 → success
7. v7 promoted (shard_count=1) vs BEAVA_SHARDS=8 → guard fires

### Task 2: Hard-Fail Boot Guard on Shard Count Mismatch

`src/state/store.rs`:
- `check_shard_count_guard(snapshot_shard_count: u16, beava_shards: u16) -> Result<(), String>`: pure guard function. Returns `Err(exact TPC-CORR-02 string)` when counts differ, `Ok(())` on match.
- `read_beava_shards() -> u16`: reads `BEAVA_SHARDS` env var, defaults to 1 on absent/malformed (T-52-01-03).

Exact error string (TPC-CORR-02 mandate):
```
snapshot shard_count=N but BEAVA_SHARDS=K — run 'tally reshard --from N --to K' then restart
```
(em dash U+2014 as required)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] SnapshotFile::Base variant type change required caller updates**
- **Found during:** Task 1 implementation
- **Issue:** Promoting `SnapshotFile::Base` from `BaseSnapshotState` to `BaseSnapshotStateV8` broke three call sites: `tcp.rs::load_base_snapshot_for_fetch`, `replica.rs::filter_base_snapshot`, and two integration test files.
- **Fix:** Updated all four callers to use `BaseSnapshotStateV8`. `filter_base_snapshot` now propagates `shard_count` and `replica_lsn_map` through filtered snapshots (correct behavior).
- **Files modified:** `src/server/tcp.rs`, `src/server/replica.rs`, `tests/test_incremental_snapshot.rs`, `tests/test_replica_snapshot_fetch.rs`
- **Commits:** d2ae67c

**2. [Rule 2 - Missing] `save_base_snapshot` needed to stay backward-compat for existing callers**
- **Found during:** Task 1 implementation
- **Issue:** Plan specified only `save_base_snapshot_v8`; many existing callers build `BaseSnapshotState` structs. Requiring all callers to switch to `BaseSnapshotStateV8` would break the snapshot rollover test, hybrid ops test, and others.
- **Fix:** `save_base_snapshot` wraps `BaseSnapshotState` into `BaseSnapshotStateV8` with `shard_count=1` before writing. This preserves backward compat for all existing call sites while ensuring everything on disk is v8.
- **Files modified:** `src/state/snapshot.rs`

## Known Stubs

- `replica_lsn_map` is always written as `HashMap::new()` — the field is scaffolded per D-11 but population deferred to Phase 52-06 (LSN dedup plan). This is intentional; `#[serde(default)]` ensures future v8 snapshots without the map load cleanly.
