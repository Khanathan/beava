### Response frame bodies (JSON payloads — content_type=CT_JSON always in v0)

**Ping response (OP_PING response, 200-equivalent):**
```json
{
  "server_version": "0.1.0",
  "registry_version": 7
}
```

**Register success response (OP_REGISTER response, 200-equivalent):**
Same shape as HTTP `RegisterSuccess`:
```json
{
  "status": "ok",
  "registry_version": 3,
  "registered_descriptors": [...],
  "added": [...],
  "already_present": [...]
}
```

**Register validation / conflict (OP_ERROR_RESPONSE):**
Same shape as HTTP `RegisterErrorBody`:
```json
{"error": {"code": "invalid_registration", "path": "nodes[0].schema", "reason": "..."}, "registry_version": 0}
{"error": {"code": "registration_conflict", "message": "...", "diff": {...}}, "registry_version": 3}
```

**Parse error (OP_ERROR_RESPONSE after CT_JSON body won't deserialize):**
```json
{"error": {"code": "invalid_registration", "path": "<body>", "reason": "..."}, "registry_version": <current>}
```

**Frame-level errors (OP_ERROR_RESPONSE, TCP-specific codes):**
```json
{"error": {"code": "unknown_op", "message": "opcode 0x4242"}, "registry_version": <current>}
{"error": {"code": "op_not_implemented", "message": "opcode 0x0010 (push) reserved for Phase 6"}, "registry_version": <current>}
{"error": {"code": "unsupported_content_type", "reason": "MessagePack payload encoding ships in Phase 6 / 12"}, "registry_version": <current>}
{"error": {"code": "frame_too_large", "limit": 4194304, "declared": 99999999}, "registry_version": <current>}
{"error": {"code": "malformed_frame", "reason": "declared length cannot cover op+content_type"}, "registry_version": <current>}
```

### tcp.rs module layout

```rust
//! TCP wire listener + accept loop + per-connection handler dispatch.
//!
//! Accept model: one tokio task spawns per connection on the SAME current_thread
//! runtime that drives HTTP. Each connection runs a strict-FIFO loop:
//! read one frame → dispatch to handler → write response frame → repeat.
//! Pipelining works because the client can send N frames without waiting;
//! the server reads/dispatches/writes one at a time, preserving order (Redis-RESP model).

use anyhow::Context;
use beava_core::registry::Registry;
use beava_core::wire::{
    decode_frame, encode_frame, opcode_name, reserved_phase, CT_JSON, CT_MSGPACK, Frame,
    FrameError, OP_ERROR_RESPONSE, OP_PING, OP_REGISTER,
};
use bytes::{Bytes, BytesMut};
use serde_json::{json, Value};
use std::net::SocketAddr;
use std::sync::Arc;
use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::net::{TcpListener, TcpStream};
use tokio_util::sync::CancellationToken;  // NEW dep — add to Cargo.toml

use crate::register::{execute_register, RegisterOutcome, RegisterPayload};

/// Handle to a bound TCP listener plus its max_frame_bytes for the accept loop.
pub struct TcpListenerHandle {
    listener: TcpListener,
    local_addr: SocketAddr,
    max_frame_bytes: u32,
}

impl TcpListenerHandle {
    pub async fn bind(host: &str, port: u16, max_frame_bytes: u32) -> Result<Self, std::io::Error> {
        let addr: SocketAddr = format!("{}:{}", host, port).parse()
            .map_err(|e| std::io::Error::new(std::io::ErrorKind::InvalidInput, e))?;
        let listener = TcpListener::bind(addr).await?;
        let local_addr = listener.local_addr()?;
        tracing::info!(
            target: "beava.tcp",
            kind = "tcp.listener_bound",
            addr = %local_addr,
            max_frame_bytes,
            "TCP listener bound"
        );
        Ok(Self { listener, local_addr, max_frame_bytes })
    }

    pub fn local_addr(&self) -> SocketAddr { self.local_addr }
}

/// Accept-loop task body. Runs until `cancel` is tripped.
pub(crate) async fn accept_loop(
    handle: TcpListenerHandle,
    registry: Arc<Registry>,
    cancel: CancellationToken,
) {
    let TcpListenerHandle { listener, local_addr: _, max_frame_bytes } = handle;

    let connections_tracker = tokio_util::task::TaskTracker::new();  // NEW: tokio-util feature

    loop {
        tokio::select! {
            biased;
            _ = cancel.cancelled() => {
                tracing::info!(target: "beava.tcp", "TCP accept loop cancelled; draining");
                break;
            }
            accept_res = listener.accept() => {
                match accept_res {
                    Ok((stream, peer)) => {
                        tracing::info!(
                            target: "beava.tcp",
                            kind = "tcp.connection_accepted",
                            peer = %peer,
                            "TCP connection accepted"
                        );
                        let reg = Arc::clone(&registry);
                        let cancel_child = cancel.clone();
                        let mfb = max_frame_bytes;
                        connections_tracker.spawn(async move {
                            if let Err(e) = handle_connection(stream, reg, cancel_child, mfb).await {
                                tracing::warn!(
                                    target: "beava.tcp",
                                    kind = "tcp.handler_error",
                                    error = %e,
                                    "connection handler exited with error"
                                );
                            }
                        });
                    }
                    Err(e) => {
                        tracing::warn!(
                            target: "beava.tcp",
                            kind = "tcp.accept_error",
                            error = %e,
                            "accept failed; continuing"
                        );
                    }
                }
            }
        }
    }

    // Drain
    connections_tracker.close();
    connections_tracker.wait().await;
    tracing::info!(target: "beava.tcp", "TCP accept loop drained");
}

/// Per-connection read→dispatch→write loop. Strict FIFO.
pub(crate) async fn handle_connection<S>(
    mut stream: S,
    registry: Arc<Registry>,
    cancel: CancellationToken,
    max_frame_bytes: u32,
) -> anyhow::Result<()>
where
    S: tokio::io::AsyncRead + tokio::io::AsyncWrite + Unpin,
{
    let mut read_buf = BytesMut::with_capacity(8 * 1024);
    let mut write_buf = BytesMut::with_capacity(8 * 1024);

    loop {
        // Try to decode a frame already in the buffer
        match decode_frame(&mut read_buf, max_frame_bytes) {
            Ok(Some(frame)) => {
                tracing::trace!(
                    target: "beava.tcp",
                    kind = "tcp.frame_received",
                    op = format!("{:#06x}", frame.op),
                    content_type = format!("{:#04x}", frame.content_type),
                    payload_len = frame.payload.len(),
                    "frame received"
                );
                let response = dispatch(&registry, frame).await;
                write_buf.clear();
                encode_frame(&response, &mut write_buf);
                stream.write_all(&write_buf).await
                    .context("write response frame")?;
                // Loop: try another frame from the buffer first.
                continue;
            }
            Ok(None) => {
                // Need more bytes — fall through to read.
            }
            Err(FrameError::TooLarge { declared_len, limit }) => {
                tracing::warn!(target: "beava.tcp", kind = "tcp.frame_error",
                    error = "too_large", declared_len, limit);
                let err_frame = build_error_frame(
                    &registry,
                    "frame_too_large",
                    json!({"limit": limit, "declared": declared_len}),
                );
                write_buf.clear();
                encode_frame(&err_frame, &mut write_buf);
                let _ = stream.write_all(&write_buf).await;
                let _ = stream.shutdown().await;
                tracing::debug!(target: "beava.tcp", kind = "tcp.connection_closed", reason = "frame_too_large");
                return Ok(());  // close connection
            }
            Err(FrameError::LengthUnderflow { declared_len }) => {
                tracing::warn!(target: "beava.tcp", kind = "tcp.frame_error",
                    error = "length_underflow", declared_len);
                let err_frame = build_error_frame(
                    &registry,
                    "malformed_frame",
                    json!({"reason": "declared length cannot cover op+content_type", "declared": declared_len}),
                );
                write_buf.clear();
                encode_frame(&err_frame, &mut write_buf);
                let _ = stream.write_all(&write_buf).await;
                let _ = stream.shutdown().await;
                tracing::debug!(target: "beava.tcp", kind = "tcp.connection_closed", reason = "malformed_frame");
                return Ok(());
            }
        }

        // Read more bytes (or cancel)
        tokio::select! {
            biased;
            _ = cancel.cancelled() => {
                tracing::debug!(target: "beava.tcp", kind = "tcp.connection_closed", reason = "shutdown");
                let _ = stream.shutdown().await;
                return Ok(());
            }
            read_res = stream.read_buf(&mut read_buf) => {
                match read_res {
                    Ok(0) => {
                        // Client closed (EOF). If buffer has pending bytes it's a truncated
                        // frame — log and close.
                        if !read_buf.is_empty() {
                            tracing::debug!(
                                target: "beava.tcp",
                                kind = "tcp.connection_closed",
                                reason = "truncated",
                                buffered_bytes = read_buf.len(),
                                "client closed mid-frame"
                            );
                        } else {
                            tracing::debug!(target: "beava.tcp", kind = "tcp.connection_closed", reason = "client_close");
                        }
                        return Ok(());
                    }
                    Ok(_n) => { /* loop back and try to decode */ }
                    Err(e) => return Err(e).context("stream read"),
                }
            }
        }
    }
}

/// Map an inbound frame to an outbound frame.
async fn dispatch(registry: &Arc<Registry>, frame: Frame) -> Frame {
    match frame.op {
        OP_PING => handle_ping(registry, &frame).await,
        OP_REGISTER => handle_register(registry, &frame).await,
        op if opcode_name(op).is_some() && reserved_phase(op).is_some() => {
            // Known but reserved (push/push_sync/…/mset/…)
            let name = opcode_name(op).unwrap();
            let phase = reserved_phase(op).unwrap();
            build_error_frame(registry, "op_not_implemented", json!({
                "message": format!("opcode {:#06x} ({}) reserved for {}", op, name, phase)
            }))
        }
        op => {
            // Unknown opcode (not in the table)
            build_error_frame(registry, "unknown_op", json!({
                "message": format!("opcode {:#06x}", op)
            }))
        }
    }
}

async fn handle_ping(registry: &Arc<Registry>, _frame: &Frame) -> Frame {
    let body = json!({
        "server_version": env!("CARGO_PKG_VERSION"),
        "registry_version": registry.version(),
    });
    let bytes = serde_json::to_vec(&body).expect("serialize ping");
    Frame { op: OP_PING, content_type: CT_JSON, payload: Bytes::from(bytes) }
}

async fn handle_register(registry: &Arc<Registry>, frame: &Frame) -> Frame {
    // Content-type policy: register requires JSON in v0.
    if frame.content_type == CT_MSGPACK {
        return build_error_frame(registry, "unsupported_content_type", json!({
            "reason": "MessagePack payload encoding ships in Phase 6 / 12"
        }));
    }
    if frame.content_type != CT_JSON {
        return build_error_frame(registry, "unsupported_content_type", json!({
            "reason": format!("unknown content_type {:#04x}", frame.content_type)
        }));
    }

    // Parse JSON → RegisterPayload
    let payload: RegisterPayload = match serde_json::from_slice(&frame.payload) {
        Ok(p) => p,
        Err(e) => {
            tracing::warn!(target: "beava.tcp", kind = "register.parse_error",
                reason = %e, "malformed register payload over TCP");
            return build_error_frame(registry, "invalid_registration", json!({
                "path": "<body>",
                "reason": e.to_string(),
            }));
        }
    };

    let outcome = execute_register(registry, payload).await;
    match outcome {
        RegisterOutcome::Success { version, registered_descriptors, added, already_present } => {
            build_success_frame(OP_REGISTER, json!({
                "status": "ok",
                "registry_version": version,
                "registered_descriptors": registered_descriptors,
                "added": added,
                "already_present": already_present,
            }))
        }
        RegisterOutcome::EmptyPayload { version } => {
            build_success_frame(OP_REGISTER, json!({
                "status": "ok",
                "registry_version": version,
                "registered_descriptors": [],
                "added": [],
                "already_present": [],
            }))
        }
        RegisterOutcome::Noop { version, registered_descriptors, already_present } => {
            build_success_frame(OP_REGISTER, json!({
                "status": "ok",
                "registry_version": version,
                "registered_descriptors": registered_descriptors,
                "added": [],
                "already_present": already_present,
            }))
        }
        RegisterOutcome::ValidationFailed { version, first_error_path, first_error_reason, .. } => {
            Frame {
                op: OP_ERROR_RESPONSE, content_type: CT_JSON,
                payload: Bytes::from(serde_json::to_vec(&json!({
                    "error": {"code": "invalid_registration", "path": first_error_path, "reason": first_error_reason},
                    "registry_version": version,
                })).unwrap()),
            }
        }
        RegisterOutcome::Conflict { version, added, changed } => {
            Frame {
                op: OP_ERROR_RESPONSE, content_type: CT_JSON,
                payload: Bytes::from(serde_json::to_vec(&json!({
                    "error": {
                        "code": "registration_conflict",
                        "message": "Registration would change or remove existing descriptors",
                        "diff": {"added": added, "removed": [], "changed": changed},
                    },
                    "registry_version": version,
                })).unwrap()),
            }
        }
    }
}

fn build_success_frame(op: u16, body: Value) -> Frame {
    Frame {
        op,
        content_type: CT_JSON,
        payload: Bytes::from(serde_json::to_vec(&body).expect("serialize success")),
    }
}

fn build_error_frame(registry: &Arc<Registry>, code: &str, extra: Value) -> Frame {
    let mut err_obj = json!({"code": code});
    // merge `extra` object keys into err_obj
    if let Some(obj) = extra.as_object() {
        for (k, v) in obj {
            err_obj[k] = v.clone();
        }
    }
    let body = json!({
        "error": err_obj,
        "registry_version": registry.version(),
    });
    Frame {
        op: OP_ERROR_RESPONSE,
        content_type: CT_JSON,
        payload: Bytes::from(serde_json::to_vec(&body).expect("serialize error")),
    }
}
```

### Server::serve: spawn both listeners on the same cancel token

```rust
pub async fn serve<F>(self, shutdown: F) -> Result<(), ServerError>
where F: Future<Output = ()> + Send + 'static,
{
    let cancel = CancellationToken::new();

    // Translate the external `shutdown` future into a cancel.
    let cancel_for_signal = cancel.clone();
    tokio::spawn(async move {
        shutdown.await;
        cancel_for_signal.cancel();
    });

    // Spawn TCP accept loop if enabled
    let tcp_task = self.tcp_listener_handle.map(|handle| {
        let reg = Arc::clone(&self.registry);
        let cancel_child = cancel.clone();
        tokio::spawn(crate::tcp::accept_loop(handle, reg, cancel_child))
    });

    // HTTP serve with graceful shutdown tied to the same cancel
    let app = router(self.readiness, self.registry, self.dev_endpoints);
    let http_cancel = cancel.clone();
    let http_shutdown = async move { http_cancel.cancelled().await; };

    let start = Instant::now();
    axum::serve(self.http_listener, app)
        .with_graceful_shutdown(http_shutdown)
        .await
        .map_err(ServerError::Serve)?;

    // Trip cancel in case HTTP exited for a reason other than cancel (error path).
    cancel.cancel();
    if let Some(t) = tcp_task {
        let _ = t.await;  // ignore join errors; accept_loop logs its own exit
    }

    let elapsed_ms = start.elapsed().as_millis() as u64;
    tracing::info!(target: "beava.shutdown", duration_ms = elapsed_ms, "shutdown complete");
    Ok(())
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Remote TCP client → handle_connection | Untrusted bytes; every frame is potentially malformed, oversized, or misrouted |
| handle_register → execute_register | Reuse of Phase 2 trust boundary; no new surface |
| accept_loop task spawn → per-connection task | Task panic in one handler must not crash the accept loop (tokio::spawn isolates panics by default) |
| CancellationToken → all TCP tasks | Shutdown signal must propagate; no task may block forever on a read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.5-04-01 | DoS | Attacker opens N connections and sends no bytes (slow-connect flood) | accept | Each idle connection buffers only the initial BytesMut (8 KiB) + stack. Tokio handles 1000s of idle tasks cheaply. v0 has no per-IP rate limit — operators front with nginx / Cloudflare if needed. Documented in PROJECT.md Constraints. |
| T-02.5-04-02 | DoS | Oversize frame → server tries to allocate max_frame_bytes | mitigate | `decode_frame` rejects oversize BEFORE any allocation (Plan 02 T-02.5-02-01). Handler writes error + shuts down connection in ≤ a few hundred bytes. |
| T-02.5-04-03 | Tampering | Malformed frame causes desync, poisoning subsequent frames | mitigate | On FrameError::LengthUnderflow / TooLarge, `handle_connection` closes the connection. There is no recovery path that resumes reading — the buffer cursor is not advanced on errors (Plan 02 contract), but the connection closes anyway to eliminate the desync risk entirely. |
| T-02.5-04-04 | Spoofing | Attacker sends OP_REGISTER with forged `registry_version` expectation (e.g., expecting optimistic concurrency) | accept | v0 has NO optimistic-concurrency check — the JSON payload is additive-only, and additive operations commute. `registry_version` in the response is informational only. Client cannot force a version mismatch. |
| T-02.5-04-05 | DoS | Handler panic crashes the entire runtime | mitigate | `tokio::spawn` isolates panics to the task that raised them. `TaskTracker::spawn` is used for per-connection tasks, so a panic in one handler leaves the accept loop and other connections intact. Plan 05 adds a smoke test that confirms this (injected-panic connection + concurrent healthy connection). |
| T-02.5-04-06 | Repudiation | TCP operations not logged / auditable | mitigate | Every TCP operation emits a structured tracing event: tcp.listener_bound, tcp.connection_accepted, tcp.frame_received, tcp.frame_error, tcp.handler_error, tcp.connection_closed. Same `kind` field as HTTP (tcp.* vs register.*). Ops dashboards key on these. |
| T-02.5-04-07 | Information disclosure | Error payloads leak server internals (paths, stack traces) | mitigate | `build_error_frame` constructs payloads from hand-picked fields (code, message, reason, limit, declared, path). Never passes `anyhow::Error` strings or stack traces. `reason` fields for parse errors reflect `serde_json::Error::to_string()` which is bounded to "expected X at line Y, column Z" — safe v0 posture. |
| T-02.5-04-08 | Elevation | TCP bypasses HTTP's auth | accept | v0 has NO auth on either listener (matches CONTEXT.md D-06 + PROJECT.md). Commercial tier adds auth on both. |
| T-02.5-04-09 | Correctness | TCP register bypasses Content-Type policing that HTTP enforces (415) | mitigate | `handle_register` checks `frame.content_type` and returns `unsupported_content_type` for CT_MSGPACK (0x02) and any non-CT_JSON. This mirrors HTTP's 415 behavior — test `handle_register_msgpack_content_type_returns_unsupported` proves it. |
| T-02.5-04-10 | DoS | Write back-pressure: slow client never drains response; server buffer grows | accept | `stream.write_all` awaits until bytes are flushed. If a slow client hangs, `write_all` blocks and the handler cannot read the next frame — the connection stalls, but no unbounded buffer growth occurs. tokio's socket buffers are OS-bounded (~64 KiB default). An idle-write timeout is deferred to v0.x. |
| T-02.5-04-11 | Lock hold across await | handle_register takes the registry write lock then awaits | mitigate | `execute_register` is async but the `apply_registration` call is synchronous — the write lock is taken and released inside a non-async block. Plan 03 already enforced this via clippy's `await_holding_lock`. Re-verified here by running `cargo clippy --workspace --all-targets -- -D warnings`. |
| T-02.5-04-12 | Shutdown: connections never drain | CancellationToken trip, in-flight frame being written | mitigate | handle_connection's main loop checks cancel BEFORE reading, but a mid-write frame completes before the next iteration checks cancel. Worst case: one extra response frame per connection finishes after SIGTERM, bounded by write time. TaskTracker::wait blocks until all per-connection tasks return — so Server::serve waits for this drain. |
