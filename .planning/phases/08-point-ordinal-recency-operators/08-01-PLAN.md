# Plan 08-01 ÔÇö Point/ordinal operators (first, last, first_n, last_n, lag)

**REQ-IDs:** AGG-POINT-01..AGG-POINT-05

## Goal

Land the 5 point/ordinal operators on top of Phase 5's `AggOp` framework.
Each ships as a `*State` struct in `agg_state.rs`, an `AggKind` + `AggOp`
variant, JSON-wire compile in `agg_compile.rs`, schema inference via
`output_type_for`, and redÔćĺgreen per-task tests.

## Tasks

### Task 1 ÔÇö `FirstState` / `LastState` (red Ôćĺ green)

**Red:** add tests in `agg_state.rs::tests` asserting:
- `FirstState` records the value of the first event's field, ignores
  subsequent updates, returns `Value::Null` when no events seen.
- `LastState` records the value of every event's field, returns
  `Value::Null` when no events seen.
- Both respect `where_matched=false` (skip the update).
- Both respect `where_matched=None` field path (skip if field missing/null).

**Green:** implement minimal structs, run tests.

### Task 2 ÔÇö `FirstNState` / `LastNState` (red Ôćĺ green)

**Red:** tests for both, parameterised over `n` (the descriptor field).
- `FirstNState`: stores the first n field values in arrival order; returns
  `Value::Str(serde_json::to_string(&list))` (D-07: JSON-array string).
  Ignores once full.
- `LastNState`: ring-buffered; cap n; returns JSON-array string of the last
  n in arrival order (oldest first).

**Green:** implement.

### Task 3 ÔÇö `LagState` (red Ôćĺ green)

**Red:** tests:
- `LagState::new(n)` with bounded VecDeque cap n+1; on event, pushes the
  current value and evicts when len > n+1; on query returns the value at
  index 0 if `len == n+1` else Null.
- Respects `where_matched`.

**Green:** implement.

### Task 4 ÔÇö Wire `AggKind` + `AggOp` + dispatch + `output_type_for` + JSON
compile (red Ôćĺ green)

**Red:** integration-style tests in `agg_op.rs::tests`:
- `AggOp::new` for `desc { kind: First, field: Some("x"), .. }` returns
  `AggOp::First(_)`.
- `AggOp::query` dispatches to per-state query for each new variant.
- `output_type_for` returns expected types per D-08.
- `agg_compile::parse_agg_kind` recognises the 5 new strings.
- JSON-wire test with `{"op": "first_n", "params": {"field": "x", "n": 3}}`
  compiles to a NamedAggOp with kind=FirstN and `n=Some(3)`.

**Green:** add `AggKind::{First, Last, FirstN, LastN, Lag}` variants;
extend `AggOp` enum with the 5 new variants; thread `AggOpDescriptor.n:
Option<u32>`; extend `parse_agg_kind`; extend `output_type_for`; extend
`extract_agg_params` to read `n`; extend `compile_aggregations_from_nodes`
to validate n (>0, ÔëĄ1024) for ops that need it.

### Task 5 ÔÇö End-to-end smoke (red Ôćĺ green)

**Red:** `crates/beava-core/tests/phase8_point_ops.rs` ÔÇö register a
GroupBy with one each of {first, last, first_n, last_n, lag}, push 5
events, assert each feature returns the expected value via
`apply_event_to_aggregations` + state-table query.

**Green:** the bench should "just work" once Tasks 1-4 land.
