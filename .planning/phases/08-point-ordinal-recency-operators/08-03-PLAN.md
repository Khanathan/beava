# Plan 08-03 — TCP `OP_PUSH` handler (Phase 8 folded scope)

**Folded from:** Phase 2.5 OP_PUSH reservation; Phase 7.5 throughput harness
could only exercise HTTP push. Phase 8+ throughput rows need TCP coverage.

## Goal

Wire the TCP `OP_PUSH` opcode to the same apply path as the HTTP push
handler, sharing `WalSink + IdemCache + state_tables` via `AppState`.

## Decisions

- **D-01:** Extract `execute_push(&AppState, event_name, body_bytes) → PushOutcome`
  from the HTTP handler. Both transports consume this.
- **D-02:** Add `accept_loop_with_app(handle, app, cancel)` variant that
  threads `Arc<AppState>` through. Legacy `accept_loop(handle, registry, cancel)`
  kept (test-only) so Phase 2.5 unit tests don't churn.
- **D-03:** Wire TCP `OP_PUSH` envelope as JSON: `{"event": "<name>",
  "body": {...}}`. MessagePack reserved (CT=0x02) — returns
  `unsupported_content_type` in v0 (matches HTTP content-type policy).
- **D-04:** On success: return frame with op=OP_PUSH + JSON body
  (`PushAck`). On idempotent replay: re-serialize cached response with
  `idempotent_replay: true`. On error: emit `OP_ERROR_RESPONSE` with the
  same `code` strings as the HTTP path (`invalid_event`, `event_not_found`,
  `wal_unavailable`).
- **D-05:** Update `server.rs` `serve()` to spawn `accept_loop_with_app`
  for production (folded from Phase 2.5 which was registry-only).
- **D-06:** Update `wire.rs::reserved_phase` to remove `OP_PUSH`; update
  the doc-table comment to reflect Phase 8 implementation.
- **D-07:** Ship a `push_json` helper on `TcpClient` so integration tests
  have parity with `register_json`.

## Tasks

1. Extract `execute_push` + `PushOutcome` enum in `push.rs`; shim HTTP
   `push_handler` through it (red → green via existing Phase 6 tests
   which all still pass).
2. Add `accept_loop_with_app` + `handle_push` in `tcp.rs`; thread
   `Option<Arc<AppState>>` through dispatcher (red → green via new tests).
3. Wire into `server.rs serve()`; update `wire.rs::reserved_phase`.
4. New integration test `tests/phase8_tcp_push.rs` — 6 tests covering:
   success + ack_lsn, unknown event, invalid body, full TCP→apply→HTTP
   loop, pipelined 3 pushes, idempotent replay with flag flipped.
