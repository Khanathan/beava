# Plan 08-02 — Recency markers + streak family + first_seen_in_window

**REQ-IDs:** AGG-POINT-06..AGG-POINT-11, AGG-RECENCY-01..AGG-RECENCY-04

## Goal

Land the 10 remaining Phase 8 operators on top of the state structs that
shipped in Plan 08-01 (necessary for the AggKind exhaustive-match
cohesion). Plan 08-02's deliverable is comprehensive unit-test coverage
for the per-op semantics.

## Operators

- `first_seen` — Datetime of first matching event (AGG-POINT-06)
- `last_seen` — Datetime of most recent matching event (AGG-POINT-07)
- `age` — ms since first matching event (AGG-POINT-08)
- `has_seen` — Bool: ever observed an event (AGG-POINT-09)
- `time_since` — ms since most recent matching event (AGG-POINT-10)
- `time_since_last_n` — ms since the n-th most recent matching event (AGG-POINT-11)
- `streak` — current count of consecutive matching events (AGG-RECENCY-01)
- `max_streak` — high-watermark of streak (AGG-RECENCY-02)
- `negative_streak` — current count of consecutive NON-matching events (AGG-RECENCY-03)
- `first_seen_in_window` — Bool: most-recent matching event within window
  (AGG-RECENCY-04)

## Tasks

### Task 1 — Recency markers state-struct unit tests (red → green)

Adds tests in `agg_state::tests` for `SeenState`, `TimeSinceLastNState`,
covering happy paths, where=false skip, query-time semantics, and
empty-state Null returns.

### Task 2 — Streak state-struct unit tests (red → green)

Adds tests for `StreakState`, `NegativeStreakState` covering current
count, max-seen tracking, reset on non-match.

### Task 3 — `first_seen_in_window` unit tests (red → green)

Tests `FirstSeenInWindowState::query` returns Bool(false) on empty,
Bool(true) for in-window event, Bool(false) for out-of-window event.

### Task 4 — JSON-wire compile coverage (red → green)

Adds tests in `agg_compile::tests` proving register accepts each new op
string and rejects:
- `first_seen_in_window` without window=
- recency op WITH window= (AGG-RECENCY ops are lifetime per D-02)
- `time_since_last_n` without n=
