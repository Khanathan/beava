<objective>
Fill every Rust-gap row from `COVERAGE-GAPS.md` with a direct unit test in `crates/beava-core/tests/`. Per D-03, MUST-FIX gaps that are op-internal correctness concerns go to Rust unit tests (cheaper than Python integration; no transport setup).

Per CLAUDE.md §Conventions TDD discipline, each task is RED-then-GREEN — but in coverage-fill the "RED" is "test asserts existing behavior, run BEFORE this commit lands". Specifically:

1. Write the test asserting expected behavior.
2. Run `cargo test -p beava-core --test <new_test_file>`.
3. **If the test PASSES against the current implementation:** the implementation satisfies the documented contract; commit as a single `test(13.7.5-10): cover <surface>` commit (no separate green commit needed because no production code change). This is the doc-only-plan-style exemption applied at task granularity for coverage fills.
4. **If the test FAILS:** stop. The implementation does NOT satisfy the documented contract → either the contract is wrong (Plan 13.7.6 / future bug-fix), or the test is wrong. Do NOT modify production code to make the test pass — coverage fill is not a license to fix bugs. Surface the failure as a discrepancy and SKIP that row (mark in commit message AND append a row to `.planning/phases/13.7.5-pre-oss-code-polish/SKIPPED-COVERAGE-ROWS.md`), routing to a follow-up bug-fix plan.

Output: One commit per surface_id (or per family if multiple ops fit one test file). Suggested commit message: `test(13.7.5-10): cover <op_or_surface> with direct unit test`. Any SKIP rows logged into `SKIPPED-COVERAGE-ROWS.md` (the same phase-local registry Plan 11 uses).
</objective>
