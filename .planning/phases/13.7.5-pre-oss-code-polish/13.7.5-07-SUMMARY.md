## What was deleted

- **Phase- / Plan-stamped opening docstrings** on every module ("Phase 13.5 Plan 04" / "Phase 13.5 Plan 11 contract" / "Phase 13.5.1 Plan 07b (Rule 3 blocking-issue auto-fix)" / etc.). Replaced with what-the-module-is descriptions.
- **Section markers** like `# ── Core (8) ──`, `# ── Sketch (5) ──`, `# === HTTP transport ===`, `# Arithmetic ─`, `# Comparisons ─`, `# Boolean ─`, `# ── 7 wire-mapped methods ──` — removed entirely.
- **Inline phase tags** in deviation-comments: `# Phase 13.5.1 Plan 05 (Rule 3 — blocking issue auto-fix)`, `# Phase 13.5.2 D-02 marker`, `# Phase 13.5.2 D-01 (USER-LOCKED)`, `# Phase 13.4 D-03`, `# Phase 13.5 Plan 02 D-05` — replaced with version-agnostic WHY comments where the rationale was load-bearing, deleted otherwise.
- **Restated docstrings** on private helpers (e.g. ``"""Q1 Path B: `field` arg must be a string column name, not an `_Expr`."""`` in `_enforce_field_str`) — deleted or rewritten without Q1/Path-B framing.
- **Restated signature docstrings** + paraphrase descriptions like `"""See docs/operators/buffer-geo/geo_velocity.md."""` (the cross-reference is delete-pattern when the public docstring already names the operator) — collapsed into one-liners.
- **DEPRECATED docstrings** rewritten from `"""DEPRECATED: use `bv.mean` per ADR-002."""` → `"""Deprecated alias for :func:`mean`."""` (docstring becomes Sphinx-rendered "Deprecated alias for X", DeprecationWarning text simplified).
- **Multi-paragraph paraphrase blocks** in `_app.py` (the ~14-line "Phase 13.5.1 Plan 05 Rule 3 blocking-issue auto-fix" comment block above `_FIXED_OP_OUTPUT_TYPE`) — collapsed into a 6-line WHY comment that names the architectural constraint without phase tags.

## What was kept (KEEP-pattern WHY-comments preserved)

- **Public-API Sphinx-rendered docstrings** on `bv.event` / `bv.table` / `bv.col` / `bv.lit` / `bv.App` and all 53 op helpers + 5 deprecation aliases. These render on `docs.beava.dev` for `pip install beava` users.
- **Strict-TypeError WHY-comment** in `_table.py::_resolve_upstream_proxies` (the Phase 13.5.1 D-01 USER-LOCKED design choice). Rephrased without the "Phase 13.5.1 D-01 (USER-LOCKED)" tag, but the rationale ("silent fallback used to surface as `AttributeError` deep inside user code … the sharp `TypeError` here points readers straight at the unannotated parameter") is preserved verbatim in spirit.
- **No-event-time-pivot rationale** in `_events.py::_validate_class_event` (the strict TypeError against `event_time` schema fields and `tolerate_delay` / `event_time_field` decorator kwargs). Tag-free rephrasing references the locked `project_redis_shaped_no_event_time_ever` commitment by name.
- **PEP 563 forward-ref handling contract** in `_table.py::_resolve_upstream_proxies` and `_events.py::_make_event_derivation` — the 3-source resolution priority (globals → closure cells → caller f_locals) and file-identity boundary detection are documented as "the contract" without 07e plan references.
- **Architectural-invariant ADR references** — ADR-001 (locked 2026-05-03 partial-overturn → @bv.table revived for aggregation-output only), ADR-002 (Polars-rename), ADR-003 (global-aggregation Decision B + Decision A explicit-literal) are kept as named architectural decisions — these are the 8th KEEP bullet of the rubric (cite source-of-truth).
- **Chain-flatten WHY** in `_app.py::_descriptor_to_node` (the rationale for why an `@bv.table` whose parent chain contains an `@bv.event def` output flattens upstream to the root EventSource — the server's apply-time routing index keys on direct upstream name). Critical reader-context that cannot be inferred from the code alone.
- **Boolean-operator overload WHY** in `_col.py::_Expr.__and__` / `__or__` — explains that Python forbids overloading `and`/`or`, so `&`/`|` are repurposed as boolean combinators with wire-form `and`/`or`.
- **Mock anti-pattern call-out** in `_app.py::register` — the rationale for rejecting raw `EventDerivation` instances at register-time (the marker handshake with `@bv.event def`).
- **Discovery-order security note** in `_embed.py::discover_binary` — explains why `BEAVA_BINARY` is fail-fast vs. silent-fallback (silent fallback would mask user misconfiguration).
- **Disk lifecycle WHY** in `_embed.py::spawn_embedded_server` — `atexit.register` cleanup contract for per-spawn tmpdirs, with OS tmpfs reaper as backstop.
- **CPython reference-cycle break** in `_table.py::_collect_caller_frame_locals` (`del frame` in the `finally` block).
