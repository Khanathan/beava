| End user `go get github.com/beava-dev/beava/sdk/go` → compiled binary | SDK runs in user's Go process at user privilege |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13.8-03-01 | T (Tampering) | git tag at `sdk/go/v0.0.0` | mitigate | Tag is signed with the GitHub-Actions-managed signing key (Phase 13.7.6 §G if covered, else default GitHub signature) ; `go.sum` checked in; Go proxy independently verifies the tag content matches the proxied module |
| T-13.8-03-02 | R (Repudiation) | unauthorized tag push | mitigate | Branch-protection on main + tag-protection rules require signed tags pushed only by the `beava-dev` org members (operator hardening — out of this plan's automated scope; user-level GitHub config) |
| T-13.8-03-03 | I (Information disclosure) | go.sum integrity | mitigate | `go mod tidy -check` in CI fails the workflow if go.sum drifts from the imports — Tampering surface narrowed |
| T-13.8-03-04 | S (Spoofing) | wrong-form tag (`v0.0.0` instead of `sdk/go/v0.0.0`) | accept | Plan 13.8-05/06 dual-tag push covers this; if user pushes only plain `v0.0.0`, `go get` returns `module not found at v0.0.0` — fail-loud, not spoof; risk = user confusion, not security |
