    ## 60-second quickstart (curl only)

    ```bash
    docker run -p <BEAVA_HTTP_PORT>:<BEAVA_HTTP_PORT> -p <BEAVA_TCP_PORT>:<BEAVA_TCP_PORT> beava/beava:v0.0.0
    ```

    Register a feature pipeline. Push events. Query the live aggregate.

    ```bash
    # 1) Register a "Click" event with a count-by-user feature
    curl -X POST http://localhost:<BEAVA_HTTP_PORT>/register \
      -d '{"events": [{"name": "Click", "schema": {"user": "str"}}],
           "features": [{"table": "Clicks", "key": "user", "agg": {"count": {}}}]}' \
      -H "Content-Type: application/json"

    # 2) Push some events
    curl -X POST http://localhost:<BEAVA_HTTP_PORT>/push/Click -d '{"user": "alice"}' -H "Content-Type: application/json"
    curl -X POST http://localhost:<BEAVA_HTTP_PORT>/push/Click -d '{"user": "alice"}' -H "Content-Type: application/json"
    curl -X POST http://localhost:<BEAVA_HTTP_PORT>/push/Click -d '{"user": "bob"}' -H "Content-Type: application/json"

    # 3) Query
    curl -X POST http://localhost:<BEAVA_HTTP_PORT>/get -d '{"table": "Clicks", "key": "alice"}' -H "Content-Type: application/json"
    # → {"count": 2}
    ```

    No broker, no ETL, no SDK required. Same flow works in Python, TypeScript, Go, or any language that speaks HTTP.

    ## Try it from Python

    ```python
    import beava as bv

    app = bv.App(test_mode=True)  # spawns embedded beava binary, no server setup needed

    @bv.event
    class Click:
        user: str

    @bv.event
    def Clicks(c: Click):
        return c.agg(key="user", op=bv.count())

    app.register([Click, Clicks])
    app.push("Click", {"user": "alice"})
    app.push("Click", {"user": "alice"})
    print(app.get("Clicks", "alice"))   # → {"count": 2}
    ```

    See [docs/quickstart.md](docs/quickstart.md) for TypeScript and Go equivalents.
