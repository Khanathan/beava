<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → Docker Hub | DOCKERHUB_TOKEN scope; if leaked, attacker can publish malicious beava/beava image |
| GitHub Actions runner → ghcr.io | uses GITHUB_TOKEN (auto-injected, scoped to packages:write); revoked when run completes |
| End user `docker pull beava/beava` → container | runs as user `beava` (UID 1001) inside container; container privilege depends on user's `docker run` flags |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13.8-04-01 | I (Information disclosure) | DOCKERHUB_TOKEN | mitigate | Token scoped to write+read+delete on `beava/*` namespace only (NOT `*` org-wide); 90-day rotation; lives only in GitHub repo secrets |
| T-13.8-04-02 | T (Tampering) | image content | mitigate | Multi-stage build: builder stage runs cargo with `Cargo.lock` checked in (`--locked`); runtime stage copies only the compiled binary, no build tools / source; OCI labels link to source repo for traceability |
| T-13.8-04-03 | E (Elevation) | container privilege | mitigate | `USER beava` non-root; `chmod +x` minimal; no SUID; user can `docker run --user 0` but that's their privilege, not ours |
| T-13.8-04-04 | T | base image | accept | `debian:bookworm-slim` is well-known + actively maintained; `rust:1.83-slim` similarly. Base image vulnerabilities tracked via dependabot at the org level (Phase 13.7.6 §C); not in-scope for v0 to switch to distroless / scratch — defer to v0.0.x if security review surfaces concerns |
| T-13.8-04-05 | I | image layer caching | accept | `cache-from: type=gha` / `cache-to: type=gha,mode=max` uses GitHub-hosted action cache; cache poisoning surface is GitHub-managed; no local secrets in layer cache |
