<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → npmjs.com | publish endpoint; if Trusted Publishers / NPM_TOKEN compromised, attacker can publish malicious @beava/sdk |
| End user `npm install @beava/sdk` → bundled JS | SDK runs in user's Node process at user privilege; no native binary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13.8-02-01 | I (Information disclosure) | NPM_TOKEN | mitigate | Prefer Trusted Publishers (GitHub OIDC; no long-lived secret); fallback NPM_TOKEN scoped to `@beava` org publish-only with 90-day expiry |
| T-13.8-02-02 | T (Tampering) | npm tarball | mitigate | `npm publish --provenance` produces SLSA Level 3 attestation linking the tarball to the GitHub Actions run; verifiable by `npm audit signatures` post-install |
| T-13.8-02-03 | R (Repudiation) | unauthorized publish | mitigate | Trusted Publishers tied to specific GitHub repo + workflow file; only `.github/workflows/npm.yml` on the `beava-dev/beava` repo can publish |
| T-13.8-02-04 | T | source of dist/ output | mitigate | `npm run build` runs `tsc` from a checked-in tsconfig.json against checked-in sources; reproducible build; SHA-pinned setup-node ensures consistent Node version |
</threat_model>
