<objective>
Wire the Plans 01 + 02 primitives into the live server: (a) recovery on startup, (b) periodic snapshot cadence, (c) registry WAL records, (d) readiness gate that reflects recovery completion.

Purpose: Satisfies SRV-REG-04 (registry durability), SRV-RECOV-02 (WAL replay on restart), SRV-RECOV-03 (WAL truncation after snapshot), SRV-RECOV-05 (schema evolution across restart).

Output:
- `beava-persistence::WalSink::append_record(record_type, payload) -> Lsn` — generalized append (the existing `append_event` keeps its Event specialization by calling through this).
- `beava-core::registry::Registry::install_from_descriptors(&RegistryDescriptorsOnly)` — rebuilds runtime caches; idempotent.
- `beava-server::recovery::{load_snapshot_if_any, replay_wal_from_lsn, RecoveryOutcome}`.
- `beava-server::snapshot_task::spawn_snapshot_task` — periodic task; holds a weak handle to AppState + WalSink + cfg.
- `Server::bind` runs recovery BEFORE flipping readiness (replaces the 100ms stub).
- `register_router` writes a RegistryBump WAL record before applying to the in-memory Registry.
- DurabilityConfig gains `snapshot_dir`, `snapshot_interval_ms`, `snapshot_retain_count`.
</objective>

     - `pub async fn load_snapshot_if_any(snapshot_dir: &Path, app_state: &AppState) -> Result<Lsn, PersistError>` — returns the snapshot_lsn restored from (0 if none / all corrupt fall through).

       - Skip any record with `lsn <= from_lsn_exclusive`.
       - Event record: decode payload (use same encoder used by push — inspect push.rs for the format) → extract (source, event_time_ms, row) → call `apply_event_to_aggregations` under the state_tables lock.
       - RegistryBump record: bincode-decode `RegistryBumpPayload` → call `app_state.dev_agg.registry.apply_registration` (compiling chains from the descriptors; reuse the register_handler's compile path factored into a helper).
       - Bump `next_event_id` and `max_event_time_ms` as records are replayed.
