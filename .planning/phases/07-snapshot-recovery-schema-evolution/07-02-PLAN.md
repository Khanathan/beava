<objective>
Make every live aggregation state type serde-serializable, then expose a `SnapshotBody` type in beava-core that bundles everything a Phase 7 snapshot needs: registry descriptors (already serde), per-aggregation per-entity AggOp state (new serde), and scalar counters.

Purpose: Satisfies SRV-RECOV-01 (serializes in-memory state) and groundwork for SRV-RECOV-05 (schema versions stored in registry â†’ already there; this plan makes sure state also carries the metadata needed to reconstruct). All code lives in beava-core because:
(a) AggOp is defined here and serde lives next to its definition,
(b) bincode is a pure compute path (no syscalls) â€” WASM-safe,
(c) beava-persistence stays transport-only (in Plan 03 the server wires beava-core::SnapshotBody through beava-persistence::SnapshotWriter).

Output:
- All AggOp variant state types and EntityKey: `#[derive(Serialize, Deserialize)]`
- `beava_core::snapshot_body::SnapshotBody` with typed fields + encode/decode
- `SnapshotBody::from_live` constructor takes `&RegistryInner` + `&BTreeMap<String, AggStateTable>` + `next_event_id` + `max_event_time_ms` â†’ SnapshotBody
- `SnapshotBody::into_parts` returns the tuple for the loader
- Plan 03 (next) wires this with `Registry::apply_registration` to rebuild compiled caches on reload
</objective>

```rust
// beava-core/src/snapshot_body.rs
use crate::agg_op::AggOp;
use crate::agg_state_table::{AggStateTable, EntityKey};
use crate::registry::{EventDescriptor, TableDescriptor, DerivationDescriptor};
use serde::{Deserialize, Serialize};
use std::collections::BTreeMap;

/// Serialized form of the in-memory registry inner â€” EXCLUDES runtime caches
/// (compiled_chains, compiled_aggregations, feature_index), which are rebuilt
/// via `Registry::apply_registration` on load.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RegistryDescriptorsOnly {
    pub version: u64,
    pub events: BTreeMap<String, EventDescriptor>,
    pub tables: BTreeMap<String, TableDescriptor>,
    pub derivations: BTreeMap<String, DerivationDescriptor>,
}

/// Full snapshot body â€” serialized with bincode by plan 03, which then hands
/// the bytes to beava-persistence::SnapshotWriter.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SnapshotBody {
    /// Monotonic format version for evolution detection; v0 = 1.
    pub body_format_version: u16,
    pub registry: RegistryDescriptorsOnly,
    pub state_tables: BTreeMap<String, Vec<(EntityKey, Vec<AggOp>)>>,
    pub next_event_id: u64,
    pub max_event_time_ms: i64,
}

#[derive(Debug, thiserror::Error)]
pub enum SnapshotBodyError {
    #[error("bincode: {0}")] Bincode(#[from] Box<bincode::ErrorKind>),
    #[error("unsupported snapshot body_format_version {0}")] UnsupportedVersion(u16),
}

pub const SNAPSHOT_BODY_FORMAT_VERSION: u16 = 1;

impl SnapshotBody {
    /// Build a SnapshotBody from live pointers. No I/O, no locking beyond
    /// the caller's; caller is expected to hold the state_tables mutex.
    pub fn from_live(
        registry: &crate::registry::RegistryInner,
        state_tables: &BTreeMap<String, AggStateTable>,
        next_event_id: u64,
        max_event_time_ms: i64,
    ) -> Self;

    /// Encode to bincode (size-prefixed default config).
    pub fn encode(&self) -> Result<Vec<u8>, SnapshotBodyError>;

    /// Decode from bincode. Rejects unknown body_format_version.
    pub fn decode(bytes: &[u8]) -> Result<Self, SnapshotBodyError>;
}
```

Serde derives required (this is the bulk of the plan):
- `EntityKey(pub Vec<(String, String)>)` â€” derive Serialize + Deserialize.
- `AggOp` enum with internally tagged variant (serde default: externally tagged is fine for bincode).
  - `AggOp::Windowed(Box<WindowedOp>)` â€” WindowedOp needs serde.
- Per-state structs (in `agg_op.rs` â€” find each `#[derive(Debug, Clone)] struct XState`):
  - `CountState` â†’ add `Serialize, Deserialize`.
  - `SumState` â†’ add. (f64 field; serde handles f64 as-is.)
  - `AvgState` â†’ add.
  - `MinState` â†’ add. (Stores Value â€” Value in `row.rs` must also derive serde. Check first!)
  - `MaxState` â†’ add.
  - `VarianceState` â†’ add.
  - `RatioState` â†’ add.
- `WindowedOp` in `agg_windowed.rs` â€” derive serde. Includes `inner_kind: AggKind`, `window_ms: u64`, `buckets: Vec<Option<Bucket>>`, base_bucket_idx, etc. Bucket also needs serde.
- `AggKind` enum â€” derive serde (trivial; u8-like enum).
- `Value` in `row.rs` â€” CHECK if already serde. If not, add derives. `Value::Bytes(Vec<u8>)` serde: serde serializes Vec<u8> as byte sequence; keep default.

`AggOpDescriptor` (in agg_op.rs) contains `Option<Arc<Expr>>` â€” Expr already serde-derived. Arc<Expr> is serde-compatible via the default blanket impl for Arc when T: Deserialize + Clone. Confirm with a test.
