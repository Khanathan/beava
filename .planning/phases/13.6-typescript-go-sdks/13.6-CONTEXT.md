# Phase 13.6: TypeScript + Go SDKs — Context

**Gathered:** 2026-05-03 (--auto mode)

<domain>
## Phase Boundary

Build two SDKs from scratch implementing the same wire contract as Python (Phase 13.5). Each ~1800 LOC. Two parallel sub-tracks within the phase. Per ROADMAP §13.6 + ADR-003 (`bv.lit` + global-agg surface).

**Repos / packages:**
- TypeScript: `@beava/sdk` on npm
- Go: `github.com/beava-io/beava-go`

**Both SDKs share:**
- Wire-level identical to Python (verified by `examples/wire/_validate_examples.py` running cross-SDK)
- Same 53 operator catalog
- Same chained DSL pattern (filter → group_by → agg)

</domain>

<decisions>
## Implementation Decisions — TypeScript SDK

### TS-D-01 — Stage-3 decorators (TS 5.0+) for `@bv.event` / `@bv.table`
- Use stage-3 decorators (not legacy / experimental) — modern TS 5.0+ baseline
- Class form: `@bv.event class Click { @field user_id: string; @field page: string; }`
- Function form: `@bv.table({key: "user_id"}) function UserClicks(clicks): bv.Table { return clicks.groupBy("user_id").agg({total: bv.count({window: "1h"})}); }`
- TS 4.x users get a builder-pattern fallback (documented in sdk-api/typescript.md)

### TS-D-02 — Naming: camelCase per TS idiom
- `app.batchGet(...)` (NOT `batch_get`); `app.appendDescriptor(...)` (NOT `append_descriptor`)
- Op names stay snake_case to match wire (`bv.n_unique`, `bv.first_n`) — wire is the contract
- Field names stay snake_case in user data (matches event schema)

### TS-D-03 — Async-first
- All wire methods return `Promise<T>` (no separate sync API)
- `await app.push("Click", {...})`
- Reason: TS ecosystem is async-native; sync would feel wrong

### TS-D-04 — `bv.lit(value)` factory + global aggregation (ADR-003)
- `bv.lit(value: number | string | boolean | null): Expr`
- `events.groupBy()` empty allowed; `events.agg({...})` direct
- `@bv.table` without `key=` opt → global table
- `app.get(tableName)` overloaded TypeScript signature:
  ```typescript
  get<T extends GlobalTable<infer R>>(table: T): Promise<R>;
  get<T extends EntityTable<infer R>>(table: T, entityId: string): Promise<R>;
  ```
- Total ~75 LOC

### TS-D-05 — Build + publish
- `tsc --strict` clean (Phase 13.6 success criterion)
- Output: `dist/` with both `.js` (CJS + ESM) and `.d.ts`
- Bundler: tsc only (no webpack / rollup — keep it simple); skip CommonJS if ESM-only is viable
- Publish target: `@beava/sdk` on npm; v0.0.0 cut in Phase 13.8

### TS-D-06 — Tests: jest fixtures matching Python pattern
- `tests/v0/test_<family>.test.ts` mirrors `python/tests/v0/test_<family>.py`
- Use `jest.beforeAll` / `jest.afterAll` for App lifecycle (matches Python `app` fixture)
- Same high-volume test pattern (≥500 events per test, ≥3 entities, cold-start assertion)
- Skipif until engine + SDK ready (matches Python pattern)

## Implementation Decisions — Go SDK

### GO-D-01 — Idiomatic context-aware methods
- All wire methods take `ctx context.Context` first param: `app.Push(ctx, "Click", map[string]any{...})`
- Returns explicit `error` last value (no panics in user-facing API)
- Cancellation respected (long pushes / connections check ctx.Done())

### GO-D-02 — Struct tags + builder pattern (Go has no decorators)
- Class-form events: struct with `beava:""` tags
  ```go
  type Click struct {
      UserID string `beava:"user_id"`
      Page   string `beava:"page"`
  }
  ```
- Function-form derivations: builder pattern
  ```go
  table := beava.NewTable("UserClicks").
      Key("user_id").
      Source[Click]().
      Agg(map[string]beava.AggOp{
          "total": beava.Count(beava.Window("1h")),
      })
  ```

### GO-D-03 — Naming: PascalCase exports per Go idiom
- `app.Register(...)`, `app.Push(...)`, `app.Get(...)`, `app.BatchGet(...)`, `app.Reset(...)`, `app.Ping(...)`
- Op constructor functions: `beava.Count(...)`, `beava.Sum(...)`, `beava.NUnique(...)`, `beava.Quantile(...)` (PascalCase even on snake_case underlying op-name)

### GO-D-04 — `beava.Lit(value any)` + separate `app.GetGlobal(ctx, table)` method (ADR-003)
- `beava.Lit(value any) beava.Expr`
- `events.GroupBy()` empty allowed; `events.Agg(...)` direct
- `beava.NewTable(...).NoKey().Source[T]()...` for global tables (no `Key()` call)
- **Separate method** `app.GetGlobal(ctx, tableName)` instead of arity overloading (Go's typing convention; Python's `App.get` overload doesn't translate)
- Total ~75 LOC

### GO-D-05 — Module + tests
- Module: `github.com/beava-io/beava-go` (v0.0.0 tag in Phase 13.8)
- Tests: standard `testing` + `testify/require`
- `go vet ./... && go test ./...` clean (Phase 13.6 success criterion)
- Mirror Python test files: `tests/v0/<family>_test.go`

### Claude's Discretion
- Whether to ship CommonJS for TS or ESM-only (default ESM-only; revisit if Node 16 LTS support required)
- Plan/wave shape (likely ~10 plans across 3 waves; could split TS / Go into parallel sub-streams)
- TS decorators vs builder pattern landing first (default: decorators primary, builder as fallback)
- Go generics pattern for type-safe events (Go 1.21+ baseline)
- TDD red→green per task

</decisions>
