  <name>Task 3: update docs/sdk-api/shared.md authoring=Python-only clarification</name>
  <action>
Edit `docs/sdk-api/shared.md`:

- Add a new section "**Authoring vs communicate**" near the top (after "Overview"):

> **Authoring** (decorators, expression DSL, op helpers) is **Python-only in v0**.
> The Python SDK is the canonical authoring UX — see [python.md](python.md).
>
> **Communicate** (push events, register pre-compiled JSON, read features) is **universal**.
> All three SDKs (Python, TypeScript, Go) implement the wire surface in this doc.
>
> Cross-language workflow: Python authors descriptors via the DSL → compiles to JSON →
> TypeScript / Go apps push events + read features against the same registered pipeline.
> No DSL parity is required because there is no DSL outside Python.

- Update the "Cross-language API surface map" table to drop DSL methods; keep wire methods.

- Update "ValidationError envelope" — note that TS/Go users won't see authoring-time validation errors (those happen in Python at compile time); they'll see register-time errors when pushing pre-compiled JSON to a server that rejects it.

- Update "Public expression literal (`bv.lit`)" to clarify it's Python-only in v0 (TS/Go don't have an expression layer).

- Update "Global-table sentinel (`key = ""`)" — the Python `app.get(table)` 1-arg + TS `app.get(table)` overload + Go `app.GetGlobal(ctx, table)` patterns all SURVIVE because they're wire-level (pure routing on the `key` value).
