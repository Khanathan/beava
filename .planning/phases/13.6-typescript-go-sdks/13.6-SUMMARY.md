# Phase 13.6 Summary — TypeScript + Go SDKs (communicate-only)

**Status:** CLOSED
**Closed:** 2026-05-03
**Plans landed:** 8 (all atomically committed)

## Overview

Phase 13.6 ships **communicate-only** TypeScript and Go SDKs for Beava v0. Both
clients implement the cross-language wire surface from `docs/sdk-api/shared.md`
without any pipeline DSL — pipeline authoring (decorators, expressions, op
helpers) lives in the Python SDK only per the 2026-05-03 scope amendment.

## Auto-defaults locked during planning (per CONTEXT.md "Claude's Discretion")

| Question | Resolution | Rationale |

| TS test runner? | **vitest.** | ESM-native; less config friction with `"type": "module"` D-01. |
| `pushSync` v0 behavior? | **DELEGATE to push.** | OP_PUSH_SYNC reserved per `docs/wire-spec.md`; v0 keeps API parity with Python without wiring an opcode the server doesn't yet implement. |
| TS adapter `@beava/sdk` resolution in conformance? | **In-tree dist/ import.** | Avoids npm-link state pollution; uses Node 22+ `--experimental-strip-types` to run `run_ts.ts` directly without a `tsx` runtime dependency. |

## Cross-phase gaps (deliberate handoff)

The cross-SDK conformance harness (`python/tests/conformance/test_cross_sdk.py`)
SKIPS cleanly with a clear diagnostic when the engine binary's accepted wire
shape lags `docs/wire-spec.md`. Specifically the engine still rejects
`kind: "table"` per the Phase 12.7 invariant (per `project_v0_events_only_scope`),
even though ADR-001 was partially overturned 2026-05-03 to revive `@bv.table`
for aggregation-output. Phase 13.4 owns the engine-side revival; the harness
flips GREEN automatically when that lands. See
`.planning/phases/13.6-typescript-go-sdks/13.6-07-SUMMARY.md` for the full
matrix of cross-phase gaps.

This is a deliberate parallel-phase staging — Phase 13.6 ships the SDKs at the
documented wire spec; Phase 13.4 catches the engine up; the conformance harness
asserts agreement when both are caught up.
