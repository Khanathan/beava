## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-01-01 | Denial of Service | ReadEntityBatch with unbounded keys Vec (memory DoS) | mitigate | Cap `keys.len()` in the ShardOp constructor; reject with `ShardDispatchError::ProcessingError("batch too large — max 4096 keys")` when exceeded. | T-56-01-02 | Tampering | SsjInsert poisoning the join buffer via malformed `event` map | mitigate | `apply_ssj_insert` validates `event` is a JSON object; if not, returns an empty matches Vec without inserting (already the behaviour in operators.rs for StreamStreamJoin). |
| T-56-01-03 | Denial of Service | Oneshot reply poisoning via dropped Sender (deadlock) | mitigate | All new dispatch arms must `let _ = tx.send(...)` (ignore closed-channel errors); caller's `rx.recv()` has a bounded timeout (500 ms) falling back to `BeavaError::ShardOverload`. | T-56-01-04 | Repudiation | ShardOp variant added without metric increment (silent drop) | mitigate | Dispatch arms increment `beava_enrich_cross_shard_total` / `beava_ssj_cross_shard_total` before reply; missing-row path increments `beava_enrich_missing_total`. Grep-gate in acceptance criteria. |
| T-56-01-05 | Elevation of Privilege | — | accept | No auth boundary involved; ShardOp is an in-process SPSC only. |

        impl PipelineEngine {
            /// Phase 56 D-A1 + D-A3: EnrichFromTable cross-shard read with
            /// same-shard fast path. Returns Option<EntityState>.
            ///
            /// Deadlock analysis: this helper is called from
            /// push_with_cascade_on_shard on the source shard's thread; it
            /// may try_send to the target shard's inbox and block on
            /// oneshot rx. The target shard runs on its own pinned OS
            /// thread + runtime; it reads from its own inbox, not from the
            /// source's. try_send is non-blocking on the send side. If the
            /// target's inbox is Full: return BeavaError::ShardOverload
            /// immediately (propagates to HTTP 503 / TCP SHARD_OVERLOAD).
            /// No blocking send → no wait-chain edge → no cycle possible.
            pub fn read_entity_at_shard(
                &self,
                shard_partitions: &[ShardHandle],
                target_shard_idx: usize,
                current_shard: &Shard,
                table_name: &str,
                key: &str,
            ) -> Result<Option<EntityState>, BeavaError> {
                if target_shard_idx == current_shard.shard_id() {
                    metrics::counter!(ENRICH_INTRA_SHARD_TOTAL, "table" => table_name.to_string()).increment(1);
                    if current_shard.read_entity_at(table_name, key).is_none() {
                        metrics::counter!(ENRICH_MISSING_TOTAL, "table" => table_name.to_string()).increment(1);
                    }
                    return Ok(current_shard.read_entity_at(table_name, key));
                }
                // Cross-shard: try_send + blocking recv + ShardOverload-on-Full.
                // Metric increment lives on the target's dispatch arm to avoid double-counting.
                let (tx, rx) = oneshot::channel::<ShardResult>();
                let evt = ShardEvent { op: ShardOp::ReadEntityAt { table_name: table_name.into(), key: key.into() }, response_tx: Some(tx), ..Default::default() };
                match shard_partitions[target_shard_idx].inbox_tx.try_send(evt) {
                    Ok(()) => {}
                    Err(TrySendError::Full(_)) => {
                        metrics::counter!("beava_shard_inbox_full_total", "shard" => target_shard_idx.to_string()).increment(1);
                        return Err(BeavaError::Protocol("shard inbox full — enrich cross-shard read backpressure".into()));
                    }
                    Err(TrySendError::Disconnected(_)) => {
                        return Err(BeavaError::Protocol("target shard inbox disconnected".into()));
                    }
                }
                match rx.recv_timeout(Duration::from_millis(500)) {
                    Ok(ShardResult::ReadEntityOk(v)) => Ok(v),
                    Ok(ShardResult::Err(e)) => Err(BeavaError::Protocol(format!("enrich cross-shard: {:?}", e))),
                    Ok(_) => Err(BeavaError::Protocol("enrich cross-shard: unexpected reply variant".into())),
                    Err(_) => Err(BeavaError::Protocol("enrich cross-shard: oneshot timeout 500ms".into())),
                }
            }
