This is the "no fast paths deferred" clause from ROADMAP.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source shard → target shard | SPSC dispatch carrying right_keys from source to target; validated at source (encode_group_by rejects malformed values). |
| target shard → source shard | Oneshot reply carrying Option<EntityState>; in-process only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-02-01 | Denial of Service | Coalesce BTreeMap unbounded growth for pathological batches | mitigate | Coalesce buffer is dropped at end of batch (no persistent state); per-target `keys.len()` capped at MAX_ENRICH_BATCH_KEYS=4096 (Wave-1 DoS guard on the dispatch arm). If a single batch exceeds the cap for one (target, table), split the flush into multiple ShardOps. |
| T-56-02-02 | Tampering | Malformed right_key from encode_group_by | accept | Existing encode_group_by validates + returns BeavaError::Protocol on missing fields; unchanged path. |

| T-56-02-04 | Denial of Service | Deadlock on parallel oneshots if join_all blocks the runtime | mitigate | Use `std::thread::scope` + crossbeam rx per-target OR `tokio::join_all` inside `Handle::current().block_on`. Wave-1 comment pre-approved the latter; pick consistently. |
