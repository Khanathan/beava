```rust
/// Evict individual stream entries from entities based on per-stream entity_ttl.
/// Two-phase process:
/// 1. For each entity, iterate streams and remove those whose last_event_at
///    exceeds their stream's entity_ttl.
/// 2. Remove entities that have zero remaining streams AND zero static_features.
///
/// Streams with entity_ttl=None fall back to the global TTL behavior
/// (ttl_multiplier * max_window). Streams with last_event_at=None are not evicted.
///
/// Returns the number of stream entries evicted.
pub fn evict_expired_stream_entries(
    store: &mut StateStore,
    engine: &PipelineEngine,
    now: SystemTime,
    ttl_multiplier: u32,
) -> usize
```

The implementation must:
- For each entity in the store, iterate its `streams` entries
- For each stream entry, look up the StreamDefinition via `engine.get_stream(stream_name)`
- If the stream has `entity_ttl = Some(ttl)`, use that TTL
- If the stream has `entity_ttl = None`, fall back to `engine.max_window_duration() * ttl_multiplier`
- If `max_window_duration()` is zero and entity_ttl is None, skip (don't evict)
- Check `stream_state.last_event_at` against the TTL
- Collect stream names to remove, then remove them from the entity
- After processing all entities, call `store.remove_empty_entities()`
- Return total count of removed stream entries

3. Add MGET parsing to `parse_command` after the OP_REGISTER arm:
```rust
OP_MGET => {
    if buf.len() < 4 {
        return Err(TallyError::Protocol("MGET payload too short: need 4 bytes for count".into()));
    }
    let count = u32::from_be_bytes([buf[0], buf[1], buf[2], buf[3]]) as usize;
    buf = &buf[4..];
    let mut keys = Vec::with_capacity(count);
    for _ in 0..count {
        keys.push(read_string(&mut buf)?);
    }
    Ok(Command::Mget { keys })
}
```

In src/server/tcp.rs:

5. Add MGET to the sync command dispatch in `handle_sync_command`. MGET is synchronous (not chunked like MSET) because reads are fast and non-destructive:
```rust
Command::Mget { keys } => {
    let mut app = state.lock().unwrap_or_else(|e| e.into_inner());
    let AppState {
        ref engine,
        ref mut store,
        ..
    } = *app;
    let mut result = serde_json::Map::new();
    for key in &keys {
        let features = engine.get_features(key, store, now);
        let feature_json = feature_map_to_json(&features);
        // Parse the JSON bytes back to a Value for nesting
        let value: serde_json::Value = serde_json::from_slice(&feature_json)
            .unwrap_or(serde_json::Value::Object(serde_json::Map::new()));
        result.insert(key.clone(), value);
    }
    Ok(serde_json::to_vec(&serde_json::Value::Object(result)).unwrap())
}
```

Note: Use `engine.get_features` (not `store.get_all_features`) for consistency with the GET handler -- this evaluates view derives and lookups. The get_features method may add qualified names like "Stream.feature" which are internal; strip any feature names containing "." from the result before serialization to avoid leaking internal names to clients.

In the match on `cmd`, ensure `Command::Mget { .. }` goes through `handle_sync_command` like GET does.
