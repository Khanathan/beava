Wire the new `Command::PushAsync` and `Command::Flush` variants (defined by Plan 01) into the server's connection dispatch. The async path shares all side effects with sync PUSH (engine call, event log, cascade/fan-out, dirty marking, throughput, latency histogram) but skips the response write + flush on success to enable `BufWriter`-level TCP pipelining.

Purpose: PERF-01 — deliver the fire-and-forget ingest path on the server. Preserve the sync PUSH latency budget while unlocking ~9us round-trip savings per push on the async path.
Output: `src/server/tcp.rs` with an extracted `handle_push_async` and new match arms in `handle_connection`. Zero regression on the sync PUSH path.
</objective>
