<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TCP client → tally server | Untrusted bytes enter via the TCP listener and reach `parse_command` → `decode_event_binary`. This plan introduces a new parser on that boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering / DoS | `decode_event_binary` buffer over-read | mitigate | Every length read (`u16 field_count`, `u16 key_len`, 1-byte tag, 1-byte bool, 8-byte i64/f64, string length) is bounds-checked against `buf.len()` before consuming bytes. Test coverage includes truncation at each field. |
| T-11-02 | Tampering | Unknown/malicious type tag | mitigate | `match tag { ... _ => Err(Protocol(...)) }` explicitly rejects unknown tags. Unit test `test_decode_event_binary_unknown_tag` proves it. |
| T-11-03 | DoS | Oversized `field_count` allocating huge `Map` | mitigate | `field_count` is `u16` — max 65535, bounded. `serde_json::Map::with_capacity(65535)` is safe (under 1 MB). Upstream `MAX_FRAME_SIZE = 64 MB` framing in tcp.rs further bounds the frame. |
| T-11-04 | Tampering | UTF-8 smuggling in key/string-value | mitigate | Reuses existing `read_string` which calls `str::from_utf8` and rejects invalid sequences. No new UTF-8 surface. |
| T-11-05 | Tampering | NaN/Infinity floats smuggled through f64 | mitigate | `decode_event_binary` explicitly rejects non-finite f64 via `n.is_nan() || n.is_infinite()`. Tests `test_decode_event_binary_f64_nan` and `test_decode_event_binary_f64_inf` prove it. Matches v1.1 JSON strictness. |
| T-11-06 | Information disclosure | Error-frame string leaking internals | accept | Error messages mirror the existing `read_json_payload` style — "expected N bytes, got M", "unknown type tag 0xFF". Same information shape as v1.1 JSON errors. No new leak. |
</threat_model>
