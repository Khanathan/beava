## Risks & Mitigations

- **Replay outpaces server decode → TCP backpressure → skewed eps.** → Use sync-batch `push_many` (awaits ACK), not fire-and-forget. Per RESEARCH Pitfall 2, async-batch plateaus at 178k/client; sync-batch holds 480k/client, so 8 workers × 480k = 3.8M theoretical ceiling, comfortably above the 1.1M baseline.
- **Non-determinism from `now_ms`.** → Generator pins `now_ms` as an explicit parameter defaulting to `time.time_ns() // 1_000_000` only in `__main__`; tests pass a fixed value.
- **CI flakiness from real Tally subprocess.** → Fallback path: the integration test can skip (with a clear reason) if the binary is not built, so CI passes on pure-Python runners.
- **Generator drift from `bench.py`.** → Copy constants with a source-link comment; add a grep-based pre-commit check as follow-up if drift bites.
