### `apply_shard.rs::WireRequest::HttpGet` body parser (D-01 — verb-style single-row GET)

**Analog 1:** `crates/beava-server/src/apply_shard.rs:468-504` (`WireRequest::TcpGet` block — already does codec-dispatch parse-and-call)

**Codec-dispatch parse pattern** (lines 468-504):

```rust
WireRequest::TcpGet { body, body_format } => {
    use beava_core::wire::{CT_JSON, CT_MSGPACK};
    #[derive(serde::Deserialize)]
    struct TcpGetReq {
        feature: String,
        key: String,
    }
    let req: TcpGetReq = match body_format {
        CT_JSON => match serde_json::from_slice(&body) {
            Ok(r) => r,
            Err(e) => {
                return GlueResponse::InternalError {
                    reason: e.to_string(),
                };
            }
        },
        CT_MSGPACK => match rmp_serde::from_slice(&body) {
            Ok(r) => r,
            Err(e) => {
                return GlueResponse::InternalError {
                    reason: e.to_string(),
                };
            }
        },
        other => {
            return GlueResponse::InternalError {
                reason: format!("unsupported content_type: {other:#04x}"),
            };
        }
    };
    crate::runtime_core_glue::dispatch_get_single_sync(
        &self.state,
        &req.feature,
        &req.key,
        body_format,
    )
}
```

**Migration target shape (D-01):** Replace `TcpGetReq { feature, key }` with `TcpGetReq { table, key, features: Option<Vec<String>> }` and route to `dispatch_get_single_sync` with the new signature (table + key + features). HTTP variant (`WireRequest::HttpGet { body }` at line 453) currently routes to `dispatch_get_batch_sync`; per D-01 it must route to a single-row handler since the verb-style `POST /get` body is single-row.

**Current legacy shape to replace** (`apply_shard.rs:453-457`):

```rust
WireRequest::HttpGet { body } => crate::runtime_core_glue::dispatch_get_batch_sync(
    &self.state,
    &body,
    beava_core::wire::CT_JSON,
),
```

    if reason.starts_with("unsupported content_type") {
        GlueResponse::TcpError {
            code: "unsupported_content_type",
            message: reason,
            extras: serde_json::json!({}),
        }

            code: "missing_event_name_in_body",
            registry_version: self.state.dev_agg.registry.version() as u32,
        }
    } else if reason == "invalid_json_body" {
        GlueResponse::PushError {
            code: "invalid_json_body",
            registry_version: self.state.dev_agg.registry.version() as u32,

**HTTP encoder mapping** (`crates/beava-server/src/server.rs:2441-2445`):

```rust
GlueResponse::PushError { code, .. } => {
    let body = serde_json::json!({"error": {"code": code}});
    let status = if *code == "event_not_found" { 404 } else { 400 };
    (status, serde_json::to_vec(&body).unwrap_or_default())
}
```

**Migration target (D-05):** Two options the planner should weigh:

(a) **Add a new `GlueResponse` variant** `UnsupportedRequestShape { route: &'static str, hint: String }` that the HTTP encoder maps to 400 with the locked body shape, mirroring `PushError`'s flavor.

(b) **Reuse `GlueResponse::PushError`** with code `"unsupported_request_shape"` — but the encoder currently emits `{"error":{"code":code}}` only (no `message` field) so the locked body shape (which carries a `message` with a doc hint) needs the encoder to also carry a message. Stretching `PushError` blurs its semantics.

**Recommended:** option (a) — add `GlueResponse::UnsupportedRequestShape` variant. Keeps the legacy-shape detection localized to the new dispatch arms; doesn't pollute `PushError`. Body shape locked to:

```json
{
  "error": {
    "code": "unsupported_request_shape",
    "message": "POST /get expects {table, key, features?}; received legacy {keys, features} shape — see docs/http-api.md#post-get"
  }
}
```

**Detection strategy:** First try strict-deserialize the new shape; on parse failure, attempt to deserialize as `serde_json::Value` and check for keys `keys` (legacy 2D-cell) or `feature` (legacy single-feature). On match → return `UnsupportedRequestShape` with the route-specific hint. On no-match → fall through to the existing `InternalError { reason: <serde error> }` (so genuinely malformed JSON still surfaces clearly).

### `runtime_core_glue.rs::dispatch_get_single` (D-01 + D-06 — single-row narrow-by-features)

**Analog:** `crates/beava-server/src/runtime_core_glue.rs:150-279` — current `feature/key`-based single-row dispatch with two cases (feature-name lookup vs aggregation-node lookup):

```rust
fn dispatch_get_single(
    app: &Arc<AppState>,
    feature: &str,
    key: &str,
    body_format: u8,
) -> GlueResponse {
    use beava_core::wire::{CT_JSON, CT_MSGPACK};
    // ... codec validation ...
    let registry = &app.dev_agg.registry;
    let query_time_ms = { /* atomic load max-event-time */ };

    // Case 2: `feature` is an aggregation node name.
    // Return all features for the entity as `{feat_name: value, ...}`.
    if let Some(descriptor) = registry.compiled_aggregation(feature) {
        let entity_key = match parse_entity_key(key, &descriptor.group_keys) {
            Some(k) => k,
            None => return GlueResponse::QueryNotFound { code: "key_parse_failure" },
        };
        let tables = app.dev_agg.state_tables.lock();
        let table = match tables.get(descriptor.agg_id as usize) {
            Some(t) => t,
            None => return GlueResponse::QueryNotFound { code: "key_not_found" },
        };
        let mut result = serde_json::Map::new();
        for (idx, named_op) in descriptor.features.iter().enumerate() {
            if let Some(v) = table.query_feature(&entity_key, idx, query_time_ms) {
                result.insert(named_op.feature_name.clone(), value_to_json(v));
            }
        }
        if result.is_empty() {
            return GlueResponse::QueryNotFound { code: "key_not_found" };
        }
        // ...serialize and return QueryResult...
    }
}
```

**Migration target (D-01):** Change signature from `(feature, key, body_format)` to `(table, key, features: Option<&[String]>, body_format)`. The verb-style spec is `table` (always an aggregation-node-name lookup, not feature-name lookup), so Case 1 of `dispatch_get_single` (feature-name resolution) is no longer reachable from the verb-style entry point — the legacy `WireRequest::HttpGetSingle` path that used Case 1 stays for back-compat with `GET /get/:feature/:key`. The new verb-style dispatch always hits Case 2 with `table` as the input.

**D-06 features-filter narrowing pass:** Apply at the `result.insert` site — same filter shape as the batch path. Cold-start with explicit features filter still returns `{}` (NOT QueryNotFound), per the locked spec wire-fixtures `examples/wire/get-found.response.json` and `batch_get-heterogeneous.response.json` line 5 (`{}`).

### `runtime_core_glue.rs::dispatch_get_batch` (DEPRECATE or REWIRE)

**Status:** This function (lines 306-511) currently parses the legacy 2D-cell `{keys, features}` shape and is the destination of the old `WireRequest::HttpGet` arm. After Phase 13.4.1, the verb-style `POST /get` is single-row; the verb-style `POST /batch_get` already goes to `dispatch_batch_get_sync` (in `apply_shard.rs`).

**Two viable plans the planner should weigh:**

(a) **Delete `dispatch_get_batch` entirely** — the only callers are the legacy `HttpGet` route and `TcpGetMulti`/`TcpMGet`. `HttpGet` will be redirected to single-row dispatch per D-01. `TcpGetMulti` (`OP_GET_MULTI 0x0022`) and `TcpMGet` (`OP_MGET 0x0021`) are pre-13.0 opcodes; check `docs/wire-spec.md` to confirm whether they survive into v0 or are deprecated. If deprecated, delete; if surviving, leave untouched (they aren't in the Phase 13.4.1 scope).

(b) **Keep but mark legacy** — preserve for `OP_MGET` / `OP_GET_MULTI` continuity; just not the hot path anymore.

**Recommended:** option (a) only if `OP_MGET` / `OP_GET_MULTI` are confirmed dead (planner reads `docs/wire-spec.md`); otherwise (b). Either way, the verb-style dispatch sites must NOT route through `dispatch_get_batch`.

## Cross-cutting Patterns

### 1. CT_JSON / CT_MSGPACK codec dispatch (Plan 12-09 D-A/D-B)

**Source:** `apply_shard.rs:468-497` (TcpGet) and `apply_shard.rs:1193-1215` (dispatch_batch_get_sync). Every body-parsing site uses the identical match shape:

```rust
let req: ReqType = match body_format {
    CT_JSON => match serde_json::from_slice(&body) {
        Ok(r) => r,
        Err(e) => return GlueResponse::InternalError { reason: e.to_string() },
    },
    CT_MSGPACK => match rmp_serde::from_slice(&body) {
        Ok(r) => r,
        Err(e) => return GlueResponse::InternalError { reason: e.to_string() },
    },
    other => return GlueResponse::InternalError {
        reason: format!("unsupported content_type: {other:#04x}"),
    },
};
```

**Apply to:** the new D-01 `WireRequest::HttpGet` parser (HTTP path always passes `CT_JSON` per `http_listener.rs:225`), the new D-01 `WireRequest::TcpGet` parser (forward `body_format` from frame), and `dispatch_batch_get_sync` (already correct; keep).

**Important:** D-05 structured-400 detection must run BEFORE the `Err(e) => InternalError` fallback. Try strict-deserialize first; on Err, attempt legacy-shape detection; on legacy-shape match return `UnsupportedRequestShape`; on truly-malformed JSON return `InternalError`. Three-step ladder.

### 2. Registry feature-resolve upfront for `feature_not_found`

**Source:** `runtime_core_glue.rs:382-396` — resolve all requested features against the registry BEFORE entering the per-row hot loop:

```rust
let registry = &app.dev_agg.registry;
let mut missing_features: Vec<String> = Vec::new();
let mut feature_resolutions: Vec<(String, usize)> = Vec::new();
for feat in &req.features {
    match registry.resolve_feature(feat) {
        Some(resolution) => feature_resolutions.push(resolution),
        None => missing_features.push(feat.clone()),
    }
}
if !missing_features.is_empty() {
    return GlueResponse::InternalError {
        reason: format!("feature_not_found: missing={:?}", missing_features),
    };
}
```

**Apply to:** the new D-01 `dispatch_get_single` (validate `entry.features` against the descriptor's feature names — feature-not-in-descriptor errors upfront with `feature_not_found`); the new D-02 per-entry filter in `dispatch_batch_get_sync` (validate per-entry, but per D-06 distinguish "feature in registry but absent on this entity" — that case omits silently, NOT errors).

**D-06 distinction (from CONTEXT.md lines 109-114):**

- **Feature not in registry / not in descriptor** (typo, `"foo"` doesn't exist) → reject with `feature_not_found`
- **Feature exists in descriptor but not present on this entity** (entity not yet touched by that derivation) → omit silently from the row's response

**Apply to:** D-04 alias-detection log. CONTEXT.md D-04 acknowledges that exact source-field tracking via custom deserialize is fiddly and offers a pragmatic alternative ("emit the WARN unconditionally on every batch_get request that lands during v0"). Planner should weigh:

- **Strict (custom deserialize):** track which field name was actually present in the request; emit WARN only when `entity_id` was used. More plumbing but accurate.

- **Middle (sample-on-first-occurrence per process):** `static SEEN: AtomicBool` guards a single WARN per process lifetime.

### 5. GlueResponse error-path precedents

**Existing variants used for structured errors:**

| Variant | Use | Encoder mapping |
|---------|-----|-----------------|
| `GlueResponse::InternalError { reason }` | Generic 5xx-grade error | server.rs maps to 500 (current; check) |
