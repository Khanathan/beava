## Throughput Regression Gate

Per CLAUDE.md §End-to-end throughput regression contract:

**Harness:** `beava-bench-v18 --pipeline small --transport tcp --wire-format msgpack --blast-shape zipfian --cardinality 10000 --total-events 1000000 --parallel 16 --pipeline-depth 1024 --no-ledger` (small/tcp gate cell — 3 runs, median).

**Runs (sustained_eps):** 744,250 / 631,610 / 606,174 → median **631,610 EPS**

| Cell | Phase 13.4.1 EPS | Phase 13.5 baseline EPS | Δ% | Verdict |
|------|---------------:|------------------------:|---:|---------|
| small/tcp (gate) | 631,610 | 674,108 | **−6.30%** | **PASS** (within ±10% gate band; 5K EPS clear of warn floor) |

10% warn floor: 606,697 EPS — gate cell at 631,610 has ~5K EPS headroom.
25% block floor: 505,581 EPS — gate cell well above; not at risk.

The −6.30% delta is most plausibly **measurement variance on the dev box** rather than a structural regression — consistent with the Phase 13.5 −7.1% delta on the same gate cell with no hot-path edits. Phase 13.4.1's edits are concentrated on the *read* hot path (`/get`, `/batch_get`, `OP_GET`, `OP_BATCH_GET`); the *push* hot path (`/push/:event_name`, `OP_PUSH`) — which dominates the small/tcp PUSH-shaped bench — is untouched.

Full row + root-cause analysis appended to `.planning/throughput-baselines.md` (Apple-M4 hw-class section).
