## MVCC storage shape

| Option | Description | Selected |
|--------|-------------|----------|
| `BTreeMap<EntityKey, BTreeMap<Lsn, MvccVersion>>` | Per-entity inner sorted map; `range(..=as_of).next_back()` is O(log n) | ✓ (recommended; orchestrator hint) |
| Per-entity `Vec<(Lsn, MvccVersion)>` (sorted) | Flat vec; O(log n) binary search; better cache locality | |
| B+tree / arena allocator | v1.x optimization; over-engineered for v0 | |

**Auto-selection rationale:** Orchestrator prompt explicitly suggested `BTreeMap` chains; matches pattern already used elsewhere; insert + range are both O(log n) on the inner map; outer map already established for table state.

## Retraction strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Tombstone-style (append `Retracted { undo_of: Lsn }` marker) | Preserves audit trail; correct for `as_of` straddling retract; idempotent | ✓ |
| Version-deletion (remove the original version from the chain) | Simpler; loses audit trail; incorrect for `as_of` queries before the retract | |

**Auto-selection rationale:** `as_of` query correctness is a hard requirement (Phase 12 joins consume this). Version-deletion would silently corrupt `as_of=K` queries where `K < retract_lsn` because the original version disappears. Tombstone-style costs one extra `MvccVersion` per retraction — negligible.

## Retention enforcement

| Option | Description | Selected |
|--------|-------------|----------|
| Snapshot-time sweep + on-write soft cap | Sweeps during snapshot (free I/O); hard per-entity version cap as backstop | ✓ |
| Periodic background sweep (timer-driven) | Adds complexity (lock contention; thread mgmt); marginal gain in v0 | |
| On-write only (no snapshot sweep) | Cheaper but requires writes to amortize; idle tables never sweep | |

**Auto-selection rationale:** Snapshot already walks every table — incremental sweep cost is zero. Soft cap prevents pathological retract-loop memory growth between snapshots. Background timer rejected per CLAUDE.md "single thread for event processing" invariant.

## `as_of` semantics

| Option | Description | Selected |
|--------|-------------|----------|
| `as_of=K` returns version at largest LSN ≤ K (`range(..=K).next_back()`) | Matches "what would I have seen at LSN K" | ✓ |
| `as_of=K` returns version at largest LSN < K | Off-by-one; surprising | |

**Non-temporal table + `as_of`:** Returns 400 (D-08). Silent ignore would create silent join-correctness bugs.

## Bench scope

Three criterion benches selected:
1. `temporal_store_upsert` (chain depths 1, 10, 100, 1000)
2. `temporal_store_as_of_lookup` (same depths)

Pure in-memory; no WAL/fsync. Tracks data-structure regressions cleanly.
