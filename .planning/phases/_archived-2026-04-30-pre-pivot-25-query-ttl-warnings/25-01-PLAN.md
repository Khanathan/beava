<objective>
Ship the v0 multi-table query verb `GET_MULTI` — one TCP round-trip to
assemble an ML feature vector across N Tables for a single entity key —
plus the reserved-opcode stubs for `SCAN` and `SUBSCRIBE` that v0 does
not implement. This closes the v0 query surface described in
`v0-restructure-spec.md §6` and Phase 25 CONTEXT.md §Query surface
(LOCKED decisions).

Purpose: ML inference paths need to read 5-20 Tables per prediction.
Doing that as N sequential GETs is a latency amplifier. One round-trip
with a per-table null-collapse response gives callers a single contract:
"here is the merged feature vector; missing = null."

Output:
- `OP_GET_MULTI` opcode end-to-end (Rust server + Python SDK)
- `OP_SCAN` / `OP_SUBSCRIBE` reserved at the parser boundary
- Rust and Python tests covering happy path + null-collapse + error cases
</objective>
