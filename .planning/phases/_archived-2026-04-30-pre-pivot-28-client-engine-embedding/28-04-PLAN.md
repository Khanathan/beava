## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server TCP | Admin token travels over the wire; server may be remote |
| server → client (response) | Server-filtered snapshot trusted by client |
| client CLI process → stdout | `--dump-json` exposes snapshot entities to any process that captures stdout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-04-01 | Spoofing | client → server handshake | mitigate | Bearer-token handshake mirrors `require_loopback_or_token`; token sourced from `--token` or `TALLY_ADMIN_TOKEN` env; never logged |
| T-28-04-02 | Tampering | server response payload | accept | v0: trusted transport assumption (internal network, loopback demo). TLS/mutual auth deferred — documented as a known gap in SUMMARY |
| T-28-04-03 | Repudiation | audit of snapshot fetches | accept | No audit log in v0; server-side tcp logging (exists today) is the only trail |
| T-28-04-04 | Information disclosure | `--dump-json` writes state to stdout | mitigate | Documented: `--dump-json` is a dev/test ergonomic flag. CLI help text warns that stdout may contain sensitive aggregated state; production clone without `--dump-json` prints only a summary |
| T-28-04-05 | Denial of service | retry loop hammers a down server | mitigate | Exponential backoff with cap (30s) and hard attempt limit (5); no infinite retries |
| T-28-04-06 | Denial of service | malicious server sends oversized payload_len | mitigate | Reject `payload_len > SNAPSHOT_MAX_BYTES` (set const to 10 GiB in `src/client/clone.rs`); on violation return `CloneError::Protocol` without allocating the buffer |

| T-28-04-08 | Tampering | corrupted postcard payload mid-stream | mitigate | postcard decode failure → `CloneError::Decode` → retry; no partial state written to StateStore (bulk_load only runs after successful decode) |
