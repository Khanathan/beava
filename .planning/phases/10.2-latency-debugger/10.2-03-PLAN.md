**2. Add `formatLatency(us)` helper function:**
```javascript
function formatLatency(us) {
  if (us == null || Number.isNaN(us)) return '--';
  if (us < 1) return '<1us';
  if (us < 1000) return Math.round(us) + 'us';
  if (us < 1000000) return (us / 1000).toFixed(1) + 'ms';
  return (us / 1000000).toFixed(2) + 's';
}
```

**3. Add `renderHistogramBars(container, histogram)` function:**

Use el() for all elements. No innerHTML. Maximum 30 bars (one per bin). If histogram is null/empty, show "--" text.

**4. Add `renderLatencySection(panel, node, latencyData)` function:**
This is the 6th section in the stream drill-in panel, per CONTEXT.md Decision 2. Follow the renderThroughputSection pattern exactly:
- Query existing section: `panel.querySelector('.drill-in-section[data-section="latency"]')`

- Render 3-cell stat row (same `stat-row`/`stat`/`stat-label`/`stat-value` classes as throughput section): p50, p95, p99 using formatLatency

- If existing section found, replaceWith. Otherwise appendChild.
- Only render for stream nodes (not views). If node.kind === 'view', skip or show "N/A" placeholder.

**5. Add `renderGlobalLatencyDashboard(panel, latencyData)` function:**
Per CONTEXT.md Decision 2, when no node is selected the drill-in panel shows a global latency dashboard instead of "Select a stream to see details".

Contents:

- Slow-query list: render the `slow_queries` array as a list. Each entry shows: timestamp (formatted as HH:MM:SS), command, stream (or "--"), latency (formatLatency), key_preview (truncated). Use `el('div', { class: 'slow-query-item' })` for each entry. Maximum 20 entries. If empty, show "No slow queries recorded".

The global dashboard MUST NOT set `data-empty` on the panel (it has real content). Remove or do not set `data-empty` attribute when rendering the dashboard.
