Create src/server/latency.rs with the following components, per CONTEXT.md Decision 1 (bucketed histogram) and Decision 4 (slow-query capture):

**Constants:**
- `NUM_BINS: usize = 30`
- `BIN_EDGES: [f64; 31]` — hardcoded const array with precomputed values from formula `10^(i * 4.0 / 30.0)` for i in 0..=30. This covers 1us (10^0) to 10000us (10^4) in 30 log-spaced bins. Hardcode all 31 f64 literals (no runtime computation, no lazy_static). Per RESEARCH Open Question 1.
- `SWAP_INTERVAL_SECS: u64 = 150` (2.5 minutes per CONTEXT.md Decision 4)
- `SLOW_QUERY_CAPACITY: usize = 20`

**Histogram struct:**
- `counts: [u64; NUM_BINS]`, `total: u64`
- `record(&mut self, micros: f64)` — binary search on BIN_EDGES to find bin index, increment counts[bin] and total. Clamp negative/NaN to bin 0. Per RESEARCH Pattern 1, Pitfall 2.
- `percentile(&self, p: f64) -> f64` — walk counts, find bin where cumulative >= target, linear interpolate within bin. Return 0.0 if total == 0. Per RESEARCH Pattern 1.
- `merge(&self, other: &Histogram) -> Histogram` — element-wise add counts and totals.
- Derive Default (all zeros).

**RollingHistogram struct:**
- `current: Histogram`, `previous: Histogram`, `current_start: Instant`, `swap_interval: Duration`
- `new() -> Self` — default histograms, current_start = Instant::now(), swap_interval = 150s
- `record(&mut self, micros: f64, now: Instant)` — check if now >= current_start + swap_interval; if so swap current<->previous, reset current to default, update current_start. Then current.record(micros). Per RESEARCH Pattern 2.
- `snapshot(&self, now: Instant) -> Histogram` — if now > current_start + 2*swap_interval, return empty histogram (idle decay, Pitfall 3). Otherwise merge current + previous. Per RESEARCH Pattern 2.

**CommandKind enum:**
- `#[derive(Clone, Copy, Debug, PartialEq, Eq)]` `#[repr(usize)]`
- Variants: Push=0, Get=1, Set=2, Mset=3
- Implement Display: "PUSH", "GET", "SET", "MSET"

**SlowQueryRecord struct:**
- Fields: `timestamp: SystemTime`, `command: CommandKind`, `stream: Option<String>`, `latency_us: f64`, `key_preview: String`
- Implement `Ord`/`PartialOrd` by latency_us for BinaryHeap usage (PartialEq by latency_us too).

**SlowQueryHeap struct:**
- `heap: BinaryHeap<Reverse<SlowQueryRecord>>`, `capacity: usize`
- `new(capacity: usize) -> Self`
- `insert(&mut self, record: SlowQueryRecord)` — if len < capacity, push. Else if record.latency_us > min's latency_us, pop min and push record. Per RESEARCH Pattern 3.
- `drain_sorted(&self) -> Vec<&SlowQueryRecord>` — return all records sorted descending by latency (for JSON serialization).

**LatencyTracker struct:**
- `command_histograms: [RollingHistogram; 4]` indexed by CommandKind as usize
- `stream_histograms: AHashMap<String, RollingHistogram>` (PUSH-only per-stream)
- `slow_queries: [SlowQueryHeap; 4]` indexed by CommandKind as usize
- `new() -> Self` — init all 4 rolling histograms, empty map, 4 heaps with capacity 20
- `record_push(&mut self, stream_name: &str, micros: f64, now: Instant)` — record into command_histograms[Push] AND stream_histograms entry for stream_name
- `record_command(&mut self, kind: CommandKind, micros: f64, now: Instant)` — record into command_histograms[kind] only
- `maybe_record_slow(&mut self, kind: CommandKind, stream: Option<&str>, micros: f64, key_preview: String)` — per Pitfall 5: check heap minimum FIRST (just f64 comparison), only construct SlowQueryRecord and insert if sample qualifies. This avoids heap alloc on the common path.
- `to_json(&self, now: Instant) -> serde_json::Value` — build the JSON structure from RESEARCH Pattern 5: `{ per_command: [...], per_stream: [...], slow_queries: [...] }`. Snapshot each RollingHistogram, compute p50/p95/p99, collect slow queries from all 4 heaps sorted by latency descending.

Add `pub mod latency;` to src/server/mod.rs.
