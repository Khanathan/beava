```rust
/// Read a single feature value for an entity. Used by cross-key lookups.
/// Returns Missing if entity or feature not found.
/// Takes &mut self because operator read() requires mutable access.
pub fn get_feature_value(&mut self, key: &str, feature_name: &str, now: SystemTime) -> FeatureValue {
    let entity = match self.entities.get_mut(key) {
        Some(e) => e,
        None => return FeatureValue::Missing,
    };
    // Check live operators first
    for (name, op) in entity.live_operators.iter_mut() {
        if name == feature_name {
            return op.read(now);
        }
    }
    // Check static features
    if let Some(sf) = entity.static_features.get(feature_name) {
        return sf.value.clone();
    }
    FeatureValue::Missing
}
```

**Step 2: Implement fan-out in tcp.rs PUSH handler.**

In `handle_sync_command` for `Command::Push`:

After the primary push (which returns features for the primary stream), add fan-out logic:

```rust
Command::Push { stream_name, payload } => {
    let push_start = std::time::Instant::now();
    let mut app = state.lock().unwrap_or_else(|e| e.into_inner());
    let AppState { ref engine, ref mut store, .. } = *app;

    // Primary push
    let features = engine.push(&stream_name, &payload, store, now)?;
    let result = feature_map_to_json(&features);

    // Fan-out: push to other streams whose key_field exists in the event
    let targets = engine.fan_out_targets();
    for (target_name, target_key_field) in &targets {
        if target_name == &stream_name {
            continue; // Skip primary stream
        }
        // Check if event contains this stream's key_field as a non-empty string
        if let Some(serde_json::Value::String(key_val)) = payload.get(target_key_field.as_str()) {
            if !key_val.is_empty() {
                // Push to secondary stream -- ignore errors (primary response already committed)
                let _ = engine.push(target_name, &payload, store, now);
            }
        }
    }

    let push_elapsed = push_start.elapsed();
    app.metrics.push_latency_seconds = push_elapsed.as_secs_f64();
    app.metrics.events_total += 1;
    Ok(result)
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->view registration | Untrusted ViewDefinition with derive expressions and lookup targets |
| lookup->StateStore | Cross-key read -- user controls which entity key is looked up |
| fan-out->multiple streams | Single event triggers state updates across multiple entity keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-08 | Information Disclosure | Lookup cross-key read | mitigate | Lookups only read feature values (not operator internals). The foreign key must exist in the event or entity's features. No arbitrary key enumeration possible -- lookup target stream and feature are fixed at registration. |
| T-5-09 | Denial of Service | Fan-out amplification | mitigate | Fan-out iterates registered streams (bounded by stream count, typically <10). Each secondary push is O(operators) same as primary. No recursive fan-out (each stream pushed once). |
| T-5-10 | Tampering | View registration | mitigate | Views only support derive and lookup features (no windowed operators). Derive expressions use existing parser with bounded AST. Lookup targets validated at registration (stream name must exist). |
| T-5-11 | Denial of Service | Qualified field population | accept | Building qualified feature map duplicates feature values with "Stream.name" keys. O(features * 2) memory per GET. Bounded by registered feature count. Low risk. |
</threat_model>
