<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python SDK â†’ TCP REGISTER | SDK-validated JoinSpec crosses into Rust engine; engine must NOT trust field names or types without re-validation |
| TCP PUSH â†’ cascade â†’ Table lookup | Left-side event fields used to index right-side Table state; malformed/missing fields must yield typed errors, not panics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering | `v0_join_to_stream_def` | mitigate | Re-validate `desc.join.on` fields exist in left schema AND right schema before building `EnrichFromTable`; reject at REGISTER with field-name in error |
| T-23-02 | Denial of Service | Enrichment cascade | accept | Enrichment is O(1) per event (single HashMap lookup). No amplification beyond 1:1 event emission. Rate-limit lives in the TCP layer, not here |
| T-23-03 | Info Disclosure | `_right` collision | mitigate | SDK (_join.py:compute_joined_schema) already applies suffix; engine emits verbatim field names from `right_fields` mapping. Engine MUST NOT synthesize its own suffix (would bypass SDK's collision-resolution rules) |
| T-23-04 | Elevation | Outer-type smuggling | mitigate | Engine rejects `type="outer"` at REGISTER with the exact v0.1-deferred error message, even though SDK already rejects (defense in depth for raw-JSON clients bypassing the SDK) |
</threat_model>
