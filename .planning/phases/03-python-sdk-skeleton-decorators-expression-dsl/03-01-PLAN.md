ValidationError shape (D-11, frozen dataclass):

```python
@dataclass(frozen=True)
class ValidationError:
    kind: str   # "cycle" | "missing_upstream" | "schema_mismatch" | "bad_return_type"
                # | "unknown_field_type" | "table_key_invalid" | "event_time_field_invalid"
                # | "registration_conflict" | "duplicate_name"
    path: str   # e.g., "Transaction.event_time", "Checkouts.filter[2]"
    message: str

    def __str__(self) -> str:
        return f"[{self.kind}] {self.path}: {self.message}"
```

`RegistrationError(Exception)` carries `.code: str`, `.path: str`, `.message: str`, `.errors: list[ValidationError]` (empty for single-error responses).

`BinaryNotFoundError(Exception)` — raised by `_embed.py` in Plan 03-04; created here so Plan 03-04 can import it.

From `main:python/beava/_types_core.py`: collapse nested `Optional[Optional[T]]` to `Optional[T]`; `MISSING` is a singleton sentinel with `__bool__ == False`.

From repo `crates/beava-core/src/schema.rs`: `FieldType` enum variants are `Str, F64, I64, Bool, Bytes, Datetime`. Server accepts lowercase strings via `#[serde(rename_all="snake_case")]`.
