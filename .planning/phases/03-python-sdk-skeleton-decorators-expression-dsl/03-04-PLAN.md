**Frame format (LOCKED from Phase 2.5 `crates/beava-core/src/wire.rs`):**

```
[u32 length BE][u16 op BE][u8 content_type][payload: length-3 bytes]
```

`length` = bytes of `op + ct + payload` (not counting the length field itself). Minimum length is 3 (empty payload).

**Opcode constants (Phase 3 uses only these two):**
- `OP_PING = 0x0000`
- `OP_REGISTER = 0x0001`
- `OP_ERROR_RESPONSE = 0xFFFF`

**Content types (Phase 3 uses only JSON):**
- `CT_JSON = 0x01`
- `CT_MSGPACK = 0x02` (reserved Phase 6+, NOT used here)

**TCP server address:** Server binds TCP per `cfg.tcp.enabled` + `cfg.tcp.port` (`DEFAULT_TCP_PORT = 7380`). Tests use port 0 for OS-assigned. Embed mode parses server stderr to discover ports.

**Server stderr log lines** (from `crates/beava-server/src/server.rs`):
- `{"kind":"server.http_bound","addr":"127.0.0.1:PORT", ...}` — emitted on HTTP bind
- `{"kind":"server.tcp_bound","addr":"127.0.0.1:PORT", ...}` — emitted on TCP bind

Parser: each stderr line is a JSON log record from `tracing`. Embed mode reads lines until both `kind=server.http_bound` AND `kind=server.tcp_bound` arrive; extracts `addr` field from each; timeout 5s. Remaining stderr is forwarded to Python logger at DEBUG per D-10.

**HTTP wire contract (Phase 2, LOCKED from `crates/beava-server/src/register.rs`):**

`POST /register` request body:
```json
{"nodes": [{"kind": "event", "name": "...", ...}, ...]}
```

Success (200):
```json
{"status": "ok", "registry_version": N, "registered_descriptors": [...], "added": [...], "already_present": [...]}
```

Validation failure (400):
```json
{"error": {"code": "invalid_registration", "path": "...", "reason": "..."}, "registry_version": N}
```

Conflict (409):
```json
{"error": {"code": "registration_conflict", "message": "...", "diff": {"added": [...], "removed": [], "changed": [...]}}, "registry_version": N}
```

Unsupported Content-Type (415):
```json
{"error": {"code": "unsupported_media_type", "path": "...", "reason": "..."}, "registry_version": N}
```

**TCP register wire contract (Phase 2.5, same JSON body in a frame):**
- Request: `OP_REGISTER`, `CT_JSON`, payload = UTF-8 JSON bytes identical to HTTP.
- Response on success: `OP_REGISTER`, `CT_JSON`, payload = same JSON as HTTP 200 body.
- Response on error: `OP_ERROR_RESPONSE`, `CT_JSON`, payload = same JSON as HTTP 400/409 body.

**TCP ping wire contract:**
- Request: `OP_PING`, `CT_JSON`, payload = `b"{}"` (or empty — server ignores the payload per Phase 2.5).
- Response: `OP_PING`, `CT_JSON`, payload = `{"server_version": "...", "registry_version": N}`.

         ```
         length = 2 + 1 + len(payload)  # op(2) + ct(1) + payload
         return struct.pack(">IHB", length, op, ct) + payload

       - `def decode_frame(buf: bytes, max_frame_bytes: int = MAX_FRAME_BYTES) -> Frame:`:
         ```
         if len(buf) < 7:
             raise IncompleteFrame(f"need >=7 bytes; got {len(buf)}")
         (length, op, ct) = struct.unpack(">IHB", buf[:7])
         if length > max_frame_bytes:
             raise FrameTooLarge(f"frame too_large: length={length} > max={max_frame_bytes}")
         payload_len = length - 3
         if len(buf) < 7 + payload_len:
             raise IncompleteFrame(f"need {7 + payload_len} bytes; got {len(buf)}")
         return Frame(op=op, ct=ct, payload=buf[7:7+payload_len])
         ```
