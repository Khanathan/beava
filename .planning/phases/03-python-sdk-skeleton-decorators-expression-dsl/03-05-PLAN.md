`App` public API:

```python
class App:
    def __init__(self, url: str | None = None, *, timeout: float = 30.0) -> None:
        ...
    def __enter__(self) -> "App": ...
    def __exit__(self, *exc) -> None: ...  # calls close()
    def close(self) -> None: ...
    def register(self, *descriptors) -> dict:
        """Validate locally, topo-sort, dispatch ONE REGISTER, return server body."""
    def validate(self, *descriptors) -> list[ValidationError]:
        """Run the same local checks as register; zero network I/O."""
    def ping(self) -> dict:
        """Delegates to transport.send_ping(). TCP: returns {server_version, registry_version}.
        HTTP: raises NotImplementedError — ping is a TCP-only opcode in v0 per Phase 2.5."""
```

Embed-mode constraint (D-10): `bv.App()` without URL spawns a subprocess. Users MUST use it as a context manager in embed mode or cleanup won't run. Enforce: if `url is None` and `__enter__` is not called before `register`, raise `RuntimeError("bv.App() embed mode requires 'with bv.App() as app:' pattern")`. Track `_entered: bool` state.

`validate_descriptors(descriptors) -> list[ValidationError]` mirror rules from `crates/beava-core/src/register_validate.rs`:

Rules Python validates pre-wire:
1. **duplicate_name** — two descriptors with same `_name` in one register call.
2. **cycle** — DFS three-color detection on the upstream graph.
3. **missing_upstream** — a derivation lists an upstream name not present in the batch AND not already known (Phase 3 Python can't know server state, so it ONLY flags names not in the current batch; server does the registry-merge check).
4. **schema_mismatch** — for each derivation, walk the op chain (empty in Phase 3 — stateless ops land Phase 4) and resolve any `bv.col(...)` field references. In Phase 3 with empty ops, this rule is a no-op placeholder. Keep the function so Phase 4 can extend it.
5. **event_time_field_invalid** — if descriptor has `event_time_field` set, confirm it's a schema field with type `i64` or `datetime` (already checked at decoration, but re-check post-compilation to catch manually-constructed descriptors).
6. **table_key_invalid** — table's `primary_key` list all in schema (same note as 5).
7. **unknown_field_type** — every `schema.fields.values()` is one of the 6 valid type strings.
8. **bad_return_type** — function-form derivation's return type marker matches output_kind. Phase 3 only exposes `output_kind` via `_beava_kind`; check it is one of `"event" | "table" | "derivation"` and matches the decorator's declared output_kind.

Rules 5-8 are mostly pre-caught at decoration time; include them as defense-in-depth so validate can catch malformed descriptors.

Rules NOT Python-validated (server-only because they require registry state):
- `registration_conflict` — server compares against existing registry.
- `additive_only` — server checks the diff.
- parse errors.

**Topological sort (Kahn's algo):**
```
graph: name -> set of upstream names
in_degree: name -> count of unresolved upstreams
queue: names with in_degree == 0
while queue:
    name = queue.popleft()
    output.append(name)
    for dependent of name:
        in_degree[dependent] -= 1
        if in_degree[dependent] == 0:
            queue.append(dependent)
if len(output) < len(graph):
    # cycle
    raise detect_cycle(graph)
```

`detect_cycle(graph)` — returns a `ValidationError(kind="cycle", path=" -> ".join(cycle_path), message=...)` by DFS-three-color.

**Register dispatch compilation:**
```python
def _compile_register_payload(sorted_descriptors) -> bytes:
    nodes = []
    for desc in sorted_descriptors:
        nodes.append(desc._to_register_json())
    return json.dumps({"nodes": nodes}, ensure_ascii=False).encode("utf-8")
```

Then `transport.send_register(payload_bytes)` returns the server's response dict (already parsed by Plan 03-04).
