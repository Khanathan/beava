<interfaces>
Canonical grammar (D-08, LOCKED — Phase 4 server-side parser depends on this):

```
expr      := field | literal | bin_op | unary_op | call
field     := identifier | identifier "." identifier    # e.g., x, Stream.x
literal   := number | "'" string "'" | "true" | "false" | "null"
bin_op    := "(" expr <space> op <space> expr ")"       # EVERY binary op parenthesized
op        := "+" | "-" | "*" | "/" | ">" | ">=" | "<" | "<=" | "==" | "!=" | "and" | "or"
unary_op  := "(" "not" <space> expr ")"
call      := ident "(" expr ("," <space> expr)* ")"      # e.g., cast(x, float), isnull(x)
```

String literal escaping: `'\\'` inside the value becomes `\\\\`; `'\''` becomes `\\'`. Other characters pass through.

Python → grammar op table:
```
__add__     → "+"     __sub__     → "-"   __mul__ → "*"   __truediv__ → "/"
__lt__      → "<"     __le__      → "<="  __gt__  → ">"   __ge__       → ">="
__eq__      → "=="    __ne__      → "!="
__and__     → "and"   __or__      → "or"  __invert__ → "not" (unary)
```

`cast(x, type)` — second arg is a `_Literal(_BareIdent(type_name))` so it renders as `float` not `'float'`.

`isnull()` is shorthand for `_BinOp("==", self, _Literal(None))` → canonical `(x == null)`.

Type inference (SDK-COL-08) — minimal table:
```
int  + int   -> int
int  + float -> float
float + int  -> float
float + float -> float
any  + (string | bool | bytes | datetime) -> TypeError
```
Exposed as `infer_output_type(lhs: str, rhs: str, op: str) -> str` where lhs/rhs are the server FieldType strings `"i64"|"f64"|"str"|"bool"|"bytes"|"datetime"`. Comparison ops return `"bool"`. Arithmetic ops return the widened numeric type. Non-numeric + arithmetic raises TypeError. `and`/`or` require bool+bool → bool.

Reference: `git show main:python/beava/_col.py` — read the full file end-to-end before implementing. Re-implement, do not copy.
