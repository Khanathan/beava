FieldSpec (normalized schema entry — exposed from `_schema.py`):

```python
@dataclass
class FieldSpec:
    name: str
    py_type: type       # always the inner type (e.g., str, int, datetime.datetime)
    optional: bool = False
    desc: str | None = None
    default: Any = MISSING
```

`extract_schema(cls) -> dict[str, FieldSpec]` flow:
1. `annotations = typing.get_type_hints(cls, include_extras=False)`.
2. For each `(name, py_type)`:
   - If `py_type` is a `_OptionalSpec` (our marker), set `optional=True` and `py_type = py_type.inner`.
   - If `py_type` is `typing.Optional[T]` (i.e., `Union[T, None]`): per D-07 do NOT auto-mark optional — raise TypeError directing user to `bv.Optional[T]`.
   - Validate `py_type` via `py_type_to_field_type(py_type)`; let TypeError propagate with the unsupported-type message.
   - Look up `cls.__dict__.get(name)`; if it's a `_FieldMarker`, merge `desc` + `default`.
3. Return `{name: FieldSpec(...)}` in declaration order (guaranteed by `get_type_hints` + `__annotations__` ordering on Python 3.10+).
