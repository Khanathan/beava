# PLAN 25-02: Unified /debug/warnings endpoint + SignalRegistry bus

## Goal

Build the internal `SignalRegistry` observability bus and expose it via `GET /debug/warnings`. This is the substrate that:
- Phase 25-03's config-recommendation engine will emit into (category=`config`)
- Phase 24's late-drop counter feeds (category=`data_quality`)
- REGISTER / snapshot / memory-pressure paths feed (category=`safety` / `operational`)

Locked response shape per CONTEXT §decisions. Dedupe by `id`. 7d observation window. No external alert delivery — UI reads this endpoint.

## Success Criteria

1. `cargo test --test test_signal_registry` passes with ≥10 unit tests covering: record, dedupe-by-id (first_seen preserved, last_seen advances, evidence overwrites), age-out past 7d, severity sort stability, category filter, empty registry → empty response.
2. `cargo test --test test_debug_warnings_endpoint` passes with ≥6 integration tests covering: empty registry, single signal, multi-severity sort order, dedupe visible in response, category coverage, observation_window field correctness.
3. `GET /debug/warnings` returns HTTP 200 with JSON exactly matching the locked shape:
   ```json
   {
     "generated_at": "<RFC3339 UTC>",
     "observation_window": "7d",
     "warnings": [{"id", "severity", "category", "title", "detail", "action", "first_seen", "evidence"}]
   }
   ```
4. Severity sort: `critical` > `error` > `warning` > `info`; within severity, order by `first_seen` ascending (stable).
5. Phase 24's `tally_late_events_dropped_total` counter is polled on each snapshot cycle; if the 1-minute rate exceeds a configurable threshold (default: 1% of total events), a `data_quality` signal is emitted with id `late_drop.{stream_name}`.
6. A REGISTER failure emits a `safety` / `error` signal with id `register.failure.{pipeline_name}` and the error message in `detail`.
7. A snapshot write failure emits an `operational` / `error` signal with id `snapshot.failure`.
8. Memory pressure (RSS / configured_limit > 0.85) emits an `operational` / `warning` signal with id `memory.pressure`; polled on snapshot cycle.
9. Endpoint is admin-gated (same middleware as `/debug/key`, `/debug/streams`).
10. Registry persistence: acceptable to be in-memory only (per CONTEXT deferred list). Restart loses signals; document this in the module doc comment.

## Implementation Steps

### Task 1 — SignalRegistry core + tests (Wave 1)

1. Create `/data/home/tally/src/server/signals.rs`:
   ```rust
   //! Internal signal bus for /debug/warnings + config recommendations.
   //!
   //! In-memory only in v0; restart loses state. Dedupe by Signal.id.
   //! 7-day default observation window. See CONTEXT §decisions.

   use std::sync::{Arc, RwLock};
   use std::time::{SystemTime, Duration};
   use serde::{Serialize, Deserialize};
   use ahash::AHashMap;

   #[derive(Clone, Copy, Debug, PartialEq, Eq, PartialOrd, Ord, Serialize, Deserialize)]
   #[serde(rename_all = "lowercase")]
   pub enum Severity { Info, Warning, Error, Critical } // Ord order matters: smallest→largest

   #[derive(Clone, Copy, Debug, PartialEq, Eq, Serialize, Deserialize)]
   #[serde(rename_all = "snake_case")]
   pub enum Category { Config, DataQuality, Operational, Safety, Performance }

   #[derive(Clone, Debug, Serialize)]
   pub struct Signal {
       pub id: String,
       pub severity: Severity,
       pub category: Category,
       pub title: String,
       pub detail: String,
       #[serde(skip_serializing_if = "Option::is_none")]
       pub action: Option<serde_json::Value>,
       pub first_seen: SystemTime,
       pub last_seen: SystemTime,
       pub evidence: serde_json::Value,
   }

   pub struct SignalRegistry {
       signals: AHashMap<String, Signal>,   // key = signal.id
       observation_window: Duration,
   }
   pub type SharedRegistry = Arc<RwLock<SignalRegistry>>;
   ```
   Implement methods:
   - `pub fn new(observation_window: Duration) -> Self`
   - `pub fn record(&mut self, mut sig: Signal)` — if `self.signals` already has `sig.id`, preserve existing `first_seen` and set `sig.last_seen = now()`. Overwrite title/detail/action/evidence. Severity and category can update in place (allow escalation).
   - `pub fn age_out(&mut self, now: SystemTime)` — drop any signal where `now.duration_since(last_seen) > observation_window`.
   - `pub fn snapshot_sorted(&self, now: SystemTime, filter: Option<Category>) -> Vec<Signal>` — returns signals (after age_out logic applied virtually; actual age_out happens on write paths or on scheduled tick) sorted by severity desc, then first_seen asc.
   - `pub fn categories_present(&self) -> HashSet<Category>` (test helper).

   async fn debug_warnings(
       State(state): State<SharedState>,
       Query(params): Query<AHashMap<String, String>>,
   ) -> impl IntoResponse {
       let category = params.get("category").and_then(|s| parse_category(s));

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Breaking 21 existing `test_register_json_v0` tests by changing `register()` signature | Pass `Option<&SharedRegistry>` — existing tests pass `None` and continue working unchanged. Only production callers supply Some. |
| RwLock contention on high-traffic paths emitting signals | Signal emission is off the hot path (only on REGISTER, snapshot cycle, late-drop polling — all low-frequency). Document this constraint in `signals.rs` module comment. Hot-path PUSH does NOT call record(). |
| Snapshot failure emitter recursion (snapshot fails → try to record signal → if record path also uses disk, recurse) | SignalRegistry is in-memory only; no disk I/O in record(). Explicit unit test `test_record_no_io`. |
| Clock skew / `duration_since` underflow in age_out | Use `duration_since(last_seen).unwrap_or(Duration::ZERO)` — same pattern as `src/state/eviction.rs` uses for `now.duration_since(last_event)`. |
| Memory-pressure emitter on non-linux hosts | `#[cfg(target_os = "linux")]` gate the `/proc/self/statm` read; non-linux path is a no-op. Documented in module comment. |
| Late-drop rate computation needs prior counter value | Store `prev_counter: AHashMap<String, u64>` with last-polled timestamp; on first poll for a stream, skip emission (bootstrap). |
