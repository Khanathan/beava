Frame layout (little-endian throughout):
```
| field          | size   | covers_crc |
|----------------|--------|------------|
| length         | u32    | no         |  (length of [crc..end] = 4 + 8 + 1 + payload_len)
| crc32c         | u32    | no         |  (CRC32C of lsn || record_type || payload)
| lsn            | u64    | yes        |
| record_type    | u8     | yes        |  (0x01=Event, 0x02=RegistryBump reserved)
| payload        | bytes  | yes        |  (record-type-specific; Event = serde_json of WalEventPayload)
```

Segment header (one per file, written at open):
```
| magic          | 8 bytes | b"BEAVAWAL" |
| format_version | u32     | = 1 |
| start_lsn      | u64     | first LSN in this segment |
| registry_vers  | u32     | registry version at creation |
```

Public API (crates/beava-persistence/src/lib.rs):
```rust
pub type Lsn = u64;

#[repr(u8)]
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum RecordType { Event = 0x01, RegistryBump = 0x02 }

#[derive(Debug, Clone)]
pub struct WalRecord {
    pub lsn: Lsn,
    pub record_type: RecordType,
    pub payload: Vec<u8>,
}

#[derive(Debug, thiserror::Error)]
pub enum PersistError {
    #[error("io: {0}")] Io(#[from] std::io::Error),
    #[error("crc mismatch at offset {offset}: expected {expected:#010x}, got {got:#010x}")]
    CrcMismatch { offset: u64, expected: u32, got: u32 },
    #[error("bad magic: expected BEAVAWAL, got {got:?}")] BadMagic { got: [u8; 8] },
    #[error("unsupported format_version {0}")] UnsupportedVersion(u32),
    #[error("unknown record_type {0:#04x}")] UnknownRecordType(u8),
    #[error("torn record at offset {offset}: {reason}")] TornRecord { offset: u64, reason: String },
}

pub struct WalWriter { /* file handle + current offset + next_lsn counter exposure */ }
impl WalWriter {
    pub fn open(dir: &std::path::Path, start_lsn: Lsn, registry_version: u32) -> Result<Self, PersistError>;
    pub fn append(&mut self, record: &WalRecord) -> Result<(), PersistError>;  // no fsync
    pub fn current_path(&self) -> &std::path::Path;
    pub fn bytes_written(&self) -> u64;  // since open (excludes header)
}

pub struct WalReader { /* file handle + pos */ }
impl WalReader {
    pub fn open(path: &std::path::Path) -> Result<Self, PersistError>;
    pub fn start_lsn(&self) -> Lsn;
    pub fn registry_version(&self) -> u32;
}
impl Iterator for WalReader { type Item = Result<WalRecord, PersistError>; }
```

Tail-truncation rule: if the final record has length != payload_len or CRC mismatches and the file ends exactly at the torn boundary, the iterator returns `Ok(None)` (treats torn last record as end-of-stream). Only intermediate corruption yields a `Err(CrcMismatch)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| on-disk WAL file ↔ WalReader | Reader consumes bytes potentially corrupted by fs/hardware faults or adversarial files. Must validate magic, format_version, and per-record CRC32C. |
| WAL payload bytes ↔ downstream consumers | Payload is opaque bytes in this plan; caller in Plan 03 decodes as JSON and MUST re-validate schema. Persistence crate does not sanitize payload content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | WAL segment file on disk | mitigate | CRC32C over [lsn||type||payload] on every record; header magic `BEAVAWAL` + format_version validated on open. Single-bit payload flips are detected and returned as `PersistError::CrcMismatch`. |
| T-06-02 | Tampering | Segment header | mitigate | Magic + format_version check in `segment::read_header`. Wrong magic → `BadMagic`; unsupported version → `UnsupportedVersion`. |
| T-06-03 | Denial of Service | Torn last record from crash | mitigate | Reader treats EOF or a partial-length-prefix as clean end-of-stream (returns `None` from iterator); only interior CRC mismatches raise errors. |
| T-06-04 | Information Disclosure | WAL file permissions | accept | File created with default OS permissions (typically 0644 on Unix); single-tenant by project design, operator controls fs access. Revisit if multi-tenant requirement ever emerges (currently deferred to commercial tier). |
| T-06-05 | Repudiation | LSN ordering | accept | Phase 6 relies on caller (Plan 02) to assign monotonic LSNs; this plan does not enforce ordering during append. Plan 02 adds the ordering contract via the fsync worker. |
</threat_model>
