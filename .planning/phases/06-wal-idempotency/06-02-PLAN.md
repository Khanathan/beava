```rust
#[derive(Debug, Clone)]
pub struct WalSinkConfig {
    pub dir: std::path::PathBuf,
    pub initial_start_lsn: Lsn,
    pub initial_registry_version: u32,
    pub fsync_interval_ms: u64,      // default 2; 0 = fsync every append
    pub fsync_bytes: u64,            // default 1 << 20 (1 MiB); 0 = only time-based
    pub segment_bytes: u64,          // default 128 << 20 (128 MiB)
}

impl Default for WalSinkConfig {
    fn default() -> Self { /* defaults above, dir = PathBuf::from(".") */ }
}

#[derive(Clone)]
pub struct WalSink {
    append_tx: tokio::sync::mpsc::Sender<AppendRequest>,
    durable_lsn: tokio::sync::watch::Receiver<Lsn>,
    control_tx: tokio::sync::mpsc::Sender<ControlMsg>,  // for truncate, shutdown
}

impl WalSink {
    /// Spawns the worker task; returns the sink handle + a JoinHandle for the worker.
    pub fn spawn(cfg: WalSinkConfig) -> Result<(Self, tokio::task::JoinHandle<()>), PersistError>;

    /// Staged append: caller supplies record_type + payload; worker assigns the LSN.
    /// The returned future resolves only after the assigned LSN is durably on disk.
    pub async fn append_event(&self, payload: Vec<u8>) -> Result<Lsn, PersistError>;

    /// Returns the last-durable LSN (snapshot of the watch channel).
    pub fn durable_lsn(&self) -> Lsn;

    /// Delete closed segments whose last_lsn < covered_lsn.
    pub async fn truncate_up_to(&self, covered_lsn: Lsn) -> Result<u32, PersistError>;

    /// Graceful shutdown: flush + fsync pending batch, close current segment, return.
    pub async fn shutdown(self) -> Result<(), PersistError>;
}

// Internal (not part of crate public API)
struct AppendRequest { payload: Vec<u8>, done: tokio::sync::oneshot::Sender<Result<Lsn, PersistError>> }
enum ControlMsg { TruncateUpTo { covered_lsn: Lsn, ack: oneshot::Sender<Result<u32, PersistError>> }, Shutdown(oneshot::Sender<Result<(), PersistError>>) }
```

Worker loop pseudo:
```
let mut pending: Vec<AppendRequest> = Vec::new();
let mut staged_bytes: u64 = 0;
let mut next_lsn = cfg.initial_start_lsn;
loop {
    select! {
        Some(req) = append_rx.recv() => {
            staged_bytes += estimate_encoded_len(&req.payload);
            pending.push(req);
            if staged_bytes >= cfg.fsync_bytes { flush_now = true }
        }
        _ = sleep(fsync_interval_ms), if !pending.is_empty() => { flush_now = true }
        Some(ctrl) = control_rx.recv() => { handle(ctrl); continue; }
    }
    if flush_now {
        // assign LSNs, encode + write each record, sync_data, rotate if needed, bump watermark
        let mut highest = 0;
        for req in pending.drain(..) {
            let lsn = next_lsn; next_lsn += 1;
            writer.append(&WalRecord { lsn, record_type: Event, payload: req.payload })?;
            highest = lsn;
            done_senders.push((lsn, req.done));
        }
        spawn_blocking(|| writer.sync_data()).await??;
        if writer.bytes_written() >= cfg.segment_bytes { rotate(&mut writer, next_lsn)?; }
        durable_lsn_tx.send(highest)?;
        for (lsn, tx) in done_senders.drain(..) { let _ = tx.send(Ok(lsn)); }
        staged_bytes = 0;
    }
}
```

Truncate rule (rotation.rs):
- Enumerate files matching `wal-<lsn_16hex>.log` in dir
- Parse start_lsn from filename
- Read each segment header's start_lsn to confirm
- A segment's *last_lsn* is approximated by the NEXT segment's start_lsn - 1
- Delete a segment iff it is NOT the current open one AND its last_lsn < covered_lsn
- Return count of deleted files

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| async push handler ↔ fsync worker | Cross-task boundary via tokio MPSC; worker owns the file handle. Bounded channel (1024) prevents unbounded memory growth under backpressure. |
| OS page cache ↔ durable disk | `sync_data` call is the enforceable boundary; handler's future MUST NOT resolve until syscall returns. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Denial of Service | MPSC append channel | mitigate | Channel capacity = 1024; when full, `append_tx.send().await` exerts backpressure on HTTP handler — OS TCP backpressure follows naturally. |
| T-06-07 | Tampering | Rotation race (truncate during append) | mitigate | Worker serializes append + truncate in one event loop (both arrive on different channels but processed in the same select!); no concurrent modification possible. |
| T-06-08 | Denial of Service | fsync blocking event loop | mitigate | `sync_data` runs under `spawn_blocking` to keep the tokio runtime responsive; worker doesn't starve non-fsync control messages. |
| T-06-09 | Repudiation | LSN assignment ordering | mitigate | Single worker task assigns LSNs — monotonic and unique by construction. Concurrent callers see well-defined `FIFO-at-channel-arrival` ordering. |
| T-06-10 | Tampering | Truncate deletes wrong segment | mitigate | `truncate_up_to` explicitly excludes `current_start_lsn`; only closed segments considered; covered_lsn comparison uses `next_segment_start_lsn <= covered_lsn` to avoid off-by-one that would delete a segment whose tail extends past covered_lsn. |
</threat_model>
