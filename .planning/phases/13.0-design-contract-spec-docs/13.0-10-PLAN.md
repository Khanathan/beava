For each bounded-buffer page, use the Edit tool to replace TODO markers with concrete prose. Per-op content guidance:

- **histogram.md**: count per fixed bucket; takes `field` + `buckets: list[float]` (REQUIRED) + `window`; returns dict `{bucket_label: count}`; BoundedByRequiredKwarg('buckets') memory ceiling = O(len(buckets)); cite V0-MEM-GOV-02; example: "transaction amount histogram with buckets=[0, 10, 50, 100, 500, inf]"
- **hour_of_day_histogram.md**: 24-bin count histogram per entity (one bin per hour 0-23); no kwargs needed (24 fixed bins); O(24) = O(1) memory; example: "user activity by hour of day"
- **dow_hour_histogram.md**: 168-bin (day-of-week × hour) histogram; same as hour_of_day but with weekday axis; O(168) = O(1) memory; example: "user activity by Mon-9am vs Fri-7pm pattern"
- **seasonal_deviation.md**: z-score against this entity's hour-of-day baseline; returns float; uses internal hour_of_day_histogram state; useful for detecting unusual-hour activity; null if cold-start
- **event_type_mix.md**: proportion per category over window; takes `field` (categorical) + `categories: list[str]` (optional — limits to specific categories) + `window`; returns dict `{category: proportion_in_[0,1]}`; BoundedByConfig('max_categories', 256) memory cap; cite V0-MEM-GOV-02; if more categories than 256 are seen, cap takes the most-frequent + "other" bucket per Phase 12.8 cap-hit metric
