# Phase 13.0 Plan 05: Operator catalog scaffold — 54 op page stubs + index.md + coverage tripwire Summary

## One-liner

Mechanical scaffold pass: generated 54 per-op page paths (53 unique AggKind
variants + ema alias inline in ewma.md) across 7 family subdirectories, plus a
master `docs/operators/index.md` catalog and a CI tripwire script
(`scripts/check_op_page_coverage.py`) that asserts 1:1 between the AggKind
catalogue and on-disk page files. `rate_of_change.md` correctly lives under
`velocity/` per RESEARCH §3 + §5; ADR-002 renamed ops use the NEW Polars names
as filenames + H1 with a "Previously called" note for searchability. Single
`docs(13.0-05)` commit per CLAUDE.md TDD §Note 4 doc-only-plan exemption.

## What landed

### scripts/scaffold_op_pages.py (397 LOC)

Idempotent scaffold script. Inputs:

- A hardcoded canonical `OPS` list (54 tuples: `(family, op, polars_renamed_from, brief_description)`)
  — single source of truth for Plan 05.
- A hardcoded `LIFETIME_BOUND` dict mirroring
  `crates/beava-core/src/register_validate.rs::lifetime_bound_for_op_str`
  classification (O1 / BoundedSketch / BoundedByRequiredKwarg / BoundedByConfig).
- A hardcoded `POLISH_PLAN` map (which Plan 13.0-NN polishes which family/op).

Outputs:

- 54 op page stubs at `docs/operators/<family>/<op>.md`.
- `docs/operators/index.md` (master 54-row catalog).
- `docs/operators/decay/ewma.md` augmented with an inline `## Aliases` section
  documenting `bv.ema` (idempotent — skipped if already present).

Each stub uses the 9-section template per RESEARCH §3 + CONTEXT D-02:

- `# bv.<op>` (H1)
- `> <brief>` (one-line description)
- `## Signature` (Python signature placeholder + ADR-002 rename note if applicable)
- `## Description` (TODO Plan 13.0-NN)
- `## Parameters` (TODO Plan 13.0-NN)
- `## Returns` (TODO Plan 13.0-NN)
- `## Complexity` (3-row table: CPU per event TODO + Memory per entity from
  LIFETIME_BOUND + Lifetime mode TODO)
- `## Examples` (TODO Plan 13.0-NN)
- `## Wire` (JSON wire form skeleton + cross-link to
  `examples/wire/register-fraud-team.request.json`)
- `## Edge cases` (TODO Plan 13.0-NN)
- `## See also` (cross-links: cost-class.md + TODO related ops + compilation-rules.md)

Re-running the script OVERWRITES op page stubs — so polish plans 06-11 must NOT
re-run the scaffold; they Edit individual files in place. The ewma.md
`## Aliases` append is idempotent (skipped if `bv.ema` already present) so
re-runs don't duplicate the section.

### scripts/check_op_page_coverage.py (82 LOC)

CI tripwire. Imports `OPS` from sibling `scaffold_op_pages.py` (single source
of truth — no separate canonical list to keep in sync) and computes:

- `expected_paths = { docs/operators/<family>/<op>.md for each OPS entry }`
- `actual_paths = { all *.md under docs/operators/ excluding index.md / cost-class.md / .keep }`
- `missing = expected - actual`
- `extra = actual - expected`

If either set is non-empty: prints diagnostic listing (per-file MISSING /
UNEXPECTED markers) + a hint to update `OPS` in `scaffold_op_pages.py`, then
exits 1.

Also verifies that `docs/operators/decay/ewma.md` contains both `## Aliases`
and `bv.ema` — the inline-alias special case for the 54-paths-but-53-AggKinds
invariant. If either is missing, prints a clear diagnostic and exits 1.

Verified by round-trip test in this session:

```bash
mv docs/operators/core/count.md docs/operators/core/_count_TEMP.md
python3 scripts/check_op_page_coverage.py
# FAIL — op-page coverage mismatch:
#   MISSING: docs/operators/core/count.md
#   UNEXPECTED: docs/operators/core/_count_TEMP.md
#   hint: update OPS in scripts/scaffold_op_pages.py to match...
# exit=1
mv docs/operators/core/_count_TEMP.md docs/operators/core/count.md
python3 scripts/check_op_page_coverage.py
# OK — 54 op pages match catalogue (53 unique AggKind variants + ema alias inline in ewma.md = 54 page paths)
# exit=0
```

Tripwire detects BOTH missing AND extra files — symmetric coverage. Closure
Plan 13.0-15 will run this as the final gate before phase advance.
