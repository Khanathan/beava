## Architectural outcomes

1. **Wire contract locked at machine-validatable level.** D-01 ships 13 JSON Schema 2020-12 files in `examples/wire/schemas/` plus 20 worked-example fixtures — the schemas are live-checkable artifacts (`examples/wire/_validate_examples.py` exits 0). SDK porters in Phase 13.6 read schemas as source of truth, NOT the Python implementation. Same schemas wire into Phase 13.4 engine CI for register/push/get round-trip validation.

2. **53-op catalog documented in DuckDB/Polars devex-first style.** D-02 ships 54 op page paths (53 unique AggKind + ema alias inline in ewma.md) across 7 family subdirs. Each page follows a strict template (signature / semantics / return / complexity / worked example / edge cases / cross-link to wire JSON). Highest devex per `project_v2_devex_first` — Priya (target user per `project_beava_website_ia`) googling `bv.percentile` lands on a self-contained page with a runnable Python snippet.

3. **9 runnable demos doubling as Phase 13.5/13.6 integration regression test.** D-03 ships `examples/{python,typescript,go}/{adtech,fraud,ecommerce}.{py,ts,go}` — 9 files runnable against per-language mocks during 13.0 (`examples/test_examples.sh` exits 0); re-verified against the real engine in 13.5/13.6. If a demo breaks against the real engine post-13.4, that's a contract bug — surfaces immediately rather than at v0.0.0 ship.

4. **Pipeline-DSL ambiguity matrix unblocks SDK porters (Phase 13.6).** D-04 ships per-chain-method H3 worked examples + a 20+ row ambiguity matrix in `docs/pipeline-dsl/compilation-rules.md`. Each ALLOWED row links to a JSON wire example; each FORBIDDEN row links to the structured error code SDK porters MUST raise. SDK porters writing the TS+Go compilers don't reverse-engineer the Python implementation.

5. **ADR-001 partial overturn of `project_v0_events_only_scope` formalized.** `@bv.table` revived as aggregation-output decorator (function-form `@bv.table(key=...) def Name(ev) -> bv.Table: return ev.group_by(...).agg(...)`). MVCC, TemporalStore, `app.upsert/delete/retract` STAY KILLED. Phase 13.4 amends `phase12_7_no_table_surface.rs` allowlist to permit `OpNode::Table*` for derivation `output_kind=table` (per-AST-context check, not global symbol grep).

6. **ADR-002 Polars op renames documented.** `avg→mean / variance→var / stddev→std / count_distinct→n_unique / percentile→quantile` documented as the v0 contract. Mechanical renames deferred: Rust ops + fraud-team.json → Phase 13.4; Python SDK aliases → Phase 13.5; TS+Go → Phase 13.6.

7. **ADR-003 mid-execution scope amendment: global aggregation + bv.lit (per user directive 2026-05-03).** First-class global aggregation (`@bv.table` no `key=` / `events.agg(...)` no `group_by`) and public `bv.lit(value)` export added to v0 launch surface. Implementation deferred: Phase 13.4 (~30 LOC engine sentinel routing for `entity_id=""` + accept `key_cols=[]`) + Phase 13.5 (~110 LOC Python SDK) + Phase 13.6 (~150 LOC TS+Go). Closes monitoring + dashboard use cases ("total throughput / global p95 / top-10-globally") that were blocked in the per-entity-only design.

8. **v0 critical path advances to 4-way parallel.** Phase 13.0 (CLOSED) → parallel 13.4 (engine) + 13.5 (Python+bench) + 13.6 (TS+Go) + 13.7 (docs site) → sequential 13.8 (packaging + GA tag). Total wall-clock for v0: ~3-4 weeks with parallelism (vs ~6-7 weeks fully serial).

## Lessons learned

- **Doc-only TDD §Note 4 exemption reduced commit overhead.** Single `docs(13.0-NN):` commit per plan vs paired `test:` + `feat:` commits. Saved ~30 commits across 15 plans. Convention preserved auditability via plan-level granularity (every plan = 1 commit + 1 SUMMARY = full traceability).
- **Section-ownership contract held cleanly.** Closure plan (13.0-15) touches only its own sections (SUMMARY/VERIFICATION/STATE/ROADMAP/CLAUDE.md/index.md + ADR-003 + ADR-003-derived patches). Spec docs (Plans 02-14) untouched by the closure plan in their existing sections; only additive patches per scope amendment.
- **Q-answer pre-resolution flow worked.** 8 open questions surfaced at research time → user picked 2 product-shaping (Q1 Path B + Q7 OP_RESET) → researcher resolved 6 logistics-autonomous → all 8 resolved before plan execution. Zero mid-plan blocking on architectural ambiguity.
- **Mid-execution scope amendment via single combined ADR.** ADR-003 covers BOTH global aggregation AND `bv.lit` — they're related and small enough that splitting into ADR-003 + ADR-004 would have added bookkeeping noise without information value. Pattern reusable for future mid-execution scope additions.
- **Forward-references handled by closure-plan link check.** Plan 03 (http-api.md, Wave 1) links to Plan 12 (error-codes.md, Wave 2). Cross-link check in closure-plan Task 4 catches any unresolved forward references; none found at this gate.
- **53 vs 54 op-page reconciliation: explicit citation in research + summaries prevents future confusion.** Numbers stay consistent across all docs because a one-line note in research + each plan SUMMARY + this VERIFICATION explains the count discrepancy ("53 unique AggKind variants + ema alias inline in ewma.md = 54 page paths").
- **Failing-acceptance-test suite as Phase 13.4 + 13.5 forcing function (Plan 16).** 68 tests gated by `_engine_available()` SKIP until the engine + Python SDK land. When 13.4 + 13.5 implement the renamed ops + `@bv.table` + `bv.lit` + global-agg surface, these tests turn GREEN as the acceptance gate. Pattern carries forward — failing tests as the contract, implementations make them pass.
- **Doc-tree NUKE-and-rewrite (Plan 01) was the right call.** Stale `docs/architecture.md` / `http-api.md` / `python-sdk.md` / `protocol.md` / `quickstart.md` / `operators.md` predated the no-event-time pivot + table strip + Polars rename. Preserving them would have leaked stale assumptions into spec docs. NUKE preserves git history for archaeology while clearing the slate. `docs/operators/cost-class.md` (Phase 19.2) KEPT — alive and current.

## Why this phase exists

Phase 13.0 is the bottleneck for v0. Without locked specs, every implementation phase (13.4 / 13.5 / 13.6 / 13.7) is speculative — the engine speculates the wire shape, the Python SDK speculates the op signatures, the TS+Go SDKs speculate the cross-language contract, the docs site speculates the structure. Phase 13.0 collapses all four speculations into a single locked design contract. From here forward, implementation is mechanical: read the spec, write the code, run the test, ship.

The scope amendment (ADR-003 mid-execution) demonstrates the value of doing this phase first: the user noticed the global-agg + bv.lit gap during the design-contract authoring, surfaced it as "ship both / do both", and the cost was ~150 LOC across 3 implementation phases. Catching this gap during 13.4/13.5/13.6 implementation would have cost a re-discuss + re-plan + cross-phase coordination — orders of magnitude more expensive.
