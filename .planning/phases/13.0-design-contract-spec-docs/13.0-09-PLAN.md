For each velocity/ page, use the Edit tool to replace TODO markers with concrete prose. Per-op content guidance:

- **rate_of_change.md**: rate or acceleration delta across two adjacent windows; returns float; takes `field` (numeric or count via implicit shape) + `window` + optional `sub_window`; cold-start = null; example: "transaction rate change over last hour vs prior hour"; useful for detecting acceleration in behavioral patterns; O1 memory; Tier 1 CPU
- **inter_arrival_stats.md**: returns struct/dict `{mean_ms: float, stddev_ms: float, cv: float}` of gaps between matching events; takes optional `where=` predicate + `window=` kwarg; useful for detecting bot-like cadence; example: "transaction inter-arrival stats over 1h"; cold-start with <2 events returns null fields
- **burst_count.md**: max events observed in any sub-window inside the outer window; takes `window` + `sub_window` (BOTH required); returns int; example: "max IPs in any 1-minute slice over 1-hour window"; sub_window MUST be < window or register-time error
- **delta_from_prev.md**: current value minus previous event's value; returns same type as field; null on first event; example: "amount delta vs prior transaction"; useful for detecting sudden value jumps
- **trend.md**: slope of EW linear regression over the window; returns float (+ = increasing, - = decreasing); window REQUIRED; cold-start with <2 events returns null
- **trend_residual.md**: current value minus trend-predicted value; returns float; uses same EW regression as trend.md; useful for "is this event consistent with trend?"
- **outlier_count.md**: count of events beyond `sigma * stddev` from rolling mean in the window; default `sigma=3`; takes `field` + `window` + optional `sigma` kwarg (default 3); returns int
- **value_change_count.md**: count of field value flips over window; returns int; example: "how many distinct merchant categories did the user touch in 1h"; equivalent to `count(distinct(field, window=...)) - 1` but cheaper
- **z_score.md**: current event's value vs rolling mean/stddev baseline over `baseline_window`; takes `field` + `baseline_window` (REQUIRED) + `current` (default = current event value); returns float; null if stddev=0; useful for "how unusual is this transaction amount given user's history"

For each, ensure:
- Window kwarg semantics + duration-string grammar
- Cold-start behavior explicit (typically null until enough events accumulated)
- 1-2 Python examples with concrete kwarg values
- JSON wire form

# Velocity / Trend / Z-Score Aggregation Operators

The 9 velocity-family ops cover rate-of-change between windows, behavioral cadence (inter-arrival stats, burst count), value-trajectory (delta-from-prev, trend, trend residual), outlier flags (sigma-bound counts), value-flip counts, and entity-level z-scores.

| Op | Required kwargs | Returns |
|----|-----------------|---------|
| [`bv.rate_of_change`](./rate_of_change.md) | `window` | float or null |
| [`bv.inter_arrival_stats`](./inter_arrival_stats.md) | `window` | dict `{mean_ms, stddev_ms, cv}` |
| [`bv.burst_count`](./burst_count.md) | `window`, `sub_window` | int |
| [`bv.delta_from_prev`](./delta_from_prev.md) | — | same type as field |
| [`bv.trend`](./trend.md) | `window` | float (slope) |
| [`bv.trend_residual`](./trend_residual.md) | `window` | float |
| [`bv.outlier_count`](./outlier_count.md) | `window` (sigma optional, default 3) | int |
| [`bv.value_change_count`](./value_change_count.md) | `window` | int |
| [`bv.z_score`](./z_score.md) | `baseline_window` | float or null |

All 9 are O(1) memory per entity. Tier 1 / Tier 2 CPU per [cost-class.md](../cost-class.md). Lifetime mode is allowed for trend / trend_residual / value_change_count / z_score (windowless EW versions); burst_count + outlier_count + inter_arrival_stats + rate_of_change require explicit `window=`.

Velocity-family ops use server processing-time per [`project_redis_shaped_no_event_time_ever`](../../../$HOME/.claude/projects/-Users-petrpan26-work-tally/memory/project_redis_shaped_no_event_time_ever.md). Inter-arrival stats compute Δt between consecutive arrival events (NOT event-time field).

> Note: per REQUIREMENTS.md, `z_score` is family AGG-Z-* — it lives here in `velocity/` per RESEARCH §3 directory layout (it's a velocity-related entity-level statistic). `rate_of_change` is the canonical Phase 9 velocity-family op per RESEARCH §5 (NOT decay/ — decay-family ops use half-life-based EW averaging without explicit windows).

- [cost-class.md](../cost-class.md)
- [Decay family](../decay/) — related: trend uses EW regression (see ewma); rate_of_change can be paired with EW averages
- [Sketch family](../sketch/) — related: outlier_count uses similar sigma-thresholding to entropy
