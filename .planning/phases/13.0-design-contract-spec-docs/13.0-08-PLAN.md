# Decay-Family Aggregation Operators

The 6 decay-family ops cover exponentially-weighted statistics (EWMA / EWVar / EW Z-Score), forward-decay accumulators (Cormode 2009), and time-weighted averages.

| Op | Required kwarg | Returns |
|----|----------------|---------|
| [`bv.ewma`](./ewma.md) (alias `bv.ema`) | `half_life` | float |
| [`bv.ewvar`](./ewvar.md) | `half_life` | float |
| [`bv.ew_zscore`](./ew_zscore.md) | `half_life` | float or null |
| [`bv.decayed_sum`](./decayed_sum.md) | `half_life` | float |
| [`bv.decayed_count`](./decayed_count.md) | `half_life` | float |
| [`bv.twa`](./twa.md) | `window` | float |

All 6 are O(1) memory per entity and run in Tier 1 / Tier 2 CPU per [cost-class.md](../cost-class.md). Lifetime mode is default for the EW-decay family — half-life sets the decay rate. `twa` requires an explicit window.
