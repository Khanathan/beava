# Bounded Buffer + Geo Aggregation Operators

The buffer-geo/ subdirectory hosts 11 ops grouped into two related families: **bounded buffers** (count-per-bucket histograms, deques, reservoirs, categorical mixes) and **geo** (haversine-distance aggregations on lat/lon field pairs). Both families share the property that their state is non-O(1) but bounded.

## Bounded buffers (7)

| Op | Memory class | Required kwarg |
|----|--------------|----------------|
| [`bv.histogram`](./histogram.md) | BoundedByRequiredKwarg("buckets") | `buckets` |
| [`bv.hour_of_day_histogram`](./hour_of_day_histogram.md) | O(24) | — |
| [`bv.dow_hour_histogram`](./dow_hour_histogram.md) | O(168) | — |
| [`bv.seasonal_deviation`](./seasonal_deviation.md) | O(24) (uses HoD baseline) | — |
| [`bv.event_type_mix`](./event_type_mix.md) | BoundedByConfig("max_categories", 256) | — |
| [`bv.most_recent_n`](./most_recent_n.md) | BoundedByRequiredKwarg("n") | `n` |
| [`bv.reservoir_sample`](./reservoir_sample.md) | BoundedByRequiredKwarg("samples") | `samples` |

## Geo (4)

| Op | Memory class | Required kwargs |
|----|--------------|------------------|
| [`bv.geo_velocity`](./geo_velocity.md) | O(2) | `lat`, `lon`, `window` |
| [`bv.geo_distance`](./geo_distance.md) | O(2) | `lat`, `lon`, `window` |
| [`bv.geo_spread`](./geo_spread.md) | O(2) | `lat`, `lon`, `window` |
| [`bv.distance_from_home`](./distance_from_home.md) | BoundedByConfig("samples", 100) | `lat`, `lon` |

Geo ops use haversine distance on lat/lon field pairs (NOT geohashes — `bv.unique_cells` and `bv.geo_entropy` were removed in Phase 19.2 per `project_v2_devex_first` simplification; users compose them via `count_distinct(quadkey)` + `entropy(quadkey)` if needed).

## V0-MEM-GOV-02 compliance

All 11 ops in this subdirectory have a finite memory ceiling at register-time. Three ops use BoundedByRequiredKwarg (kwarg required at decorator/build time): `histogram`, `most_recent_n`, `reservoir_sample`. Two ops use BoundedByConfig (built-in default cap): `event_type_mix` (256 categories), `distance_from_home` (100 samples). Per V0-MEM-GOV-02, register-time enforcement returns structured error code `unbounded_op_in_lifetime_mode` if a required kwarg is missing.
