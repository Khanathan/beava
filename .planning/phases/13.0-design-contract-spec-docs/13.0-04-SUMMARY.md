# Phase 13.0 Plan 04: SDK API specs (4 cross-language docs) Summary

## One-liner

Authored 4 SDK API spec docs — `shared.md` (cross-language parity contract),
`python.md` (canonical authoring UX), `typescript.md` (npm @beava/sdk port spec
with builder pattern + camelCase + Promise-based), `go.md` (Go module port spec
with context-aware methods + functional-options + struct-tag mapping). Q1 Path B
`bv.sum(field: str)` narrowing locked symmetrically across all 3 SDK languages
with the canonical two-stage `with_columns` + `sum` recipe documented inline +
cross-linked to `docs/pipeline-dsl/compilation-rules.md` (forward-ref Plan
13.0-12). Single `docs(13.0-04)` commit per CLAUDE.md TDD §Note 4 doc-only-plan
exemption.

## What landed

### docs/sdk-api/shared.md (293 LOC)

Cross-language semantic parity contract. JSON wire (per `docs/wire-spec.md`) is
the canonical contract; per-language SDKs are thin compilers from idiomatic
syntax to the wire. Sections:

- **Overview** — wire-as-contract + which doc is normative for what.
- **Wire transports** — 3 URL schemes (http/tcp/embed) + URL-scheme dispatch
  pattern across all 3 SDKs.
- **Window grammar** — `\d+(ms|s|m|h|d) | "forever"` with per-language
  client-side rejection rules.
- **Key shape** — single string + composite array of `[string|number|boolean]`
  with FxHash equality.
- **Field types** — 6-element vocab (str / f64 / i64 / bool / bytes / datetime)
  with per-language native types + nullable-marker semantics.
- **FeatureResult shape** — row-shape dict; cold-start returns `{}` (NOT an
  error).
- **ValidationError envelope** — 9 frozen `kind` values with when-thrown table.
- **Schema evolution** — `force` + `dry_run` per-language idiom translation.
- **Cross-language naming convention** — snake_case wire ↔ Python snake_case,
  TS camelCase, Go PascalCase.
- **Error semantics** — per-class error behavior + batch atomicity (no partial
  success in v0).
- **Lifetime aggregation rules** — V0-MEM-GOV-02 register-time validation;
  `unbounded_op_in_lifetime_mode` server-side default-on enforcement.
- **Cross-language API surface map** — 13-row table mapping wire opcode →
  Python / TS / Go method names for quick reference.

### docs/sdk-api/python.md (632 LOC)

v0-target Python SDK (post-13.5 rewrite shape). Module structure walkthrough
(`beava/`, `beava/test/`, `beava/cli/`), `App` class with all 7 wire-mapped
methods (`register` / `push` / `get` / `batch_get` / `reset` / `ping` /
`close`), `@bv.event` class+function form decorator with all kwargs
(`keep_events_for`, `cold_after`, `dedupe_key`, `dedupe_window`),
`@bv.table` function-form decorator per ADR-001, `bv.col` expression DSL with
operator overloading, `bv.lit`, 53-op family table (Core 8 + Sketch 5 +
Point/ordinal 5 + Recency 10 + Decay 6 + Velocity 9 + Bounded-buffer 7 +
Geo 4 = 54), ADR-002 deprecation aliases (bv.avg → bv.mean etc.) with
v0.0.x → v0.1 removal policy, `RegistrationError` / `BinaryNotFoundError`
exceptions, `beava.test` fixtures with pytest integration example.

**Q1 Path B narrowing section** (lines 295-345): `bv.sum(field: str, *, window: str | None = None)`
signature locked; inline expressions like `bv.sum(bv.col("flag").cast(int))`
are FORBIDDEN per Q1 Path B; recommended two-stage pattern documented inline:

```python
events.with_columns(flag_int=bv.col("is_fraud").cast(int)) \
      .group_by("user_id") \
      .agg(c=bv.sum("flag_int", window="1h"))
```

Cross-link to `docs/pipeline-dsl/compilation-rules.md` § Boolean-sum recipe
(forward-ref Plan 13.0-12). The narrowing applies symmetrically across TS + Go
SDK docs (cross-references in both directions).

### docs/sdk-api/typescript.md (598 LOC)

v0-target TypeScript SDK (post-13.6 ship shape). npm `@beava/sdk` module
structure, `BeavaApp` class with Promise-based methods (`register`, `push`,
`get`, `batchGet`, `reset`, `ping`, `close`), `Symbol.asyncDispose` for
`using` / `await using` ergonomics (TS 5.2+ / Node 22+), camelCase method
names (`batchGet`, `withColumns`, `groupBy`, `nUnique`), explicit builder
API (`bv.event(name, schema)` + `bv.table({...})`) since decorators are
deferred to v0.1+, method-chained expression DSL (no operator overloading
in JS — `.gt(100)`, `.and(...)`, `.isnull()`, `.cast("int")`), 53-op family
table in camelCase spelling, int64 caveat (`Number.MAX_SAFE_INTEGER` =
`2^53 - 1`), error class definitions (`RegistrationError`, `BinaryNotFoundError`,
`ValidationError`), `@beava/sdk/test` fixtures with vitest example.

**Q1 Path B section** (typescript.md lines 380-410): `bv.sum(field: string, opts?)`
signature locked; inline expressions are a TypeScript compile error; same
two-stage `withColumns` + `sum` pattern with per-language syntax.

### docs/sdk-api/go.md (609 LOC)

v0-target Go SDK (post-13.6 ship shape). Module `github.com/beava-io/beava-go`
structure, `App` struct with context-aware methods (every public method takes
`ctx context.Context` first), functional-options pattern (`WithForce`,
`WithDryRun`, `WithKey`, `WithUpstream`, `WithAgg`, `AppOption`,
`RegisterOption`), struct-tag field mapping (`beava:"<wire_field_name>"` with
explanation of why the tag is recommended for multi-letter abbreviations like
`UserID`), generic `NewEvent[T]` reflection-based descriptor constructor,
`NewTable(name, opts...)` builder, method-chained expression DSL with
PascalCase methods (`.Gt(100)`, `.And(...)`, `.IsNull()`, `.Cast("int")`),
53-op family table in PascalCase spelling, error types
(`*RegistrationError` with `errors.As`, `*BinaryNotFoundError`), `beavatest`
sub-package with `Fixture(t, opts...)` + `AssertFeaturesEq(t, got, want)`.

**Q1 Path B section** (go.md lines 415-440): `beava.Sum(field string, opts ...SumOption)`
signature locked; inline expressions are a Go compile error (cannot pass an
`Expr` where `string` is required); same two-stage `WithColumns` + `Sum`
pattern with `map[string]beava.Expr{...}` syntax.

## Decisions made

All 4 decisions in `must_haves.truths` and the addresses_decisions list were
honored. Two further decisions were made within autonomy (no user
intervention needed):

1. **TS expression DSL uses method names matching the wire op family**
   (`.gt`, `.ge`, `.lt`, `.le`, `.eq`, `.ne`, `.add`, `.sub`, `.mul`, `.div`,
   `.and`, `.or`, `.not`, `.isnull`, `.cast`, `.alias`) — this is the
   `polars-ts` / `arquero` convention and matches the wire-level expression
   string grammar emitted by the Python `_col.py::_BinOp.to_expr_string()`.
2. **Go SDK ships `beavatest` as a separate sub-package** (`github.com/beava-io/beava-go/beavatest`)
   rather than a `beava.Test` exported field — matches the `cobra` / `gorilla`
   pattern; avoids the test surface leaking into the production import.
