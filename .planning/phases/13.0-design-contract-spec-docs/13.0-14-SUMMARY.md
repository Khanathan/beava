# Phase 13.0 Plan 14: Vertical demos + mock backends + smoke test + quickstart Summary

**One-liner:** 9 vertical demo files (Python + TypeScript + Go × adtech + fraud + ecommerce) runnable against language-local mock backends with computing aggregation logic per BLOCKER 4 fix; CI smoke test fails hard on missing toolchain per WARNING 7; quickstart doc lands in this plan per BLOCKER 1; Go layout Option A (//go:build exclude reference + inline mockApp duplication).

## What Shipped

### 3 mock backends with MINIMAL AGGREGATION LOGIC

`examples/python/_mock.py` (234 LOC), `examples/typescript/_mock.ts` (206 LOC), `examples/go/_mock.go` (232 LOC). All 3 implement the same `register / push / get / batch_get / reset / ping / close` surface (per language idiom: snake_case Python, camelCase TS, PascalCase Go). The `push` method computes `count / sum / mean / min / max` aggregations via the registered descriptors — sketches (`n_unique`, `quantile`, `top_k`), decays (`ewma`, `ewvar`, `ew_zscore`, `decayed_*`), velocity (`rate_of_change`, `trend`, etc.), and geo (`geo_velocity`, `geo_distance`, etc.) are deliberate no-ops. Demos pick coverage-aware ops for assertions and document the no-op fallback inline. Phase 13.5 (Python) + 13.6 (TS+Go) re-verify against real engines as integration regression tests.

The Go `_mock.go` file is special: it carries `//go:build exclude` on the first line (Go's standard exclude-from-build idiom) so it does not participate in any binary build during Phase 13.0. It is reference code only for Phase 13.6 SDK porters. Each Go demo file inlines its own `mockApp` helper struct (~30 LOC) so `go run examples/go/<demo>.go` works zero-config without needing a multi-file package.

### 9 vertical demo files

`examples/{python,typescript,go}/{adtech,fraud,ecommerce}.{py,ts,go}` — all 9 runnable; all 9 use the register → push → query flow (NO `_seed` pre-population per BLOCKER 4 fix); all 9 have computed-value assertions.

- **adtech**: campaign-axis aggregations (impressions count + bid sum/mean/min/max). 5 events pushed across 2 campaigns. Demonstrates the simplest end-to-end shape.
- **fraud**: mirrors `crates/beava-bench/configs/fraud-team.json` shape (5 event types: Txn/Login/Signup/CardAdd/Refund + user_id key). 8 ops registered: 5 mock-supported (count/sum/mean/min/max) + 3 declared-but-no-op (n_unique, quantile, geo_velocity). 10 transactions pushed for `alice`. Documents which ops fall back to no-op so Phase 13.5/13.6 re-verification is unambiguous.
- **ecommerce**: purchase-basket aggregations (purchase_count + items_purchased + total_spend + mean_purchase_value + min/max price). 4 purchases pushed for `bob`.

All 3 fraud demos use the new ADR-002 Polars-convention op names (`mean / n_unique / quantile`) — none use the deprecated `avg / count_distinct / percentile`.

### CI smoke runner with hard-fail on missing toolchain (WARNING 7 fix)

`examples/test_examples.sh` (74 LOC, executable). Pre-flight checks all 3 toolchains (python3 + npx + go). If any toolchain is missing, exits 2 with a clear "install + retry" message. If all toolchains present, runs all 9 demos and exits 0 on clean, 1 on any failure. Plan 13.0-15 closure re-runs this script as the final D-03 gate.
