    app.register(<Event>, <FeatureTable>)

    rng = random.Random(<test-specific-seed>)

## Key design decisions captured

### D-01 — Cold-start equivalence helper

Tests use `cold_start_equivalent(value)` which accepts `{} OR None`. v0 transports surface QueryNotFound differently (embed/TCP-msgpack returns `{}`, HTTP/JSON may return `None`). Locking on one shape would couple tests to a transport. The helper preserves the cold-start contract while staying transport-agnostic — a Phase 13.5 SDK rewrite that picks one canonical shape can tighten the helper later.

### D-02 — Per-test seeded `random.Random(N)`

Per-test seed varies (42-144 across the 68 tests) so a single-test failure during execution doesn't cascade to sibling tests via shared RNG state. Reproducibility is bit-identical given the seed.

### D-03 — Time-controlled ops use coarse wall-time bounds

`age`, `time_since`, `time_since_last_n`, `first_seen`, `last_seen` capture `int(time.time() * 1000)` at push-time AND query-time, then assert `actual ∈ [test_start_ms - 50, test_end_ms + 200]`. The 50/200 ms slack absorbs apply-loop scheduling jitter on the embed-mode subprocess. FakeClock injection is not in the v0 engine API surface; coarse bounds are sufficient given sub-second test windows.

### D-04 — Sketch tolerance helpers

`assert_sketch_within_tolerance(actual, expected, rel=, abs_=)` accepts EITHER a relative (e.g. 5% for HLL) OR absolute (e.g. ±2 percentile points for DDSketch) tolerance and passes if either is met. `assert_top_k_order_preserved(actual, expected_ranking)` asserts (a) the top-1 item is correctly ranked first and (b) every expected member is present in the actual list (set inclusion). SpaceSaving may perturb tail order; this matches the operator's documented contract (top is sticky, tail less so).

### D-05 — `bv.lit` value-type test merges 5 types into one filter chain

`test_lit_value_types` pushes 500 events through ONE filter using `int / float / str / bool / None` lits, then asserts the wire payload textually references each lit form. Compact equivalent coverage vs 5 separate single-type tests.

### D-06 — `bv.lit(42) is bv.lit(42) → False` (immutability)

`test_lit_immutability` asserts `id(a) != id(b)` for two `bv.lit(42)` calls AND that they `to_expr_string() == "42"` identically. Catches the regression where a future SDK author might cache literals into a singleton table — which would break wire-payload determinism.
