## Deviations from Plan

**1. [Rule 1 - Bug] RED via main() reference, not #[test]**
- **Found during:** Task 1.a
- **Issue:** In `harness = false` bench files, `#[test]` items outside `#[cfg(test)]` are not compiled in the bench binary. The plan's `#[test] fn bench_has_nontrivial_output()` referencing `phase25_wire_benches` at top-level would not trigger a compile error.
- **Fix:** Placed the `phase25_wire_benches::CRITERION_GROUP_COUNT` reference inside `fn main()` instead, which IS compiled in the bench binary — guaranteeing the compile-fail RED signal.
- **Files modified:** `crates/beava-server/benches/phase25_wire.rs`
- **Commit:** `983b6b0`

**2. [Rule 2 - Pattern] Explicit imports in #[cfg(test)] mod tests**
- **Found during:** Task 1.b (clippy gate)
- **Issue:** `use super::*` inside `#[cfg(test)] mod tests` in a `harness = false` bench triggers `unused_imports` under clippy `-D warnings` because the bench binary's non-test compilation path can't see the `#[test]` consumers.
- **Fix:** Used `#[allow(unused_imports)]` on the glob import and added explicit per-function imports in each test body.
- **Files modified:** `crates/beava-server/benches/phase25_wire.rs`
- **Commit:** `b575ac4`
