## Decisions Made

1. **Module-level descriptors required** â€” `@bv.event` function-form reads `param.annotation` directly at decoration time via `inspect.signature`. When descriptors are defined inside a nested function with `from __future__ import annotations`, Python stores the annotation as a string (`'Tx'`) rather than the live object. Moving all 10 descriptors to module scope (with no deferred annotations in the file) resolves this. This matches the existing pattern in `test_phase3_smoke.py`.

2. **10-descriptor DAG composition** â€” 3 `@bv.event` class sources (`BenchTx`, `BenchLogin`, `BenchPageView`), 3 `@bv.event` function derivations chained on `BenchTx` (filter â†’ with_columns â†’ select), 2 `@bv.table(key=...)` class sources (`BenchUserBaseline`, `BenchUserFlags`), 1 `@bv.table` function derivation (`bench_user_filtered`), 1 standalone `@bv.table` class source (`BenchLoginHistory`). All unique names with `Bench` prefix to avoid collisions with other test modules.

3. **bench marker** â€” Added to `[tool.pytest.ini_options].markers` so `--strict-markers` doesn't error on collection. Benches are not skipped by default; developers run them via `pytest -m bench` or `--benchmark-only`.
