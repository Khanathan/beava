  ### 2. `python/beava/_client.py` — TallyClient.negotiate_wire_format

  Add method to `TallyClient`:

  ```python
  def negotiate_wire_format(self) -> tuple[int, int]:
      """Phase 59 D-B1 (TPC-PERF-09): handshake with server.

      Sends OP_NEGOTIATE_WIRE_FORMAT with the client's capability bits +
      version tag; receives server's bits + version. Caches the result in
      ``self.server_capability_bits`` (int) and ``self.server_version_tag``
      (int) for the connection lifetime.

      Returns ``(server_bits, server_version)``. On a pre-Phase-59 server
      that rejects 0x18 as unknown opcode, returns ``(0, 0)`` sentinel —
      indicating the client should fall back to "emit binary without
      handshake" (D-E4). Safe to ignore the return value; the cache is
      the primary consumer.
      """
      import struct
      from beava._protocol import (
          OP_NEGOTIATE_WIRE_FORMAT, WIRE_BINARY_PASSTHROUGH,
          WIRE_VERSION_TAG_CLIENT,
      )
      payload = struct.pack(">IH", WIRE_BINARY_PASSTHROUGH, WIRE_VERSION_TAG_CLIENT)
      status, body = self.send_command(OP_NEGOTIATE_WIRE_FORMAT, payload)
      if status != 0x00:
          # Pre-Phase-59 server: silently fall back. Per D-E4.
          self.server_capability_bits = 0
          self.server_version_tag = 0
          return (0, 0)
      if len(body) < 6:
          # Malformed response from a broken server — don't crash; fall
          # back same as pre-59 server case.
          self.server_capability_bits = 0
          self.server_version_tag = 0
          return (0, 0)
      server_bits = struct.unpack(">I", body[:4])[0]
      server_version = struct.unpack(">H", body[4:6])[0]
      self.server_capability_bits = server_bits
      self.server_version_tag = server_version
      return (server_bits, server_version)
  ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python SDK→server OP_NEGOTIATE | Python sends u32 bits + u16 version; server echoes supported bits. No auth crossing. |
| pre-59-server→Python SDK fallback | Server returns STATUS_ERROR on unknown opcode; Python caches `(0, 0)` sentinel. No data tampering possible — cache is local to Python process. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-03-01 | Spoofing | Pre-59 server pretending to be Phase 59 by sending bogus `server_bits` | accept | Python's server_capability_bits is advisory only — actual wire format (binary or JSON) is determined by what Python emits (always binary post-Phase-11). Spoofed bits cause no misdispatch. |
| T-59-03-02 | Denial-of-service | BEAVA_WIRE_NEGOTIATE=1 adds 1 round-trip per connection (latency cost) | accept | 1 RTT per connection at connection-establishment time — amortized over thousands of pushes. Acceptable. |
| T-59-03-03 | Information disclosure | Python's `server_capability_bits` attribute accessible to user code | accept | Non-secret; matches public-facing server version. |
| T-59-03-04 | Downgrade | MitM forces handshake failure → Python falls back to no-negotiate mode → server still accepts binary anyway | accept | Downgrade attack is out of scope per D-E2 (CONTEXT); TLS is operator concern. |
</threat_model>
