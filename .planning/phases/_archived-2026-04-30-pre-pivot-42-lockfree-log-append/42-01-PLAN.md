# Plan 42-01 — Lock-free event-log append via O_APPEND

## Goal

Remove the per-stream writer Mutex from the event-log append path. Use Linux `O_APPEND` atomic `write()` instead. No userspace lock on the hot path. Unlocks 1-stream concurrent write throughput from ~540k to (predicted) 2-3M eps.

## Task T1 — `LockFreeStreamLog` type

**Files:** `src/state/event_log.rs` (mainly), possibly `Cargo.toml` (if `libc` not already direct dep).

**Subtasks:**
- Define `LockFreeStreamLog { fd: OwnedFd, stream_name: String, partial_write_lock: PLMutex<()> }`.
- Constructor: takes `path: &Path`, opens via `File::options().create(true).write(true).append(true).open()`, converts to OwnedFd.
- `fn append_raw(&self, bytes: &[u8]) -> io::Result<()>`: single `libc::write` call, handles EINTR loop, falls back to `append_raw_partial_fallback` on partial write.
- `fn append_raw_partial_fallback(&self, bytes: &[u8], already_written: usize)`: takes `partial_write_lock`, loops `libc::write` until all bytes written.
- `fn fsync(&self) -> io::Result<()>`: `libc::fdatasync(fd)`.
- Unit test `parallel_appends_do_not_tear_frames`:
  - Spawn 8 threads, each appends 10,000 length-prefixed frames to the same file.
  - Join, open file, decode via length prefix loop, verify 80,000 frames, each structurally intact.
  - Use `std::thread::Barrier` to maximize concurrency window.
- Unit test `partial_write_fallback_preserves_order`:
  - Mock a writer that returns partial write on first call.
  - Verify the fallback completes the write without corruption.
  - (Can use a small test file + shimmed fd to simulate; or skip if too intricate and rely on the parallel test.)

**Done when:** both unit tests pass; `cargo build` green under default + client feature.

## Task T2 — Swap into `EventLog`

**Files:** `src/state/event_log.rs`, possibly `src/server/replica.rs` + any callers using `fsync_all`.

**Subtasks:**
- Change `EventLog.writers` field type: `DashMap<String, LockFreeStreamLog>`.
- `EventLog::register_stream`: now constructs `LockFreeStreamLog`, no BufWriter.
- `EventLog::append`:
  - DashMap lookup.
  - Encode frame into stack-allocated or small heap Vec<u8>.
  - Call `log.append_raw(&frame)`.
- `EventLog::append_many`:
  - DashMap lookup.
  - Concatenate all frames into one Vec<u8>.
  - Size check: `debug_assert!(buf.len() < 1_048_576)` — 1 MB sanity.
  - Call `log.append_raw(&buf)` ONCE (batch-atomic).
- `EventLog::fsync_all`: iterate DashMap, call `log.fsync()` on each.
- `EventLog::deregister_stream`: drop the entry (closing fd via Drop of OwnedFd).
- Remove the old BufWriter-based code paths entirely.
- Existing internal tests adapt to new API — should be minor since the public surface is unchanged.

**Done when:** `cargo test -p tally --lib state::event_log` green; existing integration tests that write-then-read the log still green (Phase 35 LOG_FETCH path, Phase 20 replay_30d integration).

## Task T3 — Benchmarks + validation

**Files:** no code changes; run benchmarks.

**Subtasks:**
1. Full `cargo test` green.
2. `scripts/check-feature-builds.sh` green (default + client + demo).
3. `cargo test -p tally --test '*'` (integration tests) green.
4. Start server with `TALLY_WORKER_THREADS=8`, pinned to CPUs 0-7.
5. Run three bench scenarios and record numbers in SUMMARY:
   - Single-stream single-client `push_many` (baseline: 553k pre-42). Expect: similar or slight gain.
   - **8-proc 1-stream `push_many` (baseline: 540k pre-42). Expect: > 1.5M, ideally 2-3M.**
   - 8-proc 8-stream `push_many` (baseline: 676k pre-42 via single-event). Expect: > 1.5M scaled with per-process count.
6. Optional: strace -c during the 8-proc bench. Expect dramatic drop in `futex` count (Mutex gone). May need sudo which was available this session; if not, skip.

**Done when:** numbers documented in `42-01-SUMMARY.md` alongside before/after tables.

## Stop-and-report triggers

- Existing `BufWriter`-based test suite assumes flush semantics that don't hold under direct `write()` — flag and adapt.
- 8-proc 1-stream bench lands between 540k and 1.5M — partial win, profile to find new bottleneck (probably cascade or DashMap contention on hot keys).
- 8-proc 1-stream bench drops below 540k (regression) — rollback and investigate.
- Partial-write fallback is untestable without unsafe mocking — ship without the dedicated test; the parallel-append test covers the common case.
- `libc` not trivially addable as dep — evaluate `nix` crate or `std::os::fd` alternatives.
