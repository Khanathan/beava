# Plan 36-01 — `tally serve --replica-from` boot mode

## Goal

Server boots in replica mode: LOG_FETCH historical → SUBSCRIBE live, all events flow through the local ingest path. Scientist registers their own pipelines (via `--replica-pipeline-file` at boot or HTTP POST after listeners open) and queries via normal `tl.Client`.

## Task T1 — CLI flag parsing

**Files:** `src/main.rs` (or wherever flags are parsed).

**Subtasks:**
- Add flags per CONTEXT: `--replica-from`, `--replica-since`, `--replica-streams`, `--replica-keys`, `--replica-key-prefix`, `--replica-token`, `--replica-block-until-catchup` (default true), `--replica-pipeline-file`.
- Parse `--replica-since` as either ISO-8601 UTC or raw `u64` milliseconds. Reject ambiguous formats loudly.
- Bundle into a `ReplicaBootConfig` struct.
- If `--replica-from` absent, legacy server behavior unchanged. If present, all other `--replica-*` flags required EXCEPT `--replica-keys`/`--replica-key-prefix` (both optional) and `--replica-pipeline-file`.
- Unit tests: flag parsing happy + error paths (missing required, ambiguous --replica-since, token from env var).

**Done when:** `cargo test --bin tally` new flag-parsing tests green.

## Task T2 — Replica client loop

**Files:** `src/server/replica_client.rs` (new) + `src/client/session.rs` (extend if helper needs broadening).

**Subtasks:**
- `pub struct ReplicaClient { config: ReplicaBootConfig, app: Arc<AppState>, catchup_done_tx: tokio::sync::oneshot::Sender<()> }`.
- `async fn run(self) -> Result<(), ReplicaError>`:
  1. Connect TCP to `config.remote`. Reuse Phase 28's handshake + framing code.
  2. Send OP_LOG_FETCH: `[u16 token_len][token][u64 from_ts_millis][scope_bytes]`. from_ts_millis = config.since_millis.
  3. Loop reading response frames:
     - On tag=EVENT (0x03): decode `(timestamp_ms, payload)`; call `ingest::replica_ingest(&self.app, stream_name, key, timestamp_ms, payload)` (new entry point — see T3).
     - On tag=END (0x04): break catchup loop; fire `catchup_done_tx` to unblock listener startup.
  4. Close LOG_FETCH connection. Open SUBSCRIBE connection per Phase 27/31-01 pattern: `[u16 token_len][token][scope_bytes]` + receive EVENT frames.
  5. Same ingest call per event. Track `last_applied_ts_ms` as an atomic on AppState.
  6. On SUBSCRIBE EOF: reconnect with LOG_FETCH{from_ts = last_applied_ts}. Retry with backoff per CONTEXT policy. After 10 consecutive failures within 1 min → log fatal + shutdown the server process cleanly.
- `ReplicaError` enum: ConnectFailed, HandshakeFailed, MalformedFrame, IngestFailed, RetryExhausted.
- Unit tests for the state machine (mock TCP streams, assert reconnect counts / catchup-done fires / ingest calls happen).

**Done when:** `cargo test -p tally --lib replica_client` green.

## Task T3 — Ingest path + listener gate wiring

**Files:** `src/engine/pipeline.rs` (expose `replica_ingest`), `src/server/app.rs` (wire replica client + gate), `src/server/tcp.rs` + `src/server/http.rs` (gate on catchup-done signal).

**Subtasks:**
- Factor out the "pushed-event-to-local-log-and-run-pipelines" path into an entry point `replica_ingest(app: &AppState, stream: &str, key: &str, ts_ms: u64, payload: &[u8])`. Reuse what `push_with_cascade_internal` already does for a locally-originated PUSH, minus:
  - No admin-auth check (replica events are pre-authenticated upstream).
  - No `notify_subscribers` call (prevents feedback loop; local scientists don't need to SUBSCRIBE to replicated events in v0 — document).
  - No `tally_events_accepted_total` bump (these aren't locally-accepted; add `tally_replica_events_ingested_total{stream}` instead).
- In `main.rs` boot path: if `ReplicaBootConfig` present:
  1. Build `AppState` as normal.
  2. If `--replica-pipeline-file` given, parse + register pipelines BEFORE spawning replica client.
  3. Spawn `ReplicaClient::run` as a tokio task; get `catchup_done_rx` handle.
  4. If `--replica-block-until-catchup=true`: `rx.await?` before starting listeners.
  5. Start TCP + HTTP listeners normally.
  6. On replica-client fatal error, log + graceful shutdown.
- Local PUSH handler: if `AppState.replica_config.is_some()`, reject PUSH with `TallyError::Protocol("replica mode: local PUSH disabled")`.

**Done when:** Feature flag smoke still works; no regressions in existing `cargo test`.

## Task T4 — Integration test

**Files:** `tests/integration/test_replica_mode.py` (new Python).

**Flow:**
1. Start a prod Tally server subprocess on port A with seed events for stream Transactions (push 20 events across two keys, u1/u2, spread over timestamps T0..T10).
2. Start a replica Tally server subprocess on port B with:
   - `--replica-from 127.0.0.1:A`
   - `--replica-since T0`
   - `--replica-streams Transactions`
   - `--replica-keys u1,u2`
   - `--replica-token ADMIN`
   - `--replica-pipeline-file test-pipeline.json` (a simple `@group_by("user").count()` aggregate REGISTER payload)
   - `--replica-block-until-catchup=true`
3. Wait for replica to report catchup-done (look for log line OR poll `/debug/ready` if that exists).
4. Open `tl.Client(remote="localhost:B")` and `client.get("u1", stream="Transactions")` — expect the count aggregate for u1 (10 events).
5. Push 5 more events to prod, wait 2s.
6. Re-query replica: count should be 15. Verifies live tail.
7. Attempt local PUSH against replica → expect rejection.

**Done when:** pytest green; replica logs show "replica caught up to {ts}; opening listeners" and "live tail active".
