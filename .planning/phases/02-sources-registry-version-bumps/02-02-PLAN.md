```rust
// op_node.rs
use serde::{Deserialize, Serialize};
use std::collections::BTreeMap;

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[serde(tag = "op", rename_all = "snake_case")]
pub enum OpNode {
    Filter      { expr: String },                                                       // Phase 4 evaluates
    Select      { fields: Vec<String> },
    Drop        { fields: Vec<String> },
    Rename      { mapping: BTreeMap<String, String> },
    WithColumns { exprs:   BTreeMap<String, String> },
    Map         { exprs:   BTreeMap<String, String> },                                  // alias for WithColumns
    Cast        { type_map: BTreeMap<String, String> },
    Fillna      { defaults: BTreeMap<String, serde_json::Value> },
    GroupBy     { keys: Vec<String>, agg: BTreeMap<String, AggSpec> },                  // Phase 5 executes
    Join        { other: String, on: Vec<String>, #[serde(default)] within_ms: Option<u64>, join_type: JoinType },  // Phase 12
    Union       { others: Vec<String> },                                                 // Phase 12
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct AggSpec {
    pub op:     String,              // "count", "sum", "ewma", "geo_velocity", ... — Phase 5 validates against catalogue
    #[serde(default)]
    pub params: serde_json::Value,   // operator-specific params; Phase 2 treats as opaque
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum JoinType { Inner, Left }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client JSON → OpNode | Untrusted `"op"` discriminator controls enum variant selection; malformed input must error cleanly, never UB |
| Client JSON → AggSpec.params | Opaque JSON value flows into the Registry and will be consumed by Phase 5 — Phase 2 must NOT execute or parse it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | OpNode tag-based deserialization | mitigate | Serde's internally-tagged enum errors on unknown variant; Test 7 asserts Err message lists valid variants. No panic path. |
| T-02-02-02 | DoS | Unbounded AggSpec.params payload | accept | Phase 2 stores params as opaque Value; size limit is enforced at the HTTP body-parsing layer (Plan 05 sets `axum::extract::DefaultBodyLimit`). Phase 2 core crate does not bound this. |
| T-02-02-03 | Information disclosure | Filter `expr` stored verbatim | accept | Expr is user-authored SQL-like code; storage in plaintext is intentional for debugging. Phase 4 parser rejects malformed expressions at evaluation time. |
| T-02-02-04 | Tampering | Join.within_ms unbounded u64 | mitigate | Phase 4 validation (Plan 04) rejects `within_ms > 7 days` with a `{code, path, reason}` error. Phase 2 core type accepts any u64 — validation is the enforcement layer. |
</threat_model>

## For Plan 02-03 (diff engine)
- Use derived `PartialEq` on `OpNode` to compare ops lists within `DerivationDescriptor::equiv_ignoring_version`. BTreeMap ordering makes this deterministic.
- Note: `AggSpec` is `PartialEq` but not `Eq` due to `params: Value`. Your diff engine's public type `RegistryDiff` should hold `String` names + `Vec<ConflictDetail>` — it does NOT need `Eq` on descriptor types.

## For Plan 02-04 (validation)
- When validating derivation ops later (Phase 4+), walk `d.ops` and match on `OpNode` variants. Phase 2 validation (this phase's Plan 04) only checks:
  - Each derivation's upstream names resolve (which doesn't inspect ops at all)
  - The derivation schema is non-empty
  - NO op-level validation (deferred to Phase 4/5)

## For Plan 02-05 (endpoint)
- The HTTP handler receives the outer JSON `{"nodes": [...]}` where each node has a `"kind"` discriminator (event/table/derivation). Plan 05 implements a custom deserializer or a two-pass parse:
  1. First pass: parse into `enum PayloadNode { Event(EventInput), Table(TableInput), Derivation(DerivationInput) }` using `#[serde(tag = "kind", rename_all = "snake_case")]`.
  2. Validate, then map to the corresponding descriptor types by adding `registered_at_version` = target version (computed post-validation).
- Unknown `kind` values → 400 with path `nodes[i].kind` per CONTEXT.md §Error messages.

## For Phase 3 (Python SDK compiler)
- Python SDK's `to_expr_string()` output (SDK-COL-06) goes verbatim into `OpNode::Filter.expr`. No escaping rules needed beyond standard JSON string escaping.
- Python SDK's `group_by(...).agg(...)` must emit `OpNode::GroupBy` with the AggSpec shape described here. Per-operator params go in `params: Value` — Phase 3 SDK is responsible for shaping per-operator params to match Phase 5's expectations.

## Stability notes
- The 11 OpNode variants are locked for v0. New operators (e.g., Phase 5's aggregation variants) are represented inside `AggSpec.op` string, NOT as new OpNode variants — keeps the transformation DAG flat.
- Within a variant, new optional fields can be added without breaking older clients (serde default), but removing/renaming fields is a breaking change that requires a new variant.
</handoff>
