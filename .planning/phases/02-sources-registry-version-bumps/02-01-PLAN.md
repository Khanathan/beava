```rust
// registry.rs
pub enum OutputKind { Event, Table }
pub enum TableMode { Append }

pub struct EventDescriptor {
    pub name:                 String,
    pub schema:               EventSchema,
    pub event_time_field:     String,
    pub idempotency_key:      Option<String>,
    pub idempotency_ttl_ms:   Option<u64>,
    pub history_ttl_ms:       Option<u64>,
    pub watermark_lateness_ms: Option<u64>,
    pub registered_at_version: u64,
}

pub struct TableDescriptor {
    pub name:                  String,
    pub primary_key:           Vec<String>,
    pub schema:                TableSchema,
    pub ttl_ms:                Option<u64>,
    pub mode:                  TableMode,
    pub registered_at_version: u64,
}

pub struct DerivationDescriptor {
    pub name:                  String,
    pub output_kind:           OutputKind,
    pub upstreams:             Vec<String>,
    pub ops:                   Vec<serde_json::Value>,   // placeholder — Plan 02 replaces with OpNode
    pub schema:                DerivedSchema,
    pub table_primary_key:     Option<Vec<String>>,
    pub registered_at_version: u64,
}

pub struct RegistryInner {
    pub version:     u64,
    pub events:      std::collections::BTreeMap<String, EventDescriptor>,
    pub tables:      std::collections::BTreeMap<String, TableDescriptor>,
    pub derivations: std::collections::BTreeMap<String, DerivationDescriptor>,
}

pub struct Registry { inner: parking_lot::RwLock<RegistryInner> }
```

Serde tags: `FieldType` uses `#[serde(rename_all = "snake_case")]` with token list `str | f64 | i64 | bool | bytes | datetime`. `OutputKind` uses the same. `TableMode` uses the same (only `append` valid in v0).

JSON shape for descriptors matches 02-CONTEXT.md §Registration payload shape exactly — NO `registered_at_version` in input JSON; it is assigned server-side. Mark with `#[serde(default)]` (defaults to 0 on deserialize; server overwrites on insert).

- The `DerivationDescriptor.ops` field is currently `Vec<serde_json::Value>`. Plan 02 swaps it for `Vec<OpNode>` by changing the type and re-running tests. The JSON wire format is unchanged (each op is a JSON object; Plan 02 adds strict variant parsing).

- Use `EventDescriptor::equiv_ignoring_version` (and the two sibling methods) for descriptor equality. Do NOT use derived `PartialEq` — it includes `registered_at_version` and would produce false conflicts.

## For Plan 02-04 (validation)
- Use `validate_descriptor_name` for name pattern + reserved-prefix checks. Import from `beava_core::schema`.
- Use `FieldType::I64` to check `event_time_field` type. `EventSchema.fields: BTreeMap<String, FieldType>` gives O(log n) lookup.

## For Plan 02-05 (endpoint)
- Use `Registry::snapshot()` for read-heavy pre-diff inspection (clones RegistryInner, drops the lock immediately — no lock-held-across-await hazard).
- Use `Registry::read()` for cheap version probes.
- The pub(crate) `install_descriptors` helper is NOT directly callable from `beava-server`. Plan 05 will either:
  - (a) export an `apply_registration(&self, new_version, events, tables, derivations)` method on Registry (recommended), OR
  - (b) re-structure to put the endpoint handler in `beava-core`.
  Go with (a): add `pub fn apply_registration(...)` in Plan 05 calling the existing helper.

## For Phase 3 (Python SDK compiler)
- The JSON DAG shape emitted by Python decorators MUST deserialize into these descriptor structs byte-compatibly. The structs are locked from this plan forward. If Phase 3 needs a shape change, open a discussion — do not silently diverge.
- `registered_at_version` is a server-assigned field; SDK MUST NOT emit it (extra field with `#[serde(default)]` so it's ignored if present — but SDK should not send it).

## Stability notes
- FieldType variants: frozen for v0. New types (e.g., Decimal, UUID) are post-v0 additions.
- TableMode::Append is v0-only. v0.1 may add Changelog; the enum is designed with room for that.
- `Vec<String>` for derivations.upstreams is intentional — preserves input order for SDK tests and topological-order validation in Plan 04.
