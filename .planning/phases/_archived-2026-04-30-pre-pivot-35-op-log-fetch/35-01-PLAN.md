# Plan 35-01 — OP_LOG_FETCH{from_ts_millis, scope}

## Goal

Ship the third replica opcode. A scoped client can pull all historical events from time `from_ts_millis` forward (per-stream order, at-least-once at boundary timestamps). Primitive for the Option M data-scientist fork flow.

## Task T1 — Protocol additions

**Files:** `src/server/protocol.rs` (modified), `src/client/wire.rs` (modified — keep duplicate in sync).

**Subtasks:**
- Add const `OP_LOG_FETCH: u8 = 0x13`.
- Add const `REPLICA_FRAME_TAG_END: u8 = 0x04` (terminal frame for the event stream).
- Add `Command::LogFetch { admin_token: String, from_ts_millis: u64, scope: Scope }` variant.
- Extend `parse_command` with an `OP_LOG_FETCH` arm: read `[u16 token_len][token][u64 from_ts_millis][scope_bytes]`.
- Mirror the const + payload shape in `src/client/wire.rs` so future client code can build requests (even though we're not shipping a Rust client for it in this phase).
- Unit tests: round-trip encode→parse for Command::LogFetch; unknown-opcode negative test still works; invalid from_ts (e.g., oversized not possible since u64, but truncated buffer) rejects cleanly.

**Done when:** `cargo test -p tally --lib protocol` adds ≥4 new tests all green; feature-build smoke script still OK.

## Task T2 — Handler + per-stream log iteration

**Files:** `src/server/replica.rs` (add `handle_log_fetch`), `src/server/tcp.rs` (dispatch), possibly `src/state/event_log.rs` (expose an iterator helper if one doesn't exist).

**Subtasks:**
- In `src/server/replica.rs`, add:
  ```rust
  pub async fn handle_log_fetch(
      state: &AppState,
      write: &mut OwnedWriteHalf,
      cmd: Command::LogFetch,
  ) -> Result<(), TallyError>
  ```
- Validate admin token (match 27-01 pattern exactly — gate on `state.admin_token.as_deref() == Some(token)`).
- Call `validate_scope(&state.streams, &cmd.scope)` — reuse Phase 27's validator.
- For each stream in `cmd.scope.streams`:
  - Open/iterate Phase 6's per-stream log file for that stream. Read each `LogEntry { timestamp, payload }`.
  - Gate on `entry.timestamp.as_millis() >= cmd.from_ts_millis`.
  - Gate on `entity_matches_scope(key_from_payload, &[stream_name], &cmd.scope)` — reuse Phase 27's predicate. Need to extract `key` from payload (event JSON contains it — reuse whatever key-extraction 27-02's `notify_subscribers` does).
  - Write a `REPLICA_FRAME_TAG_EVENT` frame: `[tag=0x03][u32 body_len][u64 timestamp_ms][u32 payload_len][payload_bytes]`.
  - Increment `tally_replica_log_entries_sent_total{stream}` counter.
- After all streams exhausted, write a `REPLICA_FRAME_TAG_END` frame (tag 0x04, body len 0).
- Flush and return.
- Auth failure: emit `emit_replica_auth_failure` signal (reuse 27-02 helper), return `TallyError::Protocol("unauthorized")`.

**Concerns to flag to the user if hit:**
- If the event log doesn't expose a per-stream iterator cheaply, adding one is in scope — look at how Phase 6 writes entries and mirror the read side.
- If the event payload doesn't carry `key` as a top-level field, need to surface the extraction — mirror 27-02's approach.

**Done when:** handler compiles, dispatch routes OP_LOG_FETCH to it, counter declared + incremented.

## Task T3 — Integration tests

**Files:** `tests/test_replica_log_fetch.rs` (new Rust integration test), `tests/integration/test_replica_log_fetch_asyncio.py` (new Python asyncio test).

**Rust integration (4 cases):**
1. Happy path: push 10 events to a stream across two timestamps; LOG_FETCH with `from_ts_millis=T0` returns all 10. With `from_ts_millis=T0+500ms` returns only the ones ≥ that boundary.
2. Scope filter: push events to streams A and B; LOG_FETCH with scope=streams=[A] returns only A's.
3. Key filter: push events with keys u1, u2, u3; LOG_FETCH with scope.keys=[u1,u2] returns only those.
4. Auth reject: no token → STATUS_ERROR before any data frames.

**Python asyncio (2 cases):**
1. Wire contract: send LOG_FETCH request, parse response frames (event + END), verify frame count + timestamps.
2. Scope isolation: two clients with disjoint scopes see disjoint events from a shared log.

**Done when:** `cargo test` stays green, `pytest tests/integration/test_replica_log_fetch_asyncio.py` green.

## Stop-and-report triggers

- Per-stream log read API doesn't exist and adding it is invasive (>100 lines).
- Key extraction from payload is ambiguous (payload schema varies per stream).
- Timestamp ordering within a stream log isn't actually monotonic (existing Phase 6 semantics) — may need a different cursor.
