# PLAN 25-03: TTL defaults + reinit bloom + config-recommendations + tally CLI

## Goal

Close out Phase 25 by:
1. Honoring per-Table `ttl` and per-Stream `history_ttl` kwargs in the engine with 30d / 90d defaults.
2. Tracking recently-evicted keys via a per-Table double-buffered bloom filter so the `tally_ttl_eviction_then_reinit_total` counter exists and is accurate.
3. Building the suggestion engine + `/debug/config-recommendations` endpoint + `tally suggest-config` CLI.
4. Wiring recommendations into the SignalRegistry (from plan 25-02) so `/debug/warnings` surfaces them under category=`config`.

This closes success criteria 5, 7, and 8 from the ROADMAP Phase 25 entry.

## Success Criteria

1. `@tl.table(ttl='60d')` evicts at 60d in engine tests; omitting `ttl` gets 30d default.
2. `@tl.stream(history_ttl='180d')` retains 180d event log; omitting gets 90d default.
3. `ttl='forever'` disables eviction for that Table (tested via 10000d worth of simulated clock).
4. A key evicted from Table T and then PUSH'd again increments `tally_ttl_eviction_then_reinit_total{table=T}` once per reinit.
5. Bloom filter memory per Table ≤ 2MB (configured for 1M evictions over 7d at ≤1% FP). Double-buffered — older generation replaced every 24h.
6. Measured FP rate ≤ 1.5% over a synthetic workload of 100k-evict-then-100k-unrelated-PUSH (test).
7. `GET /debug/config-recommendations` returns HTTP 200 with:
   ```json
   {"recommendations": [{"knob":"UserProfile.ttl","current":"30d","suggested":"60d","confidence":0.72,"reason":"...","evidence":{...},"copy_paste":"@tl.table(key=\"user_id\", ttl=\"60d\")"}]}
   ```
8. Three recommendation rules fire correctly:
   - **R1**: Table reinit-rate > 5% over 24h → recommend doubling ttl, confidence = min(reinit_rate × 10, 0.9).
   - **R2**: For each Stream with downstream Table(s), if `history_ttl < max(downstream.ttl)`, recommend `history_ttl = max(downstream.ttl)`, confidence = 1.0.
   - **R3**: Tombstone grace-expired-read rate > 10/hr → recommend doubling `tombstone_grace`.
9. Every recommendation is ALSO recorded into SignalRegistry as `Category::Config` / `Severity::Info` signal, making it visible at `/debug/warnings?category=config`.
10. `tally suggest-config` CLI:
    - Exits 0 on success.
    - Output groups recs by decorator target (one heading per Table / Stream).
    - Prints one copy-paste line per rec.
    - Empty recs → prints "No configuration recommendations at this time."
    - Connects to `localhost:6401` by default; `--host` and `--port` flags override.
11. Startup log emits one advisory line if recs exist at boot: `advisory: N config recommendations available; run 'tally suggest-config'`.
12. Installing the Python SDK provides a `tally` command on PATH (`pyproject.toml` entry point).

6. In `src/state/store.rs`, add:
   ```rust
   pub struct EvictionTracker {
       // Double-buffered: current (filling) + previous (aging out after 24h)
       current: Bloom<String>,
       previous: Bloom<String>,
       rotated_at: SystemTime,
       pub reinit_count: AtomicU64,  // exposed as tally_ttl_eviction_then_reinit_total{table}
   }
   impl EvictionTracker {
       pub fn new_for_table() -> Self { /* 1M items, 0.01 FP → ~1.2MB per bloom */ }
       pub fn insert(&mut self, key: &str) { self.current.set(&key.to_string()); }
       pub fn contains(&self, key: &str) -> bool { self.current.check(&...) || self.previous.check(&...) }
       pub fn maybe_rotate(&mut self, now: SystemTime) { if now.duration_since(self.rotated_at) > Duration::from_secs(86400) { self.previous = std::mem::replace(&mut self.current, Bloom::new_for_fp_rate(1_000_000, 0.01)); self.rotated_at = now; } }
   }
   ```

    ```python
    import argparse, sys, json, urllib.request

    def cmd_suggest_config(args):
        url = f"http://{args.host}:{args.port}/debug/config-recommendations"
        headers = {}
        if args.token:
            headers["Authorization"] = f"Bearer {args.token}"
        req = urllib.request.Request(url, headers=headers)
        with urllib.request.urlopen(req, timeout=5) as resp:
            data = json.load(resp)
        recs = data.get("recommendations", [])
        if not recs:
            print("No configuration recommendations at this time.")
            return 0
        # Group by decorator target (prefix of knob before the dot)
        groups = {}
        for r in recs:
            target = r["knob"].split(".", 1)[0]
            groups.setdefault(target, []).append(r)
        for target, items in sorted(groups.items()):
            print(f"\n{target}:")
            for r in items:
                print(f"  {r['knob']}: {r['current']} → {r['suggested']}  (confidence={r['confidence']:.2f})")
                print(f"    {r['reason']}")
                print(f"    {r['copy_paste']}")
        return 0

    def main(argv=None):
        p = argparse.ArgumentParser(prog="tally")
        sub = p.add_subparsers(dest="cmd", required=True)
        sc = sub.add_parser("suggest-config")
        sc.add_argument("--host", default="localhost")
        sc.add_argument("--port", type=int, default=6401)
        sc.add_argument("--token", default=None)
        sc.set_defaults(func=cmd_suggest_config)
        args = p.parse_args(argv)
        return args.func(args)

    if __name__ == "__main__":
        sys.exit(main())
    ```

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Bloom check on every PUSH adds hot-path latency | Bloom `contains` is ~200ns for 7 hashes; well under 100µs PUSH SLO. Only checked on new-key PUSHes (not re-PUSH for existing key). Guard with fast-path: `if store.has_entity(key) { skip bloom check }`. |
| Recommendation computation on every endpoint hit could be expensive on large pipelines | Compute is O(tables + streams); at v0 scale (< 100 tables) this is negligible. If hot in practice, cache for 60s. |
| Recommendations double-emit signals when `/debug/config-recommendations` is polled frequently | SignalRegistry dedupes by `id`. Use id = `config.rec.{knob}`; repeated emits update `last_seen` but don't duplicate. Verified by plan 25-02's dedupe test. |
| `ttl="forever"` parsing collides with existing None semantics | Map `"forever"` → `None` in `parse_duration_str` return type `Result<Option<Duration>, ParseErr>`; None = skip eviction. Existing eviction.rs already handles None. Explicit test. |
| Bloom FP rate drift as key count grows past 1M | Current-generation bloom sized for 1M items at 1% FP; at 2M items FP jumps to ~3.5%. Double-buffer rotation every 24h caps accumulation. For pathological cases, recommendation engine itself flags high eviction rate → user will extend TTL. Acceptable FP ceiling: 2% (tested). |
| `tally` CLI auth — admin router is token-gated | CLI accepts `--token`; in dev mode (loopback) token may be unnecessary. Document in CLI `--help`. |
| Python CLI breaks on Python 3.10 vs 3.12 difference | Use only stdlib (`argparse`, `urllib`, `json`, `sys`) — no third-party deps. Test against both. |
| Registered pipelines without `ttl` kwarg would have had NO eviction prior to this plan; applying 30d default retroactively could surprise existing users | This is a v0 pre-launch phase. Document in release notes. For anyone who wants to opt out, `ttl="forever"` is the explicit disable. |
