# PLAN 25-01: GET_MULTI opcode end-to-end + SCAN/SUBSCRIBE reserved

## Goal

Ship the `GET_MULTI` TCP opcode (0x0D) end-to-end — Rust server dispatch, Python SDK client method, integration tests — so ML inference callers can assemble a feature vector across N Tables for one key in a single round-trip. Reserve `SCAN` (0x10) and `SUBSCRIBE` (0x11) opcodes with a "not implemented in v0" error so clients fail cleanly today and forward-compat is locked.

This plan implements success criteria 1, 3, 4, and 9 from the ROADMAP (and criterion 2 is already met by existing MGET).

## Success Criteria

Observable and testable:

1. **Wire format works**: A client that sends `[0x0D][u8 count][string × count][string key]` receives a framed response whose payload is `{ "T1": {...}|null, "T2": {...}|null, ... }`.
2. **Null-collapse**: For each requested table, if the key is (a) never-seen in that Table, (b) tombstoned within grace, or (c) pending, the table's value is JSON `null`. No error, no field-level leakage.
3. **Atomic on malformed table**: If ANY requested table name is unknown to the registered pipeline, the whole response is a single error; no `{known: ..., unknown: error}` partial responses.
4. **Composite keys**: Passing `key` as a JSON-encoded array (matching the existing composite-key wire convention used by OP_GET / OP_PUSH_TABLE) works identically to string keys.
5. **Count cap enforced**: `count > 32` is rejected at parse time with a specific "table count exceeds limit (32)" error.
6. **Degenerate single-table**: `GET_MULTI(["T1"], k)` returns `{"T1": same_value_as_GET_for_slice}`.
7. **Python SDK**: `app.get_multi(["UserProfile", "UserSpend"], "u1")` returns `{"UserProfile": dict_or_None, "UserSpend": dict_or_None}`.
8. **Reserved opcodes**: Sending 0x10 or 0x11 returns a structured `TallyError` with message `"opcode 0x10 (SCAN) not implemented in v0"` / `"opcode 0x11 (SUBSCRIBE) not implemented in v0"`; the TCP connection stays open for the next request.
9. All existing TCP tests (`test_server`, `test_op_push_table`, Phase-24 integration) remain green. `cargo test` passes full suite.

## Implementation Steps

### Task 1 — Rust protocol + dispatch (Wave 1)

1. In `src/server/protocol.rs`:
   - Add `pub const OP_GET_MULTI: u8 = 0x0D;` next to the existing Table opcodes (0x0B/0x0C). Add a block comment noting 0x0D is the first of the query-expansion opcodes.
   - Add `pub const OP_SCAN: u8 = 0x10;` and `pub const OP_SUBSCRIBE: u8 = 0x11;` with a block comment: "Reserved in v0. TCP opcodes 0x10–0x1F reserved for future query verbs per v0-restructure-spec §6.2."
   - Extend `enum Command` with:
     ```rust
     GetMulti { tables: Vec<String>, key: String },
     Scan, // reserved
     Subscribe, // reserved
     ```
     where `key: String` is the raw payload (composite keys are JSON arrays encoded as a string and parsed downstream, mirroring existing GET behavior).
   - In `parse_command`:
     - Arm for `OP_GET_MULTI`: read `u8 count`; if `count > 32`, return `TallyError::ProtocolError("GET_MULTI table count exceeds limit of 32")`. Then read `count` length-prefixed UTF-8 strings (use the same `read_string` helper MGET uses). Then read one length-prefixed `key` string. Return `Command::GetMulti { tables, key }`.
     - Arms for `OP_SCAN` and `OP_SUBSCRIBE`: return `TallyError::ProtocolError("opcode 0x10 (SCAN) not implemented in v0")` (and similarly for 0x11). Do NOT consume payload beyond what's needed to keep the frame-level invariant; the frame length byte lets the TCP reader already advance past it cleanly — verify this by re-reading `encode_frame` contract.
2. In `src/server/tcp.rs`:
   - In the dispatch `match command` arm, add:
     ```rust
     Command::GetMulti { tables, key } => handle_get_multi(&state, tables, key).await,
     Command::Scan | Command::Subscribe => {
         /* Error already produced at parse_command; this arm is unreachable because
            parse_command returns Err for reserved opcodes. Keep for exhaustiveness. */
         unreachable!("reserved opcodes are rejected at parse_command")
     }
     ```
   - Implement `async fn handle_get_multi(state: &SharedState, tables: Vec<String>, key: String) -> Response`:
     1. Validate ALL requested table names exist in the registered pipeline. If any missing, return a single `TallyError::UnknownTable(name)` — no partial response. (Mirror the unknown-table-before-mutation pattern from Phase 24-02's OP_PUSH_TABLE.)
     2. Call `state.store.collect_merged_features(&key)` once (reuse the merged GET view from Phase 24-02).
     3. For each requested table name `T`, extract the slice of the merged map whose keys start with `T.` (i.e. `"T.field_a"`, `"T.field_b"`) AND that table's stream-operator keys if this table is a derived stream aggregation output (see merged-view shape from 24-02 SUMMARY).
        - Strip the `T.` prefix when building the per-table inner object.
        - If the slice is empty (no fields, no static, and no live `table_rows[T]`), emit `"T": null` (null-collapse per requirement QUERY-GETMULTI-02).
        - Tombstoned rows are already filtered by `collect_merged_features` per 24-02 SUMMARY; keys with ONLY tombstoned state will produce an empty slice → null (correct null-collapse).
     4. Serialize as `{table_name: row_or_null}` JSON and return as the frame payload.
   - Keep the function `#[inline]` and single-threaded-safe; it only reads StateStore.

### Task 3 — Tests (Wave 1)

   - `test_get_multi_three_tables_happy_path` — push to three Tables for key `u1`, GET_MULTI(["A","B","C"], "u1"), assert all three present.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Slice extraction from `collect_merged_features` could leak cross-table fields if a Table name is a prefix of another (e.g. "User" vs "UserProfile") | Use exact `T.` prefix check with the dot — "User." won't match "UserProfile.field". Add an explicit regression test with prefix-colliding Table names. |
| Null-collapse vs empty-dict ambiguity: a Table with a live row but all-null fields would currently serialize as `{}` not `null` | Distinguish at handler level: check if the Table has a live `table_rows[T]` entry OR any stream operator state for that `T`; if yes → emit `{}` or the populated dict; if no → emit `null`. Covered by `test_get_multi_tombstoned_returns_null` + `test_get_multi_missing_key_all_null`. |
| Connection corruption after reserved-opcode error | Because `parse_command` returns `Err` BEFORE `handle_*` runs and the TCP layer already length-framed the request, the reader advances past the frame correctly. Explicit test (`test_scan_opcode_reserved_error`) validates a subsequent GET on the same connection. |
| `count=0` semantics (empty table list) | Reject at parse with "GET_MULTI requires at least one table". Documents the floor of the `1 <= count <= 32` contract in the SDK assert and on the wire. |
| Python SDK composite-key encoding divergence from existing `app.get` | Re-read `_app.py::get` composite-key handling and mirror it byte-for-byte in `get_multi`. Add an explicit e2e test. |
