<objective>
Land the zero-cost dispatch skeleton for Phase 5's 8 core aggregation operators plus the `Windowed<Op>` bucket infrastructure — all in `beava-core` (syscall-free, WASM-portable).

Per D-01 (enum + match arm dispatch, no `Box<dyn AggOp>`), D-04 (64-bucket event-time tumbling), D-06 (deterministic apply: `event.event_time_ms` only, Welford for variance). Per locked memory `project_stateful_architecture.md`: dispatch surface stays thin (update + query only — no `serialize` on the operator trait).

Purpose: establish the in-memory operator state types and their apply/query contracts. No server wiring, no `where=`, no register-time validation, no per-entity state table. Subsequent plans bolt on: predicate threading (05-02), schema propagator (05-03), Rule 11 + caching (05-04), apply-loop hook + per-entity state (05-05), HTTP/TCP query endpoint (05-06).

Output: three new modules exported from `beava-core`, plus per-op table-driven unit tests proving each operator is correct on a fixed event stream (SC3 unit coverage starts here).
</objective>

```rust
// crates/beava-core/src/agg_state.rs
pub struct CountState  { pub n: u64 }
pub struct SumState    { pub total: f64 }
pub struct AvgState    { pub sum: f64, pub n: u64 }
pub struct MinState    { pub current: Option<Value> }
pub struct MaxState    { pub current: Option<Value> }
/// Welford's online algorithm — numerically stable, deterministic, combinable.
pub struct VarianceState { pub n: u64, pub mean: f64, pub m2: f64 }
pub struct RatioState  { pub matching: u64, pub total: u64 }

// crates/beava-core/src/agg_op.rs
#[derive(Debug, Clone)]
pub struct AggOpDescriptor {
    pub kind: AggKind,
    pub field: Option<String>,          // field for sum/avg/min/max/variance/stddev
    pub window_ms: Option<u64>,         // None → lifetime (windowless)
    // where_expr added in Plan 05-02
}
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum AggKind { Count, Sum, Avg, Min, Max, Variance, StdDev, Ratio }

/// The live state for one (feature, entity) slot. Enum dispatch; no Box<dyn>.
#[derive(Debug, Clone)]
pub enum AggOp {
    Count(CountState),
    Sum(SumState),
    Avg(AvgState),
    Min(MinState),
    Max(MaxState),
    Variance(VarianceState),
    StdDev(VarianceState),  // shares state with Variance; query takes sqrt
    Ratio(RatioState),
    Windowed(Box<WindowedOp>),
}
impl AggOp {
    pub fn new(descriptor: &AggOpDescriptor) -> AggOp;  // builds the right variant
    pub fn update(&mut self, row: &Row, event_time_ms: i64, field: Option<&str>, where_matched: bool);
    pub fn query(&self, query_time_ms: i64) -> Value;
}

/// Register-time type inference — used by Plan 05-03 schema propagator.
pub fn output_type_for(upstream: &Schema, desc: &AggOpDescriptor) -> Result<FieldType, AggTypeError>;

// crates/beava-core/src/agg_windowed.rs
pub struct WindowedOp {
    pub inner_kind: AggKind,
    pub bucket_ms: u64,
    pub buckets: [Option<Box<AggOp>>; 64],  // None = empty bucket; lazily created
    pub bucket_epoch_start_ms: [i64; 64],   // event_time of first event in bucket i
}
impl WindowedOp {
    pub fn new(kind: AggKind, window_ms: u64) -> Self;  // bucket_ms = ceil(window_ms / 64)
    pub fn bucket_index(&self, event_time_ms: i64) -> usize;  // floor(t / bucket_ms) mod 64
    pub fn update(&mut self, row: &Row, event_time_ms: i64, field: Option<&str>, where_matched: bool);
    pub fn query(&self, query_time_ms: i64) -> Value;  // folds active buckets
}
```
