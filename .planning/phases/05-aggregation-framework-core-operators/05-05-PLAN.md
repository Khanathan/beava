<objective>
Land the apply-loop hook that turns a pushed event into state updates on every matching aggregation.

Per D-06: apply is deterministic â€” only `event.event_time_ms` influences bucket index; no wall-clock; no HashMap iteration where serialization depends on it (use `BTreeMap` for per-entity state). Per locked memory `project_stateful_architecture.md`: single-writer invariant, no locks around per-entity state, no `serialize()` on the operator trait.

This plan lands the state storage + dispatch logic and a dev endpoint (`POST /dev/apply_events`) that lets Plan 05-08's acceptance gate exercise it without WAL/push-sync machinery (those land in Phase 6). Plan 05-06 layers the read-side query endpoint (`GET /get/{feature}/{key}`) on top.

Output: `AggStateTable` module + `apply_event_to_aggregations` entry point + dev endpoint scaffold. Production `POST /push` wiring is Phase 6; Plan 05-05 exposes the mechanism that Phase 6 will call.

```rust
// crates/beava-core/src/agg_state_table.rs
/// Entity key â€” a tuple of group-key values serialized canonically.
/// Uses a BTreeMap-backed vec so iteration is deterministic (D-06).
#[derive(Debug, Clone, PartialEq, Eq, Ord, PartialOrd)]
pub struct EntityKey(pub Vec<(String, String)>);  // (group_key_name, value_as_string)

impl EntityKey {
    pub fn from_row(group_keys: &[String], row: &Row) -> Option<EntityKey>;
    // returns None if any group_key field is null or missing (event is dropped)
}

/// Per-aggregation state: a BTreeMap from EntityKey -> per-feature AggOp state.
/// (Each feature of the aggregation has its own AggOp; entity row holds Vec<AggOp>.)
pub struct AggStateTable {
    pub entities: BTreeMap<EntityKey, Vec<AggOp>>,   // parallel to descriptor.features
}
impl AggStateTable {
    pub fn new() -> Self;
    pub fn get_or_init(&mut self, key: &EntityKey, descriptor: &AggregationDescriptor) -> &mut Vec<AggOp>;
    pub fn query_feature(&self, key: &EntityKey, feature_index: usize, query_time_ms: i64) -> Option<Value>;
    pub fn entity_count(&self) -> usize;
}

// crates/beava-core/src/agg_apply.rs
/// Apply a single event to every aggregation keyed by the event's source.
/// Called per-event by Phase 6's push handler; Plan 05-05 exposes via /dev/apply_events.
///
/// Semantics:
///   - Look up all aggregations for source_name (Registry::compiled_aggregations_for_source).
///   - For each: derive EntityKey from row + descriptor.group_keys.
///     If any group-key field is null/missing, drop the event for that aggregation.
///   - Look up/init entity row in the aggregation's AggStateTable.
///   - For each feature: call AggOp::update_with_row.
///
/// This function is a PURE function of (event_time_ms, row, descriptors, prior state).
/// No wall-clock reads. No RNG. Safe for WAL replay determinism (D-06).
pub fn apply_event_to_aggregations(
    source_name: &str,
    row: &Row,
    event_time_ms: i64,
    registry: &Registry,
    state_tables: &mut BTreeMap<String /* agg node_name */, AggStateTable>,
);
```

       ```rust
       pub fn apply_event_to_aggregations(
           source_name: &str,
           row: &Row,
           event_time_ms: i64,
           registry: &Registry,
           state_tables: &mut BTreeMap<String, AggStateTable>,
       ) {
           for desc in registry.compiled_aggregations_for_source(source_name) {
               let entity_key = match EntityKey::from_row(&desc.group_keys, row) {
                   Some(k) => k,
                   None => continue,  // drop events with null/missing group-key values

                   entity_row[i].update_with_row(
                       row,
                       event_time_ms,
                       feat.descriptor.field.as_deref(),
                       feat.descriptor.where_expr.as_ref(),
                   );
               }
           }
       }
       ```
