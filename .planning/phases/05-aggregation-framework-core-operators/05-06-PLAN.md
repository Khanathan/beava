<objective>
Land the read-side of Phase 5. HTTP clients (including Plan 05-07's Python SDK and Plan 05-08's acceptance gate) query aggregation feature values via:

1. **`GET /get/{feature}/{key}`** — single-feature lookup. Returns `{value: <JSON>}` per D-02.
2. **`POST /get`** — batch lookup accepting `{keys: [...], features: [...]}`. Returns `{key: {feature: value}}` per SRV-API-08 shape.

Both endpoints live in the dev-gated surface for Phase 5 (they reuse the `DevAggState` from Plan 05-05). Production `/get` endpoints (with full push-ACK wiring, WAL, idempotency) land in Phase 6/12. Phase 5's goal is end-to-end *mechanism*: push via /dev/apply_events, query via /get, acceptance gate runs.

Resolution of `feature_name → (agg_descriptor, feature_index)`:
- A feature name is globally unique across aggregations (Rule 11 in Plan 05-04 enforces this WITHIN an aggregation via `AggregationDuplicateFeatureName`; add a new cross-aggregation uniqueness check if needed, OR scope lookup to `{feature}` = `"{aggregation_node}.{feature_name}"` string).
- **Decision for Phase 5:** the feature URL is the feature name as registered — i.e. the output column of an aggregation Table. If two aggregations define a feature named `cnt`, they collide. We add a new Rule 11 check (`feature_name_global_collision`) OR scope it as `{agg_node}/{feature}` → use the scoped form to avoid new Rule-11 work. **Chosen form: `GET /get/{feature}/{key}` where `{feature}` resolves against a flat `feature_name -> (agg_node, feature_index)` index built at register time; on collision, return 400 at register time.**

Output: new HTTP module for feature queries + feature-name index maintained alongside `compiled_aggregations`.
</objective>

       ```rust
       pub fn feature_query_router(state: FeatureQueryState) -> Router {
           Router::new()
               .route("/get/:feature/:key", get(get_feature))
               .route("/get", post(post_get_batch))
               .with_state(state)

       async fn get_feature(
           Path((feature, key)): Path<(String, String)>,
           State(state): State<FeatureQueryState>,
       ) -> (StatusCode, Json<serde_json::Value>) {
           let (agg_node, feature_index) = match state.registry.resolve_feature(&feature) {
               Some(x) => x,
               None => return (StatusCode::NOT_FOUND,
                   Json(serde_json::json!({"error": {"code": "feature_not_found"}}))),
           };
           let descriptor = state.registry.compiled_aggregation(&agg_node).expect("index consistency");
           let entity_key = parse_entity_key(&key, &descriptor.group_keys);  // pipe-split on "|" for multi-key
           let tables = state.state_tables.lock();
           let query_time_ms = compute_query_time_ms(&tables);  // max seen event_time across all state_tables, or 0
           let value_opt = tables.get(&agg_node)
               .and_then(|t| t.query_feature(&entity_key, feature_index, query_time_ms));
           match value_opt {
               Some(v) => (StatusCode::OK, Json(serde_json::json!({"value": value_to_json(v)}))),
               None => (StatusCode::NOT_FOUND,
                   Json(serde_json::json!({"error": {"code": "key_not_found"}}))),
           }
       }
       ```
