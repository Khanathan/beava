Model the `Event.group_by(*keys).agg(**features)` descriptor at the beava-core level and derive the output `DerivedSchema` at register time.

Per D-05: aggregations produce a **TableDerivation** with `key = group_keys` and `schema = group_keys (inherited types) + each feature's output_type_for(upstream_schema)`. Temporal = false (SDK-AGG-05 rejects aggregation-on-Table; that rejection lives in Plan 05-04's Rule 11).

Purpose: provide the structural data the apply loop and the HTTP query endpoint will use to look up aggregations by source + feature. Plan 05-04 caches `Arc<AggregationDescriptor>` in `RegistryInner.compiled_aggregations`. Plan 05-05 uses `source_node_name` to route events. Plan 05-06 looks up features by name.

Output: two new `beava-core` modules, zero server wiring (that's 05-04), zero apply-loop wiring (that's 05-05).

    ```rust
    pub fn propagate_aggregation_schema(
        upstream: &Schema,
        descriptor: &AggregationDescriptor,
    ) -> Result<DerivedSchema, Vec<AggSchemaError>> {
        let mut errors = Vec::new();
        let mut fields = BTreeMap::new();

        // 1. Validate group keys exist in upstream schema, copy their types into output.
        let mut group_key_set: std::collections::HashSet<&str> = std::collections::HashSet::new();
        for key in &descriptor.group_keys {
            match upstream.fields.get(key) {
                Some(ty) => { fields.insert(key.clone(), *ty); group_key_set.insert(key.as_str()); }
                None => errors.push(AggSchemaError::GroupKeyMissing { key: key.clone() }),
            }
        }

        // 2. Validate feature uniqueness + collision with group keys; infer each type.
        let mut seen_feature_names: std::collections::HashSet<&str> = std::collections::HashSet::new();
        for feat in &descriptor.features {
            // Duplicate feature name?
            if !seen_feature_names.insert(feat.feature_name.as_str()) {
                errors.push(AggSchemaError::DuplicateFeatureName { name: feat.feature_name.clone() });
                continue;
            }
            // Collides with group key?
            if group_key_set.contains(feat.feature_name.as_str()) {
                errors.push(AggSchemaError::GroupKeyCollidesWithFeature { name: feat.feature_name.clone() });
                continue;
            }
            // Type inference via output_type_for.
            match output_type_for(upstream, &feat.descriptor) {
                Ok(ty) => { fields.insert(feat.feature_name.clone(), ty); }
                Err(AggTypeError::FieldMissing { field }) => {
                    errors.push(AggSchemaError::FeatureTypeError {
                        feature: feat.feature_name.clone(),
                        kind: feat.descriptor.kind,
                        field_missing: Some(field),
                    });
                }
                Err(AggTypeError::FieldRequired { kind }) => {
                    errors.push(AggSchemaError::FeatureTypeError {
                        feature: feat.feature_name.clone(),
                        kind,
                        field_missing: None,
                    });
                }
            }
        }

        if !errors.is_empty() {
            return Err(errors);
        }

        // Optional-fields: aggregation output is NON-NULL for group keys (implicit from group_by
        // semantics — null-keyed events are dropped at apply time in Plan 05-05). Feature values
        // MAY be null only for avg/min/max/variance/stddev/ratio when no matching rows. Document
        // by leaving optional_fields empty in v0 (all features default to non-null; apply loop
        // returns Null for empty-state queries but the SCHEMA contract is "field present").
        Ok(DerivedSchema {
            fields,
            optional_fields: Vec::new(),
        })
    }
    ```
