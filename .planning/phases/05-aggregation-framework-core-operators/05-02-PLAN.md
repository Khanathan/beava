<objective>
Thread the `where=` predicate through the Phase 5 apply path. Per D-03: every core aggregation accepts an optional `where: bv.col(...)` predicate; at apply time, evaluate against the event Row using Phase 4's `eval::eval_with_depth`. `Bool(true)` → update; `Bool(false)` / `Null` / any other value → skip the update (three-valued null drop). For Ratio: the predicate still gates the numerator only (total always increments when a row is seen).

Purpose: make `bv.count(where=bv.col("status")=="ok", window="5m")` expressible at the apply layer. Register-time validation (Rule 11) + end-to-end wire semantics come in Plans 05-04 and later. Plan 05-02 is pure apply-path mechanics.

**lib.rs is not edited by this plan.** Plan 05-01 pre-declared `pub mod agg_where;` in `lib.rs` pointing at a placeholder docstring file. This plan overwrites the body of `agg_where.rs` with the real implementation, removing the wave-2 fan-in conflict with Plan 05-03.

Output: `AggOpDescriptor` gains `where_expr: Option<Arc<Expr>>`, apply methods gain a predicate-eval wrapper, and unit tests cover null, false, true, and the Ratio "gate numerator only" semantics.
</objective>

    2. **`agg_op.rs::AggOp::update_with_row`**:
       ```rust
       pub fn update_with_row(
           &mut self,
           row: &Row,
           event_time_ms: i64,
           field: Option<&str>,
           where_expr: Option<&Arc<Expr>>,
       ) {
           let where_matched = match where_expr {
               Some(e) => evaluate_where_predicate(e, row),
               None => true,
           };

           match self {
               AggOp::Ratio(state) => {
                   // Ratio semantics: total increments on any seen row; matching gated by predicate.
                   state.total += 1;
                   if where_matched { state.matching += 1; }
               }
               AggOp::Windowed(w) => {
                   // Windowed delegates; per-bucket inner op will gate itself.
                   w.update_with_row(row, event_time_ms, field, where_expr);
               }
               _ => {
                   if where_matched {
                       self.update(row, event_time_ms, field, true);  // existing Plan 05-01 path
                   }
                   // No-op on predicate-false for Count/Sum/Avg/Min/Max/Variance/StdDev
               }
           }
       }
       ```

       Note: for Ratio we bypass `AggOp::update` to implement the D-03 "gate numerator only" semantics. Alternatively, teach RatioState's update to take two flags — but threading it here is cleaner.
