<objective>
Make event-time primary and wall-clock explicit. Parse `_event_time` from
every event payload, track per-stream watermarks with the fixed 5-second
lateness model, drop late events with an observable counter, propagate
watermarks at join/agg boundaries using the γ model (alignment only at
those boundaries — stateless ops pass through), route window buckets by
event-time, and expose watermarks via `/debug`.

Per CONTEXT.md §Watermarks, §Event-time wire format, §Builtins,
§Late event handling (all LOCKED).

Purpose: The final correctness change required for v0. Without event-time
+ watermarks, Tally is arrival-time-only and cannot be sold as a
production streaming feature server.

Output: Per-stream watermark state, event-time parsing at ingest, γ
propagation through the operator graph, late-event drop + counter,
event-time-aware window buckets, `event_time()` builtin, debug visibility.
</objective>

Change: every caller must pass `event_time` (the parsed `_event_time` or
the wall-clock fallback) as the `now` argument. Rename the parameter to
`event_time` and update all call sites in operators.rs.

Current event shape on the wire (OP_PUSH): a JSON object. Reserved fields
already present: the key field per the stream's decorator, plus
application fields. `_event_time` is a reserved-but-unused field in v2.0;
Phase 24 makes it active.

Event-time parsing rule (LOCKED, v0-restructure-spec §4.1):
- JSON key: `_event_time`
- Accepted formats: ISO8601 string ("2026-04-14T12:34:56.789Z"), Unix
  milliseconds (i64), Unix seconds (f64 — if value &lt; 2^31, interpret as
  seconds; if &gt;= 2^31, interpret as milliseconds for backwards-compat
  with large epoch values)
- Absent / unparseable → fall back to wall-clock arrival, do NOT error
- Parsed value must be convertible to SystemTime

Watermark tracking surface (NEW, plan 04):

```rust
pub struct WatermarkTracker {
    // Per-stream: max observed event_time
    observed_max: AHashMap&lt;String, SystemTime&gt;,
    // Per-stream: watermark = observed_max - 5s
    // (stored implicitly; computed on read via observed_max - LATENESS)
}

pub const WATERMARK_LATENESS: Duration = Duration::from_secs(5);

impl WatermarkTracker {
    pub fn observe(&mut self, stream: &str, event_time: SystemTime);
    pub fn watermark(&self, stream: &str) -> Option&lt;SystemTime&gt;;
    // γ propagation helpers
    pub fn propagate_stateless(&mut self, from: &str, to: &str);
    pub fn propagate_join(&mut self, left: &str, right: &str, output: &str);
    pub fn attach_to_table(&mut self, source_stream: &str, output_table: &str);
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `_event_time` JSON field | Attacker-controlled timestamp — can claim far-future or far-past |
| watermark state | Concurrent updates across multiple connections |
| expression evaluator | event_time() bound to evaluator context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-04-01 | S | Far-future `_event_time` jumps watermark | accept | v0 takes user's event_time at face value; post-v0 will clamp to `now + tolerance`. Documented limitation in CONTEXT.md §Late event handling. |
| T-24-04-02 | D | Watermark poisoning via a single far-future event | accept | Same as above; mitigation deferred to v0.1 (bounded clock-skew tolerance, 1h cap) |
| T-24-04-03 | T | `_event_time` parse confusion between seconds and milliseconds | mitigate | Explicit threshold at 2^31 (~2038 if seconds) — documented + tested |
| T-24-04-04 | I | `/debug/streams/:name` leaks timing info | accept | Management endpoint only (separate port, localhost-only in v0 defaults) |
| T-24-04-05 | D | Late-drop counter unbounded label cardinality | mitigate | Label is `stream` name only — cardinality = # registered streams, bounded at register time |
| T-24-04-06 | T | Expression `event_time()` evaluated out of context | mitigate | Parser binds event_time only in event-scoped evaluator; standalone /query path (v0.1) will reject event_time() at parse time |
</threat_model>
