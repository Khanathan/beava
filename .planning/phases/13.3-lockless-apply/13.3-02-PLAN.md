<objective>
Cut over every WRITE path in the server from the `state_tables.lock()` Mutex to the `ApplyActor` scaffolded in Plan 01. After this plan, the Mutex fields on `DevAggState` are deleted and the actor is the sole writer to `ApplyState`. Read paths (GET /get, POST /get, GET /temporal) stay on a temporary read-side compatibility shim until Plan 03 migrates them.

Purpose: unlock the ~10× throughput lift projected in the idea doc. Today's 17k EPS at parallel=64 is apply-Mutex-bound; this plan removes that bound.

Output: every `/push`, `/push-sync`, `/push-table`, `/delete-table`, `/retract`, `/register`, TCP OP_PUSH, snapshot, and recovery write-path routes through the actor. WAL fsync stays on `spawn_blocking`. No functional behavior change — the same tests pass with the same responses.
</objective>
