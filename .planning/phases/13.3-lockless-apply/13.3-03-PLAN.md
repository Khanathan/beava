<objective>
Plan 02 removed the Mutex but left single-thread fairness as a code-review
concern. This plan codifies the fairness discipline: chunked yield in batch
handlers, DAG-phase yield in register, and a lint (if available) or review
checklist (if not) enforcing "no borrow held across `.await`".

Purpose: in a single-thread cooperative runtime, one slow handler can starve
every other connection. The Redis analogue is `SLOWLOG` + the
"don't block the event loop" discipline. This plan encodes that discipline as
runtime behavior (yields) and static checks (lint / checklist).

Output: push_many chunks + register phase-yields + clippy lint enabled +
fairness test suite.
</objective>

Target for `push_many`:
```rust
pub async fn push_many_handler(
    State(state): State<Rc<RefCell<AppState>>>,
    Json(body): Json<PushManyBody>,
) -> Response {
    const CHUNK: usize = 128;
    let mut last_lsn = 0;
    for chunk in body.events.chunks(CHUNK) {
        {
            let mut s = state.borrow_mut();
            for ev in chunk {
                s.apply_event(...);
                last_lsn = s.wal_buffer.append_inmem(...);
            }
        } // drop borrow
        tokio::task::yield_now().await;
    }
    Ok(Json(json!({ "last_lsn": last_lsn })))
}
```
