<objective>
Scaffold the lockless-apply machinery without changing any call-site behavior. Introduce the `apply` module (state + actor + command), wire `LocalSet` into `main.rs`, and prove the actor drains commands via a smoke test. No Mutex is removed in this plan — the old `DevAggState.state_tables.lock()` code path stays live. This is pure foundation work.

Purpose: de-risk the architecture by landing the `!Send`-friendly runtime wiring and the actor skeleton BEFORE any handler cutover. If axum's `!Send` state support fails here (D-15 grey area), we find out on a contained surface instead of during a bigger refactor.

Output: a new `apply` module + `LocalSet` wiring in `main.rs` + one smoke test proving the actor receives and ACKs `ApplyCommand::Ping`.
</objective>
