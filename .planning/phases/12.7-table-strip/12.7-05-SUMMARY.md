## Decisions Made

- **D-01 RESET (not bump):** all 3 constants reset 2 -> 1 per CONTEXT.md D-01. The reset is the canonical move for an unreleased product; bump is the canonical move for a released product. v0 falls in the former bucket, so reset wins. (Rationale: saves a forward-version slot; pre-12.7 dev WALs are dev artifacts that operators clear before booting the new binary; existing `SchemaVersionMismatch` / `UnsupportedVersion` error paths surface the rejection without a new error code.)
- **D-02 error framing:** from_u8(0x03|0x04|0x05) routes to the existing generic `PersistError::UnknownRecordType(b)` variant. No new feature_removed_no_tables_v0 code in persistence — that retrospective framing was right for 12.6's `event_time` strip (a feature that pre-pivot users had used) but is wrong for 12.7's table strip (a feature that no v0 user ever knew existed because v0 isn't released). Forward-looking "not supported in v0" framing is correct here.
- **FORMAT_VERSION re-export:** added a single line `pub use record::FORMAT_VERSION;` to `beava-persistence/src/lib.rs` so the external Plan-05 test could name the constant. Sibling `SNAPSHOT_FORMAT_VERSION` / `SNAPSHOT_BODY_FORMAT_VERSION` were already re-exported; this aligns the public API and is a Rule-3 (auto-fix blocking issue) move that lives with the RED commit because it's required infrastructure for the test.
- **Symbol-free docstrings:** the architectural sub-test `temporal_record_type_variants_deleted` does literal `src.contains()` against `TableUpsert = 0x` / `RecordType::TableUpsert` etc. — it does NOT strip line comments. So my initial post-deletion docstrings (which referenced the deleted symbols by name to explain WHAT was deleted) tripped the test. Fix: rewrote all explanatory docstrings in `lib.rs` / `record.rs` / `recovery.rs` to use prose (e.g., "the Phase 11.5 table-write and stream-retract record types" instead of "TableUpsert / TableDelete / Retract"). The architectural test's grep stays simple; the design intent stays in the code; both are happy.
