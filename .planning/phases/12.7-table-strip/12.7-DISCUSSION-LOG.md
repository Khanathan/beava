## WAL/snapshot schema strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Bump v2→v3, hard SchemaVersionMismatch (Recommended) | Mirror 12.6 D-03 verbatim — bump FORMAT_VERSION 2→3 across record.rs + snapshot_body.rs + snapshot_header.rs. Pre-12.7 WALs/snapshots fail with SchemaVersionMismatch at the version-byte check. RecordType variants TableUpsert/Delete/Retract deleted; from_u8 returns UnknownRecordType. | |
| Keep v=2, strict-deny only on table record types | FORMAT_VERSION stays 2; only the RecordType variants deleted; from_u8(0x03|0x04|0x05) hard-errors with feature_removed_no_tables_v0 mid-stream. WALs without table records still replay. | |

**User's choice:** "Just make it version 1 no need to care backward compat" — captured as **D-01: Reset FORMAT_VERSION to 1**.
**Notes:** v0 launches at version=1 across record.rs + snapshot_body.rs + snapshot_header.rs. Reset rather than bump. Consistent with the user's stance that pre-pivot artifacts are dev-only and shouldn't be version-numbered for compat.

## Strictness boundaries (SDK + register + HTTP)

| Option | Description | Selected |
|--------|-------------|----------|
| Mirror 12.6 D-03 verbatim (Recommended) | Python TypeError at decoration time. RuntimeError at app.retract call. feature_removed_no_tables_v0 at register JSON-prelude shim. 404 at deleted mio routes. UnknownRecordType at WAL. | |
| Same, but mio routes return structured 410 Gone | Same as Recommended, but instead of plain 404 return 410 with structured body. | |
| Decorator-only deny; lazy server-side | TypeError at decoration only; no JSON-prelude shim; serde "unknown variant" surfaces. | |
| **Hard errors at all layers, framed "not supported in v0" not "removed" (user-supplied free text)** | Same hardness as Recommended, but error messages drop retrospective "removed" / "feature_removed_*" framing. v0 is the FIRST release; users never knew tables existed. Forward-looking "not supported in v0" framing is correct. | ✓ |

**User's choice:** "error but no mention of remove" — captured as **D-02: Hard errors at all layers, framed as "not supported in v0"**.
**Notes:** Concrete framing shifts:
- Python `@bv.table` → `AttributeError` / `TypeError` ("`@bv.table` is not supported in v0; v0 ships events-only via `@bv.event`"). Decorator-time error per Plan 08 pattern.
- `app.retract` → `AttributeError: 'App' object has no attribute 'retract'` (natural Python error from method removal).
- Register-time JSON-prelude shim → structured code `unsupported_node_kind` (NOT `feature_removed_no_tables_v0`). Message lists supported kinds (`["event"]`).
- Mio HTTP `/upsert /delete /retract /table` → plain 404 (mio default; no special handler).
- WAL recovery → `UnknownRecordType` (existing generic variant).
