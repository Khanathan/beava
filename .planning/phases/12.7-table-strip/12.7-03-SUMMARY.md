## Decisions Made

- **D-02 framing applied verbatim:** Deleted routes return plain 404 from mio (no special deny-handler, no `feature_removed_no_tables_v0` retrospective error code). Stale curl-callers see `{"error": {"code": "not_found", "path": "/upsert/X"}}` — same as any other unknown route. v0 is the FIRST public release; users never knew tables existed in v0, so retrospective framing would confuse fresh users.
- **Variant deletion order respected:** Plan 01's `pre_check_unsupported_node_kind` shim was already wired at HEAD, so deleting the wire variants here doesn't break the rejection path. Register payloads with `{"kind": "table"}` get caught at the JSON layer before serde reaches the (now-deleted) `OpNode::Table*` / `WireRequest::Http*` variants. Each commit is unambiguous; rollback is a clean revert.
- **Doomed-test #[ignore] (not delete):** 5 tests across 2 test files asserted on the old temporal_http response shapes (`{"error": "table_not_found"}` / `"event_id_not_found"` / 501 stream retract); after route deletion they hit mio's default 404 shape. `#[ignore]`-marked with rationale `Phase 12.7-03: <route> route deleted; file slated for deletion in 12.7-06` — avoids a Plan-03 deletion that would conflict with Plan 04/06's planned deletions of these test files. Follows the existing pattern from Plan 01's `Phase 12.7-01: registers kind=table; file slated for deletion` markers in the same files.
- **GlueResponse::TemporalResponse variant kept alive at this commit:** No producers remain (4 dispatch arms deleted), but the variant declaration in `runtime_core_glue.rs:109` and its encoder in `server.rs:2186` stay until Plan 04 deletes `temporal_http.rs` (along with these consumers). Workspace stays GREEN nonetheless — clippy doesn't flag the dead variant under `#[derive(Debug)]` because the enum is `pub`.
- **Body-shape discriminator for RED tests (planner discretion):** Three of the four deleted routes (`/upsert`, `/retract`, `/table`) already returned 404 pre-deletion (handler-side `table_not_found` / `event_id_not_found` 404). Status code alone wasn't enough discrimination. Used body-shape assertion (`error.code == "not_found"` only present in mio's default route-not-found shape; string-valued `error` in temporal handlers fails) — RED tests fail with clean diagnostic: `left: None, right: Some("not_found")` or `body={"error":"table_not_found"}`. Reusable pattern for future variant-delete plans.

## Issues Encountered

- **Body-shape RED test discriminator design (resolved during RED gate):** Initial RED test asserted on status code alone (`assert_eq!(status, 404)`), but 3 of the 4 deleted routes already returned 404 pre-deletion via the temporal_http handler (`table_not_found` / `event_id_not_found` 404). Test was insufficient — only 1 of 4 routes (the `/delete/X` 400 case) was actually RED. **Resolution:** Strengthened the assertion to also check body shape (`error.code == "not_found"`); this discriminates the temporal-handler string-valued `error` (e.g. `{"error": "table_not_found"}`) from mio's default object-valued `error` (`{"error": {"code": "not_found", "path": "/upsert/X"}}`). All 4 RED tests now fail with clean diagnostics. Pattern captured under `patterns-established` for future variant-delete plans.
- **No surprises at the GREEN gate.** Variant deletion was mechanical (4 contiguous declarations + 4 contiguous parse arms + 4 contiguous translate arms + 4 contiguous dispatch arms in a single block); no surprising callers needed cleanup.
