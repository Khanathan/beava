# Phase 12.7 Plan 04: Server-Side Temporal Strip Summary

Whole-module delete of `temporal_http.rs` (~756 LOC) and `temporal.rs` (~394 LOC) plus surgery on `DevAggState` to remove the `temporal_stores` + `event_id_index` fields, the `EventIdEntry` enum, the `GlueResponse::TemporalResponse` variant + consumer, the apply-shard retract-bookkeeping write site, and the orphan `temporal_mvcc` criterion bench. Net 1,282 LOC of MVCC-shaped state removed; v0 ships events-only at the server-side per `project_v0_events_only_scope` (locked 2026-04-30).

## What Changed

### Source files deleted (whole-module delete, ~1,150 LOC of removed surface)

- **`crates/beava-server/src/temporal_http.rs`** — `~756 LOC`. The Phase 12.6 Plan 14 mio-bridge module that handled `POST /upsert/{table}` / `POST /delete/{table}` / `POST /retract` / `GET /table/{name}` via `*_via_mio` helpers. After Plan 03 deleted the wire/router/dispatch arms that called these helpers, the module was an orphan with zero callers. Whole-file delete.
- **`crates/beava-core/src/temporal.rs`** — `~394 LOC`. The Phase 11.5 MVCC-store core (`TemporalStore`, `MvccVersion`, `RetractError`, retention sweep). With `temporal_http.rs` gone (the only caller in production code) and the `DevAggState.temporal_stores` map deleted, this module had no production consumers. Whole-file delete.
- **`crates/beava-core/benches/temporal_mvcc.rs`** — `~69 LOC`. Orphan criterion bench that imported `beava_core::temporal::TemporalStore`. With `temporal.rs` gone the bench fails to compile; deleted along with its `[[bench]] name = "temporal_mvcc"` entry in `crates/beava-core/Cargo.toml`.

### Module declarations removed

- **`crates/beava-server/src/lib.rs`** line 22 — deleted `pub mod temporal_http;`.
- **`crates/beava-core/src/lib.rs`** line 40 — deleted `pub mod temporal;`.

### `DevAggState` surgery (`crates/beava-server/src/registry_debug.rs`)

Three field/import sites + the `EventIdEntry` enum + a stale unit-test:

- Removed `use beava_core::temporal::TemporalStore;` import.
- Removed `pub temporal_stores: Arc<Mutex<HashMap<String, TemporalStore>>>` field + initializer in `DevAggState::new()`.
- Removed `pub event_id_index: Arc<Mutex<hashbrown::HashMap<u64, EventIdEntry, fxhash::FxBuildHasher>>>` field + initializer in `DevAggState::new()`.
- Removed `pub enum EventIdEntry { Stream { event_name: Arc<str> }, TableWrite { table_name, entity_key, retracted } }` declaration. After this plan the enum has zero non-test consumers; the bookkeeping write-site in `apply_shard.rs` was the only producer (deleted alongside).
- Removed `event_id_entry_stream_takes_arc_str` test (consumed the now-deleted enum).
- Removed `use std::collections::HashMap` (was used only by `temporal_stores`).
- Removed `use super::*` from the surviving `tests` module (was used only by the deleted test).
- Updated module docstring to reflect post-12.7-04 state (only `state_tables` + `registry` + atomic counters survive in `DevAggState`).

### `GlueResponse::TemporalResponse` deletion

- **`crates/beava-server/src/runtime_core_glue.rs`**: deleted the `TemporalResponse { http_status: u16, body: Bytes }` variant from `enum GlueResponse`. The variant's only producer was Plan 03's now-deleted dispatch arms in `apply_shard.rs`. Updated module docstring (line 10) to drop the `temporal_http::*_via_mio` reference.
- **`crates/beava-server/src/server.rs`** lines 2183-2186: deleted the consumer `match` arm that called `body.to_vec()` on the variant. With no producer and no consumer the variant is fully gone.

### `apply_shard.rs` orphan-lock surgery

- **`crates/beava-server/src/apply_shard.rs`** lines 782-797 (now 782-790): deleted the entire "step 10 — Record event ID entry for retract routing" block. The block held the only surviving lock site for `event_id_index`; with the field gone it would not compile, and with the retract path gone it has no purpose. The `t_bk_evid` timing capture is preserved (a no-op `Instant::now` call) so the `TRACE_APPLY ns push` line keeps its fixed stage-label shape.

### Test maintenance (Rule 3 deviations)

- **`crates/beava-server/tests/phase12_6_legacy_axum_killed.rs::temporal_http_axum_handlers_deleted`**: rewritten. Original test asserted `temporal_http.rs` had zero `axum::` references and that `*_via_mio` helpers were preserved. With the file deleted the original `read_to_string` panics. Repointed body at the new invariant: `temporal_http.rs` MUST NOT exist on disk. Cross-references the canonical 12.7-side `phase12_7_legacy_table_handlers_killed::legacy_table_files_deleted` test.
- **`crates/beava-server/tests/phase18_12_arc_str_bookkeeping_test.rs`**: gated out with `#![cfg(any())]` (a never-true cfg attribute on the file). The test was a Plan 18-12 RED contract that asserted `event_id_index.lock().get(&ack_lsn)` returns an `EventIdEntry::Stream` whose `Arc<str>` is `Arc::ptr_eq` to the descriptor's `name_arc`. With the side-table and enum gone the test cannot compile. Per critical_constraints note #7 the file is slated for deletion in Plan 12.7-06; the cfg-out preserves the original test text + the deletion is centralized in the canonical SDK-strip wave.

## Why This Plan Wave-Ordered Where It Did

Plan 04 sits in Wave 2 alongside Plan 03. Plan 03 deleted the wire/router/dispatch entry points (`WireRequest::HttpUpsert` / `HttpDelete` / `HttpRetract` / `HttpTableGet`, the `Route::*` table variants, and the corresponding dispatch arms in `apply_shard.rs`). That left two large source files (`temporal_http.rs` + `temporal.rs`) and two `DevAggState` fields (`temporal_stores` + `event_id_index`) as orphans with zero production callers. Plan 04 deletes them.

Plans 05 (persistence schema reset — `RecordType::TableUpsert/TableDelete/Retract` + `FORMAT_VERSION` reset) and 06 (Python SDK strip + `_tables.py` deletion + final architectural-test green) are independent of Plan 04's scope and can run in parallel after this plan lands.
