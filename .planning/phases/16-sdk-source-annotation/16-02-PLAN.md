<objective>
Ship `app.upsert(target, row)` and `app.delete(target, *, key={...})` as the
first-class Python-SDK verbs for writing to source tables. Enforce D-05 at
two layers: (1) client-side `BV-E-PUSH-TO-DERIVATION` guard before network
I/O; (2) server-side 400 `cannot_push_to_derivation` for out-of-band
(curl / other SDKs) callers. No wire format changes — uses the existing
Phase 11.5 `POST /push-table/{name}` + `POST /delete-table/{name}`
endpoints.

Purpose: Give v0 users the named, symmetric verbs they reason about
(`upsert` / `delete`) rather than URL-pathy names (`push_table`). Makes
Plan 03's migration concrete: tests demonstrate the intended surface.

Output: 2 new App methods; 2 new Transport methods (HTTP + TCP); 1 new
server error code + its 400 response shape; 4 new REQ-IDs closed.

**Scope note:** Per CONTEXT GA-2, we do NOT add `app.push_table` /
`app.delete_table` as deprecated aliases — they do not exist today and
shipping them just to deprecate them is dead weight. If the user rejects
GA-2, Plan 02 adds ~20 LoC of DeprecationWarning aliases.
</objective>

      def upsert(self, target: Any, row: dict[str, Any]) -> dict[str, Any]:
          transport = self._require_transport()
          self._check_upsert_target(target)  # raises ValueError/TypeError per rules
          payload = json.dumps(row, ensure_ascii=False).encode("utf-8")
          return transport.send_push_table(target._name, payload)

      def delete(self, target: Any, *, key: dict[str, Any]) -> dict[str, Any]:
          transport = self._require_transport()
          self._check_upsert_target(target)
          if not isinstance(key, dict) or not key:
              raise TypeError("app.delete requires non-empty key= dict kwarg")
          payload = json.dumps({"key": key}, ensure_ascii=False).encode("utf-8")
          return transport.send_delete_table(target._name, payload)

      def _check_upsert_target(self, target: Any) -> None:
          if not hasattr(target, "_beava_kind") or not hasattr(target, "_name"):
              raise TypeError(f"app.upsert/delete target must be a @bv.table descriptor; got {type(target).__name__}")
          kind = target._beava_kind
          if kind == "event":
              raise ValueError(f"app.upsert targets tables; got event source {target._name!r} — use app.push for events (Phase 12)")
          if kind != "table":
              raise ValueError(f"BV-E-PUSH-TO-DERIVATION: app.upsert/delete target {target._name!r} is a derivation; only @bv.source @bv.table sources accept pushes")
          if not getattr(target, "_beava_is_source", False):
              raise ValueError(f"BV-E-PUSH-TO-DERIVATION: app.upsert/delete target {target._name!r} is a derivation (no @bv.source); only @bv.source @bv.table sources accept pushes")

<threat_model>
## Trust boundaries

| Boundary | Description |
|----------|-------------|
| Python SDK user → `app.upsert/delete` | In-process; SDK does kind-check as fail-fast UX. Not a security boundary. |
| Any HTTP client (curl, SDK, other language) → beava server `/push-table/{name}` | Untrusted. Server must enforce kind-check independently. |

## STRIDE register

| ID | Category | Component | Disposition | Mitigation |
|----|----------|-----------|-------------|------------|
| T-16-04 | Tampering | Client bypassing `_check_upsert_target` via monkey-patch | accept | SDK is an in-process helper; server-side 400 is the authoritative gate (Task 2). |
| T-16-05 | Tampering | Forged POST to `/push-table/{derivation_name}` via curl | mitigate | `push_table_handler` rejects with 400 `cannot_push_to_derivation` when descriptor is derivation (Task 2). Integration smoke enforces. |
| T-16-06 | Denial of service | Floods of 400 responses | accept | No new cost over existing 400 responses; server's existing rate-limiting (if any) unchanged. |
| T-16-07 | Information disclosure | Error body naming the derivation | accept | Descriptor names are not secrets (same reasoning as T-16-02). |
| T-16-08 | Elevation of privilege | Using upsert to bypass a derivation's intended aggregation logic | mitigate | The 400 gate makes this impossible — derivations cannot be written to directly, so their computed values remain authoritative. |
</threat_model>
