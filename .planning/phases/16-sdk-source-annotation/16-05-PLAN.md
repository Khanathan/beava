<objective>
Ship `with app.pipeline() as pipe:` — a framed-TCP strict-FIFO batching
context manager (D-11). Queues mixed ops (push / get / upsert / delete)
during the `with` block; flushes all queued ops in order on
context-manager `__exit__`; returns responses via `pipe.results()`
parallel to submission order.

Purpose: Python SDK users running a Redis-style multi-op batch (e.g. 20
feature `get`s for a scoring request) pay 20 RTTs today. Pipeline
collapses to ~1 RTT by leveraging Phase 2.5's framed-TCP strict-FIFO
primitive — no new server opcodes, no wire changes. HTTP users get a
clear error directing them to a TCP URL.

Output: new `_pipeline.py` module; `App.pipeline()` factory; HTTP
rejection with `BV-E-PIPELINE-HTTP`; 2 REQ-IDs closed.

**Scope note:** Python SDK only; zero server changes. Uses existing
Phase 2.5 framing + strict-FIFO request/response correlation.
</objective>

      ```python
      class Pipeline:
          def __init__(self, app):
              self._app = app
              self._queued: list[tuple[str, tuple]] = []  # (op_name, args) pairs
              self._results: list = []
              self._entered = False

          def __enter__(self):
              self._entered = True
              return self

          def __exit__(self, exc_type, exc, tb):
              # Always flush — even on exception path, so user sees any server-side
              # responses for the queued ops. Suppress no exceptions.
              if exc_type is None:
                  self._flush()
              else:
                  # On exception we choose NOT to send queued ops (user's code
                  # decided the batch is poisoned). Clear queue; let exception propagate.
                  self._queued.clear()
              return False

          def push(self, event_desc, row: dict) -> None:
              self._check_entered()
              self._app._check_push_target(event_desc)  # Phase 12 hook; reject derivation
              self._queued.append(("push", (event_desc._name, row)))

          def upsert(self, target, row: dict) -> None:
              self._check_entered()
              self._app._check_upsert_target(target)     # fail-fast: BV-E-PUSH-TO-DERIVATION
              self._queued.append(("upsert", (target._name, row)))

          def delete(self, target, *, key: dict) -> None:
              self._check_entered()
              self._app._check_upsert_target(target)
              self._queued.append(("delete", (target._name, key)))

          def get(self, feature_desc, key) -> None:
              self._check_entered()
              self._queued.append(("get", (feature_desc._name, key)))

          def results(self) -> list:
              return list(self._results)

          def _check_entered(self):
              if not self._entered:
                  raise RuntimeError("Pipeline used outside its `with` block")

          def _flush(self):
              # Single TCP connection; send all frames back-to-back, then recv in order.
              transport = self._app._transport  # TcpTransport instance
              frames = []
              for op_name, args in self._queued:
                  frames.append(_encode_op(op_name, args))
              responses = transport.send_framed_batch(frames)  # strict-FIFO recv
              self._results = [_decode_response(op_name, r) for (op_name, _), r in zip(self._queued, responses)]
      ```
