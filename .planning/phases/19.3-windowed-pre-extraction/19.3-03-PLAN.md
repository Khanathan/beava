The inline Variance arm MUST follow the exact algebraic ordering of `VarianceState::update_pre` at `agg_state.rs:357-369` — `delta = x - mean; n += 1; mean += delta / n; delta2 = x - mean; m2 += delta * delta2;` — line-for-line. No reordering, no algebraic shortcuts. Failure mode: Welford "close enough" but bit-different state silently corrupts variance/stddev queries that compare across windowed and non-windowed registrations.

Purpose: collapse the per-call dispatch tax for the six highest-frequency Tier-1 windowed kinds.
Output: 6 inline match arms inside `WindowedOp::update_at` + 2 unit tests (Welford bit-identical, parametrized invariants).

```rust
pub struct VarianceState {
    pub n: u64,
    pub mean: f64,
    pub m2: f64,
}

impl VarianceState {
    /// Welford online variance update.
    pub fn update_pre(&mut self, pre_val: Option<&Value>, where_matched: bool) {
        if !where_matched { return; }
        let Some(v) = pre_val else { return; };
        let Some(x) = v.as_f64() else { return; };
        // Lines 364-368 — the canonical 5-line Welford ordering:
        let delta = x - self.mean;
        self.n += 1;
        self.mean += delta / (self.n as f64);
        let delta2 = x - self.mean;
        self.m2 += delta * delta2;
    }
}
```

CountState / SumState / MinState / MaxState / AvgState all expose similar `update_pre(pre_val, where_matched)` methods at agg_state.rs (lines 126, 175, 231, 289, 458). Reference algorithms:
- Count: `if where_matched { self.n += 1; }`
- Sum: `if where_matched && let Some(v) = pre_val.and_then(|v| v.as_f64()) { self.total += v; }`
- Min: `cur = cur.map_or(v, |c| c.min(v))`
- Max: symmetric
- Avg: `(total, n)` tuple accumulator

From `crates/beava-core/src/agg_op.rs:181` (`AggKind::supports_windowed_wrap`): the 6 Tier-1 kinds Count/Sum/Min/Max/Avg/Variance ALL support windowed wrap. (Researcher confirmed in RESEARCH.md §2 Q3.)

From `crates/beava-core/src/agg_op.rs` — `AggOp::Count(CountState)`, `AggOp::Sum(SumState)`, etc. The windowed bucket holds `Box<AggOp>` → match on the inner variant.
