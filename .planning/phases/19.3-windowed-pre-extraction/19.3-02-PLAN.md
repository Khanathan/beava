<objective>
Land sub-goal **19.3-A**: extend Plan 19.2-01's pre-extraction protocol across the WindowedOp wrapper. Add `WindowedOp::update_at` that mirrors the non-windowed pre-extracted signature and dispatches per-bucket via `AggOp::update_with_extracted` instead of `AggOp::update_with_row`. Reroute `AggOp::update_with_extracted::Windowed(…)` arm at `agg_op.rs:867-869` to call the new method.

This eliminates two per-call costs that Phase 19.2 measured at ~3,900 ns/event total across fraud-team's 60 windowed updates per event (per `19.2-INVESTIGATION.md` §4):
1. Inner `row.get(fname)` linear scan over 19-field Txn rows (~50 ns × 60 = 3,000 ns)
2. Inner `evaluate_where_predicate` re-evaluation (~5 ns × 60 = 300 ns)
3. Outer→inner double-dispatch tax (~10 ns × 60 = 600 ns)

Predicted: 14,059 ns → ~10,150 ns agg-stage; 70k → ~95k EPS. **Floor (75% of predicted): -2,925 ns** — if measured lift undershoots, Plan 19.3-03 does not start (D-02 measurement gate).

Purpose: protocol extension across the SOLE wrapper layer (per RESEARCH.md Q4 — no RetractableOp exists; WindowedOp is the only wrapper variant of AggOp).
Output: working fast-path method + rerouted dispatch arm + measured EPS lift.
</objective>

pub fn update_with_row(
    &mut self,
    row: &Row,
    event_time_ms: i64,
    field: Option<&str>,
    where_expr: Option<&std::sync::Arc<crate::expr::Expr>>,
) {
    let epoch = self.bucket_epoch(event_time_ms);
    if let Some(pos) = self.position_for_epoch(epoch) {
        self.buckets[pos].1.update_with_row(row, event_time_ms, field, where_expr);
        return;

    if self.buckets.len() >= self.max_buckets {

    new_op.update_with_row(row, event_time_ms, field, where_expr);
    self.buckets.push((epoch, new_op));
}
