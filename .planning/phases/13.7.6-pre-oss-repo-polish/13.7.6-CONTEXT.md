### D-01 (7.6-A) — History shape = keep + strip `.planning/` + strip `CLAUDE.md` via `git filter-repo` (USER-LOCKED)

**Procedure:**
```bash
# Run on a fresh clone — NEVER on the working tree
git clone --bare <source> beava-bare.git
cd beava-bare.git
git filter-repo \
  --invert-paths \
  --path .planning/ \
  --path CLAUDE.md \
  --path .claude/ \
  --commit-callback '...trailer-strip per D-02...' \
  --mailmap mailmap.txt
```

Result: real ~2400-commit engineering narrative preserved; `git blame` works to design decisions; `.planning/` + `CLAUDE.md` + `.claude/` never appear in any tree of any commit. Force-push to fresh `github.com/beava-dev/beava` repo.

**Combined effects:**
- Real history visible (~2400 commits)
- Author = `Hoang Phan <hoang@beava.dev>` on every commit (D-04)
- No `Co-Authored-By: Claude` trailers anywhere (D-02)
- No `🤖 Generated with [Claude Code]` markers anywhere (D-02)
- No `.planning/` business reasoning visible
- No `CLAUDE.md` AI-tool branding visible
- No `.claude/` skill / worktree noise visible

**Private dev branch retention:** keep current `v2/greenfield` (or whatever local branch) as the private-dev branch with `.planning/` + `CLAUDE.md` + `.claude/` intact. Public push goes to a separate fork / origin.

**Mechanical risk:** `filter-repo` is destructive on the working tree it runs against. Plan 13.7.6-09 uses `--bare` clone + force-push pattern, never operates on the dev clone.

### D-02 (7.6-B) — Subject-rewrite scope = trailer-only surgical strip (USER-LOCKED)

`git filter-repo --commit-callback` removes ONLY:
- The `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` trailer line(s).
- The `🤖 Generated with [Claude Code]` marker line(s).
- Any blank trailing line that becomes orphaned by trailer removal.

**Preserve:** every commit subject + body otherwise. Including:
- `chore: update CLAUDE.md with TDD rule` (legitimate filename reference; survives even though `CLAUDE.md` itself is stripped from trees).
- `fix: align Phase 12.7 tripwire after Claude resumed mid-Phase` (subject mentions the AI tool by name; preserved — historical accuracy > cosmetic purity).
- Phase / plan / task / wave numbers in commit subjects (engineering narrative).

**Rationale:** trailer-only is surgical — predictable, scriptable, no risk of corrupting legitimate filename / phase references. Aggressive subject rewriting was rejected (corrupts `CLAUDE.md` filename refs even though the file gets stripped).

### D-06 (7.6-E follow-up) — Public-facing distillation of CLAUDE.md content = SKIP (USER-LOCKED)

Per user override: architectural invariants are NOT publicly documented in v0. CONTRIBUTING.md ships only:
- Test commands (`cargo test --workspace --features testing`)
- Clippy + fmt commands (`cargo clippy -D warnings`, `cargo fmt --check`)
- Python checks (`pytest python/tests`, `mypy --strict python/beava`)

Architectural invariants (mio-only, events-only, memory-governance) are enforced by tripwire tests already in CI. First PR breaking one will surface the rule organically. Saves ~200 lines of CONTRIBUTING / docs/architecture/ writing; lower maintenance cost.
