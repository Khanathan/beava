### `cargo deny check bans` — wildcards relaxed for workspace-internal path deps

Workspace-internal `path = "../<crate>"` dependencies don't carry version specs (idiomatic for crates not published to crates.io). v0 publishes only one Python wheel — internal crates (`beava-core`, `beava-runtime-core`, `beava-persistence`, `beava-bench`) stay path-only. Set `wildcards = "allow"` + `allow-wildcard-paths = true` in deny.toml. Tighten to `"deny"` + add `version = "0.x.y"` to each `path = ...` dep in v0.1+ if any internal crate gets published.
