# Phase 13.7.6 Plan 28: Honest 8-cell sustained throughput rebaseline (post-Plan-27)

Reruns the 4-shape × 2-transport throughput matrix using the post-Plan-13.7.6-27 fixed bench (sustained-mode no longer deadlocks at deadline; `sustained_eps:` label only emits when elapsed ≥ 95% of duration). Appends a new section to `.planning/throughput-baselines.md` with the honest sustained numbers and headline figure for the Phase 13.8 ship-pitch + README rewrite.

## Why this plan

Plan 13.7.6-27 fixed two related launch-blocker bugs in `beava-bench-v18`: (1) the deadline-only sustained path deadlocked at shutdown when the sender was parked on `Semaphore::acquire_owned()`; (2) the human-report `sustained_eps:` label was emitted unconditionally, mislabelling 1.5-second burst rates from `--total-events`-capped runs as 60-second sustained rates. Most committed numbers in `.planning/throughput-baselines.md` through Phase 13 were therefore burst rates. The Plan 27 SUMMARY explicitly defers rebaselining to Plan 28 — this plan.

## What ran

The plan's 8-cell sweep, single run per cell, `--duration-secs 60 --parallel 16 --pipeline-depth 1024 --no-ledger` (no `--total-events` cap):

```bash
cargo build --workspace --release  # cache-hit; 15.11s
cd crates/beava-bench
for SHAPE in small medium large fraud-team; do
  for TRANSPORT in tcp http; do
    WIRE=$([ "$TRANSPORT" = "tcp" ] && echo "msgpack" || echo "json")
    timeout 90 ../../target/release/beava-bench-v18 \
      --pipeline "$SHAPE" \
      --transport "$TRANSPORT" \
      --wire-format "$WIRE" \
      --blast-shape zipfian \
      --cardinality 10000 \
      --duration-secs 60 \
      --parallel 16 \
      --pipeline-depth 1024 \
      --no-ledger
  done
done
```

All 8 cells completed; 0 deferred. Per Plan 27 SUMMARY § Out of scope, `--parallel 32 --pipeline-depth 1024` would still hang on a separate server-side `semaphore_wait_trap` starvation issue; moderate parallelism (`--parallel 16`) is the canonical rebaseline shape and is what this plan ran.

## Headline

**The honest sustained rebaseline tells a more nuanced story than "burst rates are inflated by 5-10×."**

- **TCP small / medium / large** sustained ≈ committed burst within ±7%. Committed numbers were already honest within measurement variance; the post-Plan-27 sustained-rate methodology mostly **confirms** them.
- **TCP fraud-team** sustained at **125,061 EPS** is **+48% above** the prior committed burst median (84,290 EPS). Prior runs were genuinely depressed by box-load — Phase 13.7.5's own SUMMARY tags the median −20% measurement variance. The honest 60-second figure is **125 K EPS sustained on the 14-node × 110-feature primary tuning bench**.
- **HTTP small** sustained at **69,421 EPS** is **−25% below** the WEBSITE-GAPS Gap-43 user-measured number (92,088 EPS). The difference is fan-out (Gap 43 used `--parallel 32 --duration 30s`; this run uses `--parallel 16 --duration-secs 60` per Plan 27's moderate-parallelism constraint). Lower fan-out → fewer concurrent HTTP requests → lower throughput; this is the honest moderate-fan-out figure.

The headline number for v0 ship-pitch is **660 K EPS sustained / 60 s on small/tcp** (Apple-M4 / 10 cores). This is the figure that should land in the README rewrite (Plan 13.7.6-29; WEBSITE-GAPS §43).
