<objective>
Run THREE secret-detection tools against the full git history (~2443 commits) and surface every finding into a single auditable report. Tools: `gitleaks` (default rules), `trufflehog` (default + entropy heuristics), and `git secrets` (configured AWS + GitHub patterns). Every finding gets disposition. TRUE-POSITIVEs (real leaked secrets) get fed into Plan 11's filter-repo `--path` invocation list AND the underlying credential is rotated at the provider BEFORE the rewrite runs.

Per CONTEXT.md D-08 Workstream-C-8 + CONTEXT.md "Claude's Discretion": run BOTH tools (gitleaks + trufflehog) regardless of order; planner picks order.

Specific patterns to flag (per CONTEXT.md "Critical constraints" + D-08): `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `AWS_*` (AKIA / aws_secret_access_key), `GITHUB_TOKEN` / `GH_TOKEN` / personal-access-tokens (`ghp_*`, `github_pat_*`), `BEAVA_*` (defensive — should be config not credentials), private-service URLs (internal grafana / prometheus / dashboards), employee-PII patterns.

Purpose: Workstream-C requirement; the filter-repo run in Plan 11 is the LAST chance to strip anything from history. If a real secret has been committed and pushed, it's already considered compromised — but stripping limits future discovery.

Output: Secrets-sweep report; extra-strip-paths file (possibly empty); any rotated-credentials log inside the report.
</objective>
