## Rollback procedure

This rehearsal made **zero changes to the dev tree**. The rehearsal clone at
`/tmp/beava-rehearsal-bare.git` was created with `git clone --bare`, mutated with `git
filter-repo`, and then `rm -rf`'d at the end of Task 2. The dev tree's HEAD before and after
the rehearsal is unchanged (verified post-tear-down).

**If anything goes wrong in Plan 11 (the actual run):**

1. **The dev tree is the source of truth and is never touched.** Plan 11 also runs against
   `/tmp/beava-bare.git/` (or equivalent), so a failed Plan 11 means: `rm -rf /tmp/beava-bare.git`
   and start over from this plan (re-rehearse with corrected artifacts).
2. **If Plan 11 already force-pushed to `github.com/beava-dev/beava` and the result is wrong,**
   the public repo can be deleted via GitHub UI, the artifacts (`13.7.6-09-mailmap.txt` /
   `13.7.6-09-trailer-strip.py`) corrected, this plan re-rehearsed, then Plan 11 re-run. The
   dev tree provides the original `~2600`-commit history at any time.
3. **Pre-Plan-11 belt-and-braces snapshot:** before Plan 11 runs filter-repo against its bare
   clone, tag the dev tree's current HEAD (`git tag pre-13.7.6-rewrite-backup`) and push that
   tag to a private remote (or just keep it locally). If catastrophe strikes, `git checkout
   pre-13.7.6-rewrite-backup` recovers exactly the pre-rewrite state.
4. **Why the dev tree is safe:** `git filter-repo`'s only mode of corruption is to operate on
   the wrong repo. By contract (CONTEXT.md "DESTRUCTIVE OPERATIONS" + plan-checker rule 1),
   Plan 11 runs against `/tmp/beava-bare.git/` — never the dev tree. The rehearsal in this plan
   confirms the procedure is mechanically sound; the only remaining risk class for Plan 11 is
   "wrong cwd at filter-repo invocation," and that's mitigated by Plan 11's preamble guard
   (`pwd` must end in `.git` and contain no `objects/info/alternates` linking back to the dev
   tree's `.git/objects/`).
