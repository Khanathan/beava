# Plan 13.7.6-24 — Strip lying `--parallel` flag from `beava-bench throughput`

**Status:** PASS
**Wave:** 2 (Workstream-G — bench-tooling fix from WEBSITE-GAPS.md §44)
**Type:** code change with TDD red-then-green per CLAUDE.md §item 1

## What landed

Removed the `--parallel <u32>` flag from the `beava-bench throughput` subcommand. Pre-fix, `beava-bench throughput --parallel 32` produced ~1K EPS while `beava-bench-v18 --parallel 32` produced ~125K EPS on the same workload — the polished CLI's `--parallel` was advertised in `--help` but silently discarded by the harness at `let _ = parallel;`.

## Changes

- `crates/beava-bench/src/cli/throughput.rs` — removed `pub parallel: u32` field from `ThroughputArgs`; removed `args.parallel` from the `harness::run_throughput_acks_one(...)` call. Added explicit module-doc note pointing users to `beava-bench-v18` for production benchmarking.
- `crates/beava-bench/src/harness/mod.rs::run_throughput_acks_one` — removed the `parallel: u32` parameter; removed the `let _ = parallel;` no-op site. Added doc note about the strip.
- `crates/beava-bench/tests/parallel_flag_removed.rs` (new) — 2 integration tests via `assert_cmd`:
  - `throughput_help_does_not_advertise_parallel_flag` — asserts `beava-bench throughput --help` stdout does not contain `--parallel`
  - `throughput_rejects_parallel_argument` — asserts `beava-bench throughput --parallel 32` is rejected by the clap parser
