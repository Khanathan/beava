## Decisions Made

1. **Match the plan-prescribed structure verbatim.** The plan body specified the four-row decision-tree table and per-binary section layout; I used that exact structure with two additions (caveats footer; bundled-pipelines list with `fraud-team.json`). Rationale: §44 is a docs-discoverability bug; the plan structure already resolves it. Inventing a different shape would expand scope without closing the gap any better.
2. **Document `fraud-team.json` + historical configs.** `configs/` ships 10 JSON files; if the README only lists `small/medium/large` users will assume the others are stale or experimental. Listing them with one-line provenance (primary tuning bench / sketch variants / phase-locked / geo) matches the project memory note `project_fraud_team_primary_bench` and prevents follow-on questions.
3. **Surface the `--parallel N` no-op in prose, not just the table.** WEBSITE-GAPS.md §44 names this as the immediate trigger for the whole plan. The decision-tree table already routes users away from `beava-bench` for production benchmarks, but a bolded sentence in the smoke-test section is the durable warning until the engine fix (§44 option A or B) lands.
4. **Cross-link `.planning/throughput-baselines.md` and `.planning/perf-baselines.md`.** These are the source-of-truth files for the regression-gate rules; the README is the on-ramp. Two relative `../../.planning/` links (no absolute paths, no GitHub-org-specific URLs) so the README works locally and on github.com identically.
5. **Single `chore:` commit, no test pair.** Per CLAUDE.md §TDD Discipline item #7 doc-only-plan exemption. The plan body explicitly invokes this exemption.
