**2. [Rule 2 — Missing critical functionality] No explicit `permissions:` block on 4 of 6 workflows**
- **Found during:** Task 1 audit.
- **Issue:** `ci.yml`, `pr.yml`, `bench-nightly.yml`, `homebrew-bump.yml` had no top-level `permissions:` block, so `GITHUB_TOKEN` defaulted to whatever the repo / org permissive-default is. OWASP / GitHub-Actions hardening guide best-practice: default-deny + minimal grants per workflow.
- **Fix:** Added `permissions: contents: read` workflow-level on all 4 (none of these workflows write to the repo via `GITHUB_TOKEN`; `homebrew-bump.yml` writes to the external tap repo over an SSH deploy key, not the GitHub token).
- **Files modified:** `.github/workflows/{ci,pr,bench-nightly,homebrew-bump}.yml`.
- **Verification:** `grep -q '^permissions:' .github/workflows/*.yml` — all 6 pass.
- **Committed in:** combined `chore(13.7.6-22): ...` commit.

**3. [Rule 1 — Bug] `pr.yml` + `release.yml` test commands silently skip `testing`-gated integration tests**
- **Found during:** Task 1 audit (CLAUDE.md TDD-§ command parity check).
- **Issue:** `cargo test -- --test-threads=1` (without `--workspace --all-features`) does NOT enable the `testing` feature on `crates/beava-server`. Roughly 50+ test entries under `crates/beava-server/Cargo.toml` gate on `required-features = ["testing"]` (`reqwest` + `tempfile`-using integration tests); cargo silently skips those tests when the feature is off. Result: every PR check + every release pre-flight had been running with that test surface dark for an unknown length of time.
- **Fix:** Updated to canonical `cargo test --workspace --all-features -- --test-threads=1` in both `pr.yml` and `release.yml`. Inline comment added explaining why both flags are load-bearing. `--test-threads=1` retained (serializes integration tests that bind ports).
- **Files modified:** `.github/workflows/pr.yml`, `.github/workflows/release.yml`.
- **Verification:** `grep -E 'cargo test --workspace --all-features -- --test-threads=1'` matches in both files.
- **Committed in:** combined `chore(13.7.6-22): ...` commit.

**4. [Rule 1 — Bug] `pr.yml` + `ci.yml` clippy lines drop `--workspace`**
- **Found during:** Task 1 audit (TDD-§ command parity).
- **Issue:** `cargo clippy --all-targets ...` only lints the current crate; CLAUDE.md canonical requires `cargo clippy --workspace --all-targets --all-features ...` to lint every crate in the workspace. Without `--workspace`, lint coverage of `crates/beava-persistence`, `crates/beava-runtime-core`, etc. depends on whichever crate cargo picks as default.
- **Fix:** Added `--workspace` to `ci.yml` line 40 and `pr.yml` line 41. `pr.yml` additionally gained `--all-features` to match canonical.
- **Files modified:** `.github/workflows/ci.yml`, `.github/workflows/pr.yml`.
- **Verification:** `grep -E 'cargo clippy --workspace --all-targets --all-features -- -D warnings'` matches in both files.
- **Committed in:** combined `chore(13.7.6-22): ...` commit.

**5. [Rule 1 — Bug] `pr.yml` `cargo fmt --check` drops `--all`**
- **Found during:** Task 1 audit.
- **Issue:** `cargo fmt --check` on a workspace only checks the manifest's package; misses other workspace members. CLAUDE.md canonical: `cargo fmt --all --check`.
- **Fix:** Added `--all`.
- **Files modified:** `.github/workflows/pr.yml`.
- **Verification:** `grep -E 'cargo fmt --all --check' .github/workflows/pr.yml` matches.
- **Committed in:** combined `chore(13.7.6-22): ...` commit.
