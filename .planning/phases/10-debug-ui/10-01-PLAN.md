## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build-time fetch → repo | `curl` pulls three files from unpkg.com once at vendor time. Boundary crossed exactly once, by a human, under supervision. No runtime fetch. |
| Repo → binary | `cargo build` walks `src/server/ui/` via `rust-embed` and embeds the exact bytes committed to git. No external input at build time. |
| Binary → browser | Future Plan 03/04 work. Not in scope for this plan. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-A01 | Tampering | Vendored JS bundle replaced by a malicious version between download and commit | mitigate | VENDOR.md records SHA256 at commit time; Plan 05 tests re-hash and compare at test time, failing CI if bytes drift (RESEARCH §Pitfall 6). |
| T-10-A02 | Tampering | Vendor file updated on-disk in a future PR without updating VENDOR.md | mitigate | Same drift-detection tests in Plan 05 fail the build if any hash mismatches. |
| T-10-A03 | Information Disclosure | Source-map files leaking minified JS internals | accept | Source maps are not downloaded (no `*.map` file), nothing to disclose. |
| T-10-A04 | Tampering | `rust-embed` picking up a stray file outside `src/server/ui/` | accept | `#[folder = "src/server/ui/"]` in Plan 03 scopes the embed walker. No path-traversal possible at compile time. |
| T-10-A05 | Denial of Service | Dependency resolution pulling an unexpected `rust-embed` major version via `^8.11` | mitigate | Plan 05's `cargo test` covers the full embed path; any resolver breakage surfaces there before merge. |
</threat_model>
