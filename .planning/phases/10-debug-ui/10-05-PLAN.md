**Helper B — `http_get`:** raw HTTP/1.1 GET over tokio TcpStream. Returns `(status_code, headers_lowercased, body_bytes)`.

```rust
async fn http_get(port: u16, path: &str) -> (u16, std::collections::HashMap<String, String>, Vec<u8>) {
    let mut stream = TcpStream::connect(("127.0.0.1", port)).await.unwrap();
    let req = format!(
        "GET {} HTTP/1.1\r\nHost: localhost\r\nConnection: close\r\n\r\n",
        path
    );
    stream.write_all(req.as_bytes()).await.unwrap();
    stream.flush().await.unwrap();
    let mut buf = Vec::new();
    stream.read_to_end(&mut buf).await.unwrap();

    // Split headers from body at \r\n\r\n
    let sep = buf.windows(4).position(|w| w == b"\r\n\r\n")
        .expect("no header/body separator");
    let head = std::str::from_utf8(&buf[..sep]).unwrap().to_string();
    let body = buf[sep + 4..].to_vec();

    // Parse status line
    let mut lines = head.lines();
    let status_line = lines.next().unwrap();
    let status_code: u16 = status_line.split_whitespace()
        .nth(1).unwrap().parse().unwrap();

    let mut headers = std::collections::HashMap::new();
    for line in lines {
        if let Some((k, v)) = line.split_once(':') {
            headers.insert(k.trim().to_lowercase(), v.trim().to_string());
        }
    }
    (status_code, headers, body)
}

fn body_string(bytes: &[u8]) -> String {
    String::from_utf8_lossy(bytes).to_string()
}

fn body_json(bytes: &[u8]) -> serde_json::Value {
    serde_json::from_slice(bytes).expect("valid JSON body")
}
```

fn push_event(state: &SharedState, stream: &str, event: &serde_json::Value) {

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test process → live axum server | Tests send raw HTTP/1.1 over loopback. Same trust surface as any local HTTP client; no untrusted input. |
| Test process → VENDOR.md manifest | Filesystem read of a committed manifest. If the manifest is tampered with AND the vendored JS is also tampered with the same way, the hash check passes. This is why Plan 01 commits both the bytes and the hash in the same PR — a meaningful attack has to modify both files consistently. |
| CI runner → vendored JS | If CI mutates the vendored files between `cargo test` runs (unlikely), the hash check fails loudly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-08 | Tampering | Vendored JS swapped for a malicious version post-vendoring | mitigate | Three tests (`static_*_is_vendored_and_hashed`) compute the SHA256 of the served bytes at test time and compare against `src/server/ui/VENDOR.md`. Any single-byte change fails the test. |
| T-10-11 | Information Disclosure | Test server binds to 0.0.0.0 and listens on a well-known port | accept | `start_debug_ui_server` binds to `127.0.0.1:0` (loopback + random port). No external exposure. |
| T-10-12 | Tampering | The stale `tests/test_server.rs` AppState literal hides a real-world bug by bypassing the compile step | mitigate | Task 1 restores the file to a full 12-field literal so every future AppState change must update this site or fail `cargo test`. |
| T-10-13 | DoS | A test leaves the axum task running and leaks ports across tests | accept | Each `start_debug_ui_server` spawns a fresh tokio task bound to port 0 (random). Tokio's test runtime shuts the runtime down at the end of each `#[tokio::test]`. No cross-test leakage. |
| T-10-14 | Repudiation | A flaky `throughput_decays_when_idle` test masks a decay regression | mitigate | Use a fixed 250 ms sleep and assert monotone decay (strict less-than), not a specific magnitude. If flaky in CI, the acceptance criteria allows widening to 500 ms — document the wider value in the test comment, do not add a retry loop. |
