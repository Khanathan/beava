## Critical Anti-Patterns (do NOT repeat these)

| Pattern | Severity | Prevention |
|---------|----------|-----------|
| Adding a 16px typography token or a 12px spacing token | advisory | UI-SPEC §3.2 and §2.1 explicitly prohibit them. Plan-checker already verified plan acceptance criteria forbid these. If you need 16px for a metric, use 14px mono; if you need 12px padding, use 8px or 16px. |
| Double-counting throughput on cascade/fan-out | advisory | Plan 10-02 mandates `HashSet<String>` per-push dedup with a dedicated unit test (`throughput::does_not_double_count_cascade`). |
| Forgetting `list_views()` when building /debug/topology nodes | advisory | Views live in a separate AHashMap. Plan 10-03 acceptance criteria reference `list_views()` explicitly. |
| Adding a new /debug/memory/per_stream endpoint | advisory | CONTEXT.md and RESEARCH §Pitfall 8 require additive extension of the existing /debug/memory response. |
| Using older axum path syntax `*file` or `:file` | advisory | axum 0.8 requires `{*file}`. Plan 10-03 task has the exact route literal in acceptance criteria. |
| Committing vendored d3/dagre-d3 bytes without the browser smoke test | **blocking** | 10-01 Task 3 is the checkpoint:human-verify. The orchestrator MUST pause and wait for user sign-off. RESEARCH §A3 flagged this as the only HIGH-risk assumption — fall back to d3 v5 if dagre-d3 0.6.4 breaks. |
| Using `.innerHTML` for user-provided data in the Entity tab | **blocking** | Plan 10-04 requires `.textContent` for DOM writes of entity keys and feature values. 10-04 Task 4 includes an explicit paste-XSS smoke test with `<script>alert(1)</script>`. |
| Leaving `tests/test_server.rs` AppState literal stale | advisory | Plan 10-05 Task 2 is the fix. Without it, `cargo test` will fail after the new AppState fields land in Wave 2. |
