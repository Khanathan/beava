## V11 Snapshot Wire Layout

```
[V11_FORMAT=11][TYPE_TAG_BASE=0x00][postcard(TypedStateSnapshotV11)]

TypedStateSnapshotV11 {
    typed_entities: Vec<((String /* stream */, String /* entity_key */), TypedAggState)>,
    value_entities: Vec<(String /* entity_key */, SerializableEntityState)>,  // V10-compat fallback body
}

TypedAggState {
    schema_id: u32,
    schema_name: String,   // resilience across schema_id reassignment (T-59.6-05-02)
    payload: Vec<u8>,      // Row.payload bytes (byte-identical)
    arena: Vec<u8>,        // Row.arena bytes (byte-identical)
}

TypedRowEnvelope {          // same shape as TypedAggState; attached to SerializableEntityStateV11
    schema_id: u32,
    schema_name: String,
    payload: Vec<u8>,
    arena: Vec<u8>,
}
```

### Dispatch Rule

```
load_typed_state_v11(bytes):
    outer_byte == V11_FORMAT → postcard-decode TypedStateSnapshotV11 directly.
    outer_byte ∈ {V9_FORMAT, legacy V10-era byte (still V9 envelope)} →
        load_snapshot_file(bytes) → SnapshotFile::Base(v8) →
        wrap entities into TypedStateSnapshotV11.value_entities (typed_entities = empty).
    unknown byte → None.
```

**Transparent upgrade:** V10 bytes decode via fallback → next write emits V11 outer byte → subsequent reads hit the typed branch. Mirrors Phase 55/56/57 v7→v8→v9 bumps.

## Fjall Typed Persistence Layout

```
fjall PartitionHandle key layout (per-entity typed row):

    [TYPED_ROW_KEY_PREFIX=0xFF][stream_name_bytes][0x00][entity_key_bytes]

body (pure memcpy — no serde_json / postcard):

    [schema_id: u32 BE][payload_len: u32 BE][payload: payload_len bytes]
    [arena_len:  u32 BE][arena:   arena_len bytes]
```

- `encode_typed_row_body(row) → Vec<u8>`: memcpy-only encoder.
- `decode_typed_row_body(bytes) → Result<Row, BeavaError>`: validates every offset boundary (threat `T-59.6-05-01`); returns `BeavaError::Protocol` on short / out-of-bounds bodies — no panic.
- `put_entity_typed(partition, stream, key, row)`: encode + `PartitionHandle::insert`.
- `get_entity_typed(partition, stream, key) → Option<Row>`: `PartitionHandle::get` + decode.

**Coexistence with Value keyspace (SC-7):** Value-path entities are stored under `key = entity_key.as_bytes()` (raw UTF-8). Typed keys start with `0xFF` (invalid UTF-8 leading byte) — zero collision on the same partition. SC-7 test asserts both paths roundtrip cleanly side-by-side.

## Typed SSJ Dispatch (SC-9 + sharding_parity)

```
source shard (L-side event, schema'd Txns)
   ↓ shard_hint_for_event(event, Some("user_id")) → join-owning shard
   ↓ ShardOp::SsjInsertTyped { join_id, side=Left, join_key, row, schema_id, within_ms }
join-owning shard
   ↓ target-arm (Wave 5: empty reply + SSJ_CROSS_SHARD_TOTAL++; Wave 6 wires the match)
   ↓ target-arm (Wave 6+): apply_ssj_insert_typed
        ├─ TypedSsjBuffer.insert_left_and_match(op, join_key, row, now)
        │    ├─ evict_expired(right-side, within)
        │    ├─ for (r, _) in right[join_key]: op.match_typed(&row, r) → joined Row
        │    └─ left[join_key].push((row, now))
        └─ emit joined Rows via run_typed_cascade (Wave 7 hot path)
```

**SC-9 parity contract:** Operator-boundary parity — `StreamStreamJoinTyped.match_typed(left, right)` produces byte-identical Rows on the same event stream across N=1 and N=8 partitioning. The test partitions by `shard_hint_for_event(event, Some("user_id")) % N`, routing each event to a `TypedSsjBuffer` instance; the sorted joined output Vec<Row> is equal across shard counts.
