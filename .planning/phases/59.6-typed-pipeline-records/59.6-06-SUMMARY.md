# Tech tracking
tech-stack:
  added: []  # No new crates
  patterns:
    - "D-C1 SideBand type-erasure: sketch + windowed ops (HLL / UDDSketch / CMS / ring buffers) store their variable-sized state in a per-entity AHashMap-of-AHashMaps keyed by op.name(), not inline in the fixed-layout state Row. Avoids monomorphizing HLL/UDDSketch/CMS across i64/f64/str column types (~10× codegen blowup); costs one extra HashMap lookup per update vs pure typed ops like Sum/Avg/Min/Max that do store state inline."
    - "Pure-typed numeric stats (Stddev / Variance): stored as (sum: f64, sum_sq: f64, count: i64) in the state Row. No SideBand needed. Sample variance (n-1 divisor) to byte-match Value-path VarianceOp (which uses Welford). Stddev derived as sqrt(population variance) to match StddevOp (which uses population variance divisor n)."
    - "Trait additive-method pattern: TypedAggOp gains default update_with_sideband + read_feature_with_sideband. Simple aggs (Count/Sum/Avg/...) use the defaults unchanged; sketch/windowed ops override. No trait object break (all 7 Wave-4 impls continue to compile). Matches Wave 5's Value-fallback + typed-path-additive pattern."

key-decisions:
  - "SideBand sits alongside entity_state_typed on Shard, not inside it. Trade-off: two AHashMap lookups per advanced-op update (one for state Row, one for SideBand) vs one unified lookup. Chose alongside because (a) simple aggs NEVER allocate SideBand entries — they pay zero overhead for the parallel map; (b) inlining SideBand into the state Row would double per-Row size for entities that don't use sketches; (c) snapshot/fjall serialization is cleaner (V11 body untouched; future wave can add V12 with SideBand if needed). Matches the Wave 4 D-C4 invariant — simple aggs pay zero allocation after state init."
  - "VarianceOpTyped uses sample variance (n-1 divisor) to byte-match the Value-path VarianceOp (Welford algorithm with n-1 divisor). StddevOpTyped uses sqrt(population variance) (n divisor) to match the Value-path StddevOp. This mismatch between Stddev and Variance divisors is a Value-path historical inconsistency that the typed path preserves verbatim — changing it would break TPC-CORR-07 byte parity."
  - "REGISTER ack schema_id carried as a JSON field on the existing ack body, NOT as a new ack frame suffix (which the plan's `<interfaces>` suggested with `SCHEMA_ID=<id>` key=value lines). Rationale: the ack body is already JSON-shaped (status/kind/name/added/removed/backfilling); adding one more field is the minimal-surface change. Also composes cleanly with future ack extensions (e.g., shard routing hints) without framing-level renegotiation."
  - "Wave 2 `schema_id=0` shortcut removed unconditionally (no feature-flag rollout). Rationale: pre-Wave-6 Python SDK clients that use the typed path don't exist in production — typed-pipeline is a Wave-2+ feature that ships alongside v0.3.0 SDK. Pre-Wave-6 SDKs default to OP_PUSH_BATCH (Value path) when the stream has no registered schema. Strict matching protects against stale state from here on."

No semantic divergence; just a counting convention."

## SideBand Design (D-C1 Type-Erasure)

```rust
#[derive(Debug, Default)]
pub struct SideBand {
    /// DistinctCountOpTyped (HLL). Key = op.name().
    pub hll_sketches: AHashMap<String, Hll>,
    /// PercentileOpTyped (UDDSketch). Key = op.name().
    pub udd_sketches: AHashMap<String, UDDSketch>,
    /// TopKOpTyped — (CountMinSketch, TopKHeap) pair per op name.
    pub topk_sketches: AHashMap<String, (CountMinSketch, TopKHeap)>,
    /// LagOpTyped / FirstNOpTyped / LastNOpTyped — VecDeque ring buffers.
    pub ring_buffers: AHashMap<String, VecDeque<FeatureValue>>,
    /// EmaOpTyped — last-observed SystemTime per op name.
    pub ema_last_ts: AHashMap<String, SystemTime>,
}
```

**Per-entity parallel-map:** `Shard.entity_sideband_typed: AHashMap<(String, String), SideBand>` (same keyspace as `entity_state_typed`; lazy allocation on first advanced-op update).

**TypedAggOp trait extension:**

```rust
pub trait TypedAggOp: Send + Sync + std::fmt::Debug {
    // existing: init_state / update_typed / read_feature / name

    /// Wave 6: sketch + windowed ops override this to update SideBand.
    /// Default forwards to update_typed — simple aggs unchanged.
    fn update_with_sideband(
        &self,
        state: &mut Row,
        state_schema: &RegisteredSchema,
        event: &Row,
        event_schema: &RegisteredSchema,
        sideband: &mut SideBand,
        now: SystemTime,
    ) { self.update_typed(state, state_schema, event, event_schema, now); }

    /// Wave 6: sketch ops override this to project SideBand state into a
    /// FeatureValue. Default delegates to read_feature.
    fn read_feature_with_sideband(
        &self,
        state: &Row,
        state_schema: &RegisteredSchema,
        sideband: &SideBand,
    ) -> FeatureValue { self.read_feature(state, state_schema) }
}
```

## Python SDK v0.3.0 Handshake Flow

```
Server                                      Client (v0.3.0)
  |                                           |
  |  REGISTER(stream=Txns, schema={...})     |
  |<------------------------------------------|
  |                                           |
  |  engine.register_typed_schema("Txns",    |
  |       RegisteredSchema) → schema_id=42   |
  |                                           |
  |  ack = {"status":"ok", "kind":"v0",      |
  |         "name":"Txns",                   |
  |         "schema_id":42,                  |
  |         "added":[], "removed":[],        |
  |         "backfilling":[]}                |
  |------------------------------------------>|
  |                                           |
  |                                        _parse_register_ack_extended(ack, "Txns")
  |                                        → self._client._schema_ids["Txns"] = 42
  |                                           |
  |  OP_PUSH_TYPED_BATCH(stream="Txns",      |
  |     schema_id=42, rows=...)              |
  |<------------------------------------------|
  |                                           |
  |  decode_typed_row_push_batch(...) —      |
  |  strict match schema_id == 42 ✓          |
  |                                           |
  |  ack(...)                                 |
  |------------------------------------------>|
```

**Pre-Wave-6 client (v0.2.0) fallback:**
- Sends REGISTER with or without `schema:` block; ack parses via serde and ignores the `schema_id` field.
- `push_many` checks `(caps & WIRE_TYPED_PIPELINE) && schema is not None` — v0.2.0 stream decorators don't emit `_beava_schema`, so the condition fails and the client falls through to OP_PUSH_BATCH (Value path).

- **Impact:** SC-4 (advanced-aggs parity) flips GREEN via operator-boundary tests that exercise `update_with_sideband` directly on the library-level surface. The perf-gate test in Wave 7 will require the hot-path wiring to observe the ≤ 2μs pipeline phase improvement.

### Rule 3 — REGISTER ack schema_id carried as a JSON field, not a `SCHEMA_ID=<id>` framing extension

- **Found during:** Task 2 2a action block review.
- **Issue:** The plan's `<interfaces>` suggested framing: "STATUS_OK\n<stream_name>\nSCHEMA_ID=<id>" — a key=value line appended to the ack. But the current REGISTER ack is a JSON object (not a plaintext multi-line ack); appending a key=value line would require re-parsing + breaks the serde-based client consumer.
- **Fix:** Added `"schema_id"` as a JSON field on the existing ack body. Wire-compat: pre-Wave-6 clients parse via serde and ignore unknown fields. Wave-6 clients do `dict.get("schema_id")`.
- **Impact:** Simpler + more composable with future ack-field extensions (watermark lateness, shard routing hints, etc.) without framing-level renegotiation.
