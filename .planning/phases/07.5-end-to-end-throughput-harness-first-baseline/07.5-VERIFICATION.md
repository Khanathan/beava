## Open WARNINGs

1. **macOS fsync ceiling** (inherited from Phase 6). Throughput baseline
   is fsync-bottlenecked at ~1k EPS across all pipeline sizes because
   `F_FULLSYNC` is the bottleneck and per-op aggregation work is
   sub-microsecond. Documented in `throughput-baselines.md` notes; not
   a phase blocker — Phase 13 is the final ship gate where Linux numbers
   matter.

2. **TCP `OP_PUSH` not implemented**. Harness CLI supports `--transport tcp`
   and the TCP wire is bound; server-side handler returns
   `op_not_implemented`. The harness reports n/a honestly. Not a
   Phase 7.5 blocker — was always future-phase scope per the wire spec.

3. **`cli_smoke` port-race flake** (carried from Phase 7). Pre-existing;
   passes with `--test-threads=1`; deferred to Phase 8+ test-infra cleanup.
