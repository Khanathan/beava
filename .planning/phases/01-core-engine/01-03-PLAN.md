1. Define EvalContext -- the context passed to the evaluator containing feature values and event data:
```rust
use crate::types::FeatureValue;

/// Context for evaluating expressions. Provides feature values and event data.
pub struct EvalContext<'a> {
    /// Features for the current entity (from all streams).
    /// Key format: "feature_name" for local, "StreamName.feature_name" for qualified.
    pub features: &'a ahash::AHashMap<String, FeatureValue>,
    /// The current event JSON (for _event.field access).
    pub event: Option<&'a serde_json::Value>,
}

impl<'a> EvalContext<'a> {
    pub fn resolve_field(&self, field_ref: &FieldRef) -> FeatureValue {
        match field_ref {
            FieldRef::Local(name) => {
                self.features.get(name).cloned().unwrap_or(FeatureValue::Missing)
            }
            FieldRef::Qualified(stream, field) => {
                let key = format!("{}.{}", stream, field);
                self.features.get(&key).cloned().unwrap_or(FeatureValue::Missing)
            }
            FieldRef::Event(field) => {
                match self.event {
                    Some(ev) => match ev.get(field) {
                        Some(serde_json::Value::Number(n)) => {
                            if let Some(i) = n.as_i64() {
                                FeatureValue::Int(i)
                            } else if let Some(f) = n.as_f64() {
                                FeatureValue::Float(f)
                            } else {
                                FeatureValue::Missing
                            }
                        }
                        Some(serde_json::Value::String(s)) => FeatureValue::String(s.clone()),
                        Some(serde_json::Value::Bool(b)) => FeatureValue::Int(if *b { 1 } else { 0 }),
                        _ => FeatureValue::Missing,
                    }
                    None => FeatureValue::Missing,
                }
            }
        }
    }
}
```

2. Implement `eval` function:
```rust
pub fn eval(expr: &Expr, ctx: &EvalContext) -> FeatureValue
```

Evaluation rules per CONTEXT.md locked decisions:
- **Missing propagation (ENG-08):** Any arithmetic/comparison operation with a Missing input returns Missing. SQL NULL semantics, NOT JavaScript falsy.
- **Division by zero (ENG-08):** Returns Missing (not panic, not NaN, not Inf).
- **NaN/Infinity guard (ENG-08, RESEARCH.md threat register):** After every f64 arithmetic result, check: `if result.is_nan() || result.is_infinite() { return FeatureValue::Missing }`. This catches edge cases like `f64::MAX + f64::MAX` (overflow to infinity) or pathological sub-expressions that bypass the div-by-zero check. This is a defense-in-depth measure required by the STRIDE threat register.
- **Type promotion:** Int + Int -> Int. If either operand is Float, promote to Float. String in arithmetic -> Missing.
- **Comparison:** Returns Int(1) for true, Int(0) for false. String == String works. String > Int -> Missing.
- **Boolean:** `and`/`or` operate on Int(0)/Int(1). `not` flips Int(0)<->Int(1). Missing in boolean -> Missing (Pitfall 6).
- **Builtins:**
  - `abs(x)`: absolute value. Missing -> Missing.
  - `min(a, b)`: minimum of two values. Missing -> Missing.
  - `max(a, b)`: maximum of two values. Missing -> Missing.
  - `now()`: returns Float of current Unix timestamp in seconds. Use `SystemTime::now().duration_since(UNIX_EPOCH).unwrap().as_secs_f64()`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Expression string -> AST | User-defined expression strings parsed at registration time |
| Event JSON -> EvalContext | Event field values used in _event.field access during evaluation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Denial of Service | Expression parser | mitigate | Expressions are parsed once at registration time, not per-event. Deeply nested expressions are bounded by stack depth (winnow recursive descent). Consider adding a max-depth check at registration if needed in Phase 2. |
| T-01-08 | Denial of Service | Expression evaluator | mitigate | eval() is called per-event but walks a finite AST. No loops, no recursion beyond AST depth. Bounded by parse-time AST structure. |
| T-01-09 | Tampering | EvalContext field resolution | accept | Fields resolve from in-memory feature map. No external I/O. Missing field returns Missing, not an error. |
| T-01-10 | Information Disclosure | TallyError::Parse | accept | Parse errors include the problematic expression string. This is user-supplied and user-visible. No internal state leaked. |
</threat_model>
