# Plan 18-10 — Hand-rolled msgpack + sonic-rs JSON envelope parsing (target: 80 / 150 ns)

## Context (read first)

Plan 18-09 landed msgpack-on-TCP but profiling with `BEAVA_TRACE_SRV_TIMING=1` (Apple M4, parallel=1, n≥5,884) revealed msgpack is **2.3× slower** per-event than JSON, contradicting the plan's premise:

| Stage | TCP/JSON mean | TCP/MSGPACK mean | Δ msgpack vs JSON |
|---|---:|---:|---:|
| parse_wire_request | **583 ns** | **1,928 ns** | +1,345 ns (3.3× slower) |
| dispatch_push_sync | 2,428 ns | 5,041 ns | +2,613 ns (2.1× slower) |
| encode | 209 ns | 552 ns | +343 ns (2.6× slower) |
| **TOTAL** | **3,220 ns** | **7,522 ns** | **+4,302 ns** |

Root cause investigation (`crates/beava-runtime-core/src/tcp_listener.rs` + `crates/beava-server/src/apply_shard.rs`):

**`parse_wire_request` CT_MSGPACK branch (lines 129-150):**
```rust
rmp_serde::from_slice::<serde_json::Value>(&frame.payload)  // decode entire envelope to JsonValue (heap alloc)
extract event + body from Value::Object map
rmp_serde::to_vec_named(&body)                              // RE-ENCODE body back to msgpack
```

**`parse_wire_request` CT_JSON branch (lines 99-121):**
```rust
serde_json::from_slice::<PushEnvelopeJson>(&frame.payload)  // decode envelope (body as JsonValue)
serde_json::to_vec(&env.body)                               // RE-ENCODE body to JSON
```

**`dispatch_push_sync` (lines 197-220 in apply_shard.rs):**
```rust
let parsed: JsonValue = if body_format == CT_MSGPACK {
    rmp_serde::from_slice::<JsonValue>(&body)?              // decode body AGAIN to JsonValue
} else {
    sonic_rs::from_slice(&body)?                            // decode body to JsonValue
};
// then walk JsonValue → Row via json_value_to_beava_value()
```

The plan's stated goal "parse DIRECTLY into beava_core::row::Row (skip the serde_json::Value intermediate)" was NOT delivered. msgpack does **2× rmp_serde decode + 1× rmp_serde encode** through JsonValue; JSON does **1× sonic-rs decode + 1× serde_json decode + 1× serde_json encode** through JsonValue.

## Architecture decisions

### D-1 — Hand-rolled msgpack envelope via `rmp::decode` primitives

Bypass `rmp_serde` entirely for the envelope. The envelope is a fixed 2-key fixmap (`{event: str, body: any}`); serde's visitor dispatch and type erasure are pure overhead for a known shape.

```rust
pub fn parse_msgpack_envelope(payload: &[u8]) -> Result<(&str, &[u8]), ParseError> {
    let mut cursor = std::io::Cursor::new(payload);
    let map_len = rmp::decode::read_map_len(&mut cursor)?;
    if map_len != 2 { return Err(ParseError::EnvelopeShape); }

    let mut event_name: Option<&str> = None;
    let mut body_slice: Option<&[u8]> = None;

    for _ in 0..2 {
        let key_str_len = rmp::decode::read_str_len(&mut cursor)? as usize;
        let key_start = cursor.position() as usize;
        let key = std::str::from_utf8(&payload[key_start..key_start + key_str_len])?;
        cursor.set_position((key_start + key_str_len) as u64);

        match key {
            "event" => {
                let val_len = rmp::decode::read_str_len(&mut cursor)? as usize;
                let v_start = cursor.position() as usize;
                event_name = Some(std::str::from_utf8(&payload[v_start..v_start + val_len])?);
                cursor.set_position((v_start + val_len) as u64);
            }
            "body" => {
                let body_start = cursor.position() as usize;
                skip_msgpack_value(&mut cursor, payload)?;
                let body_end = cursor.position() as usize;
                body_slice = Some(&payload[body_start..body_end]);
            }
            _ => return Err(ParseError::UnknownKey(key.to_string())),
        }
    }
    Ok((
        event_name.ok_or(ParseError::MissingField("event"))?,
        body_slice.ok_or(ParseError::MissingField("body"))?,
    ))
}
```

`skip_msgpack_value` walks one msgpack value of any type by reading its marker byte and recursively skipping per the rmp spec (fixmap/map16/map32, fixarray/array16/array32, fixstr/str8/str16/str32, fixint/int8/int16/.../uint64, float32/float64, nil/bool, bin8/bin16/bin32). Implementation reads the rmp::Marker enum or matches on the first byte directly. ~50 LoC.

**Critically: `body_slice` is the EXACT raw msgpack bytes the client sent — passed through to dispatch_push_sync unchanged, then directly to WAL v=2 record body.** No re-encode. No allocation.

### D-2 — sonic-rs LazyValue for JSON envelope (with hand-rolled fallback)

Preferred approach: `sonic_rs::LazyValue` captures the raw byte range of a JSON value during deserialization. Use a struct deserialize:

```rust
#[derive(serde::Deserialize)]
struct EnvelopeLazy<'a> {
    event: &'a str,
    body: sonic_rs::LazyValue<'a>,
}

pub fn parse_json_envelope(payload: &[u8]) -> Result<(&str, &[u8]), ParseError> {
    let env: EnvelopeLazy<'_> = sonic_rs::from_slice(payload)?;
    Ok((env.event, env.body.as_raw_slice()))
}
```

If `LazyValue` isn't usable in derive(Deserialize) structs (sonic-rs 0.5 may not expose this; verify at implementation time), fall back to a hand-rolled brace-counting scanner:
- find `"event":"X"` via byte search (memchr or short scan)
- find `"body":{...}` and walk braces with string-state to capture body byte range
- ~120 LoC, target still ~150 ns

The fallback is decision-deferred to implementation. Either approach satisfies the must_have.

### D-3 — `dispatch_push_sync` skips JsonValue intermediate

Replace the `parsed: JsonValue` indirection with direct Row deserialization:

```rust
let row: Row = match body_format {
    CT_JSON    => sonic_rs::from_slice::<Row>(&body)?,    // uses RowVisitor (Task 9.3)

```

This requires `Row::Deserialize` (already exists from Plan 18-09 Task 9.3) to NOT internally re-allocate JsonValue. Audit `RowVisitor` impl — if it currently goes Row→JsonValue→Row, rewrite it to walk fields directly during `visit_map`.

The `validate_row_against_descriptor` helper replaces `validate_obj_against_descriptor` — walks `Row.fields` directly.

## Risks & mitigations

- **`sonic_rs::LazyValue` not derive-Deserialize-friendly in 0.5** — drops to hand-rolled fallback (brace-counting scanner). Adds 80-100 LoC but still hits the 150 ns target. Risk acknowledged in D-2.
- **`Row::Deserialize` (Task 9.3) may internally allocate JsonValue** — the SUMMARY says it walks via `serde_json::Value` field values. Audit + rewrite to walk MapAccess directly during 10.3.b. If the rewrite is non-trivial (>200 LoC), split into Task 10.3.5 to avoid bloating 10.3.
- **Hand-rolled msgpack `skip_msgpack_value` correctness** — must handle every rmp marker variant. Crib from rmp's existing `read_marker` walker. Property test (rmp_serde::to_vec roundtrip) is the safety net.
- **80 ns msgpack envelope target may be unreachable on M4 due to UTF-8 validation** — `std::str::from_utf8` does branch-heavy work even for ASCII. Mitigation: if profiling shows utf-8 dominates, gate validation behind a `debug_assertions` cfg or use `unsafe { from_utf8_unchecked }` (we control both ends — the SDK only ever sends UTF-8 event names). If we must use unsafe, add a property test that random fuzz inputs return Err instead of crashing.
- **Microbench-vs-trace divergence** — microbench measures pure parser cost; trace measures with surrounding mio loop overhead. 20% headroom in the verification line is intentional.
