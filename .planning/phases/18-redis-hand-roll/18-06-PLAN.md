# Plan 18-06 — Wire polish + VERIFICATION

## Goal

Final stage of Phase 18. Three micro-optimizations that individually look
small but compound to close the last gap between "we hit the spec target on
simple-fraud" (achieved at 18-05) and "we hit the spec target on every shape
in the beava-bench matrix" (the actual ship requirement). Then run the full
verification matrix and author the phase SUMMARY.

The three micro-opts targeted here:
1. **Zero-copy argv** — parsed commands point into the BytesMut read buffer;
   no string materialization, no `Row` allocation
2. **Hand-rolled response formatters** — pre-formatted byte templates spliced
   with `BytesMut::extend_from_slice`; no `format!`/`write!`
3. **Static response strings** — common responses (`HTTP/1.1 204 No Content\r\n...`)
   interned at compile time; reference, not copy

Each opt is gated by PERF GATE 6.2: the criterion microbench MUST show 5-10%
individual uplift before merge. If any opt fails to demonstrate uplift, that
opt is reverted (code budget belongs to opts that earn it).

Then PERF GATE 6.1: full Phase 13 spec on Linux Xeon across the bench matrix.

Reference: `18-redis-research.md` §"Wire-format details" + `18-rust-translation.md`
rows 9-12.

  helpers using `BytesMut::extend_from_slice`:
  ```rust
  const HTTP_204_PREFIX: &[u8] = b"HTTP/1.1 204 No Content\r\nContent-Length: ";
  const HTTP_204_KEEPALIVE_SUFFIX: &[u8] = b"\r\nConnection: keep-alive\r\n\r\n";
  const HTTP_204_CLOSE_SUFFIX: &[u8] = b"\r\nConnection: close\r\n\r\n";

  pub fn format_http_204(out: &mut BytesMut, content_length: usize, keep_alive: bool) {
      out.extend_from_slice(HTTP_204_PREFIX);
      // itoa or hand-rolled u64 → ascii (no format!)
      let mut buf = itoa::Buffer::new();
      out.extend_from_slice(buf.format(content_length).as_bytes());
      out.extend_from_slice(if keep_alive { HTTP_204_KEEPALIVE_SUFFIX } else { HTTP_204_CLOSE_SUFFIX });
  }
  ```
- Same pattern for 200, 400, 404, 405, 500. JSON body is taken as a `&Bytes`
  ref and spliced verbatim — caller serializes once.
- `runtime_core_glue.rs`: replace existing `format!("HTTP/1.1 ...")` /
  `serde_json::to_writer(&mut Vec::new(), ...)` patterns with the new
  formatters.

## Risks & mitigations

- **Micro-opt fails to demonstrate uplift in isolation**: revert that opt;
  the bar is real. Don't keep complexity that doesn't pay. Document in
  SUMMARY which opts were reverted and why.
- **Full matrix uncovers shape-specific regression**: complex-fraud or
  recommendation may need shape-specific tuning (e.g., different
  `io_threads` setting, different WAL buffer size). Acceptable to ship per-
  shape config defaults; document in `18-SUMMARY.md`.
- **Test rewrites bloat the diff**: ~200 LoC of integration test rewrites is
  expected. Concentrate them in one or two commits clearly tagged
  `test(18-redis-hand-roll-18-06): rewrite <suite> as sync` so reviewer
  can isolate.
- **`itoa` adds a dependency**: 0 deps, ~700 LoC, zero unsafe in safe API.
  Trivial. Already in the dep tree via `serde_json` likely.
- **Static response strings drift from spec**: add a `tests/wire_format_golden_test.rs`
  with golden bytes for each static response. Updating the static must update
  the golden in the same commit. CI catches drift.
- **`cargo tree` assertion is fragile to feature combinations**: write the
  test using `cargo metadata --format-version 1` parse instead — it's stable
  JSON. Document the chosen approach.
- **Admin tokio path retains attack surface**: admin endpoints are bound to
  127.0.0.1:8081 by default; document binding semantics in `18-SUMMARY.md`
  deployment section.
