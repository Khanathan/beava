# Phase 10.1 — UI Design Contract

> Visual and interaction contract for the interactive Debug UI redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Inheritance:** Phase 10.1 is a structural rewrite of the same five static files owned by Phase 10 (`index.html`, `app.css`, `app.js`, `icons.svg`, `favicon.svg`). The design system tokens (spacing, typography, color, motion, fonts) are **inherited verbatim from Phase 10 UI-SPEC §13.1** and the live `src/server/ui/app.css` `:root` block. This phase does not reopen any locked token. What changes is the layout paradigm (split view instead of four tabs), the interaction model (click-to-select node → drill-in panel), and the 1 Hz edge-label updater.

## Typography

Exactly 4 type sizes, exactly 2 weights. Locked by Phase 10 UI-SPEC §13.1 and already in `src/server/ui/app.css:12-15`. **No 16px slot** — the scale deliberately skips it to avoid a middle-weight ambiguity.

| Role | Size | Weight | Line Height | Token | Phase 10.1 usage |
|------|------|--------|-------------|-------|-----------------|
| Body | 14px | 400 | 1.5 | `--text-sm` | Default body, button labels, stream header metadata, operator-item summary, drill-in section body copy |
| Label / micro | 12px | 400 | 1.5 | `--text-xs` | Footer text, poll-control label, operator summary secondary text, edge throughput labels (`1.2k/s`), section eyebrow labels |
| Heading | 20px | 600 | 1.2 | `--text-xl` | Wordmark, drill-in section titles (`Memory`, `State`, `Throughput`, `Entity lookup`), empty-state heading "No pipelines registered" |
| Display | 24px | 600 | 1.2 | `--text-2xl` | Stream-header h2 inside the drill-in panel (selected stream name) |

Font-weight tokens: exactly 2 values — `400` (body/default) and `600` (semibold headings + active states).

**Monospace exception:** the wordmark (`tally'`) uses `--font-mono` at `--text-xl` weight 600 per the locked brand treatment in Phase 10. Inline `<code>` uses `--font-mono` at `--text-sm`. No other element uses mono.

**Line height:** body 1.5, heading 1.2 — matches the `line-height: 1.5` set on `body` and headings' tighter leading through the default browser UA.

## Interaction Contract

### Click-to-select node → drill-in

1. **Click a node** → `setSelected(name)` runs. The node gains a `.selected` class (2px `--accent-primary` outer ring), `aria-pressed="true"`. The drill-in panel re-renders with that stream's 5 sections.
2. **Click the same node again** → `setSelected(null)` runs. Ring removed, panel returns to placeholder.
3. **Click empty SVG background** → `setSelected(null)`.
4. **Press Esc** (when no `<input>` has focus) → `setSelected(null)`.
5. **Press Tab** → cycles through interactive elements: Pause button → (each focusable DAG node, using the `tabindex="0"` applied by `app.js` in the click-handler binding loop) → entity input (only if a node is selected) → Look up button → results.
6. **Press Enter or Space on a focused DAG node** → same as click.

Animation: node selection transitions use `transition: stroke-width var(--motion-fast), stroke var(--motion-fast)` — 120ms ease-out. Under `prefers-reduced-motion: reduce`, duration collapses to `0ms` via the `:root` override already in `app.css:68-71`.

### Pause toggle

Click the Pause button OR press `Space` (no input focused) → `setPaused(!state.paused)`. When paused:
- `.poll-control[data-paused="true"]` → poll-dot and label go warm
- Pause button text flips to `Resume`
- Footer live-status label becomes `Paused · last update HH:MM:SS`
- htmx containers get `data-hx-disable="true"` on every `[hx-trigger*="every"]` element
- Vanilla `setInterval` in the edge-label updater keeps firing but the `tick()` callback short-circuits on `state.paused === true` (no fetch, no DOM write)
- aria-live region announces `Polling paused`

When resumed: inverse. aria-live announces `Polling resumed`.

### Edge label updater

- Runs at 1 Hz via `setInterval` (1000ms)
- Each tick: `fetch('/debug/throughput')` → update each `g.edgeLabel > g.label > text > tspan` via `d3.select(...).text(formatFps(ewma_5s_of_downstream))`
- **Never calls `render()`** — dagre-d3 is rendered exactly once on initial topology load (RESEARCH Pattern 6)
- Network errors silently swallow — next tick retries. Sustained failure should surface via a footer "Lost connection" banner at planner's discretion (not required for this phase)

### Entity lookup

- Input + button inside drill-in section 5 (or 3 for views)
- Submit on Enter-in-input or click-the-button
- Clears result container, calls `fetch('/debug/key/' + encodeURIComponent(key))`
- Filters `computed_features` to entries whose `{source}.{label}` prefix matches the selected stream, strips the prefix for display
- Esc in the input clears the input value and the result container back to idle state
- All result strings rendered via `.textContent`
