## Decisions Made

- **Verbatim token inheritance**: The `:root` block in app.css is a byte-for-byte copy of the Phase 10 version (with a handful of Phase 10 extras like `--accent-primary-hover`, `--bg-overlay`, `--radius-sm`, `--radius-lg`, `--chart-operator`, `--chart-grid`, `--motion-med` carried over because they're still referenced by the new component rules). No hex changed, no px changed, no font stack changed. This directly honors the phase CONTEXT decision "Design tokens (spacing, typography, color, motion, fonts) are inherited verbatim from Phase 10 UI-SPEC Â§13.1".
- **CSS Grid escape hatch explicit**: `grid-template-columns: minmax(0, 1fr) 360px` + `.dag-canvas { min-width: 0; min-height: 0 }` is non-negotiable. Without the `minmax(0, ...)` wrap, a wide dagre-d3 SVG would inflate the track past viewport. The four critical gotchas from RESEARCH Pattern 4 are inlined as comments in the CSS for future maintainers.
- **Always-visible drill-in panel**: Per phase CONTEXT "Claude's Discretion" and the UI-SPEC "always visible with placeholder when no selection â€” avoids layout shift". The panel has `data-empty="true"` by default; Plan 03's app.js will flip it to `false` and render sections on node click. The empty state centers the placeholder copy via `align-items: center; justify-content: center` on the panel when `[data-empty="true"]`.
- **Favicon href mismatch with plan example**: The plan's inline HTML example used `<link rel="icon" href="/favicon.svg">`, but the axum router (`src/server/http.rs`) only routes `/static/{*file}`, and the existing Phase 10 index.html already used `/static/favicon.svg`. Using `/favicon.svg` would 404 the icon. Preserved `/static/favicon.svg` as a Rule 3 (blocking-issue auto-fix).
- **Script order correction**: Phase 10 loaded htmx first, then d3, then dagre-d3. The new shell loads d3 first (dagre-d3 is a d3 plugin and segfaults at import time if d3 isn't on the window), then dagre-d3, then htmx, then app.js. The `split_view_shell_has_dag_canvas_and_drill_in_panel` test asserts this order via `body.find` position comparison so future regressions trip the test.
- **Two tests, not one**: The DELETE side (forbidden markup gone) and the ADD side (required markers present) each get their own test. This gives clearer failure messages when a future refactor partially reverts one side, and mirrors the existing `app_js_has_no_innerhtml_or_eval_sinks` single-purpose pattern.
- **Test tuple destructuring**: The plan's sample test code used `resp.status` / `resp.body` struct access, but the actual `http_get` helper in this file returns `(u16, HashMap<String, String>, Vec<u8>)`. Adapted to `let (status, _headers, body) = http_get(...).await;` to match the existing idiom â€” same pattern used by all 19 existing tests.

**2. [Rule 3 - Blocking] Adapted test code to existing tuple helper shape**
- **Found during:** Task 3 (new shell tests)
- **Issue:** The plan's sample Rust test code used `resp.status` and `resp.body` struct field access on an `HttpResponse`, but the actual `http_get` helper in `tests/test_debug_ui.rs` returns a tuple `(u16, HashMap<String, String>, Vec<u8>)` â€” a `type HttpResponse = ...` alias, not a struct. - **Fix:** Adapted to `let (status, _headers, body) = http_get(...).await;` + `let body = body_string(&body);`, matching the 19 existing tests in the same file.
- **Files modified:** tests/test_debug_ui.rs

**3. [Rule 2 - Missing critical] Kept Phase 10 companion tokens that new selectors still reference**
- **Found during:** Task 2 (app.css rewrite)
- **Issue:** The plan's reference `:root` block listed the minimum tokens the new shell selectors reference, but several Phase 10 companion tokens (`--accent-primary-hover`, `--accent-primary-dim`, `--bg-overlay`, `--radius-sm`, `--radius-lg`, `--chart-operator`, `--chart-grid`, `--motion-med`, `--text-on-accent`, `--status-info`) are still referenced by other rules in the new file (e.g., `.btn-primary:hover { background: var(--accent-primary-hover) }`, `code { border-radius: var(--radius-sm) }`). - **Fix:** Preserved every Phase 10 companion token byte-for-byte in the new `:root` block alongside the minimum set. Net result: the `:root` block is a superset of what the plan required, not a subset.
- **Files modified:** src/server/ui/app.css
- **Verification:** Every `var(--...)` lookup in the new file resolves to a declared token (manual scan + `cargo test --test test_debug_ui static_css_is_embedded` green).
