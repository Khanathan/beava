## Critical Anti-Patterns (do NOT repeat)

Re-read before resuming. TL;DR blocking items:

- **DO NOT use `.innerHTML` for any user-supplied string** in the rewritten app.js. The Phase 10 WR-03 regression test `app_js_has_no_innerhtml_or_eval_sinks` enforces this at `cargo test` time. Forbidden sinks: `.innerHTML`, `.outerHTML`, `insertAdjacentHTML`, `document.write`, `eval(`, `new Function(`, `labelType: 'html'`.
- **DO NOT call `dagre-d3 render()` on every /debug/throughput poll.** The DAG must render ONCE on initial load; edge label updates flow through `d3.select('.edgeLabel text').text(newValue)` in-place. Re-rendering at 1 Hz re-layouts the graph and produces visual flashing. RESEARCH §2 has the canonical pattern.
- **DO NOT break the existing `features` field** on /debug/topology. The new `operators` field is ADDITIVE. Phase 10 integration tests assert on the `features` string-list shape; removing or renaming that field fails `cargo test`.

- **DO NOT skip smart discuss for Phase 10.2.** Histogram estimator choice (t-digest vs HDR vs bucketed) needs explicit user input. Do NOT run `plan-phase 10.2 --skip-discuss`.
- **DO NOT fold Phase 10.2 into Phase 10.1.** They are separate phases per user decision. 10.1 ships before 10.2 discuss runs — 10.2 needs 10.1's interactive layout on disk to decide where latency UI surfaces live.
- **DO NOT touch the Phase 10 tests that assert on the old `features` string-list shape.** The new `operators` field is a new key; existing assertions ignore new keys. Verify this before writing Wave 1's tests.
