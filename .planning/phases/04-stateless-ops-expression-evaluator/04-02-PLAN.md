<objective>
Hand-rolled recursive-descent parser for the canonical parenthesized expression grammar emitted by `python/beava/_col.py::to_expr_string()`. Produces a typed AST with byte-offset spans and column-pointing parse errors. Pure beava-core module, syscall-free. Partial implementation of SDK-COL-07 (this plan ships the parser + field-reference collector; Plan 04-03 + 04-04 wire it into register-time schema validation to complete SDK-COL-07's "field references resolve to known schema fields" clause).

Purpose: Phase 4's central parser. Every later Phase-4 plan (evaluator, op chain, register integration, acceptance) consumes `Expr` + `ParseError` + `Span`. This plan's contract is "accept exactly what the SDK emits; reject everything else with a pinpoint error."

This plan also owns the canonical AST normalization pass: after parsing, the parser rewrites `BinOp("==", e, Literal::Null)` and `BinOp("==", Literal::Null, e)` into `Call("isnull", [e])`. Rationale: SDK-COL-04 says `.isnull()` produces `(x == null)`; but CONTEXT.md D-04 says `null == null` → Null (strict SQL semantics). Rather than making eval.rs special-case `==` with a null operand, we fold the rewrite into the parser so eval's BinOp("==") stays strict-null and users who type `.isnull()` (or literal `(x == null)`) get deterministic Bool results. Plan 04-03's evaluator assumes this rewrite has already happened; no null-equality special-case in eval.rs.

Output: `crates/beava-core/src/expr.rs` with the parser, Expr AST, Span struct, ParseError, unit tests, a proptest that round-trips SDK-shaped strings, and the `(x == null)` → `isnull(x)` post-parse rewrite.
</objective>
