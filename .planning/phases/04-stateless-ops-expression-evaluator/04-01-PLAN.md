Three-valued logic (CONTEXT.md D-04):
- `null AND false = false` (short-circuit)
- `null AND true  = null`
- `null OR  true  = true`  (short-circuit)
- `null OR  false = null`
- `NOT null       = null`
- `null == null   = null` (NOT true — use isnull(x) to test)

    1. `impl PartialEq for Value`:
       - Null==Null → true.
       - F64(a)==F64(b) → `!a.is_nan() && !b.is_nan() && a == b` (NaN-safe; two NaNs are never equal).
       - Str/I64/Bool/Bytes/Datetime compare by inner equality.
       - Cross-variant comparisons → false.
       - Bytes uses `Vec<u8>` equality.
    2. `impl Value::type_of`:
       - Returns `Option<FieldType>`: None for Null; Some(FieldType::Str) for Str(_); etc. Every non-Null variant maps.
    3. `and_three_valued(&self, other: &Self) -> Self`:
       - If either operand is not `Bool(_)` or `Null` → return `Value::Null` (runtime coercion per CONTEXT.md D-04 last line: runtime ignores type errors and propagates Null).
       - Bool(true) AND Bool(true) → Bool(true).
       - Bool(_) AND Bool(false) or Bool(false) AND Bool(_) → Bool(false) (short-circuit).
       - Otherwise (at least one Null, no short-circuit false) → Null.
    4. `or_three_valued(&self, other: &Self) -> Self`:
       - Non-bool/non-null operands → Null.
       - Bool(_) OR Bool(true) or Bool(true) OR Bool(_) → Bool(true) (short-circuit).
       - Bool(false) OR Bool(false) → Bool(false).
       - Otherwise (at least one Null, no short-circuit true) → Null.
    5. `not_three_valued(&self) -> Self`:
       - Bool(b) → Bool(!b).
       - Null → Null.
       - Anything else → Null (runtime-tolerant per D-04).
    6. `Row::new() -> Self` → `Row(BTreeMap::new())`.
    7. `Row::get(&self, field: &str) -> Option<&Value>` → `self.0.get(field)`.
    8. `Row::with_field(mut self, field: &str, value: Value) -> Self` — owning API: mutates `self.0` in-place (ownership semantics — the returned Row supersedes the original), returns `self`. Document inline: "Consumes self; returns the updated Row. Callers that need to preserve the upstream Row must `.clone()` first. This matches SDK-OPS-09: derivations wrap a new copy per op step, they do not mutate shared state."

    10. `Row::renamed(mut self, old: &str, new: &str) -> Self` — if `old` exists, pop the entry and re-insert under `new`; if `new` already exists, overwrite it (matches SDK-OPS-04 semantics, though the op-level SDK-OPS-04 collision check happens in 04-04). No-op when `old` missing.
    11. `Row::iter`, `Row::len`, `Row::is_empty` — delegate to `self.0`.
