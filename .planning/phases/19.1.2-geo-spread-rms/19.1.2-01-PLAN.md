<objective>
Replace `GeoSpreadState`'s O(n)-per-push samples-Vec walk with O(1) Welford online second-moment accumulators. New `query()` returns RMS dispersion in km (lat/lon variance with local-mean-latitude cos-correction). Drops `samples: Vec<(f64, f64)>` and `max_km: f64` fields; adds `m2_lat: f64`, `m2_lon: f64`.

This is a v0 spec evolution: the user-visible value `bv.geo_spread` returns changes from "max distance (km) from running mean" to "RMS dispersion (km) — sqrt(var_lat_km² + var_lon_km²)". Beava v0 not shipped publicly per memory `project_beava_product`, so no external consumer migration path needed. fraud-team.json's `geo_spread_*` features auto-pick up new semantics on next register.

Snapshot format bumps because `GeoSpreadState` shape changes. Pre-fix snapshots fail deserialization with `missing field: m2_lat` — correct loud-failure behavior.
</objective>

```rust
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct GeoSpreadState {
    pub lat_field: String,
    pub lon_field: String,
    pub n: u64,
    pub mean_lat: f64,
    pub mean_lon: f64,
    /// Welford online sum of squared deviations from running mean (lat, deg²).
    pub m2_lat: f64,
    /// Welford online sum of squared deviations from running mean (lon, deg²).
    pub m2_lon: f64,
}
```

**Step 2**: Rewrite `update()` to use Welford online (no samples vec, no for-loop):

```rust
pub fn update(&mut self, row: &Row, where_matched: bool) {
    if !where_matched {
        return;
    }
    let Some((lat, lon)) = read_lat_lon(row, &self.lat_field, &self.lon_field) else {
        return;
    };
    self.n += 1;
    let inv_n = 1.0 / self.n as f64;
    let prev_mean_lat = self.mean_lat;
    let prev_mean_lon = self.mean_lon;
    self.mean_lat += (lat - prev_mean_lat) * inv_n;
    self.mean_lon += (lon - prev_mean_lon) * inv_n;
    // Welford online second-moment update
    self.m2_lat += (lat - prev_mean_lat) * (lat - self.mean_lat);
    self.m2_lon += (lon - prev_mean_lon) * (lon - self.mean_lon);
}
```

**Step 3**: Rewrite `query()` to return RMS-km using local-mean-latitude cos-correction:

```rust
pub fn query(&self) -> Value {
    if self.n < 2 {
        return Value::Null;
    }
    let var_lat_deg2 = self.m2_lat / self.n as f64;
    let var_lon_deg2 = self.m2_lon / self.n as f64;
    const KM_PER_DEG_LAT: f64 = 111.32;
    let km_per_deg_lon = KM_PER_DEG_LAT * self.mean_lat.to_radians().cos();
    let var_lat_km2 = var_lat_deg2 * KM_PER_DEG_LAT * KM_PER_DEG_LAT;
    let var_lon_km2 = var_lon_deg2 * km_per_deg_lon * km_per_deg_lon;
    Value::F64((var_lat_km2 + var_lon_km2).sqrt())
}
```
