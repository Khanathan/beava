```python
# sdk/python/beava/app.py
class App:
    def push_and_get(
        self,
        event_type,
        row: dict,
        *,
        entity_key: dict[str, str],
        features: list[str],
        sync: bool = False,
    ) -> tuple["PushAck", dict[str, Any]]:
        """
        Combined push + feature query in one round-trip (Phase 12.5).

        Returns (ack, features_dict). features_dict maps each requested feature
        name to its value, or None for unknown features or unknown entities.
        Unknown feature names emit a warning via the `warnings` module (D-06).

        When sync=True, uses acks=all (/push-sync-and-get); waits for fsync.
        """
```

Response parsing — `features_dict` is derived from JSON `features` object with
`None` values kept as Python None. Warnings array is iterated; each
"unknown_feature: X" → `warnings.warn(f"unknown_feature: {X}", UserWarning)`.

Transport dispatch: inspect `self._url` scheme. `http://` → POST via existing
httpx/requests session. `tcp://` → encode OP_PUSH_AND_GET (0x0015) /
OP_PUSH_SYNC_AND_GET (0x0016) frame using existing TCPTransport class.
