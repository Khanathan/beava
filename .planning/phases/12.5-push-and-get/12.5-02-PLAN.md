<!-- TCP frame body envelope for these ops (CT_JSON only in v0): -->
```json
// Request payload (same shape as HTTP POST body + event_name hoisted inside):
{
  "event": "Transaction",
  "row": { ...event fields... },
  "query": {
    "entity_key": {"user_id": "alice"},
    "features": ["cnt"]
  }
}

// Response payload (CT_JSON, op echoes request op):
{"ack_lsn": N, "registry_version": N, "features": {"cnt": 1}, "warnings": []}
```

Rationale for hoisting `event` into the payload: TCP doesn't have URL path
params, so the event name lives in the JSON envelope alongside row+query.
Matches the Phase 8 OP_PUSH envelope shape (`{"event": "...", "body": {...}}`).
