#[derive(serde::Deserialize)]
pub struct PushAndGetRequest {
    pub row: serde_json::Value,                 // event row (object)
    pub query: PushAndGetQuery,
}
#[derive(serde::Deserialize)]
pub struct PushAndGetQuery {
    pub entity_key: BTreeMap<String, String>,   // group_key name -> value
    pub features: Vec<String>,

pub struct PushAndGetResponse {
    pub ack_lsn: u64,
    pub registry_version: u32,
    pub features: BTreeMap<String, serde_json::Value>, // null for unknown feat / unknown entity
    #[serde(skip_serializing_if = "Vec::is_empty")]
    pub warnings: Vec<String>,
}

pub async fn execute_push_and_get(
    app: &AppState,
    event_name: &str,
    body_bytes: &[u8],
    sync_mode: SyncMode,
) -> Result<(PushAndGetResponse, StatusCode), PushAndGetError>;

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12.5-01 | T (Tampering) | PushAndGetRequest parser | mitigate | Strict serde Deserialize with exact field shape; reject unknown body shape with 400 invalid_event. BTreeMap<String,String> for entity_key enforces stringy values. |
| T-12.5-02 | D (DoS) | query.features length | mitigate | Cap features.len() × 1 entity ≤ 1000 (soft cap; reuse Phase 5 BATCH_CAP concept at smaller scale). Return 400 batch_too_large beyond cap. Push itself still subject to existing execute_push policies. |
| T-12.5-03 | I (Info disclosure) | Unknown feature warning | accept | Warning reveals whether a feature name is registered. Same info available via `GET /registry` already; no net-new disclosure. |
| T-12.5-04 | E (Escalation) | Atomic borrow scope | mitigate | Single `state_tables.lock()` guard spans apply + query; no await inside; clippy `await_holding_lock` lints enforced via `-D warnings`. Prevents partial-state escalation. |
| T-12.5-05 | S (Spoofing) | event_name path param | accept | Same surface as existing /push/:event_name; unknown event name → 404 event_not_found; path traversal not possible with axum's Path extractor typing. |
