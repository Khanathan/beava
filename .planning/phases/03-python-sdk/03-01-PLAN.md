pub const OP_PUSH: u8 = 0x01;
pub const OP_GET: u8 = 0x02;

    - encode_string("hi") == b'\x00\x02hi' (u16 BE length + UTF-8)
    - encode_string("") == b'\x00\x00' (empty string)
    - encode_push("Transactions", {"user_id": "u1"}) starts with encode_string("Transactions") followed by JSON bytes
    - encode_get("u123") == encode_string("u123")
    - encode_set("u123", {"x": 1}) == encode_string("u123") + JSON bytes of {"x": 1}
    - encode_mset({"u1": {"a": 1}, "u2": {"b": 2}}) == [u32 count=2][entry1][entry2] where each entry = [u16-string key][u32 json_len][json bytes]
    - encode_register({"name": "Tx", "key_field": "uid", "features": []}) == JSON bytes
    - parse_response(b'\x00\x00\x00\x05\x00{"a":1}') should fail -- wrong length. parse_response on status=0x00 returns payload. status=0x01 raises ProtocolError with payload as message.
    - MAX_FRAME_SIZE = 64 * 1024 * 1024 (64MB cap for DoS protection)
  </behavior>

    1. Write tests FIRST in `python/tests/test_protocol.py` with known-good byte sequences derived from the Rust protocol.rs encoding rules:
       - Frame encoding: verify exact bytes for encode_frame with various opcodes and payloads
       - String encoding: verify encode_string produces [u16 BE len][UTF-8 bytes]
       - PUSH: encode_push("Transactions", {"user_id": "u123", "amount": 50.0}) and verify it starts with \x00\x0cTransactions (12 bytes for "Transactions") followed by valid JSON
       - GET: encode_get("u123") == b'\x00\x04u123'
       - SET: encode_set produces [u16-string][JSON]
       - MSET: encode_mset produces [u32 count][entries with u16-string key + u32 json_len + json bytes]
       - REGISTER: encode_register produces raw JSON bytes
       - Response parsing: parse_response extracts (status, payload) from [4-byte len][status][payload]
       - Response error: status 0x01 payload is error message string

    2. Create `python/tally/_protocol.py` with:
       - Constants: `OP_PUSH = 0x01`, `OP_GET = 0x02`, `OP_SET = 0x03`, `OP_MSET = 0x04`, `OP_REGISTER = 0x05`, `STATUS_OK = 0x00`, `STATUS_ERROR = 0x01`, `MAX_FRAME_SIZE = 64 * 1024 * 1024`
       - `encode_frame(opcode: int, payload: bytes) -> bytes`: `struct.pack('>I', 1 + len(payload)) + bytes([opcode]) + payload`
       - `encode_string(s: str) -> bytes`: `struct.pack('>H', len(s_bytes)) + s_bytes` where `s_bytes = s.encode('utf-8')`
       - `encode_push(stream_name: str, event: dict) -> bytes`: `encode_string(stream_name) + json.dumps(event).encode('utf-8')`
       - `encode_get(key: str) -> bytes`: `encode_string(key)`
       - `encode_set(key: str, features: dict) -> bytes`: `encode_string(key) + json.dumps(features).encode('utf-8')`
       - `encode_mset(entries: dict[str, dict]) -> bytes`: `struct.pack('>I', len(entries))` + for each: `encode_string(key) + struct.pack('>I', len(json_bytes)) + json_bytes`
       - `encode_register(definition: dict) -> bytes`: `json.dumps(definition).encode('utf-8')`
       - `parse_response(data: bytes) -> tuple[int, bytes]`: read 4-byte u32 BE length, then status byte, then payload bytes. Raise ProtocolError if length > MAX_FRAME_SIZE.

    3. Update `python/tally/__init__.py` to also export protocol constants: `from tally._protocol import OP_PUSH, OP_GET, OP_SET, OP_MSET, OP_REGISTER`
