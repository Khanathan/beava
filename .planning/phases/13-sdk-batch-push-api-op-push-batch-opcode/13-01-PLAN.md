**Part A — ConnAccumulator.advance_seq (src/server/tcp.rs):**

Add method to ConnAccumulator impl block (after the existing `drain` method):

```rust
/// Reserve `n` consecutive seq numbers for a client-side batch that
/// bypasses the accumulator buffer. Returns the base seq; the batch
/// events get seqs [base, base+1, ..., base+n-1]. The accumulator's
/// internal counter advances to base+n so subsequent push() calls
/// continue from there. (Phase 13 D-10, Pitfall 5 in RESEARCH.md)
pub fn advance_seq(&mut self, n: u64) -> u64 {
    let base = self.next_seq;
    self.next_seq += n;
    base
}
```

**Part B — handle_connection dispatch (src/server/tcp.rs):**

In the `handle_connection` function, add a dispatch arm for `Command::PushBatch` in the same pattern as the existing non-async command dispatch. The batch must:

1. Force-flush any pending accumulator entries (same as sync commands — H-2 consistency):
   ```rust
   // Already handled by the is_async_push check above — PushBatch is NOT
   // PushAsync, so the existing force-flush at line ~294 fires automatically.
   ```

2. After the existing force-flush block, add the PushBatch arm in the command match/dispatch section:
   ```rust
   Command::PushBatch { stream_name, batch_id, events } => {
       // Reserve consecutive seq numbers (D-10, Pitfall 5)
       let base_seq = accumulator.advance_seq(events.len() as u64);
       let now = SystemTime::now();

       // Convert to Vec<PendingAsync> for handle_push_batch (D-06)
       let batch: Vec<PendingAsync> = events.into_iter().enumerate().map(|(i, (payload, raw_payload))| {
           PendingAsync::new(
               base_seq + i as u64,
               stream_name.clone(),
               payload,
               raw_payload,
               now,
           )
       }).collect();

       // Dispatch to Phase 12 handler — zero new hot-path logic
       let results = handle_push_batch(&state, &batch);

       // Collect errors with [batch:{id} event:{idx}] attribution (D-09)
       for (i, (ev, res)) in batch.iter().zip(results.iter()).enumerate() {
           if let Err(err) = res {
               pending_drain.push((ev.seq, format!("[batch:{} event:{}] {}", batch_id, i, err)));
           }
       }

       // Fire-and-forget: no response frame. Continue loop.
       continue;
   }
   ```

   **CRITICAL placement:** This arm must go in the main command dispatch area, NOT inside the tight inner loop. The tight inner loop (lines 305-340) is only for OP_PUSH_ASYNC frames. PushBatch bypasses the accumulator entirely.

   **Also handle PushBatch in the tight inner loop's non-async branch:** The tight inner loop (lines 313+) has a branch for non-async frames that arrive while draining the BufReader. If a PushBatch arrives there, it should be handled inline the same way (flush accumulator first, then dispatch). Look at how the existing non-async inline handler works (around line 327-340) and add PushBatch there too.
