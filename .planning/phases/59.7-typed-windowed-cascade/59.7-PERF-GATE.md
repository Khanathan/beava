### 3. Aggregate-EPS +9.78 % is bounded by the same measurement ceiling as 59.6

Every one of the 6 runs (baseline × 3 + candidate × 3) exits with `shard inbox full — backpressure` on all 8 clients (see per-client sections in each run's evidence txt). Per the 59.6-PERF-GATE.md §1 "Python SDK is the measurement ceiling on this host," this means the server is consuming events faster than 8 × Python processes can produce them — the aggregate EPS measured here is the Python-client ceiling on this 10-core macOS host, not the server ceiling. The same roadmap item applies: Phase 64 Rust bench client is the authoritative vehicle for aggregate-EPS verification at > 3 M EPS.

The **+9.78 %** lift candidate-vs-59.6 (and +9.28 % baseline-vs-59.6) reflects wave-0..4 throughput improvements that survive the client ceiling — the typed-cascade-direct walker + windowed ops remove work from the server side, widening the backpressure margin.

## Interpretation

Phase 59.7 closes the two gaps Phase 59.6 identified in its hot-path profile:

1. **Typed windowed aggregations (Waves 1-2).** 10 windowed typed agg ops landed (CountOpTypedWindowed / SumOpTypedWindowed{I64,F64} / AvgOpTypedWindowedF64 / MinOpTypedWindowed{I64,F64} / MaxOpTypedWindowed{I64,F64} / LastOpTypedWindowedInlineStr / FirstOpTypedWindowedInlineStr) backed by 8 packed TypedRingBuffer variants, side-mapped on Shard.entity_ringbuffers_typed, V11-snapshotted via the `typed_ringbuffers` field. Every fraud-pipeline `window=1h/24h/7d` feature now has a typed path available.
2. **Typed cascade-direct (Waves 3-4).** `run_typed_direct_cascade` walker replaces the Wave-3 same-shard proxy; it dispatches `ShardOp::RunTypedAggCascadeStep` over SPSC inboxes to cross-shard downstreams (fire-and-forget), runs same-shard typed ops inline, and falls back to the Value bridge for (a) whole-cascade retraction-capable primary streams and (b) any cascade containing a non-typed-compatible downstream. Per-non-typed-hop `typed_cascade_value_fallback` counter bump is shared via Arc<AtomicU64> with ConcurrentAppState; `typed_cascade_direct_dispatched` bumps on every dispatched ShardOp.

**Structural wins (ground truth for this phase):**

1. **Gap-2 profile gate cleared decisively.** `push_internal_on_shard` drops from 17.8 % leaf samples (Phase 59.6 state-inmem baseline) to 0.00 % in Phase 59.7 candidate — a reduction of 2 orders of magnitude in the hottest leaf function. 3. **Aggregate-EPS +9.78 % vs 59.6 baseline** — bounded by the same macOS Python-client ceiling 59.6 hit; +10 % gate narrowly missed (0.20 % gap, within 0.66 % run variance). Linux-host / Phase 64 Rust bench client will definitively verify the aggregate lift (same hand-off as 59.6 SC-5).
