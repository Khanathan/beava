## Deviations from Plan

### Rule 1 — auto-fixed bug

**1. [Rule 1 - Bug] Parity test reference floor computation**

- **Found during:** Task 1 verification — `parity_first_inline_str_typed_vs_value_windowed_5s_bucket_1s` failed on first run.
- **Issue:** Initial reference computation used `floor = cp.checked_sub(window)` which included one extra bucket at checkpoint boundaries where `cp` is bucket-aligned. Ring buffer semantics retain exactly `num_buckets` buckets starting at `bucket_start(now) - (num_buckets - 1) * bucket_duration` — not `now - window`. For window=5s/bucket=1s at cp=6s, the ring holds buckets starting at 2s..6s, but `cp - window = 1s` erroneously includes bucket [1,2).
- **Fix:** Changed reference to compute `floor = aligned(cp) - (num_buckets - 1) * bucket` via integer math on the `duration_since(UNIX_EPOCH).as_secs()` basis. Match the same `bucket_start_for` alignment the ring uses.
- **Files modified:** tests/typed_windowed_aggregation_parity.rs (replace_all=true on the `let floor = cp.checked_sub(window()).unwrap_or(UNIX_EPOCH);` site — both Last and First tests share the same reference shape).
- **Commit:** eb0d84b (fix rolled into Task 1 commit).

### Rule 2 — critical correctness adds

**2. [Rule 2 - Correctness] Byte-format golden regression test**

- **Found during:** Task 2 test authoring — plan specified "add a test_v11_byte_format_unchanged_with_default_typed_ringbuffers regression test."
- **Rationale:** Without this test, a future additive field landing without `#[serde(default)]` would silently break backward compat — existing V11 snapshots on disk would fail to decode. The golden test pins the empty-snapshot wire format to exactly 5 bytes so the breakage would surface immediately.
- **Fix:** Added `test_v11_byte_format_unchanged_with_default_typed_ringbuffers` asserting `bytes == [V11_FORMAT, 0x00, 0x00, 0x00, 0x00]`.
- **Files modified:** tests/typed_snapshot_v11_migration.rs.
- **Commit:** 2b998b6.
