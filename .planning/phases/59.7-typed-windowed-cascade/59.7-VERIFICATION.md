No engineering blocker is introduced — this is the same macOS-ceiling caveat the roadmap has acknowledged since Phase 58.

**Pre-existing proptest regression (`typed_snapshot_v11_migration::roundtrip_typed_ringbuffers`):**
Documented in `deferred-items.md` item 1. Seed-0 failure on `last_drop: Some(TooOld) != None` caused by `#[serde(skip)]` on a field that derived PartialEq still compares. Fix is local: either add `#[serde(default = "DropReason::default_none")]` or implement custom `PartialEq` that skips `last_drop`. Does not affect runtime correctness (skip is observability-only per W2 key-decision 2). Out of scope for W5 perf-gate close.

**SC-10 semantic-tightening deferral:**
The `is_typed_cascade_compatible` predicate remains structural today — it returns true for any FeatureDef variant with a Count/Sum/Avg/Min/Max/Last/First tag, regardless of window semantics. This is preserved intentionally: `FeatureDef` carries `window: Duration` (not `Option<Duration>`), so a `window.is_none()` tightening has no semantic hook. The real typed-vs-Value gating happens at `build_typed_agg_op` factory time (W3/W4) — if no windowed typed impl exists for a given (FeatureDef, input schema) pair, the factory refuses and the cascade falls back to Value. Functionally equivalent; reviewed in Waves 1, 2, and 4 key-decisions.
