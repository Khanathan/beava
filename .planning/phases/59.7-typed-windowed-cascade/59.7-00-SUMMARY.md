### Rule 3 scope fixes (blocking)

**1. [Rule 3 - Blocking] Counters placed on `ConcurrentAppState`, not a nonexistent `src/metrics.rs`**
- **Found during:** Task 2 file-layout scan
- **Issue:** Plan Task 2 Action 3 specified "In src/metrics.rs: add two new counters using the same pattern as TYPED_ROW_PATH_TOTAL." No such file exists in the repo — `ls src/metrics*` returns nothing.
- **Root cause:** Plan author mistakenly assumed module-level `pub static` counters; the actual Phase 59.6 convention (established by `typed_row_path_total` / `value_fallback_path_total`) is per-field `AtomicU64` on `ConcurrentAppState` in `src/server/tcp.rs`.
- **Fix:** Followed the actual convention — added `typed_cascade_direct_dispatched: AtomicU64` + `typed_cascade_value_fallback: AtomicU64` as `pub` fields on `ConcurrentAppState` at the exact site next to the 59.6 counters; pre-seeded to zero in the same `make_concurrent_state_full` constructor. Field names use lowercase (struct-field convention) vs the plan's uppercase (module-static convention); grep-gate still satisfied (3 occurrences each for field decl + pre-seed + doc reference).
- **Files modified:** src/server/tcp.rs
- **Commit:** bc2d138
