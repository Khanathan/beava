<objective>
Ship the foundational Table row storage primitive that the rest of Phase 24
builds on. Add `EntityState.table_rows: AHashMap<String, TableRow>` with
`TableRowState::Live | Tombstoned { since: SystemTime }`, the StateStore
methods that manipulate it, and snapshot codec v6→v7 with a backward-compat
load path. No TCP protocol, SDK, or engine-cascade changes in this plan —
pure foundation layer. Per CONTEXT.md §Storage model (locked).

Purpose: Phase 23 deferred proper per-Table row storage because retraction
semantics need real `(table_name, key)` identity that the marker-based
cascade can't express. This plan lays the identity layer so plan 02 can
wire opcodes and plan 03 can migrate the cascade off markers.

Output: `TableRow` type, 4 new StateStore methods, v7 snapshot codec with
migration from v6, 2 dedicated test files covering round-trip and migration.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| snapshot file → process | Deserialization of attacker-controlled bytes on startup |
| in-memory table_rows → GC sweep | Concurrent mutation during gc_tombstones traversal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01-01 | T (Tampering) | v6 snapshot migration | mitigate | Validate version byte before bincode decode; unknown byte → None (no panic, no arbitrary deserialization) |
| T-24-01-02 | D (Denial) | `gc_tombstones` | mitigate | Use DashMap `iter_mut` (per-shard locking); no global lock held across sweep |
| T-24-01-03 | I (Info disclosure) | tombstone leak | mitigate | `get_table_row` returns full TableRow including Tombstoned variant; consumers (Task 3 cascade, plan 02 GET) must filter; doc-comment the contract |
| T-24-01-04 | R (Repudiation) | tombstone timestamp truthfulness | accept | `since: SystemTime` is server wall-clock; no notary; acceptable for v0 |
| T-24-01-05 | E (Elevation) | arbitrary FeatureValue payload in TableRow.fields | accept | FeatureValue enum is the same surface SET already accepts; no new attack surface |
</threat_model>
