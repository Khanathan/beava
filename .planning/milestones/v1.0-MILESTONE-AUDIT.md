## Cross-Phase Integration

### MISSING-1: Python Lookup operator serializes wrong field names (BREAKING)

**Affects:** SDK-03, XSTR-02
**Files:** `python/tally/_operators.py` (Lookup.to_json)

Python sends `"expr"` and `"field"` but Rust expects `"target"` and `"on"`. Lookups silently fail when registered from Python.

### MISSING-2: Python @view decorator omits "type":"view" (BREAKING)

**Affects:** SDK-03, XSTR-01, XSTR-02
**Files:** `python/tally/_stream.py` (_to_register_json)

View definitions registered from Python are treated as stream definitions. The Rust handler dispatches on `definition_type == Some("view")`, but Python never sets this field.

### MISSING-3: View definitions not persisted in snapshots (DATA LOSS)

**Affects:** PERS-01, PERS-03, XSTR-01, XSTR-02, XSTR-03
**Files:** `src/main.rs`, `src/server/http.rs`

Snapshot collection only iterates `list_streams()`, not `list_views()`. Views are lost on server restart. Startup recovery has no path for view re-registration.
