   #[derive(Debug, Deserialize)]
   pub struct FeatureDefRequest {
       pub name: String,
       #[serde(rename = "type")]
       pub feature_type: String,
       #[serde(default)]

       #[serde(default)]

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Wire -> Parser | Untrusted bytes from TCP clients cross into frame/string/command parsing |
| JSON -> Domain | Untrusted JSON from REGISTER payloads cross into StreamDefinition types |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | protocol::read_string | mitigate | Validate u16 length against remaining buffer before reading; return TallyError::Protocol on truncation |
| T-02-02 | Denial of Service | protocol::parse_command MSET | mitigate | u32 count field limits entries to 4 billion max; actual memory bounded by frame length (u32 max ~4GB). Plan 02 adds frame size limit. |
| T-02-03 | Tampering | protocol::parse_frame | mitigate | Length field (u32 BE) validated: length=0 rejected, payload allocation bounded by advertised length |
| T-02-04 | Information Disclosure | TallyError messages | accept | Error messages contain field names and values from user input; no server internals leaked. Acceptable for v1. |
| T-02-05 | Tampering | convert_register_request | mitigate | Validate non-empty stream name and key_field; parse_expr validates expression syntax; unknown feature types rejected |
| T-02-06 | Denial of Service | parse_expr in REGISTER | mitigate | Expression parser (winnow) operates on bounded input (frame size limited); deeply nested expressions bounded by stack depth. Acceptable for v1 registration-time-only parsing. |
</threat_model>
