```rust
use std::time::SystemTime;
use ahash::AHashMap;
use serde::{Serialize, Deserialize};
use crate::types::{EntityKey, FeatureValue, FeatureMap};

/// Per-entity state. Holds live features (from streaming operators)
/// and static features (from direct SET/MSET writes).
#[derive(Debug)]
pub struct EntityState {
    /// Features computed by streaming operators. Keyed by feature name.
    /// The value is the operator instance itself (holds ring buffer state).
    pub live_operators: Vec<(String, Box<dyn crate::engine::operators::Operator>)>,
    /// Features from direct writes (SET/MSET). Bypass pipeline engine.
    pub static_features: AHashMap<String, StaticFeature>,
    /// Last event timestamp for TTL eviction (Phase 4).
    pub last_event_at: Option<SystemTime>,
}

/// A directly-written feature value.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct StaticFeature {
    pub value: FeatureValue,
    pub updated_at: SystemTime,
}

/// The top-level state store. Maps entity keys to their state.
/// Uses AHashMap per STATE.md locked decision (not std HashMap).
#[derive(Debug)]
pub struct StateStore {
    entities: AHashMap<EntityKey, EntityState>,
}
```

StateStore methods:
- `new() -> Self`: Creates empty store.
- `get_or_create_entity(&mut self, key: &str) -> &mut EntityState`: Returns existing or creates new EntityState with empty operators and static features.
- `get_entity(&self, key: &str) -> Option<&EntityState>`: Read-only access. Returns None if key not found.
- `get_entity_mut(&mut self, key: &str) -> Option<&mut EntityState>`: Mutable access.
- `set_static(&mut self, key: &str, feature_name: &str, value: FeatureValue)`: Write a static feature. Creates entity if absent.
- `get_all_features(&mut self, key: &str, now: SystemTime) -> FeatureMap`: Collects all feature values for an entity. Iterates live_operators calling `read(now)` (`read` takes `&mut self` to advance time and expire stale buckets), then overlays static_features. Static features with the same name override live features (direct writes take precedence). This method takes `&mut self` because operator `read()` requires mutable access to advance ring buffer time.
- `entity_count(&self) -> usize`: Number of tracked entities.

EntityState methods:
- `new() -> Self`: Empty state.
- `update_last_event(&mut self, now: SystemTime)`: Sets last_event_at.

NOTE: The live_operators field uses `Box<dyn Operator>`. This is NOT serializable via serde (trait objects). That is fine for Phase 1 -- serialization of operator state is a Phase 4 concern. For Phase 4, the approach will change to use an enum wrapper. For now, focus on correctness.

Write unit tests in `#[cfg(test)] mod tests` at the bottom of store.rs.

Stream definition types (these will be constructed from JSON in Phase 2's REGISTER command):
```rust
use std::time::{Duration, SystemTime};
use ahash::AHashMap;
use crate::types::{FeatureValue, FeatureMap};
use crate::error::TallyError;
use crate::state::store::StateStore;
use super::operators::{Operator, CountOp, SumOp, AvgOp};
use super::expression::{Expr, EvalContext, parse_expr, eval};

/// Definition of a single feature within a stream.
#[derive(Debug, Clone)]
pub enum FeatureDef {
    Count {
        window: Duration,
        bucket: Duration,
    },
    Sum {
        field: String,
        window: Duration,
        bucket: Duration,
        optional: bool,
    },
    Avg {
        field: String,
        window: Duration,
        bucket: Duration,
        optional: bool,
    },
    Derive {
        expr: Expr,  // Parsed at registration time (ENG-06)
    },
}

/// A stream definition: a named stream with a key field and a list of named features.
#[derive(Debug, Clone)]
pub struct StreamDefinition {
    pub name: String,
    pub key_field: String,
    pub features: Vec<(String, FeatureDef)>,  // (feature_name, definition)
}

/// The pipeline engine. Holds registered stream definitions and coordinates
/// the push-through flow: event -> extract key -> update operators -> eval derives -> return features.
#[derive(Debug)]
pub struct PipelineEngine {
    streams: AHashMap<String, StreamDefinition>,
}
```

PipelineEngine methods:

- `new() -> Self`: Creates engine with no registered streams.

- `register(&mut self, stream: StreamDefinition) -> Result<(), TallyError>`: Registers a stream. Validates derive expressions are parseable (they should already be parsed in the StreamDefinition, but verify). Stores the definition.

- `push(&self, stream_name: &str, event: &serde_json::Value, store: &mut StateStore, now: SystemTime) -> Result<FeatureMap, TallyError>`:
  The synchronous push-through flow:
  1. Look up stream definition by name. Return error if not found.
  2. Extract entity key from event JSON using the stream's key_field. Return error if missing.
  3. Get or create EntityState for this key via store.get_or_create_entity().
  4. For each feature in the stream definition:
     - If it's an operator (Count/Sum/Avg): find or create the operator in the entity's live_operators. Call operator.push(event, now).
     - If it's a Derive: skip during push phase (evaluated on read).
  5. Collect all feature values: iterate live_operators calling read(now), then evaluate derive expressions using EvalContext with the collected features + event.
  6. Update entity's last_event_at.
  7. Return the complete FeatureMap.

  IMPORTANT: The entity's live_operators list must be initialized from the stream definition on first push for that key. If the entity already has operators (from a previous push), use the existing ones. This means operator instantiation happens lazily on first event, not at registration time.

- `get_features(&self, key: &str, store: &mut StateStore, now: SystemTime) -> FeatureMap`:
  Feature retrieval for GET path. Calls store.get_all_features(key, now) (requires `&mut StateStore` because operator `read()` takes `&mut self` to advance time and expire stale buckets -- this is the resolution of the GET-only staleness problem). For any registered streams that have derive features, evaluates them.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Event JSON -> key extraction | Entity key extracted from untrusted event JSON |
| Stream name -> lookup | Stream name provided by caller, looked up in registered streams |
| Static feature write | Values written directly to state store |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-11 | Spoofing | Entity key extraction | mitigate | Key field must be a string in the event JSON. Non-string key values return TallyError. Empty string keys are rejected. Validation in PipelineEngine::push(). |
| T-01-12 | Denial of Service | StateStore unbounded growth | accept | Phase 1 has no TTL eviction (Phase 4). Unbounded entity count is a known limitation. Memory grows proportionally to unique keys. Documented in STATE.md blockers. |
| T-01-13 | Tampering | Static feature overrides live | accept | By design: SET/MSET writes take precedence over computed features. This is the intended behavior for batch feature injection. |
| T-01-14 | Denial of Service | Stream registration | mitigate | Stream names are validated as non-empty. Duplicate registration replaces the previous definition (idempotent). No unbounded allocation from registration itself. |
</threat_model>
