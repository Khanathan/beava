  <action>
Implement `RingBuffer<T>` in src/engine/window.rs per RESEARCH.md Pattern 2 and CONTEXT.md locked decisions.

The struct definition:
```rust
use std::time::{Duration, SystemTime};
use serde::{Serialize, Deserialize};

/// A fixed-capacity ring buffer with time-based bucket selection.
/// Used by all windowed operators (count, sum, avg).
/// Buckets are lazily expired on advance_to() -- no background timers.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RingBuffer<T: Default + Copy> {
    buckets: Vec<T>,
    head: usize,
    bucket_duration: Duration,
    window_duration: Duration,
    current_bucket_start: Option<SystemTime>,
}
```

Key implementation details:
- `new(window_duration, bucket_duration)`: bucket count = `ceil(window_duration_secs / bucket_duration_secs)` as usize. Initialize all buckets to `T::default()`. `current_bucket_start` is `None` (initialized on first event).
- `advance_to(&mut self, now: SystemTime) -> usize`: If `current_bucket_start` is None, initialize from `bucket_start_for(now)`, set head=0, return 0. Otherwise compute elapsed using `now.duration_since(start).unwrap_or(Duration::ZERO)`. Calculate `buckets_to_advance = elapsed_secs / bucket_duration_secs`. If 0, return head. If >= num_buckets, zero ALL buckets. Otherwise zero only the skipped buckets (indices head+1 through head+advance, mod num_buckets). Update head and current_bucket_start.
- `add_to_current(&mut self, value: T, now: SystemTime)` where T: AddAssign: Calls advance_to(now), then adds value to `self.buckets[self.head]`.
- `sum_all(&self) -> T` where T: Sum: Returns `self.buckets.iter().copied().sum()`.
- `count_nonzero(&self) -> usize` where T: PartialEq + Default: Count buckets != T::default().
- `num_buckets(&self) -> usize`: Returns `self.buckets.len()`.
- `bucket_start_for(&self, time: SystemTime) -> SystemTime`: Aligns time down to bucket boundary. `since_epoch_secs / bucket_secs * bucket_secs`.

Per CONTEXT.md: "Bucket granularity is configurable per-operator, with global default fallback." The RingBuffer accepts bucket_duration as a parameter -- the per-operator vs global default logic lives in the operator/pipeline layer (Plan 02/04).

Per RESEARCH.md Pitfall 1: Always use `duration_since().unwrap_or(Duration::ZERO)` for SystemTime arithmetic.
Per RESEARCH.md Pitfall 2: Bucket count = ceil(window/bucket). Test boundary cases.
Per RESEARCH.md Pitfall 3: If gap > full window, zero ALL buckets.

Write comprehensive unit tests in a `#[cfg(test)] mod tests` block at the bottom of window.rs covering all behaviors listed above. Use `SystemTime::UNIX_EPOCH + Duration::from_secs(N)` to create deterministic timestamps.
